# Prompt 生产 SOP

**定位：** 将章纲 + writing-style + genre-profile 生产为 subagent 可执行的 prompt.md（`prompts/vol-{N}-ch-{M}-prompt.md`），并对产物做合格判定。只对 prompt 负责，不涉及正文写作。

**全局约束：**
- writing-style 四字段必须全部注入 prompt——role 定叙事身份，core_principles 定不可违背的写作信条，possible_mistakes 定 AI 易犯错误列表，depiction_techniques 定描写层次和手法。缺任何一个 subagent 都会在最关键的地方放飞。
- 正文字数目标、情绪目标等执行参数从 `settings/writing-style.md` 和 `chapters/vol-{N}-ch-{M}.md` 读取，不在 prompts/ 中重复定义——一处修改全局生效。

**设计原则：每语义单一定义**
- "情绪"只在方法层·场景原材料定义，角色初始状态和质感不重复定义情绪
- "场景"只在方法层·场景原材料定义，前情上下文仅引用场景名
- "爽点"只在任务层·叙事目标定义，不重复出现


## 一、Prompt 工程结构（5 层）

按 prompt engineering 规范组织：先定义身份和目标，再提供上下文和约束，最后自检。

```
## 角色

你是一个 ______ 题材小说作家。______。


## 任务

创作第 ______ 章正文（在全卷中的角色：______）。

目标字数：______ ±200 字
本章驱动力：______
信息释放节奏：______

叙事目标：
1. 核心悬念：______ → 本章结束时：______（解决/加深/转移）
2. 读者情绪：______
3. 爽点设计：______，铺垫于场景______，释放于场景______


## 方法

### 写作能力（可用工具，每项填一句话说明"写什么时用这项"）
- 场景构筑与情绪节奏控制：______
- 对话设计与角色区分度：______
- 视角控制与信息差释放：______
- 反AI痕迹写作：______

### 前情上下文
上章结尾画面：______

读者情绪残留：______

上章留下的缺口：______

### 角色初始状态
______：
  起点：______
  → 经历：______
  → 终点：______
  本章微习惯/小动作：______

______：
  ...

### 场景原材料

场景 N｜______｜______基调｜权重：______
  核心事件：______
  情绪意义：______
  信息差：______
  情绪拐点：______（情绪从 X 变 Y 的转折）
  → 衔接：______
  本场景权重：______

### 写作方法论

动笔前按以下顺序执行：
1. 焦点锁定：识别本场景唯一核心冲突/人物情绪/信息差。70% 笔墨集中在焦点对象，次要元素简笔带过。高权重场景细化，低权重场景快速转场
2. 感官分层取舍：每场景只选 2 种感官细化（如听觉+触觉），其余一笔略写。禁止全维度均匀堆砌
3. 叙事节奏交替：一段密集动作/对话 + 一段短心理留白 + 一句轻环境过渡循环。禁止连续长描写、连续对话、通篇等长段落
4. 信息释放控制：关键伏笔、人物秘密拆分碎片化带出，不一次性完整交代。允许留白


## 要求

### 约束红线
冲突阶梯：______

情节红线（不可删减）：
  1. ______
  2. ______

边界禁止（不涉及）：
  1. ______
  2. ______

角色禁区：
  ______ 不会 ______
  ______ 不会 ______

### 写作规范

视角：______

描写手法：______

节奏要求：______

反 AI 规则：
  - 疲劳词阈值：______
  - 句式规则：______
  - 元叙事禁止：______
  - 情绪外化：______
  - 重复检测：______
  - 标点规范：______

题材反 AI：______

场景技法：______

### 质感要求

无用细节（不服务主线的细碎生活化细节）：
  1. ______
  2. ______

对话节奏：允许半截话、语气词、无意义停顿、重复口头禅，贴合人设习惯

段落精度分层：
  - 高权重场景：细节饱满，动作/微表情/心理细化
  - 低权重场景：简略潦草，允许流水句，不刻意美化

额外约束（本章特有）：
  1. ______
  2. ______


## 验收

### 写作前确认
- [ ] 视角已锁定，偏差即重写
- [ ] 四步方法论已理解：本场景焦点是什么？选哪 2 种感官？节奏如何交替？信息如何释放？
- [ ] 红线已确认：哪些不能删、哪些不能碰、角色不能做什么

### 写作中自检
- [ ] 每完成一个场景，确认核心事件已发生
- [ ] 段落检查：是否有 AI 疲劳词/元叙事/均匀堆砌
- [ ] 信息差是否维持到该释放的时刻

### 完稿后确认
- [ ] 字数在目标范围内
- [ ] 情绪目标已达成
- [ ] 章末停在画面/动作上，无升华总结
- [ ] 无 meta 泄漏（不包含"以下是小说的正文"类短语）
- [ ] 无均匀堆砌——高权重场景饱满，低权重场景简略
- [ ] 禁止配替代已执行——不直述情绪结论、不用副词修饰对话
```

**第一章特殊规则（自动检测 vol-x-ch-1 时启用）：**
1. 前情上下文固定填写"无前置章节，开篇直接切入角色当下行动"，禁用大段世界背景介绍
2. 场景原材料第一个场景强制标记高权重，优先刻画主角外貌、下意识微习惯、当下困境
3. 写作规范追加："前三段以人物动作作为开篇第一句，不堆砌多层环境描写"


## 二、核心填充方法

每层有独立的填充方法，定义输入源、填充原则、产出要求。

**全局原则：** 全部字段填完后全局通读一次，掩住字段名看每条读起来是叙事画面（✅）还是写作指令（❌）。


### 角色

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 题材 | genre-setting.md | 照抄，每章不变 |
| 叙事身份 | writing-style.md#role | 照抄，不展开 |


### 任务

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 章号 | 卷号+章号 | 第 {M} 章 |
| 在全卷中的角色 | 章纲 memo | 铺垫/推进/高潮/收束 |
| 目标字数 | 章纲 memo | 直接取，允许 ±200 浮动 |
| 本章驱动力 | 卷纲 主导驱动力 + 章纲信息差关系 | 与卷纲一致 |
| 信息释放节奏 | 章纲信息差关系 + 驱动力类型 | 悬疑→每章末留钩子；威胁→安全期缩短；目标→阶段性推进 |
| 核心悬念 | 章纲 memo | 是问题形式，不是情节概述 |
| 悬念状态 | 章纲 memo | 三选一：解决了/加深了/转移了 |
| 读者离场感受 | 章纲 情绪曲线落点 | 一个情绪词或画面描述 |
| 爽点类型 | 章纲 memo | 打脸/成长/暧昧/反转/解谜/绝境翻盘 |
| 铺垫位置/释放位置 | 章纲 memo | 场景编号，与场景原材料一致 |

**Goals 必须是可验证的**——写完后能判断"达成了/没达成"。


### 方法

#### 写作能力

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 场景构筑 | depiction_techniques + scene-craft 目录 | 提炼 2-3 条本章最相关的描写能力 |
| 对话/角色 | scene-craft/dialogue/ | 体现角色区分度技巧 |
| 视角/信息差 | scene-craft/pov/ | 体现视角控制能力 |
| 反AI | writing-style + anti-ai 库 | 体现 AI 痕迹规避能力 |

**原则：** 能力是对 writer 能力的声明，不是约束。每项用一句话说明"写什么时用这项"。

#### 前情上下文

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 上章结尾画面 | 上章 archives/*.md 最后 100 字 | 感官细节（看到/听到/感受到什么），30-50 字 |
| 上章读者情绪残留 | 上章情绪曲线落点 | 一个情绪词（不是分析） |
| 上章留下的缺口 | 章纲 memo | 读者期待但还没发生的事，1 句 |

上章为 ch-1 时：固定填写"无前置章节，开篇直接切入角色当下行动"，禁用大段世界背景介绍

#### 角色初始状态

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 起点 | 角色设定 + 上章归档 | 具体状态（含微动作），不只是情绪词 |
| → 经历 | 章纲 人物状态 | 1-2 个关键转折点 |
| 终点 | 章纲 人物状态 | 可感知的状态变化 |
| 微习惯/小动作 | 章纲 memo + 角色设定 | 本章特有的，与情节相关的 |

关键配角（如出场）额外填印象和必须完成的一件事。

#### 场景原材料

场景是叙事基本单元。场景切分依据情绪线拐点：

**边界信号（满足任意一个即切分）：**

| 信号 | 判定 | 例 |
|------|------|----|
| 换地点 | 角色走到另一个地方 | 早餐铺 → 书店 |
| 跳时间 | 有不可见的时间流逝 | 上午 → 黄昏 |
| 情绪转折 | 基调变了 | 尴尬 → 自然 |

**种子字段结构：**

| 字段 | 写什么 | 控制力 |
|------|--------|--------|
| 入口情绪 | 读者进入时的感受 | 一个词 |
| 核心事件 | 必须发生的，不可删减 | 保剧情不跑偏 |
| 核心事件的情绪意义 | 这个事件对读者意味着什么 | 不是分析，是感受 |
| **信息差** | **{谁知道什么} ↦ {谁不知道什么}** | **注入本场景的信息不对称——紧张/期待来源** |
| 情绪拐点 | 情绪从 X 变 Y 的瞬间 | 转折点 |
| 出口情绪 | 离开时的感受 | 一个词 |
| → 下一场景的衔接 | 情绪过渡方式 | 不生跳 |
| 本场景权重 | 根据核心事件重要性标注 | 高=细化动作/心理占70%笔墨；中=正常叙事；低=快速转场 |

**第一章特殊规则：** 第一个场景强制标记高权重，优先刻画主角外貌、下意识微习惯、当下困境

#### 写作方法论

四步执行流程直接照抄模板，不做调整。


### 要求

#### 约束红线

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| **冲突阶梯** | **章纲 memo** | **第1-4层，与卷纲冲突阶梯一致。决定了压迫感强度** |
| 情节红线（不可删减） | 章纲 memo | 2-4 个关键事件/台词/动作，subagent 的铁轨 |
| 边界禁止（不涉及） | 章纲 memo | 写"不做什么"，不写"做什么" |
| 角色禁区 | 章纲 memo + 作者补充 | 每个角色本章不会做的事 |

**禁止配替代：**

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| 直述情绪结论 | 用动作/神态/环境外化 |
| 章末升华总结 | 停在画面/动作上 |
| 副词修饰对话 | 去掉副词，用动作暗示 |

#### 写作规范

| 来源 | 文件 | 填入字段 |
|------|------|---------|
| 基础约束 | genre-example/{genre}.md#prompt_segment | 视角/描写/节奏 |
| 通用反AI | anti-ai/common-rules.md | 疲劳词阈值 + 句式规则 + 元叙事禁止 |
| 题材反AI | anti-ai/{genre}.md（如存在） | 题材正反例 |

**场景写作方法论注入：** 根据场景原材料识别的场景类型（对话/战斗/环境/心理/群像/过渡），从 `knowledge/scene-craft/` 读取对应文件。**必须经过上下文过滤**——结合本章的角色状态、信息差、情绪走向改写方法论，不能生搬硬套。**每个场景类型只抽取 1-2 条，不要求全部覆盖。**

**第一章特殊处理：** 视角/描写追加"前三段以人物动作作为开篇第一句，不堆砌多层环境描写"；第一个场景强制高权重。

#### 质感要求

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 无用细节 | 章纲 memo + 作者脑补 | 1-2 个不服务主线的细碎生活化细节（捏衣角、指尖蹭桌面、走神发呆） |
| 对话节奏 | writing-style.md | 允许半截话、语气词、无意义停顿、重复口头禅，贴合人设习惯 |
| 段落精度分层 | 场景权重映射 | 高权重场景细节饱满；低权重转场段落可简略潦草，允许流水句，杜绝全文精致统一 |


### 验收

writer 自检清单直接照抄模板。prompt-crafter 执行填入后不做额外修改。

## 三、验收（prompt-crafter 质量门禁）

> 注：以下验收流程是 prompt-crafter 的操作性检查，**不注入 prompt 文件**。prompt 中的验收层是 writer 自检清单（见第一章）。

### 快速嗅探

读场景原材料场景 1 的**核心事件**——掩住字段名，读起来是"发生了什么"（✅）还是"去写什么"（❌）？前者合格，后者是写作指令，重填。

### 完整性校验

通过以下各项 = prompt 合格：

| # | 层 | 校验项 | 通过标准 |
|---|----|-------|---------|
| 1 | 角色 | 身份明确 | 题材+叙事身份，writer 读完知道"我是谁" |
| 2 | 任务 | 参数完整 | 章号/字数/驱动力/节奏 4 字段均有值 |
| 3 | 能力 | 能力声明 | 3-4 项写作能力，与本章内容相关 |
| 4 | 前情上下文 | 画面感 | 上章结尾画面是感官细节（ch-1 标注无前文） |
| 5 | 角色初始状态 | 有变化 | 每角色有起点→经历→终点，可感知的变化 |
| 6 | 场景原材料 | 权重+字段 | 每场景有权重标注，有入口情绪/核心事件/信息差/出口情绪 |
| 7 | 约束红线 | 硬性约束 | 有冲突阶梯层位 + 红线 + 禁区 |
| 8 | 写作规范 | 四步逻辑+稀疏 | 含四步执行逻辑 + 反 AI 规则 + 每类型 1-2 条技法指引 |
| 9 | 质感要求 | 不完美约束 | 含半截话/生活化细节/段落精度分层 |

**判定：** 9 项全部 ✅ → 合格。1-2 项 ❌ → 针对性修改。3 项及以上 ❌ → 整份重填。

### 冲突检测

填完后通读，检查各层引用一致性：

| 检查项 | 一致则 ✅ |
|--------|---------|
| 前情上下文的情绪残留 = 角色初始状态第一角色的起点 | — |
| 角色状态转折落点 = 场景原材料某场景的情绪拐点 | — |
| 叙事目标悬念状态 = 场景原材料最后一场景的出口情绪 | — |
| 爽点释放位置在场景原材料中存在 | — |

### 逐节检查

| 层 | 合格标准 | 不合格信号 |
|---|---------|-----------|
| 角色 | 一读就知道作家身份和调性 | 模板级空话 |
| 任务 | 参数清晰，知道要写什么章、多少字 | 缺字数或驱动力 |
| 能力 | 能力声明与本章内容相关 | 空泛或与本章无关 |
| 前情上下文 | 像正常回忆，有画面感（ch-1 标注无前文） | 写成了剧情梗概 |
| 角色初始状态 | 写的是"这章TA经历了什么变化" | 复制了角色百科，或只有状态没有变化 |
| 场景原材料 | 每场景有权重标注，种子是叙事画面 | 种子是"这场景写…"写作指令 |
| 约束红线 | 红线+禁区齐全 | 约束空泛（"禁止跑偏"） |
| 写作规范 | 含四步执行逻辑 + 反AI + 稀疏技法 | 无四步逻辑或技法全覆盖 |
| 质感要求 | 含"不完美"约束（半截话/生活化/精度分层） | 空着或只写"无" |

全部通过 → 进入正文生成。任一不合格 → 先修再写。
