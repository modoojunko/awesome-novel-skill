# Prompt 生产 SOP + Prompt 质检标准

**定位：** 将章纲 + writing-style + genre-profile 生产为 subagent 可执行的 prompt.md（`prompts/vol-{N}-ch-{M}-prompt.md`），并对产物做合格判定。只对 prompt 负责，不涉及正文写作。

**全局约束：**
- writing-style 四字段必须全部注入 prompt——role 定叙事身份，core_principles 定不可违背的写作信条，possible_mistakes 定 AI 易犯错误列表，depiction_techniques 定描写层次和手法。缺任何一个 subagent 都会在最关键的地方放飞。
- 正文字数目标、写作模型（writing_model）、情绪目标等执行参数从 `settings/writing-style.md` 和 `chapters/vol-{N}-ch-{M}.md` 读取，不在 prompts/ 中重复定义——一处修改全局生效。

按 prompt engineering 规范组织：先定义身份和目标，再提供上下文和约束。

### 写作四步执行逻辑（所有章节生效，writer 动笔前必须执行）

1. **焦点锁定**：识别本场景唯一核心冲突/人物情绪/信息差。70% 笔墨集中在焦点对象，次要元素简笔带过。高权重场景细化，低权重场景快速转场
2. **感官分层取舍**：每场景只选 2 种感官细化（如听觉+触觉），其余一笔略写。禁止全维度均匀堆砌
3. **叙事节奏交替**：一段密集动作/对话 + 一段短心理留白 + 一句轻环境过渡循环。禁止连续长描写、连续对话、通篇等长段落
4. **信息释放控制**：关键伏笔、人物秘密拆分碎片化带出，不一次性完整交代。允许留白

  - 高权重场景：细节饱满，动作/微表情/心理细化

### 额外约束（本章特有）
  1. ______
  2. ______

**设计原则：每语义单一定义**
- "情绪"只在场景序列定义，角色状态和质感不重复定义情绪
- "场景"只在场景序列定义，前情回顾仅引用场景名
- "爽点"只在 Goals 定义，不重复出现

---

**第一章特殊规则（自动检测 vol-x-ch-1 时启用）：**
1. 前情回顾固定填写"无前置章节，开篇直接切入角色当下行动"，禁用大段世界背景介绍
2. 场景序列第一个场景强制标记高权重，优先刻画主角外貌、下意识微习惯、当下困境
3. Goals 末尾追加"章节末尾留白悬念，不完整解决本章核心冲突"
4. 写作规范中视角/描写要求追加"前三段以人物动作作为开篇第一句，不堆砌多层环境描写"


## 二、核心填充方法

每层有独立的填充方法，定义输入源、填充原则、产出要求。

**全局原则：** 全部字段填完后全局通读一次，掩住字段名看每条读起来是叙事画面（✅）还是写作指令（❌）。

---

### Role

从 genre-setting 和 writing-style 提取写作身份：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 题材 | genre-setting.md | 照抄，每章不变 |
| 叙事身份 | writing-style.md#role | 照抄，不展开 |

---

### Task

从 genre-profile 和 writing-style 提取本章执行参数：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 章号 | 卷号+章号 | 第 {M} 章 |
| 在全卷中的角色 | 章纲 memo | 铺垫/推进/高潮/收束 |
| 目标字数 | 章纲 memo | 直接取，允许 ±200 浮动 |
| 本章驱动力 | 卷纲 主导驱动力 + 章纲信息差关系 | 与卷纲一致 |
| 信息释放节奏 | 章纲信息差关系 + 驱动力类型 | 悬疑→每章末留钩子；威胁→安全期缩短；目标→阶段性推进 |
| 建议模型 | writing-style.md#model | 照抄 |

---

### Skills

从 writing-style 和 scene-craft 目录提炼：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 场景构筑 | depiction_techniques + scene-craft 目录 | 提炼 2-3 条本章最相关的描写能力 |
| 对话/角色 | scene-craft/dialogue/ | 体现角色区分度技巧 |
| 视角/信息差 | scene-craft/pov/ | 体现视角控制能力 |
| 反AI | writing-style + anti-ai 库 | 体现 AI 痕迹规避能力 |

**原则：** Skills 是对 writer 能力的声明，不是约束。每项用一句话说明"写什么时用这项"。

---

### Goals

从章纲 memo + 情绪曲线提取本章必须达成的目标：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 核心悬念 | 章纲 memo | 是问题形式，不是情节概述 |
| 悬念状态 | 章纲 memo | 三选一：解决了/加深了/转移了 |
| 读者离场感受 | 章纲 情绪曲线落点 | 一个情绪词或画面描述 |
| 爽点类型 | 章纲 memo | 打脸/成长/暧昧/反转/解谜/绝境翻盘 |
| 铺垫位置/释放位置 | 章纲 memo | 场景编号，与场景序列一致 |

**Goals 必须是可验证的**——写完后能判断"达成了/没达成"。

---

### 前情回顾

从上章章尾提取，回答"读者带着什么情绪进本章"：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 上章结尾画面 | 上章 archives/*.md 最后 100 字 | 感官细节（看到/听到/感受到什么），30-50 字 |
| 上章读者情绪残留 | 上章情绪曲线落点 | 一个情绪词（不是分析） |
| 上章留下的缺口 | 章纲 memo | 读者期待但还没发生的事，1 句 |

---

### 角色状态

从章纲人物状态 + 角色设定文件提取，每角色一条：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 起点 | 角色设定 + 上章归档 | 具体状态（含微动作），不只是情绪词 |
| → 经历 | 章纲 人物状态 | 1-2 个关键转折点 |
| 终点 | 章纲 人物状态 | 可感知的状态变化 |
| 微习惯/小动作 | 章纲 memo + 角色设定 | 本章特有的，与情节相关的 |

关键配角（如出场）额外填印象和必须完成的一件事。

---

### 场景序列

场景是叙事基本单元。场景切分依据情绪线拐点：

**边界信号（满足任意一个即切分）：**

| 信号 | 判定 | 例 |
|------|------|----|
| 换地点 | 角色走到另一个地方 | 早餐铺 → 书店 |
| 跳时间 | 有不可见的时间流逝 | 上午 → 黄昏 |
| 情绪转折 | 基调变了 | 尴尬 → 自然 |

**种子字段结构：**

| 字段 | 写什么 | 控制力 |
|------|--------|--------|
| 入口画面 | 进入这个场景的第一画面 | 定基调 |
| 入口情绪 | 读者进入时的感受 | 一个词 |
| 核心事件 | 必须发生的，不可删减 | 保剧情不跑偏 |
| 核心事件的情绪意义 | 这个事件对读者意味着什么 | 不是分析，是感受 |
| **信息差** | **{谁知道什么} ↦ {谁不知道什么}** | **注入本场景的信息不对称——紧张/期待来源** |
| 情绪拐点 | 情绪从 X 变 Y 的瞬间 | 转折点 |
| 出口情绪 | 离开时的感受 | 一个词 |
| 出口画面 | 离开时的感官定格 | 画面感 |
| → 下一场景的衔接 | 情绪过渡方式 | 不生跳 |
| 本场景权重 | 根据核心事件重要性标注 | 高=细化动作/心理占70%笔墨；中=正常叙事；低=快速转场 |

```
场景1｜梧桐巷早餐铺（抑）｜高
入口画面：苏晚提前到了，坐在靠窗老位置，手指来回摩挲菜单边缘。
入口情绪：局促

核心事件：陆屿推门进来，两人客气打招呼。张奶奶端豆浆过来，念叨小时候糗事。苏晚低头笑了。
核心事件的情绪意义：旧时光浮现，尴尬被打破

情绪拐点：苏晚笑了那一刻
出口情绪：松懈

出口画面：梧桐叶在窗外轻轻晃动
→ 下一场景：两人气氛自然，转入书店选书
```

### 约束红线

从章纲 memo 提取，目标是回答"什么绝对不能碰"：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| **冲突阶梯** | **章纲 memo** | **第1-4层，与卷纲冲突阶梯一致。决定了压迫感强度** |
| 情节红线（不可删减） | 章纲 memo | 2-4 个关键事件/台词/动作，subagent 的铁轨 |
| 边界禁止（不涉及） | 章纲 memo | 写"不做什么"，不写"做什么" |
| 角色禁区 | 章纲 memo + 作者补充 | 每个角色本章不会做的事 |

**禁止配替代：**

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| 直述情绪结论 | 用动作/神态/环境外化 |
| 章末升华总结 | 停在画面/动作上 |
| 副词修饰对话 | 去掉副词，用动作暗示 |

### 写作规范

从 genre-example 和 anti-ai 库提取，不自己发明：

| 来源 | 文件 | 填入字段 |
|------|------|---------|
| 基础约束 | genre-example/{genre}.md#prompt_segment | 视角/描写/节奏 |
| 通用反AI | anti-ai/common-rules.md | 疲劳词阈值 + 句式规则 + 元叙事禁止 |
| 题材反AI | anti-ai/{genre}.md（如存在） | 题材正反例 |

**场景写作方法论注入（按需 + 上下文过滤 + 稀疏抽取）：** 根据场景序列识别的场景类型（对话/战斗/环境/心理/群像/过渡），从 `knowledge/scene-craft/` 读取对应文件。**读取后必须经过上下文过滤**——结合本章的角色状态、信息差、情绪走向改写方法论，不能生搬硬套。**每个场景类型只抽取 1-2 条，不要求全部覆盖。** 同一方法论在不同章节因角色不同而写法不同。

**写作四步执行逻辑（强制注入 subagent 执行流程）：** 每章必须按焦点锁定→感官分层→节奏交替→信息控制四步执行。填充时确保四条全部写入写作规范节，缺一不可。

**第一章特殊处理（卷号=1且章号=1时）：**
- 视角/描写追加："前三段以人物动作作为开篇第一句，不堆砌多层环境描写"
- 第一个场景强制高权重

**额外约束：** 本章特有、不从规则库来的额外要求，1-2 条。

**视角：** 严格锁定，偏差即重写。

### 质感要求

从章纲 memo 和作者补充提取，目标是回答"怎么写得不像 AI"。

**追加"不完美"约束——抑制 AI 工整感：**

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 无用细节 | 章纲 memo + 作者脑补 | 1-2 个不服务主线的细碎生活化细节（捏衣角、指尖蹭桌面、走神发呆） |

## 三、验收

### 快速嗅探

读场景序列场景 1 的**核心事件**——掩住字段名，读起来是"发生了什么"（✅）还是"去写什么"（❌）？前者合格，后者是写作指令，重填。

### 完整性校验

通过以下各项 = prompt 合格：

| # | 节 | 校验项 | 通过标准 |
|---|----|-------|---------|
| 1 | Role | 身份明确 | 题材+叙事身份，writer 读完知道"我是谁" |
| 2 | Task | 参数完整 | 章号/字数/驱动力/节奏/模型 5 字段均有值 |
| 3 | Skills | 能力声明 | 3-4 项写作能力，与本章内容相关 |
| 4 | Goals | 可验证 | 悬念是问题形式 + 爽点释放位置存在，写完后能判断是否达成 |
| 5 | 前情回顾 | 画面感 | 上章结尾画面是感官细节（ch-1 标注无前文） |
| 6 | 角色状态 | 有变化 | 每角色有起点→经历→终点，可感知的变化 |
| 7 | 场景序列 | 权重+字段 | 每场景有权重标注，有入口/核心事件/信息差/出口 |
| 8 | 约束红线 | 硬性约束 | 有冲突阶梯层位 + 红线 + 禁区 |
| 9 | 写作规范 | 四步逻辑+稀疏 | 含四步执行逻辑 + 反 AI 规则 + 每类型 1-2 条技法指引 |
| 10 | 质感要求 | 不完美约束 | 含半截话/生活化细节/段落精度分层 |

**判定：** 10 项全部 ✅ → 合格。1-2 项 ❌ → 针对性修改。3 项及以上 ❌ → 整份重填。

### 冲突检测

填完后通读，检查各节引用一致性：

| 检查项 | 一致则 ✅ |
|--------|---------|
| 前情回顾的情绪残留 = 角色状态第一角色的起点 | — |
| 角色状态转折落点 = 场景序列某场景的情绪拐点 | — |
| Goals 悬念状态 = 场景序列最后一场景的出口情绪 | — |
| Goals 爽点释放位置在场景序列中存在 | — |

### 逐节检查

填充阶段用，写正文前拦截：

| 节 | 合格标准 | 不合格信号 |
|----|---------|-----------|
| Role | 一读就知道作家身份和调性 | 模板级空话 |
| Task | 参数清晰，知道要写什么章、多少字 | 缺字数或驱动力 |
