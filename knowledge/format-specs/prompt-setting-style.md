# Prompt 生产 SOP + Prompt 质检标准

**定位：** 将章纲 + writing-style + genre-profile 生产为 subagent 可执行的 prompt.md（`prompts/vol-{N}-ch-{M}-prompt.md`），并对产物做合格判定。只对 prompt 负责，不涉及正文写作。

**全局约束：**
- writing-style 四字段必须全部注入 prompt——role 定叙事身份，core_principles 定不可违背的写作信条，possible_mistakes 定 AI 易犯错误列表，depiction_techniques 定描写层次和手法。缺任何一个 subagent 都会在最关键的地方放飞。
- 正文字数目标、写作模型（writing_model）、情绪目标等执行参数从 `settings/writing-style.md` 和 `chapters/vol-{N}-ch-{M}.md` 读取，不在 prompts/ 中重复定义——一处修改全局生效。

---

## 一、通用底层骨架

单一职责的 9 层结构，每层只定义自己维度的信息，不跨层重复定义语义。

```
## L1. 元信息（静态，一章只用一次）

题材：______（来自 genre-profile）
写作风格：______（来自 writing-style，不展开）
目标字数：______（允许 ±200 字浮动）
本章在全卷中的角色：______（铺垫 / 推进 / 高潮 / 收束）
本章驱动力：______（悬疑驱动 / 威胁驱动 / 目标驱动 / 关系驱动 / 信息差驱动）
信息释放节奏：______（每章末留钩子 / 安全期缩短 / 阶段性推进 / 张力累积到揭穿）
建议模型：______（来自 writing-style.model）

---

## L2. 来龙（只回答：从哪来）

上章结尾画面（30-50 字）：______（上章最后场景的感官细节）
上章读者情绪残留：______（读者带着什么感受进本章）
上章留下的缺口：______（读者期待什么还没发生）

---

## L3. 去脉（只回答：到哪去）

本章核心悬念（1 个）：______（这章必须回答或加深的问题）
悬念状态在本章结束时：______（解决了 / 加深了 / 转移了）
读者离开时的感受：______（本章留给读者的余韵）

---

## L4. 角色弧光（每角色一条，不跨角色定义）

主角A：
  上章结束时：______（具体状态，不只是情绪词）
  → 本章经历：______（关键转折点，1-2 个）
  → 本章结束时：______（可感知的状态变化）
  本章特有的微习惯/小动作：______

主角B：
  上章结束时：______
  → 本章经历：______
  → 本章结束时：______
  本章特有的微习惯/小动作：______

关键配角（如出场）：
  本章给读者留下的印象：______
  本章必须完成的一件事：______

---

## L5. 场景序列（每场景 = 一个情绪单元）

场景数：2-4 个（按情绪单元分段）
情绪类型：压抑 / 升温 / 爆发 / 缓和 / 悬念 / 释然

---

场景N｜场景名｜基调类型｜字数区间｜情绪弧（起点→落点）

  入口画面：______（进入这个场景的第一画面）
  入口情绪：______（读者进入时的感受）

  核心事件：______（必须发生的，不可删减）
  核心事件的情绪意义：______（这个事件对读者意味着什么）

  信息差：______（{谁知道什么} ↦ {谁不知道什么} —— 本场景的信息不对称来源）

  情绪拐点：______（情绪从 X 变 Y 的转折瞬间）
  出口情绪：______（离开这个场景时的感受）
  出口画面：______（离开时的感官定格）

  → 下一场景的情绪过渡：______（自然衔接，不生跳）

---

## L6. 约束（只回答：什么不能碰）

本章在卷冲突阶梯中的位置：______（第1层·入门 / 第2层·升级 / 第3层·转折 / 第4层·终极）
  → 决定了本章的障碍强度和读者压迫感：入门级不宜写生死危机，终极级不能写轻松过渡。

情节红线（不可删减的事件/台词/动作）：
  1. ______
  2. ______

边界禁止（本章不涉及的内容）：
  1. ______
  2. ______

角色禁区（每个角色本章不会做的事）：
  主角A：不会 ______
  主角B：不会 ______

---

## L7. 爽点设计（每章 1-2 个，不贪多）

本章核心爽点：
  类型：______（打脸 / 成长 / 暧昧 / 反转 / 解谜 / 绝境翻盘...）
  铺垫位置：______（本章第 N 场景）
  释放位置：______（本章第 M 场景）
  释放方式：______（具体怎么写，不是概念）

克制点（为爽点服务的压抑）：
  位置：场景 X 的情绪拐点前
  写法：______（具体手法，不是"要有张力"这种废话）

---

## L8. 文字规则（来自 genre-example + anti-ai，不自己发明）

**必须填入具体内容，subagent 不读外部文件。**

视角：______（来自 genre-example/{genre}.md#prompt_segment，严格锁定）

描写手法：______（来自 genre-example/{genre}.md#prompt_segment）

节奏要求：______（来自 genre-example/{genre}.md#prompt_segment）

**通用反AI（来自 anti-ai/common-rules.md，必须填入具体值）：**
- 疲劳词阈值：______（表一，如"突然≤3/章；然而≤4/章"）
- 句式规则：______（表二，如"连续4句主语不得相同"）
- 元叙事禁止：______（表三，如"禁止'读者''本章'"）

**题材反AI（来自 anti-ai/{genre}.md，如存在）：**
- 高频AI病句正反例：
  ______

额外约束（本章特有，不从规则库来）：
  1. ______
  2. ______

---

## L9. 质感（只回答：怎么不像 AI）

无用细节（1-2 个，与主线无关但有温度）：
  1. ______
  2. ______

对话节奏：
  连续对话不超过 ______ 句
  对话之间必须有：动作 / 沉默 / 环境 / 内心（至少 1 种）

真人痕迹：
  允许词穷：______
  允许重复：______
  允许停顿：______

---

**设计原则：每语义单一定义**
- "情绪"只在 L5（场景）定义，L1-4 不重复定义情绪
- "场景"只在 L5 定义，L2 仅引用列表名称
- "爽点"只在 L7 定义，L1 仅引用类型标签

## 二、核心填充方法

每层有独立的填充方法，定义输入源、填充原则、产出要求。

**全局原则：** 全部字段填完后全局通读一次，掩住字段名看每个种子读起来是叙事画面（✅）还是写作指令（❌）。

---

### L1 元信息

从 genre-profile 和 writing-style 直接提取：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 题材 | genre-setting.md | 照抄，每章不变 |
| 写作风格 | writing-style.md | 照抄四字段名，不展开内容 |
| 目标字数 | 章纲 memo | 直接取，允许 ±200 浮动 |
| 本章在全卷中的角色 | 章纲 memo | 三选一：铺垫/推进/高潮/收束 |
| **本章驱动力** | **卷纲 主导驱动力 + 章纲信息差关系** | **与卷纲一致，说明"本章用什么力推读者走"** |
| **信息释放节奏** | **章纲信息差关系 + 驱动力类型** | **悬疑→每章末留钩子；威胁→安全期缩短；目标→阶段性推进；关系/信息差→张力累积到揭穿** |
| 建议模型 | writing-style.md#model | 照抄 |

---

### L2 来龙

从上章章尾提取，目标是回答"读者带着什么情绪进本章"：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 上章结尾画面 | 上章 archives/*.md 最后 100 字 | 感官细节（看到/听到/感受到什么），30-50 字 |
| 上章读者情绪残留 | 上章情绪曲线落点 | 一个情绪词（不是分析） |
| 上章留下的缺口 | 章纲 memo | 读者期待但还没发生的事，1 句 |

---

### L3 去脉

从本章章纲 memo 提取，目标是回答"本章结束时要给读者留下什么"：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 本章核心悬念 | 章纲 memo | 本章必须回答或加深的问题，1 个 |
| 悬念状态在本章结束时 | 章纲 memo | 三选一：解决了/加深了/转移了 |
| 读者离开时的感受 | 章纲 情绪曲线落点 | 一个情绪词或画面描述 |

---

### L4 角色弧光

从章纲人物状态 + 角色设定文件提取，每角色一条：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 上章结束时 | 角色设定 + 上章归档 | 具体状态（含微动作），不只是情绪词 |
| → 本章经历 | 章纲 人物状态 | 1-2 个关键转折点 |
| → 本章结束时 | 章纲 人物状态 | 可感知的状态变化 |
| 微习惯/小动作 | 章纲 memo + 角色设定 | 本章特有的，与情节相关的 |

**原则：** 不写角色百科，只写"这章里这个角色经历了什么变化"。

---

### L5 场景序列

场景是叙事基本单元。场景切分依据情绪线拐点：

**边界信号（满足任意一个即切分）：**

| 信号 | 判定 | 例 |
|------|------|----|
| 换地点 | 角色走到另一个地方 | 早餐铺 → 书店 |
| 跳时间 | 有不可见的时间流逝 | 上午 → 黄昏 |
| 情绪转折 | 基调变了 | 尴尬 → 自然 |

**字数估算：**

| 情绪浓度 | 典型字数 |
|---------|---------|
| 浓（高潮/冲突/深度对话） | 1500-2500 |
| 中（日常互动/推进剧情） | 800-1500 |
| 淡（过渡/铺垫/气氛渲染） | 500-1200 |

**种子字段结构：**

| 字段 | 写什么 | 控制力 |
|------|--------|--------|
| 入口画面 | 进入这个场景的第一画面 | 定基调 |
| 入口情绪 | 读者进入时的感受 | 一个词 |
| 核心事件 | 必须发生的，不可删减 | 保剧情不跑偏 |
| 核心事件的情绪意义 | 这个事件对读者意味着什么 | 不是分析，是感受 |
| **信息差** | **{谁知道什么} ↦ {谁不知道什么}** | **注入本场景的信息不对称——紧张/期待来源** |
| 情绪拐点 | 情绪从 X 变 Y 的瞬间 | 转折点 |
| 出口情绪 | 离开时的感受 | 一个词 |
| 出口画面 | 离开时的感官定格 | 画面感 |
| → 下一场景的衔接 | 情绪过渡方式 | 不生跳 |

```
场景1｜梧桐巷早餐铺（抑）
入口画面：苏晚提前到了，坐在靠窗老位置，手指来回摩挲菜单边缘。
入口情绪：局促

核心事件：陆屿推门进来，两人客气打招呼。张奶奶端豆浆过来，念叨小时候糗事。苏晚低头笑了。
核心事件的情绪意义：旧时光浮现，尴尬被打破

情绪拐点：苏晚笑了那一刻
出口情绪：松懈

出口画面：梧桐叶在窗外轻轻晃动
→ 下一场景：两人气氛自然，转入书店选书
```

---

### L6 约束

从章纲 memo 提取，目标是回答"什么绝对不能碰"：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| **本章在卷冲突阶梯中的位置** | **章纲 memo** | **第1-4层，与卷纲冲突阶梯一致。决定了本场景的压迫感强度** |
| 情节红线（不可删减） | 章纲 memo | 2-4 个关键事件/台词/动作，subagent 的铁轨 |
| 边界禁止（不涉及） | 章纲 memo | 写"不做什么"，不写"做什么" |
| 角色禁区 | 章纲 memo + 作者补充 | 每个角色本章不会做的事 |

**禁止配替代：**

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| 直述情绪结论 | 用动作/神态/环境外化 |
| 章末升华总结 | 停在画面/动作上 |
| 副词修饰对话 | 去掉副词，用动作暗示 |

### L7 爽点设计

从章纲 memo 提取，本章 1-2 个核心爽点：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 类型 | 章纲 memo | 打脸/成长/暧昧/反转/解谜/绝境翻盘... |
| 铺垫位置 | 章纲 memo | 本章第 N 场景 |
| 释放位置 | 章纲 memo | 本章第 M 场景 |
| 释放方式 | 章纲 memo | 具体怎么写，不是概念 |

**克制点（为爽点服务的压抑）：**

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 位置 | 章纲 情绪曲线 | 场景 X 的情绪拐点前 |
| 写法 | 章纲 memo | 具体手法 |

### L8 文字规则

从 genre-example 和 anti-ai 库提取，不自己发明：

| 来源 | 文件 | 填入字段 |
|------|------|---------|
| 基础约束 | genre-example/{genre}.md#prompt_segment | 全部替换通用版 |
| 通用反AI | anti-ai/common-rules.md | 疲劳词阈值 + 句式规则 + 元叙事禁止 |
| 题材反AI | anti-ai/{genre}.md（如存在） | 题材正反例 |

**额外约束：** 本章特有、不从规则库来的额外要求，1-2 条。

**视角：** 严格锁定，偏差即重写。

### L9 质感

从章纲 memo 和作者补充提取，目标是回答"怎么写得不像 AI"：

## 三、验收

### 快速嗅探

读 L5（场景序列）场景 1 的**核心事件**——掩住字段名，读起来是"发生了什么"（✅）还是"去写什么"（❌）？前者合格，后者是写作指令，重填。

### 九层硬校验

通过这 9 条 = prompt 合格。每条可人工打勾 ✅/❌：

| # | 层 | 校验项 | 通过标准 |
|---|----|-------|---------|
| 1 | L1 元信息 | 字段齐全 | 题材/风格/字数/角色/驱动力/节奏/模型 7 字段均有内容 |
| 2 | L2 来龙 | 画面感 | 上章结尾画面是感官细节，不是剧情概括 |
| 3 | L3 去脉 | 悬念明确 | 核心悬念是问题形式，悬念状态三选一 |
| 4 | L4 角色弧光 | 状态变化 | 每角色有起点→转折→落点，可感知的变化 |
| 5 | L5 场景序列 | 字段完整 | 每场景有入口/核心事件/信息差/出口，无缺位 |
| 6 | L6 约束 | 双侧闭合+定位 | 有冲突阶梯层位 + 2-4 条红线 + 边界禁止写"不做什么" |
| 7 | L7 爽点设计 | 具体释放 | 铺垫位置/释放位置/释放方式均有具体描述 |
| 8 | L8 文字规则 | 来源标注 | 标注了 genre-example + anti-ai 的来源文件 |
| 9 | L9 质感 | 真人痕迹 | 有无用细节 + 对话节奏 + 真人痕迹三字段 |

**判定：** 9 项全部 ✅ → prompt 合格。1-2 项 ❌ → 针对性修改。3 项及以上 ❌ → 整份重填。

### 冲突检测

填完后通读，检查各层引用一致性：

| 检查项 | 一致则 ✅ |
|--------|---------|
| L2 上章情绪残留 = L4 第一角色的起点 | — |
| L4 转折落点 = L5 某场景的情绪拐点 | — |
| L3 悬念状态 = L5 最后一场景的出口情绪 | — |
| L7 铺垫位置/释放位置在 L5 场景序列中存在 | — |

### 逐层检查

填充阶段用，写正文前拦截：

| 层 | 合格标准 | 不合格信号 |
|----|---------|-----------|
| L1 元信息 | 一读就知道题材、字数、调性 | 模板级空话（如"待填充"） |
| L2 来龙 | 像正常回忆，有画面感 | 写成了剧情梗概，没有感官细节 |
| L3 去脉 | 悬念是问题形式 | 悬念写成情节概述 |
| L4 角色弧光 | 写的是"这章TA经历了什么变化" | 复制了角色百科，或只有状态没有变化 |
| L5 场景序列 | 每场景种子是叙事画面非指令 | 种子是"这场景写…"写作指令 |
| L6 约束 | 红线 2-4 条，边界写"不做什么" | 约束空泛（"禁止跑偏"）或写成"应该做什么" |
| L7 爽点设计 | 释放方式具体 | 释放方式写成"要有张力"等废话 |
