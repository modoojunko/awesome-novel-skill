# awesome-novel v4.7 → v4.8 视频脚本输入文档

> 用途：视频脚本素材。聚焦近两个版本的核心变更。
> 目标观众：正在用或考虑用 AI 写小说的作者。

---

## 一、v4.7 补全 scene-craft 方法论：解决"场景写不好"

### 痛点
写小说场景时，环境描写只会"阳光洒进来"，外貌只会"肤白貌美"，转场永远是"三天后"。不是不想写好，是没有一套可复用的方法论。

### 解决：三个知识库文件

#### 环境描写——三套方法
| 方法 | 反面（别这么写） | 正面（要这么写） |
|------|----------------|----------------|
| 五感通写 | "破庙脏乱，满是灰尘蛛网" | "木门吱呀作响，殿内回荡回音，空气中飘着腐草与积尘气味，寒气顺着脚踝向上蔓延" |
| 情绪渲染 | 心情好和心情差用同一套景物 | 心情愉悦→"阳光如金"；内心低落→"阳光刺目" |
| 细节互动 | "他家境贫寒"（直白概括） | "搪瓷缸边缘磕出缺口，牙膏管被反复挤压至扁平" |

#### 角色外貌——新增 [侧面烘托] 标签
已有[锚点]（抓核心特征）和[动态]（融入动作），新增：
- **侧面烘托法**：不直接写长相，借旁人反应反衬
  - ❌ "她容貌绝美，倾国倾城"
  - ✅ "她踏入大厅，喧嚣声戛然而止。举杯的客人悬着酒杯忘了饮，服侍生的步伐凭空顿住"

#### 场景转场——七种技法
| 技法 | 反面（硬切） | 正面（顺滑） |
|------|------------|------------|
| 情绪顺延 | "男主打完架很生气，两天后去拍卖行" | "打斗残留的躁意盘踞心底，直至豪车停在拍卖行门前，车内沉香抚平戾气" |
| 危机钩子 | "女主聚餐接到病重电话，立刻去医院" | "烛火未熄，急诊来电弹出，女主慌乱打翻水杯冲出包厢" |
| 五感反差 | "女主晕倒，醒来在民宿" | "林间嘈杂消散，碎石触感褪去，艾草茶香漫入鼻腔——睁眼暖灯民宿" |
| 物件线索 | "五年过去，女主重回老街" | "指尖摩挲褪色桃木平安扣，凹凸触感一如当年，街边景象叠印出巷口少年身影" |
| 动线镜头 | "男主离开茶馆，到河畔凉亭" | "碾灭烟蒂走出檐下，沿青石河畔慢行，河水凉意拂面，抬眼已是凉亭" |
| 空镜景物 | "男主闭关二十天，伤势痊愈出关" | "闭关之初门缝涌入初夏热风，待院中槐花落尽，晚风裹挟秋凉，他推开石门" |
| 对话承接 | "酒楼众人计划夺宝，反派也得知消息" | "'明日前往拍卖行夺宝'——酒楼话音落下的同一刻，幽暗地牢里，反派重复了相同一句话" |

转场口诀：
```
转场不靠跳时间，不靠符号来敷衍；
要么情绪连着走，要么物件牵两头；
感官换环境，对话接双线；
空镜带流年。
```

### 这些方法论怎么用
不是整篇塞进提示词。prompt-crafter 会根据当前场景类型，按标签匹配，每次只选 1-2 条最相关的注入。

## 二、v4.8 角色推演沙盘：解决"有剧情但展不开"

### 痛点
很多作者卡在"知道这章要发生什么——卷纲一句话写好了——但就是写不出章纲"。他们缺的不是剧情结构，而是角色行为的具体画面。

### 推演沙盘是什么
一个交互式工具。作者设好舞台和演员、给出触发条件，AI 逐个角色串行推演。

```
第1轮：
  ─ 张三（主角）："你再说一遍？"拍案而起，目光紧逼
     ↑ 作者可以改张三的台词/动作
  ─ 李四（反派）：向后靠椅背，双手抱胸——"说了又如何？"
     ↑ 作者可以改李四的反应
  ─ 王五（中立）：默默把凳子往门口挪了挪

第2轮：
  ─ 张三：看到李四的强硬，换了策略……
  ─ 李四：……
```

### 关键设计
- **串行推演**：A→B→C，不是同时。因为人的行为是相互影响的
- **每轮可修改**：改了 B，会影响 B 后面的 C 的推演
- **多轮迭代**：直到剧情走到作者满意
- **信息差透明**：AI 只用角色当前知道的信息
- **认知层溯源**：每轮标注"这个选择来自角色的哪一层设定"

### 推演产出
不是章纲，是**推演记录**（角色行为素材）：谁在什么时候说了什么、做了什么表情、场景氛围怎么变。作者拿着这份记录去写章纲。

### 触发时机
- 作者主动说："跑一下推演"
- 作者说"卡住了""写不下去"——AI 主动建议

### 注意事项
推演记录是给**作者**看的参考素材，不自动注入任何后续流程。创作决策始终是作者的。

## 三、如果做对比展示

### 没有推演沙盘 vs 有推演沙盘
```
没有：
  卷纲一句话 → 硬写章纲 → 卡住 → 放弃/硬写 → 正文生硬

有：
  卷纲一句话 → 跑推演 → 看到角色具体行为 → 修改 →
  拿到推演记录 → 写章纲 → 正文有素材可用
```

### 没有 scene-craft vs 有 scene-craft
```
没有：
  环境描写 → "阳光洒进来，空气中弥漫着"（AI 套路）
  外貌描写 → "肤白貌美大长腿"（通用模板）
  转场 → "三天后"（硬切）

有：
  环境 → 五感通写或情绪渲染（按场景匹配）
  外貌 → 锚点/动态/侧面烘托（按角色匹配）
  转场 → 七种技法（按场景类型匹配）
```

## 四、话术片段

> "有时候你卡文，不是故事有问题，是你知道要发生什么，但不知道角色会怎么演。推演沙盘就是排练室。"

> "一个角色演完，你看效果，不行就改。改完下一个角色基于修改后的版本继续演。这种'看到过程'的体验，是直接让 AI 写一段文本给不了你的。"

> "scene-craft 就是场景工艺——环境怎么写、外貌怎么写、转场怎么接，每一件都有方法，不用从零想。"

> "v4.7 补方法论，v4.8 补推演能力。一个是给你工具箱，一个是给你排练室。"

> "有人问我为什么不对接 AI 自动写——因为推演记录是给你的，不是给 AI 的。你会用它写出你自己的章纲，而不是让 AI 替你做决定。"
