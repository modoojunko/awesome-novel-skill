# 生成章纲 skill

校准后的参考材料 → 一份可执行的章纲。格式规范参考 `knowledge/format-specs/chapter-setting-style.md`。

## 流程

立情绪核心 → 设冲突阶梯 → 建信息差 → 拆场景卡 → 禁止清单 → Memo 填充 → Hooks → 设变通知（可选）→ 产出

## 细化章纲

按 `chapter-setting-style.md` 的格式规范逐项填充。以下为执行顺序，具体格式和写法参考该规范。前三步（情绪核心/冲突阶梯/信息差）与卷纲方法论一致，只是粒度从"卷"细化到"章"。

**a. 立情绪核心** — 分两层：**本章在卷情绪走向中的位置** + **本章内部的微弧线**。

> Step 1：卷的情绪走向现在走到哪一步了？——"压抑阶段的中段"还是"到了打脸的释放阶段"？

本章的情绪主色调由卷的弧线位置决定。压抑阶段写压抑基调，释放阶段写爽基调。

> Step 2：这章内部读者情绪怎么变化？——从什么情绪开始、中间因为什么事转折、以什么情绪结束？至少三步。

卷走向的一部分 → 章内微弧线（平静→不安→确认困境→压抑），章末落点必须支撑卷走向的下一步。

**b. 设冲突阶梯** — 分两层：**本章在卷冲突阶梯中的层位** + **章内小冲突阶梯**。

> Step 1：卷的冲突阶梯走到哪一层了？——"第1层·入门"还是"第4层·终极"？

本章的障碍强度由卷的层位决定。入门级写信息受阻，终极级写正面对决。

> Step 2：这章内部障碍怎么递进？——主角在章内面对的不止一个障碍，是一组越来越难的障碍。至少 2-3 步。

卷的某一层 → 章内小阶梯（试探→遭遇→升级），每一步比前一步更难：

```
卷冲突阶梯：第1层 入门 → 第2层 升级 → 第3层 转折 → 第4层 终极
                        ↑
本章位置：第2层·升级

章内小阶梯：
  场景1：试探——主角尝试接触，发现对方有防备        ← 入门级
  场景2：遭遇——主角被警告/阻拦，确认阻力存在        ← 升级
  场景3：冲突升级——主角坚持调查，对方加大施压       ← 章内高潮
  结束：主角处于更危险的境地 → 支撑卷的下一步（第3层 转折）
```

**底线：** 章内阶梯不是重复同一件事——如果三个场景都是"主角查资料→被阻止→换个方式查资料"，这是平的，不是阶梯。

**c. 建信息差** — 信息差的定义在卷级（起点→终点），章级是信息差**动态变化**的地方。

> Step 1：本章继承卷信息差弧线的当前位置——卷走到"反派有优势"阶段还是"主角在反追"阶段？

> Step 2：本章内部信息差怎么变化？——从什么信息差开始、以什么新信息差结束？

卷的弧线的一段 → 章内信息差变化（设→用→揭→新），每次揭露产生新的信息差：

```
卷信息差弧线：主角不知道↦逐步揭露 → 信息逐步向主角靠近
                       ↑
本章位置：主角开始发现矛盾，但反派的优势仍然存在

章内信息差变化：
  开场：主角不知道现场线索意味着什么（读者和主角一样）
  中段：发现矛盾细节→主角开始觉得不对（信息差在缩小）
  结尾：被跟踪→主角知道有人在盯着（新信息差：反派知道↦主角不知道是谁）
```

明确本章出场角色谁知道什么↦谁不知道什么，同时标注变化过程：

```markdown
信息差关系（开场 → 结尾）：
· 开场：主角不知道苏棠去哪了 ↦ 读者和主角一样
· 中段：主角发现矛盾 ↦ 信息差缩小（主角追上了一步）
· 结尾：反派知道在跟踪 ↦ 主角不知道对方身份 ⬅ 新信息差，驱动下一章
```

信息差变化决定了本章的驱动力节奏（悬疑/威胁/目标/关系/信息差驱动）。

**d. 拆场景卡** — 按 §二(从方向到纲要点) 的三段锚点法拆解关键点。每条关键点对应一个场景卡，底层逻辑与卷纲一致：

```
场景卡三要素 → 关键点中的体现
  主角想干啥 → 动作锚点（角色具体做什么）
  有什么拦着他 → 判断锚点（角色遇到什么阻碍/发现什么矛盾）
  有什么悬念让读者想看下去 → 全文的情绪钩子+信息差
```

拆出 8-12 条 key_points，每条用三段锚点法（感官+动作+判断）写 2-3 句。作者确认后进入下一步。

**e. 禁止清单** — 按 §(不做什么——硬约束红线) 询问并记录 prohibitions。

**f. Chapter Memo 填充** — 按 §三(章纲内容清单) 的 7 段逐段填充（情绪核心/信息差/冲突阶梯已在 a-c 完成，此处填充其余字段）：
1. current_task（当前任务）
2. reader_expectation（读者期待）
3. payoff_plan（兑现计划）
4. downtime_functions（过渡功能）
5. key_choices（关键抉择）
6. knowledge_state（角色信息状态——已在 c 完成信息差关系，这里补充各角色知道/不知道清单）
7. required_changes（章尾改变——与 b 冲突阶梯的"结束时什么变了"一致）

填完后展示完整 memo 供作者确认。

**g. Emotional Design（完善）** — 按 §四(情绪设计) 填充情绪细节：mood_progression（已在 a 完成）、intensity_peak、intensity_level、emotional_hook、micro_payoffs。

**h. Hooks 操作** — 写入本章的钩子操作（埋/推进/收束）。不读写全局 hooks.md——真相源在各 chapter.md 的 hooks 字段。

**i. 设变通知（可选）** — 规划/校准过程中发现的设定变更需求。用于通知 novel-agent 调度 updater。按需在章纲末尾追加：

```
---
## 设定变更通知

- **目标：** settings/character-setting/{id}.md
- **类型：** 状态更新 / 新角色 / 世界观更新
- **原因：** {为什么需要这个变更}
- **详情：** {具体变更描述}
---
```

只在有变更时追加，无则省略。

## 产出

写入 `chapters/vol-{N}-ch-{M}.md`，status → `outline`
