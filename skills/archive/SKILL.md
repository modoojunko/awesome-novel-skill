# Novel Archive — 归档

> 摘要：去 draft 标记 → 更新角色状态历史 + hooks → 卷完成检测。完整性已在正文验收（3.3a）通过。

## Overview

正文落盘后，更新项目状态——角色状态、情绪弧线、钩子、滑动审视。

**When NOT to use:** 正文未写入 archives/、质量检查未通过、章状态已是 archived。

**Announce at start:** "我来归档第{N}卷第{M}章。"

## 工具契约

| 工具 | 用途 | 限制 |
|------|------|------|
| Read | 读 chapter.md 章纲、角色设定文件、前文 hooks | — |
| Write/Edit | 写 archives/ 定稿、更新 chapter.md status、追加角色状态历史/情绪弧线 | 不改正文内容 |
| Bash | 草稿重命名去 draft 标记（`mv`） | 仅文件重命名操作 |

## 错误恢复

| 失败场景 | 恢复策略 |
|---------|---------|
| 角色文件不存在/格式异常 | 跳过该角色，标记"待人工补"，继续其他角色 |
| 角色状态追加写入冲突 | 在末尾新行追加，不修改已有历史 |
| 去 draft 时同名定稿已存在 | 用 `-v2` 后缀重命名，告知作者 |
| 卷边界检测时部分章未归档 | 正常——列出未归档章节号，仅对已归档章做卷完成判断 |

## 执行前提

归档前 read_file 执行 Step 0 完整性检查表。任意一项不通过 → 报告作者缺少哪项。

## 归档前检查（Step 0）

| 检查项 | 操作 |
|--------|------|
| 正文草稿存在？ | `archives/vol-{N}-ch-{M}-*.draft.md` 存在？缺失 → **STOP**，正文尚未生成 |
| 正文验收已通过？ | 未通过 → **STOP**，返回主流程走正文验收 |
| chapter.md 完整？ | memo + emotional_design 有值？缺失 → 返回主流程补章纲 |

## 归档步骤

1. **保存 AI 原版快照**（如果不存在）：
   - 检查 `.agent/{chapter}-draft-ai.md` 是否存在
   - 不存在 → 从 `archives/vol-{N}-ch-{M}-{slug}.draft.md` 复制一份到 `.agent/`
2. 将草稿文件重命名，去掉 `-draft` 标记：
   `archives/vol-{N}-ch-{M}-{slug}.draft.md` → `archives/vol-{N}-ch-{M}-{slug}.md`
3. 复核 archives/ 中归档文件内容无误
4. 分析角色变化，追加 `状态历史`
5. 更新角色设定文件当前状态字段（位置、能力、关系、世界观、概要）
6. 追记 `情绪弧线`（情绪状态、触发事件、强度、弧线方向、表达方式）
7. 更新角色设定文件中的 hooks 引用（全局 hooks.md 不再维护——真相源在各 chapter.md 的 hooks 字段），运行钩子健康检查
8. 运行情节推进停滞检测（最近 3 章是否有实质性推进）
9. 将 chapter.md status 更新为 `archived`
10. 更新 story.md 分卷规划中的实际章节数（与预估对比）
11. **触发动态记忆回顾**（详见"动态记忆"章节）
12. **检测卷边界**：
    - 读 `chapters/` 目录，筛选当前卷的章节文件（`vol-{N}-ch-*.md`）
    - 检查这些章节的 status 是否全部为 `archived`
    - 未全部完成 → 更新 `.agent/status.md`：`current_phase` 保持 `chapter-loop`，`current_chapter` 前移
    - 全部完成 → **更新 `.agent/status.md`：** `last_volume_completed` → `true`，`current_phase` → `review`（等待选择下一步）。然后输出卷完成报告：

      ```
      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        卷 {N} 《{title}》全部 {M} 章已完成
      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

      已归档章节：
      {逐章列出标题和归档状态}

      下一步选项：
      1. 规划卷 {N+1} — 进入 `novel-volume`，AI 推理方向选项
      2. 回顾整卷 — 触发 novel-review 做整卷回顾评审
      3. 修改某章 — 指定章节重新生成

      卷 {N+1} 的新故事从当前角色状态自然生长。建议先走选项 1——让 AI 推理方向选项——基于当前角色状态推导下一卷。
      ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
      ```

## 状态历史格式

追加到角色设定文件末尾：

```markdown
### 状态历史

#### 第 1 卷第 1 章
- **位置**：角色当前位置
- **状态**：角色当前状态简述
- **变化**：
  - 具体变化 1
```

## 情绪弧线格式

跟在状态历史后面：

```markdown
### 情绪弧线

#### 第 1 卷第 1 章
- **情绪状态**：愤怒/压抑/释然/期待/恐惧/温情/决心
- **触发事件**：触发事件描述
- **强度**：7/10
- **弧线方向**：上升/回落/持平/转折
- **表达方式**：身体反应/行为/对话/环境互动
```

## 动态记忆（Dynamic Memory）

归档完成后触发动态记忆流程，记录作家修改偏好，用于后续提示词生成。

### Step 1: 执行 Diff 对比

如果 `.agent/{chapter}-draft-ai.md` 存在，执行对比：

1. 读取 AI 原版：`.agent/{chapter}-draft-ai.md`
2. 读取当前草稿：注意此时草稿已被重命名为 `.md`（去掉 `-draft`）
3. 提取差异段落，写入 `.agent/{chapter}-draft-diff.md`

**Diff 输出格式：**
```markdown
# {chapter} Diff

## 差异 1
### AI 原文
[被替换的 AI 写法]

### 作家修改
[作家修改后的写法]

### 场景
[这段在什么情况下出现，如：紧张对话、内心独白、场景描写]

## 差异 2
...
```

### Step 2: 追问作家确认

向作家展示 diff 结果，追问：

```
这章你修改了几处。是否记录以下模式？

1. [段1摘要] — 保留/跳过
2. [段2摘要] — 保留/跳过

或者直接说"这段满意/不满意"。
```

**作家回应处理：**
- 作家说"保留" → 记录该条到 `.memory/`
- 作家说"跳过" → 不记录
- 作家说"记一下这个" → 记录上下文
- 作家说"不满意" → 记录为负面模式

### Step 3: 记录模式

根据作家确认，将有效模式写入 `.memory/anti-ai.md` 或 `.memory/writer-style.md`：

**`.memory/anti-ai.md` 格式：**
```markdown
## 情绪描写类

- 场景：紧张/恐惧描写
- ❌ AI原文：[心跳加速，感到恐惧]
- ✅ 作家改法：[他站在原地，喉咙像被什么东西堵住了]
- 原因：AI喜欢用抽象情绪词，作家偏好用身体反应暗示情绪
```

**`.memory/writer-style.md` 格式：**
```markdown
## 对话风格

- 场景：人物对话
- 作家偏好：动作和对话自然交织，少用情绪标签
- 示例：直接引语，不用"抬起头，惊讶地说道"等标签式引导
```

### Step 4: 清理临时文件

记录完成后，删除临时文件：

- `.agent/{chapter}-draft-ai.md` — 删除
- `.agent/{chapter}-draft-diff.md` — 删除

### Step 5: 贡献社区（可选）

如果作家说"贡献这个模式"：

1. 读取 `.memory/anti-ai.md` 或 `.memory/writer-style.md`
2. 提取适合社区的条目，生成 community-ready 格式
3. 引导作家提 PR 到 `references/anti-ai/{genre}/defaults.md` 或 `references/writer-style/{genre}/defaults.md`

### 作家主动记录

作家可在任何时候说"记一下这个"，Skill 立即记录上下文到 `.memory/`。

## 下一步

**状态汇报 + 自动路由：**
- ✅ 本章归档：`chapters/vol-{N}-ch-{M}.md` → `status: archived`
- ✅ 角色状态已更新、情绪弧线已追记
- 📁 定稿：`archives/vol-{N}-ch-{M}-{slug}.md`
- → 主流程检测到 `status=archived`，**分发到下一章章纲**（`skills/chapter/SKILL.md`）或询问"下一章继续？"

归档完成后告知作者，回到主流程由主 SKILL.md 分发下一步。
