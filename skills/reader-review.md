# reader skill


## 流程概览

```text
Step 1: 确定评审目标 → Step 2: 收集全部上下文（10+ 文件）→ Step 3: 10 维 60+ 细项诊断 → Step 4: 输出报告
```

## 进入门禁

| 检查项 | 操作 |
|--------|------|
| `archives/vol-{N}-ch-{M}-*.md` 存在？ | 不存在 → **STOP**，正文尚未生成 |
| `chapters/vol-{N}-ch-{M}.md` 存在且 outline 完整？ | 不存在/不完整 → **STOP**，无评审依据 |
| `volumes/volume-{N}.md` 存在？ | 不存在 → 标注"无卷纲，跳过卷级评审" |
| `settings/writing-style.md` 存在？ | 不存在 → 标注"无风格指南，跳过风格评审" |
| `settings/world-setting.md` 存在？ | 不存在 → 标注"无世界设定，跳过世界观评审" |

## Step 1: 确定评审目标

若作者指定了卷号和章号 → 评审指定章节。若作者只说"评审这章" → 读取 `story.md` 找到 `current_volume` 和 `current_chapter`。

## Step 2: 收集全部上下文

| 文件 | 评审用途 |
|------|---------|
| `archives/vol-{N}-ch-{M}-*.md` | 评审对象——章正文全文 |
| `chapters/vol-{N}-ch-{M}.md` | 章纲——outline、memo、emotional_design、micro_payoffs |
| `volumes/volume-{N}.md` | 卷纲——核心冲突、章节摘要、主线走向 |
| `settings/world-setting.md` | 世界设定 |
| `settings/character-setting/*.md` | 角色设定 |
| `settings/writing-style.md` | 写作风格 |
| `settings/hooks.md`（若存在） | 伏笔/钩子 |
| 上一章正文结尾 200 字 | 前文延续 |
| `prompts/vol-{N}-ch-{M}-prompt.md`（若存在） | 写作指令对照 |

若有上下文文件缺失，标注"该维度无对照依据"，不阻断评审流程。

## Step 3: 10 维评审

### 维度 1：章纲执行度（对照 chapters/vol-{N}-ch-{M}.md）

逐条检查章正文是否兑现了章纲中的每一项规划。

| # | 细项 | 评审方法 |
|---|------|---------|
| 1.1 | 剧情概要执行 | 正文是否完整执行了 summary 描述的剧情？ |
| 1.2 | 关键情节点覆盖 | 逐条检查每个 key_point 在正文中是否有对应段落 |
| 1.3 | 出场角色兑现 | 列出的角色是否全部出场？ |
| 1.4 | 地点设定遵守 | 场景地点是否与 location 一致？ |
| 1.5 | 时间设定遵守 | 时间跨度是否与 time 一致？ |
| 1.6 | 叙事视角遵守 | 是否严格遵循指定视角？视角跳跃→摘出违规句 |
| 1.7 | 当前任务完成 | 本章必须完成的动作是否达成？ |
| 1.8 | 读者期待管理 | 是否按策略管理了读者期待？ |
| 1.9 | 伏笔兑现执行 | 必须兑现伏笔是否收束？ |
| 1.10 | 伏笔压制执行 | 必须压住的伏笔是否意外泄露？ |
| 1.11 | 伏笔推进执行 | 需推进伏笔是否有推进段落？ |
| 1.12 | 过渡段功能映射 | 过渡段落是否完成规划功能？ |
| 1.13 | 关键抉择检验 | 角色重要选择是否三连问通过？ |
| 1.14 | 章尾改变兑现 | 章尾改变是否实际发生？ |
| 1.15 | 禁止清单遵守 | 是否触犯 prohibitions？ |
| 1.16 | 主情绪达成 | 正文情绪基调是否与 primary_mood 一致？ |
| 1.17 | 情绪走向执行 | 情绪变化轨迹是否符合 mood_progression？ |
| 1.18 | 情绪峰值场景 | 峰值是否出现在规划位置？ |
| 1.19 | 爽点/情绪回报 | 小爽点是否兑现？ |
| 1.20 | 章尾情绪钩子 | 章尾是否留下对应类型的情绪钩子？ |
| 1.21 | 情绪强度匹配 | 是否接近规划的 1-10 级别？ |
| 1.22 | 微兑现交付 | 每个 micro_payoff 是否兑现？ |

### 维度 10：场景技法执行度（对照 prompt L8）

**检查方式：** 读 `prompts/vol-{N}-ch-{M}-prompt.md` 的 L8 场景写作指引段，提取本次实际注入了哪些方法论。**只检查已注入的，未注入的不检查。**

#### 若 L8 包含对话技法（[身份]/[潜台词]/[拉扯]/[动作]）

- [ ] 10.1 对话动作穿插——连续纯对话是否超过 6 句无动作/神态间隔
- [ ] 10.2 角色说话区分度——遮住角色名能否分辨谁在说话

#### 若 L8 包含战斗技法（[动词]/[反馈]/[感官]/[节奏]/[博弈]）

- [ ] 10.3 战斗反馈替代形容词——是否有"威力巨大、速度极快"等空词未配环境反馈
- [ ] 10.4 战斗动词精度——是否出现"打、踢、捶"等通用词（凡人级）或"砍、刺、挥"（仙侠级）

#### 若 L8 包含外貌描写（[锚点]/[动态]/[故事]）

- [ ] 10.5 外貌锚点执行——角色首次登场是否有核心视觉锚点，非流水账五官堆砌

#### 若 L8 包含心理技法（[生理]/[碎片]/[崩塌]）

- [ ] 10.6 心理不直写情绪词——是否出现"害怕、恐惧、愤怒"等直接情绪标签
- [ ] 10.7 心理生理通感——情绪表达是否靠生理反应（寒意/耳鸣/指尖抖）而非情绪词

#### 始终检查（prose + pov 始终加载）

- [ ] 10.8 文笔抽象具体化——是否有"很贵的、名牌、精致的"等模糊词未替换
- [ ] 10.9 文笔感官叠加——关键场景是否只用了视觉一种感官
- [ ] 10.10 视角一致性——同一自然段内是否出现两个以上人物视角
- [ ] 10.11 视角转场自然度——视角切换是否有物理契机（视线/声音/动作）

### 维度 11：首章留存力（仅第一卷第一章触发）

**触发条件：** `volume-1` 且 `chapter-1`。非首章自动跳过。

**检查方式：** 读 `prompts/vol-1-ch-1-prompt.md` 的 L8 确认用了哪种开篇钩子，再对照正文检查。

- [ ] 11.1 **前 500 字有钩子**——第一段直接抛出了冲突/金手指/反转之一，没有平淡开场
- [ ] 11.2 **无设定说明书**——前 500 字没有世界观介绍、修炼等级、家族背景、人物生平
- [ ] 11.3 **无静态写景**——前 500 字没有无冲突的风景/环境描写作为开场
- [ ] 11.4 **钩子类型可识别**——能明确指出本章用了哪种钩子（悬念留白/极度反差/矛盾前置/颠覆设定/极致情绪）
- [ ] 11.5 **第一段有人有事**——第一段就出现了具体人物和具体行动，不是环境铺垫
- [ ] 11.6 **投石入水模型**——开篇抛出了"石头"（冲突/金手指/反转），且激起了"水花"（对立画面/猎奇反差/情绪冲击）

### AI 味手检（补充信息，不阻断评审）

| # | 检查项 |
|---|--------|
| AI.1 | 「了」字病——每段不超过 2 个 |
| AI.2 | 感知词模板——「感到/看到/听到/闻到/想到」超过 3 处 |
| AI.3 | 「不是……而是……」句式——全文超过 1 处 |
| AI.4 | 身体部位+情绪模板——「心头一紧」「嘴角上扬」等超过 2 处 |
| AI.5 | 副词密度——每 100 字超过 3 个「地」 |
| AI.6 | 对话占比——低于 20% |
| AI.7 | 破折号违规——全文出现任何破折号（对话打断除外） |

## Step 4: 输出报告格式

```markdown
## 第{N}卷第{M}章评审报告：《{title}》

**总评分：X.X/10**（维度 1 权重 2.0，其余权重 1.0）

### 维度 1：章纲执行度（X/10）
| # | 细项 | 结果 | 说明 |
|---|------|------|------|
| 1.1 | 剧情概要执行 | ✅/⚠️/❌ | 具体说明 |

（维度 2-9 同上格式）

### 关键问题（必须修复）
每条格式：[维度.细项编号] 问题简述 + 原文引用 + 修改方向

### 改进建议
每条格式：[维度.细项编号] 建议简述 + 原文引用 + 优化方向

### 亮点
每条格式：[维度.细项编号] 亮点描述 + 佐证

### 评审统计
适用细项总数 | ✅ 通过 | ⚠️ 基本通过 | ❌ 未通过 | N/A 不适用 | 通过率
```

### 评分标准

| 评级 | 含义 |
|------|------|
| ✅ | 正文完全符合该细项要求 |
| ⚠️ | 大部分符合，有轻微偏差 |
| ❌ | 明显不符合 |
| N/A | 因上下文缺失无法评估 |

**维度评分：** 按通过率计算 1-10 分。
**总评分：** 10 个维度评分的加权平均（维度 1 权重 2.0，其余权重 1.0）。

## 作者决策

展示报告后等待作者选择：
- "按关键问题修改" → 主 agent 直接编辑 archives/ 正文
- "部分修改" → 指定要改的条目编号，逐条修改
- "已知悉，继续归档" → 跳过修改，进入归档
- "重新生成" → 退回写作流程，将关键问题作为补充约束注入
