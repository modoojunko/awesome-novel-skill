# prompt-crafter skill


## 流程概览

```text
Step 1: 读取输入源（6 类文件）
Step 2: 按 9 层骨架填充（L1-L9）
Step 3: 冲突检测
Step 4: 验收自检
```

## Step 1: 读取输入源

1. **writing-style.md** → 提取四字段（core_principles, possible_mistakes, depiction_techniques）+ genre + model
2. **volume.md** → 提取前一章的章名、结尾画面、情绪落点
3. **chapter.md** → 提取 memo, emotional_design, outline（场景分解）, payoff_plan
4. **角色设定文件** → 读取涉及的角色，提取本章相关状态
5. **genre-example** → 取对应题材的提示词注入段（L8 用）
6. **anti-ai 库** → common-rules.md + 题材特定规则

**注意：L2 来龙只读卷纲中的前章摘要，不读上一章全文。**

## Step 2: 按 9 层骨架填充

### L1 元信息

从 writing-style.md + chapter.md + volume.md 提取：
- 题材、写作风格、字数目标、本章角色、建议模型
- **本章驱动力**（从卷纲主导驱动力继承）、信息释放节奏
- 字数目标和叙事视角必须明确

### L2 来龙

从 volume.md 前章摘要提取（只读摘要，不读全文）：
- 上章结尾画面、情绪残留、读者信息缺口

### L3 去脉

从 chapter.md memo 提取：
- 本章核心悬念、悬念状态、读者感受预期

### L4 角色弧光

从 chapter.md 人物状态 + 角色文件提取：
- 每角色起点→转折→落点 + 微习惯

### L5 场景序列

从 chapter.md outline + 信息差关系提取：
- 按边界信号（换地点/跳时间/情绪转折）切分场景（2-4 个）
- 每场景填四字段：入口画面 / 核心事件 / **信息差** / 出口画面 + 情绪拐点
- 信息差从章纲 knowledge_state 的"信息差关系"中取本场景对应的那对关系

**场景类型识别：** 分析每个场景的核心事件，识别场景类型（对话/战斗/环境/心理/群像/过渡）。记录为列表，用于下一步加载场景方法论。

### 场景方法论加载（L5→L6 之间）

根据 L5 识别的场景类型列表，动态加载对应的场景写作方法论。加载后的方法论**必须经过上下文过滤再注入 L8**，不能生搬硬套。

**加载机制：**

1. 从 L5 场景类型列表中**去重**，得到本章涉及的不重复场景类型（如 ["dialogue", "environment"]）
2. 对每个类型，按以下顺序读取：
   a. 通用方法论：`knowledge/scene-craft/{类型}/universal.md`（必须存在）
   b. 题材特化：`knowledge/scene-craft/{类型}/{当前题材}.md`（存在则读，否则跳过）
3. 通用方法论 + 题材特化合并为该场景类型的方法论来源
4. 如果某个类型的目录不存在（如 `inner-mono/` 暂为空框架），跳过该类型

**上下文过滤（关键步骤）：**

读取方法论后，不能直接复制到 L8。必须经过 **四步转化法**，把通用方法论变成具体场景的写法指引。

---

### 四步转化法：通用方法论 → 场景写法

以 `dialogue/universal.md` 中的"动作穿插"方法论为例，演示四步转化流程：

```
收到的方法论原文：
  "对话需要动作穿插——纯对话超过 6-8 句，读者就丢了画面感"

角色设定中读到的（当前章节涉及的角色）：
  方岩：性格谨慎，回避冲突，紧张时低头喝茶/目光移开
  陆征：前刑警，直接，追问时身体前倾/用手指敲桌面

信息差中读到的：
  方岩知道三年前压案的内幕 ↦ 陆征不知道

本章情绪节奏：
  紧张段（陆征在追问）
```

#### Step 1：锚定角色

不把方法论套在"角色A""角色B"上，套在具体的人上。

> **问：这个角色是什么性格？TA 在这个场景里是什么状态？**

- 谨慎的人回避方式 vs 直接的人追问方式
- 傲慢的人不耐烦的方式 vs 自卑的人躲闪的方式
- 同一场景、同一方法论，换一个角色写出来完全不一样

#### Step 2：锚定信息差

确定角色之间的**信息不对称**——这决定了"不说真话"的方向和方式。

> **问：谁在瞒？瞒什么？为什么瞒？**

| 信息差类型 | 对对话写法的影响 |
|-----------|----------------|
| 方岩知道↦陆征不知道 | 方岩在回避、在防守；陆征在试探、在逼近 |
| 陆征知道↦方岩不知道 | 陆征在引导、在布局；方岩在被动回应 |
| 读者知道↦角色不知道 | 读者焦急等角色拆穿，对话中埋"观众看得见角色看不见"的伏笔 |

#### Step 3：锚定情绪与节奏

方法论的具体写法要匹配本章情绪指向。

> **问：这段场景的情绪是紧张、舒缓还是转折？**

| 情绪段 | 对话节奏 | 动作穿插频率 | 句子长度 |
|--------|---------|-------------|---------|
| 紧张/对峙 | 快、互相打断 | 少（动作干扰紧张感） | 短句碎片 |
| 压抑/隐瞒 | 慢、长停顿 | 多（用动作代替回答） | 话说到一半 |
| 舒缓/和解 | 自然、有停顿 | 中等（间歇性动作） | 完整句子 |

#### Step 4：融合输出

把前三步的结论合起来，写成一条**可直接写入 L8 的场景写法指引**。

```
❌ 错误案例（通用方法论，没有转化）：
  "对话要有动作穿插，每 6-8 句插一个动作。"
  → 所有章节的对话都按一个模板写，角色没有区分。

✅ 正确案例（经四步转化后）：
  "这场对话的信息差是方岩知道内幕↦陆征不知道。
  方岩性格谨慎，回避时会低头喝茶——陆征问一句，他喝一口，不接话。
  陆征前刑警的追问方式是身体前倾、缩短距离——每问一句就往前挪一点。
  情绪是紧张段，对话节奏要快——陆征短句追问，方岩用动作回避，不用长句解释。"
  → 方法论落地到具体角色、具体信息差、具体情绪，每个角色写法不同。
```

---

**四步转化法流程图：**

```
场景-craft 方法论
     ↓
Step 1：锚定角色（从角色设定中提取性格、微习惯）
     ↓
Step 2：锚定信息差（从 knowledge_state 中提取谁在瞒谁）
     ↓
Step 3：锚定情绪节奏（从 emotional_design 中提取当前情绪段）
     ↓
Step 4：融合输出 → 写入 L8
```

**底线：**
- ❌ 每条方法论生搬硬套，所有角色的对话写法都一样
- ✅ 经过 Step 1-4 转化，同一方法论在不同角色身上表现不同
- ❌ 整段复制 scene-craft 文件的内容
- ✅ 从方法论中选取 2-4 条最相关的，经四步转化后精炼输出
- ❌ 跳过转化直接写"角色A说...角色B说..."的通用指引
- ✅ 转化后的指引能让 writer 直接写出符合角色性格的对话

> 如果本章只有对话场景，从 `dialogue/universal.md` + `dialogue/{题材}.md`（如果存在）选 2-4 条方法论，每条经四步转化后注入 L8。
> 如果同时有对话和战斗，两个场景目录各选 2-3 条方法论（含题材特化），分别结合各自场景的角色和冲突转化后注入。
> 如果某条方法论对应的角色没有参与当前场景，跳过该条。

### 转化示例库（few-shot）

以下展示两条方法论从"通用 → 四步转化 → L8 注入"的完整过程。你的转化输出应该达到同样的具体度和上下文感知度。

---

#### 示例一：对话场景 · 方岩与陆征（悬疑 · 隐瞒与试探）

```
【场景-craft 方法论原文】
"角色不说真话。表层话和深层意之间的差距就是对话的张力。"

【当前上下文】
角色：
  方岩：警队内部人员，知道三年前压案内幕，性格谨慎，回避冲突
  陆征：前刑警，正在调查此案，直接、敏锐，追问时身体前倾
信息差：
  方岩知道内幕 ↦ 陆征不知道（方岩在瞒、陆征在追）
情绪节奏：
  紧张段——陆征在施压，方岩在防守

【四步转化过程】
Step 1 锚定角色：
  方岩谨慎→回避方式：低头喝茶、看窗外、不接话
  陆征直接→追问方式：身体前倾、缩短距离、盯着对方
Step 2 锚定信息差：
  方岩知道但不能说→他在防守，用"这个案子我不清楚"推脱
  陆征不知道但想确认→他在试探，用具体细节逼对方反应
Step 3 锚定情绪节奏：
  紧张段→短句、快节奏，动作穿插少（但关键时刻的动作要放大）
Step 4 融合输出：
  方岩知道三年前的事但不能说（信息差），陆征在追问。
  陆征身体前倾，短句逼问；方岩低头喝茶不接话，用回避动作代替回答。
  紧张段节奏快——没有长句解释，只有逼问和沉默交替。

【注入 L8 的最终文本】
"信息差：方岩知道内幕↦陆征不知道。陆征短句逼问，身体前倾；
方岩回避，低头喝茶、不接话。紧张段快节奏，没有长句解释，
只有逼问和沉默交替——方岩的回避不是沉默不语，是每次陆征问到关键
他就用动作岔开（喝茶/看窗外/反问'你查到哪了'）。"
```

#### 示例二：战斗场景 · 叶秋与王虎（修仙 · 实力压制）

```
【场景-craft 方法论原文】
"结果 > 过程。读者关心的是谁赢了、代价是什么，不是每一招的细节。
代价可视化——不写'主角受伤了'，写伤口的后果。"

【当前上下文】
角色：
  叶秋：练气三层，刚突破不久，还不熟悉新获得的力量
  王虎：练气二层，混混头目，仗着修为欺负人
信息差：
  叶秋刚突破↦王虎不知道（王虎以为叶秋还是上次见面的修为）
情绪节奏：
  爽点段——读者期待叶秋展示实力

【四步转化过程】
Step 1 锚定角色：
  叶秋：刚突破还不适应新力量→出手会有"没收住"的生涩感
  王虎：以为对方还是弱鸡→一开始轻敌，吃亏后才认真
Step 2 锚定信息差：
  王虎不知道叶秋突破了→他的轻敌是信息差造成的，不是故意
  叶秋知道自己变强了→但他也不确定到底强了多少，有试探成分
Step 3 锚定情绪节奏：
  爽点段→节奏先快（王虎被碾压的震惊）后慢（展示后果）
Step 4 融合输出：
  不写一招一式，聚焦"结果→王虎的反应→代价可视化"。
  用王虎的视角写——他还没反应过来已经飞出去了，这才看到叶秋的
  境界已经不一样了。

【注入 L8 的最终文本】
"这场战斗是信息差驱动的——王虎不知道叶秋已经突破了。
不写招数细节，用结果+反应代替：王虎还没看清怎么回事就已经飞出去了。
代价可视化：叶秋没收住力，拳头砸穿墙壁，虎口裂开渗血。
节奏：快（王虎飞出去）→慢（他抬头看叶秋，发现境界变了）
→余韵（叶秋看自己的手，不太确定自己现在多强）。"
```

### L6 约束

从 chapter.md memo 提取：
- **本章在卷冲突阶梯中的位置**（第1层入门/第2层升级/第3层转折/第4层终极）
- 情节红线（2-4 条）+ 边界禁止 + 角色禁区

### L7 爽点设计

从 chapter.md memo 提取：
- 类型、铺垫位置、释放位置/方式、克制点

### L8 文字规则

组装以下内容：
- 视角：从 genre-example 提取
- 描写要求：描写手法/节奏/禁止项
- 疲劳词阈值：如"突然≤3/章"
- 句式规则：如"连续4句主语不得相同"
- 元叙事禁止：如"禁止'读者'"
- 题材正反例：从题材特定 anti-ai 复制
- **场景写作指引**（从上一步"场景方法论加载"的结果中选取）：本章涉及 {N} 种场景类型（{类型列表}）。针对每种类型，注入 2-4 条具体写法指引。通用方法论来自 `scene-craft/{类型}/universal.md`，题材特化来自 `scene-craft/{类型}/{题材}.md`（存在时覆盖通用）
- **必须填入具体内容，不只是标注来源**

### L9 质感

从 chapter.md memo + 作者补充提取：
- 无用细节、对话节奏、真人痕迹

## Step 3: 冲突检测

填完后检查：
- L2 情绪残留 = L4 第一角色起点（一致）
- L4 落点 = L5 拐点（一致）
- L5 信息差与整体驱动力一致（悬疑驱动→每场景有信息缺口）
- L6 冲突阶梯层位与 L1 驱动力匹配（威胁驱动→层位不应是入门级）
- L7 释放位置在 L5 场景中存在
- **L8 场景指引与 L5 场景类型匹配**——L8 注入的指引只覆盖了 L5 中出现的场景类型，没有多注入也没有漏注入
- L9 质感字段已填充（非空）

全部填完后全局通读一次——掩住字段名，看每个种子读起来是叙事画面还是写作指令。

## Step 4: 验收自检

逐条检查 prompt.md：

| 检查项 | 标准 |
|--------|------|
| 9 层完整 | L1-L9 全部存在，不缺层 |
| 字段填充完整 | 每层字段有值，无 `______` 占位符残留 |
| writing-style 注入 | role/core_principles/possible_mistakes/depiction_techniques 四字段全部注入 |
| writing-style 完整 | 四个字段全部注入（非遗漏） |
| L2 来龙画面感 | 有具体结尾画面描述 |
| L5 场景种子叙事性 | 种子读起来是画面而非写作指令 |
| L6 约束双侧闭合 | 红线+禁区两者都有 |
| L7 爽点释放具体 | 释放位置和方式具体可落地 |
| L8 来源标注 | 反 AI 规则标注来源 |
| L8 场景指引匹配 | 注入的场景类型与 L5 场景类型一致，不多不漏 |
| L9 质感字段 | 有真实可执行的质感要求 |
| 无 meta 泄漏 | 不包含"以下是小说的正文"类自我引用短语 |
| 无整段复制章纲 | 各层内容是提炼后注入，非原文复制 |
| 无自由指令 | 不添加 9 层骨架之外的额外指令 |

**全部通过后**，写入 `prompts/vol-{N}-ch-{M}-prompt.md`。
不通过则回到 Step 2 修正对应层级。
