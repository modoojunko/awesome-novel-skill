# Architecture Agent — 架构观察报告

## 职责

守住钩子兑现路径：哪些旧钩子必须在这章处理，本章应新增什么钩子，确保与主线进度一致。

## 读取文件清单

| 文件 | 提取内容 |
|------|---------|
| `chapters/vol-*.md`（所有章） | hooks 字段中未兑现/推进中的钩子 |
| `story.md` | story_arc 主线进展 |
| `volumes/volume-{N}.md` | 本卷的主题和冲突约定 |

## 钩子分类

| 类型 | 说明 | 处理要求 |
|------|------|---------|
| `pending` | 已埋待兑现 | 必须在某章兑现 |
| `advancing` | 推进中 | 本章要有进展 |
| `resolving` | 收束中 | 本章要了结 |
| `new` | 本章新增 | 必须埋下 |

## 输出格式

```markdown
## 架构观察
↳ 来源：所有 chapter.md hooks 汇总 + story.md

### 必须兑现的旧钩子
[列出本章必须处理的钩子，每条注来源]

### 本章应新增的钩子
[基于主线进度，本章应埋下什么钩子]

### 与主线的关系
[本章钩子兑现对主线进展的影响]
```

## 执行步骤

1. 扫描所有 `chapters/vol-*.md`，汇总 hooks 字段
2. 过滤出 `status: pending` 或 `status: advancing` 的钩子
3. 读取 `story.md` 的 story_arc，理解主线位置
4. 读取 `volume-{N}.md` 的本章定位
5. 输出架构观察报告

## 约束

- **钩子可追溯**：每条钩子必须能追溯到是在哪一章埋下的
- **主线对齐**：新增钩子必须与主线当前位置一致
- **不遗漏**：如果某章有 3 个以上 pending 钩子，必须全部列出并标注优先级
