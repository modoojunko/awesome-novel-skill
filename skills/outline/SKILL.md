# Novel Outline — 主线拆纲 + 卷纲规划

> 流程：主线拆纲（仅首次）→ 卷方向确定（角色发声 / 结构模板）→ 卷纲讨论 → 生成卷提示词。逐章章纲已移至 `skills/chapter-loop/SKILL.md`。

## Overview

主线拆纲 + 引导作者规划卷纲。先判断作者类型，再走对应路径：

- **类型 A（知道全书走向）：** 先拆全书主线（总主线→定终点→找断点→定每卷冲突），再逐卷展开
- **类型 B（只知道开头）：** 只定第一卷方向 → 后续卷标"待定" → 写完后角色发声自然推导下一卷

两类路径的详细对话脚本见 `references/story-arc-style.md#先判断作者类型`。

**对规划能力弱的作者：** Agent 直接提案故事结构模板作为骨架。模板不是约束——是起点。**Agent 主动提案，不问"你想要什么结构"。**

**When NOT to use:** 设定未完成（world-setting 为空）、只是想查看已有卷纲。

**Announce at start:** "我来引导你规划故事主线。先定全书框架，再展开到每一卷。"

## 执行前提

进入本流程前，确认 world-setting.md、各角色 yaml 已填写。首次进入还需走主线拆纲（步骤 0）。
逐章章纲和提示词由 `skills/chapter-loop/SKILL.md` 处理。

## 流程

### 0. 主线拆纲（仅首次进 Phase 2 时执行）

先判断 `story.md` 是否已有 `story_arc` 定义：
- **有** → 跳过此步，进入步骤 1 卷方向确定
- **无** → 开始主线拆纲讨论

为什么先拆主线再逐卷规划？没有全书框架就规划第一卷，容易写出"第一卷完了不知道第二卷该写什么"的断裂故事。主线拆纲让全书有骨架，每卷知道自己在整体中的位置。

**详细指南和验收标准在 `references/story-arc-style.md`**。以下为流程概览，具体对话脚本与条件分支请读指南文件执行。

#### 0.0 先判断作者类型

进入拆纲，先问一句话判断作者类型：

> "你目前对全书的故事走向有完整概念吗——从开头到结局都想好了？还是只有第一卷的思路，后面边走边看？"

- **类型 A：都清楚** → 走 0.1→0.4 全流程
- **类型 B：只知道开头** → 跳过 0.2 定终点和 0.3 找断点；0.1 总主线窄化为"第一卷方向"；0.4 只定第一卷冲突，后续卷标"待定"
- 详见 `references/story-arc-style.md#先判断作者类型`

#### 0.1 总主线

- **类型 A：** 作者一句话说清全书核心冲突。**不做完所有卷答不出来的那种。**
- **类型 B：** 只聊第一卷方向——"第一卷大概讲什么？主角要做什么、遇到什么？"，不需要全书主线

不要问"你的全书核心冲突是什么？"——作者说不出来。**先听故事，再提取结构。**

> "设定都完成了，现在来规划整个故事的结构。用你的话说说，这个故事从头到尾讲的是什么？主角最想达成什么？"

从作者的回答中提取三要素：**谁 + 追求什么 + 对抗什么**。浓缩为一句话让作者确认。详细对话脚本（含 5 种作者回答的接法 + 特殊场景处理）见 `references/story-arc-style.md#总主线`。

#### 0.2 定终点：从结局倒推

> **类型 B 跳过此步。** 终点标"待定"，断点不找，直入 0.4 定每卷冲突。

不要问"几卷"，先问终点。

> "你想象的故事最后一幕是什么画面？主角最终站在什么位置？"

根据作者回答类型（有画面 / 有调性没画面 / 完全不知道）走不同分支，详见 `references/story-arc-style.md#从结局倒推`。

终点确定后反向验证与总主线是否一致——如果主线说"揭露真相"但终点说"主角不公开"，这恰恰是故事张力的来源。

#### 0.3 找自然断点

> **类型 B 跳过此步。**

从起点到终点之间，故事会经历几个大的阶段。每个断点应该是**某个根本性的变化**：

- 主角知道了什么关键信息（信息变化）
- 主角和某方的关系破裂/建立（关系变化）
- 主角的实力/地位跃升（状态变化）
- 发生了无法挽回的事件（事件变化）

> "从开头到结局，中间会经历几个大的转折？比如'主角第一次发现真相→遭遇重大失败→反击→最终对决'这样的阶段。"

根据作者回答处理（说得出 2-4 阶段 / 说"一路查到底" / 列了太多 / 完全没概念），详见 `references/story-arc-style.md#找自然断点`。Agent 可根据题材+总主线推理提案让作者确认。

#### 0.4 定每卷冲突

**类型 A：** 每个断点之间就是一卷。**一卷一卷确认**，不要一次列完。
**类型 B：** 只定第一卷。跳过断点，直接聊"第一卷大概几章？核心冲突是什么？"。

> "第一卷《失踪》：从接案到发现旧案被压，核心是追查苏棠失踪案但遭遇阻力。大约 10 章。对吗？"

| 字段 | 填写标准 |
|------|---------|
| 标题 | 2-4 字，能概括本卷核心事件 |
| 核心冲突 | 谁 + 做什么 + 被什么阻碍 |
| 预估章节 | 允许 ±30% 浮动 |

确认后写入 `story.md`。

**类型 A：** 一卷确认完再聊下一卷，允许后卷模糊——后卷至少写一句话核心冲突 + 浮动章数，写到那卷时再细化。
**类型 B：** 只写第一卷。后续卷全部标 title="待定" + core_conflict="待定" + estimated_chapters="?"。回到 Phase 2 时角色发声推导后续卷方向，替换"待定"。

**弧线位置和结构类型是最后归纳的标签**：所有卷定完后，回头看每卷在全篇中的位置标注弧线功能（"起—引入谜团"），结构类型是归纳标签（"你这三卷正好是三卷式"），不是出发点。

### 1. 卷方向确定（先于卷纲讨论）

先判断是本卷是不是首卷：

#### 1.1 卷 N+1：让角色说话

有前卷存档（`archives/` 非空）→ **直接从角色驱动分析开始**，不推模板。已存在的角色有自己的状态和意图，他们告诉作者下一卷该写什么。

**操作流程：**
1. 读 `hooks.md` 确认未收束的钩子和主角当前状态
2. 读各活跃角色的最新 `状态历史`——上一卷结尾他们在哪儿、经历了什么、掌握了什么信息
3. 对每个活跃角色（含 antagonist/重要配角），生成角色"发声"：

   ```
   【{角色名}的处境】卷末落位——和谁在一起、掌握了什么信息、目前处于什么局面
   【{角色名}未完成的事】上一卷留下的执念/目标/恐惧——什么还没解决
   【{角色名}卷间思考】在卷与卷之间的间隙里，ta 反复在想什么
   【{角色名}现在想做的事】基于当前处境，ta 下一卷最可能推动什么
   ```

4. 展示给作者。**声音越多，卷的方向越自然**——几个角色的意志冲突会自然形成主线
5. 作者根据角色发声确定卷方向

> **为什么卷 N+1 不让模板引导？** 角色已经活了一卷以上，他们的状态比任何结构模板都更精确地指向"接下来该写什么"。模板是骨架，角色状态才是血肉——角色有历史就不用骨架。

**兜底：** 角色发声后作者仍然没方向 → 回到下方模板，辅助找结构。

#### 1.2 卷 1：展示故事结构模板

首次规划卷纲（`archives/` 为空）→ 使用结构模板提供骨架。

读 `settings/writing-style.md` 的 `genre_profile` 字段：

- **有值** → 在 `references/genre-example/` 中查找匹配的 `story_arc_templates`
- **无值** → 使用下方通用故事结构模板

Agent 直接输出提案（**不问"你想要哪种结构"**——默认选第一个 variant，或根据 story.md 的题材描述推测最匹配的 variant），展示格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  卷 1 结构建议：{variant.label}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{variant.description}

建议章节数：{total_chapters_range}

各阶段概览：
1. {phase}（约 {default_chapters} 章）→ {description}
2. {phase}（约 {default_chapters} 章）→ {description}
...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
也可换成：{其他 variant.label} / {其他 variant.label}
可直接调整阶段数量、顺序或章节分配——这是骨架，不是铁律。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

确认用这个结构，还是想换一个？
```

作者可能的选择：
- "就这个" → 用确认的模板作为卷纲讨论骨架。**将各 phase 名称写入 volume.md 的 chapters_summary 作为占位章纲**，后续逐章讨论时填充
- "换 {variant}" → 展示选中的 variant 格式同上，重新确认
- "我自己来" → 跳过步骤 1，直入 ### 2. 卷纲讨论
- "我改一下" → 按作者调整后的结构走

##### 通用故事结构模板（genre_profile 无值）

Agent 根据 story.md 题材信息和世界设定推测最适合的模板，直接展示不询问：

| 模板 | 适合 | 默认分卷 |
|------|------|---------|
| **三幕式**（setup→confrontation→resolution） | 冲突驱动、有明确对手或目标的故事 | setup 3-4章 / confrontation 4-5章 / resolution 2-3章 |
| **起承転結**（起→承→転→結） | 东方叙事结构，适合慢热细腻的故事 | 起 2-3章 / 承 3-4章 / 転 2-3章 / 結 1-2章 |
| **悬疑递进**（疑问→线索→反转→答案→余味） | 解谜向、有信息差的故事 | 疑问 1-2章 / 线索 2-3章 / 反转 2章 / 答案 1章 / 余味 1章 |
| **人物弧线**（日常→转折→挣扎→蜕变→新常态） | 人物驱动、成长向的故事 | 日常 2章 / 转折 2章 / 挣扎 2-3章 / 蜕变 1-2章 / 新常态 1章 |

**输出格式同上**——Agent 直接展示选定的模板提案，由作者确认。

### 2. 卷纲讨论：定卷的故事 → 拆章节 → 章的故事+冲突载体

上一步（方向确定）已经定了"这卷大概往哪走"。这一步把它变成章节级规划。一卷只有一个主线，所有章合起来讲完这个主线。

**验证原则：** 所有章的故事加在一起 = 这卷讲完。少了任何一章，这卷的故事就不完整。这是卷级版的"每卷是总主线的子集"。

1. **定卷的故事** — 从方向确定的结果（模板阶段/角色发声交汇点），写出本卷核心冲突一句话。这一步是确认，不是重新讨论。详细指南见 `references/volume-setting-style.md#二核心冲突`
2. **拆章节** — 以模板各 phase 为骨架（首卷）或角色发声推导的冲突路径（卷 N+1），拆成具体章节数，每章分配一句话方向。详细指南见 `references/volume-setting-style.md#三章节列表`
3. **填章的故事+冲突载体** — 每章写一句话 summary（谁做啥+冲突事件+章末变化）+ 冲突载体（这章结束后什么东西变了）
4. 创建 `volumes/volume-{N}.md`，写入卷标题、核心冲突、chapters_summary

### 3. 生成卷提示词
读取 volume-N.md + story.md → 组装 `prompts/volume-{N}-prompt.md`：
- 本卷核心冲突与主线
- 章节列表（标题 + 概要，未写留空）
- "已完成章节摘要"区（初始为空，归档时追加——见 chapter-loop Step 6）
