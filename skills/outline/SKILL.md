# Novel Outline — 主线拆纲 + 卷纲规划

> 流程：主线拆纲（仅首次）→ 卷方向确定（角色发声 / 结构模板）→ 卷纲讨论 → 生成卷提示词。逐章章纲已移至 `skills/chapter-loop/SKILL.md`。

## Overview

主线拆纲 + 引导作者规划卷纲。先拆全书主线（总冲突→拆几卷→每卷冲突），再逐卷展开——确定卷方向、讨论核心冲突与章节列表、生成卷提示词。卷 N+1 走角色发声，首卷走故事结构模板。

**对规划能力弱的作者：** Agent 直接提案故事结构模板作为骨架。模板不是约束——是起点。**Agent 主动提案，不问"你想要什么结构"。**

**When NOT to use:** 设定未完成（world-setting 为空）、只是想查看已有卷纲。

**Announce at start:** "我来引导你规划故事主线。先定全书框架，再展开到每一卷。"

## 执行前提

进入本流程前，确认 world-setting.yaml、各角色 yaml 已填写。首次进入还需走主线拆纲（步骤 0）。
逐章章纲和提示词由 `skills/chapter-loop/SKILL.md` 处理。

## 流程

### 0. 主线拆纲（仅首次进 Phase 2 时执行）

先判断 `story.md` 是否已有 `story_arc` 定义：
- **有** → 跳过此步，进入步骤 1 卷方向确定
- **无** → 开始主线拆纲讨论

为什么先拆主线再逐卷规划？没有全书框架就规划第一卷，容易写出"第一卷完了不知道第二卷该写什么"的断裂故事。主线拆纲让全书有骨架，每卷知道自己在整体中的位置。

#### 0.1 总主线

作者一句话说清全书核心冲突。标准：**不做完所有卷答不出来的那种**。

> "陆征追查失踪案发现背后横跨二十年的阴谋——每卷揭开一层，最后一卷才看到全貌。"

**判断标准：** 这句话包含了"谁 + 追求什么 + 对抗什么"，且做完一卷答不完、需要多卷才能讲完。

| 作者说 | 怎么接 |
|--------|--------|
| "这就是个复仇故事" | "复仇对象是谁？主角为此要经历几个阶段？——每个阶段可能就是一卷" |
| "我还不知道全貌" | "没关系，先聊聊开头——主角最初的动机是什么？这个故事大概需要几卷来写完？" |
| "就一个案子，一卷搞定" | 那就是单卷结构——跳到 0.3 直接定卷内容 |

#### 0.2 多卷结构选型

根据题材 + 作者讲述欲，推荐最适合的多卷结构：

| 结构 | 卷数 | 特点 | 适合 |
|------|------|------|------|
| **短篇/单卷** | 1 | 起承转合在一卷内完成 | 独立短篇、试水作、命题征文 |
| **三卷式** | 3 | 起(引入)→承(展开)→转合(高潮+结局) | 中篇悬疑、都市、仙侠升级、中等长度故事 |
| **五卷式** | 5 | 起→承1(深化)→承2(转折前奏)→转(高潮)→合(余韵) | 长篇史诗、多势力博弈、深度人物弧线 |
| **多卷连载** | 3+ | 每卷自闭环 + 大主线推进 | 超长篇网文、开放式世界观 |

Agent 直接推荐（**不问"你想要哪种"**——根据题材和对话中感知的故事体量推测最匹配的）：

> "听起来你的故事大概需要 **三卷**——第一卷建立谜团和冲突，第二卷深入对抗、揭示更多层面，第三卷收束真相。你觉得这个体量合适吗？"

**兜底：** "我自己也不确定" → 先从第一卷开始聊，边写边感知整体体量。在 story.md 中将结构类型标记为"待定"。

#### 0.3 逐卷拆解

逐卷讨论核心冲突和章节数，填入 `story.md`：

| 字段 | 填写标准 |
|------|---------|
| 标题 | 卷标题，有信息量 |
| 弧线位置 | 在全篇弧线中的位置（起/承/转/合），标注本卷功能 |
| 核心冲突 | 本卷核心冲突一句话——"谁 + 想做什么 + 被什么阻碍" |
| 预估章节 | 预估章节数，允许浮动 |

确认后写入 `story.md`：

```markdown
### 卷1：失踪

- **弧线位置：** 起—引入谜团
- **核心冲突：** 追查苏棠失踪案，发现三年前旧案被压
- **预估章节：** 10

### 卷2：深水

- **弧线位置：** 承—深入对抗
- **核心冲突：** 陆征触及警队内部保护伞，每一步都有人抢先一步
- **预估章节：** 12

### 卷3：破局

- **弧线位置：** 转合—真相与代价
- **核心冲突：** 与幕后黑手正面交锋，所有线索汇聚到二十年前那件事
- **预估章节：** 8
```

> **完成主线拆纲后，标记 story.md story_arc 为已定义，后续每次进 Phase 2 跳过步骤 0，直接从步骤 1 开始。**

### 1. 卷方向确定（先于卷纲讨论）

先判断是本卷是不是首卷：

#### 1.1 卷 N+1：让角色说话

有前卷存档（`archives/` 非空）→ **直接从角色驱动分析开始**，不推模板。已存在的角色有自己的状态和意图，他们告诉作者下一卷该写什么。

**操作流程：**
1. 读 `hooks.yaml` 确认未收束的钩子和主角当前状态
2. 读各活跃角色的最新 `状态历史`——上一卷结尾他们在哪儿、经历了什么、掌握了什么信息
3. 对每个活跃角色（含 antagonist/重要配角），生成角色"发声"：

   ```
   【{角色名}的处境】卷末落位——和谁在一起、掌握了什么信息、目前处于什么局面
   【{角色名}未完成的事】上一卷留下的执念/目标/恐惧——什么还没解决
   【{角色名}卷间思考】在卷与卷之间的间隙里，ta 反复在想什么
   【{角色名}现在想做的事】基于当前处境，ta 下一卷最可能推动什么
   ```

4. 展示给作者。**声音越多，卷的方向越自然**——几个角色的意志冲突会自然形成主线
5. 作者根据角色发声确定卷方向

> **为什么卷 N+1 不让模板引导？** 角色已经活了一卷以上，他们的状态比任何结构模板都更精确地指向"接下来该写什么"。模板是骨架，角色状态才是血肉——角色有历史就不用骨架。

**兜底：** 角色发声后作者仍然没方向 → 回到下方模板，辅助找结构。

#### 1.2 卷 1：展示故事结构模板

首次规划卷纲（`archives/` 为空）→ 使用结构模板提供骨架。

读 `settings/writing-style.yaml` 的 `genre_profile` 字段：

- **有值** → 在 `references/genre-example/` 中查找匹配的 `story_arc_templates`
- **无值** → 使用下方通用故事结构模板

Agent 直接输出提案（**不问"你想要哪种结构"**——默认选第一个 variant，或根据 story.md 的题材描述推测最匹配的 variant），展示格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  卷 1 结构建议：{variant.label}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{variant.description}

建议章节数：{total_chapters_range}

各阶段概览：
1. {phase}（约 {default_chapters} 章）→ {description}
2. {phase}（约 {default_chapters} 章）→ {description}
...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
也可换成：{其他 variant.label} / {其他 variant.label}
可直接调整阶段数量、顺序或章节分配——这是骨架，不是铁律。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

确认用这个结构，还是想换一个？
```

作者可能的选择：
- "就这个" → 用确认的模板作为卷纲讨论骨架。**将各 phase 名称写入 volume.yaml 的 chapters_summary 作为占位章纲**，后续逐章讨论时填充
- "换 {variant}" → 展示选中的 variant 格式同上，重新确认
- "我自己来" → 跳过步骤 1，直入 ### 2. 卷纲讨论
- "我改一下" → 按作者调整后的结构走

##### 通用故事结构模板（genre_profile 无值）

Agent 根据 story.md 题材信息和世界设定推测最适合的模板，直接展示不询问：

| 模板 | 适合 | 默认分卷 |
|------|------|---------|
| **三幕式**（setup→confrontation→resolution） | 冲突驱动、有明确对手或目标的故事 | setup 3-4章 / confrontation 4-5章 / resolution 2-3章 |
| **起承転結**（起→承→転→結） | 东方叙事结构，适合慢热细腻的故事 | 起 2-3章 / 承 3-4章 / 転 2-3章 / 結 1-2章 |
| **悬疑递进**（疑问→线索→反转→答案→余味） | 解谜向、有信息差的故事 | 疑问 1-2章 / 线索 2-3章 / 反转 2章 / 答案 1章 / 余味 1章 |
| **人物弧线**（日常→转折→挣扎→蜕变→新常态） | 人物驱动、成长向的故事 | 日常 2章 / 转折 2章 / 挣扎 2-3章 / 蜕变 1-2章 / 新常态 1章 |

**输出格式同上**——Agent 直接展示选定的模板提案，由作者确认。

### 2. 卷纲讨论
1. 创建 `volumes/volume-{N}.yaml`，写入卷标题和卷概要（步骤 1.1 已做角色发声时，概要直接从角色推导）
2. 若步骤 1.2 已确认模板：以模板各 phase 为骨架，讨论核心冲突、主要事件、角色弧光走向。**模板 phase 名称可调整——作者觉得某段不合适就改或删**
3. 若跳过步骤 1：从零讨论卷纲——核心冲突、主要事件、角色弧光走向（步骤 1.1 角色发声的结果应直接作为讨论基础）
4. 将章节摘要列表写入 chapters_summary（已确认模板时：phase 名称作为占位章纲，待逐章讨论时补充详情）

### 3. 生成卷提示词
读取 volume-N.yaml + story.md → 组装 `prompts/volume-{N}-prompt.md`：
- 本卷核心冲突与主线
- 章节列表（标题 + 概要，未写留空）
- "已完成章节摘要"区（初始为空，归档时追加——见 chapter-loop Step 6）
