# Novel Outline — 卷纲与章纲

## Overview

引导作者拆分故事线、讨论卷纲、逐章设计章纲。每章 Agent 先基于前文推理 3-4 个情节方向，作者选一个后自动填充 outline/memo/情绪设计。**不给分析，给选项。**

**对规划能力弱的作者：** Agent 直接提案故事结构模板作为骨架，逐章给方向选项。模板不是约束——是起点。**Agent 主动提案，不问"你想要什么结构"。**

**When NOT to use:** 设定未完成（world-setting 为空）、章纲已完成（status = draft 或 archived）、只是想查看已有章纲。

**Announce at start:** "我来引导你规划章节。先确认卷纲，再逐章讨论。每章我给你几个方向，你选一个。"

## HARD-GATE

```
章纲 memo（7 段）和 emotional_design 逐章讨论，不可跳过。
memo 缺失 → Phase 4 subagent 不知道"读者此刻在等什么"。
```

## 进入门禁

| 检查项 | 操作 |
|--------|------|
| author-intent.md | 核心主题已有 / 仍是模板？ |
| current-focus.md | 1-3章聚焦已有 / 仍是模板？ |
| 任一仍是模板 | **STOP** — 先问作者填写 |

## 流程（以首卷为例，卷 N+1 参见 0.1）

### 0. 卷方向确定（先于卷纲讨论）

先判断是本卷是不是首卷：

#### 0.1 卷 N+1：让角色说话

有前卷存档（`archives/` 非空）→ **直接从角色驱动分析开始**，不推模板。已存在的角色有自己的状态和意图，他们告诉作者下一卷该写什么。

**操作流程：**
1. 读 `hooks.yaml` 确认未收束的钩子和主角当前状态
2. 读各活跃角色的最新 `state_history`——上一卷结尾他们在哪儿、经历了什么、掌握了什么信息
3. 对每个活跃角色（含 antagonist/重要配角），生成角色"发声"：

   ```
   【{角色名}的处境】卷末落位——和谁在一起、掌握了什么信息、目前处于什么局面
   【{角色名}未完成的事】上一卷留下的执念/目标/恐惧——什么还没解决
   【{角色名}卷间思考】在卷与卷之间的间隙里，ta 反复在想什么
   【{角色名}现在想做的事】基于当前处境，ta 下一卷最可能推动什么
   ```

4. 展示给作者。**声音越多，卷的方向越自然**——几个角色的意志冲突会自然形成主线
5. 作者根据角色发声确定卷方向

> **为什么卷 N+1 不让模板引导？** 角色已经活了一卷以上，他们的状态比任何结构模板都更精确地指向"接下来该写什么"。模板是骨架，角色状态才是血肉——角色有历史就不用骨架。

**兜底：** 角色发声后作者仍然没方向 → 回到下方模板，辅助找结构。

#### 0.2 卷 1：展示故事结构模板

首次规划卷纲（`archives/` 为空）→ 使用结构模板提供骨架。

读 `settings/writing-style.yaml` 的 `genre_profile` 字段：

- **有值** → 在 `genre-corpus/` 中查找匹配的 `story_arc_templates`
- **无值** → 使用下方通用故事结构模板

Agent 直接输出提案（**不问"你想要哪种结构"**——默认选第一个 variant，或根据 story.yaml 的题材描述推测最匹配的 variant），展示格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  卷 1 结构建议：{variant.label}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{variant.description}

建议章节数：{total_chapters_range}

各阶段概览：
1. {phase}（约 {default_chapters} 章）→ {description}
2. {phase}（约 {default_chapters} 章）→ {description}
...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
也可换成：{其他 variant.label} / {其他 variant.label}
可直接调整阶段数量、顺序或章节分配——这是骨架，不是铁律。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

确认用这个结构，还是想换一个？
```

作者可能的选择：
- "就这个" → 用确认的模板作为卷纲讨论骨架。**将各 phase 名称写入 volume.yaml 的 chapters_summary 作为占位章纲**，后续逐章讨论时填充
- "换 {variant}" → 展示选中的 variant 格式同上，重新确认
- "我自己来" → 跳过步骤 0，直入 ### 卷纲
- "我改一下" → 按作者调整后的结构走

##### 通用故事结构模板（genre_profile 无值）

Agent 根据 story.yaml 题材信息和世界设定推测最适合的模板，直接展示不询问：

| 模板 | 适合 | 默认分卷 |
|------|------|---------|
| **三幕式**（setup→confrontation→resolution） | 冲突驱动、有明确对手或目标的故事 | setup 3-4章 / confrontation 4-5章 / resolution 2-3章 |
| **起承転結**（起→承→転→結） | 东方叙事结构，适合慢热细腻的故事 | 起 2-3章 / 承 3-4章 / 転 2-3章 / 結 1-2章 |
| **悬疑递进**（疑问→线索→反转→答案→余味） | 解谜向、有信息差的故事 | 疑问 1-2章 / 线索 2-3章 / 反转 2章 / 答案 1章 / 余味 1章 |
| **人物弧线**（日常→转折→挣扎→蜕变→新常态） | 人物驱动、成长向的故事 | 日常 2章 / 转折 2章 / 挣扎 2-3章 / 蜕变 1-2章 / 新常态 1章 |

**输出格式同上**——Agent 直接展示选定的模板提案，由作者确认。

### 卷纲
1. 创建 `volumes/volume-{N}.yaml`，写入卷标题和卷概要（步骤 0.1 已做角色发声时，概要直接从角色推导）
2. 若步骤 0.2 已确认模板：以模板各 phase 为骨架，讨论核心冲突、主要事件、角色弧光走向。**模板 phase 名称可调整——作者觉得某段不合适就改或删**
3. 若跳过步骤 0：从零讨论卷纲——核心冲突、主要事件、角色弧光走向（步骤 0.1 角色发声的结果应直接作为讨论基础）
4. 将章节摘要列表写入 chapters_summary（已确认模板时：phase 名称作为占位章纲，待逐章讨论时补充详情）

### 生成卷提示词
读取 volume-N.yaml + story.yaml → 组装 `prompts/volume-{N}-prompt.md`：
- 本卷核心冲突与主线
- 章节列表（标题 + 概要，未写留空）
- "已完成章节摘要"区（初始为空，Phase 6 归档时追加）

### 逐章章纲（每章必做）

#### 0. 情节方向提案（先于任何讨论）

Agent 基于前文积累推理 3-4 个本章情节方向。每个方向附带推理链——说明为什么得出这个方向，方便作者判断推理是否合理、哪个方向更贴合意图。

**推理依据：**
- **读者期待**（第一优先级）：上一章结尾读者在担心什么、好奇什么、期待什么兑现——从上一章的 `emotional_design.emotional_hook` 和 `memo.reader_expectation.detail` 推导。读者等了三章的事，本章该回应还是继续吊着？
- 前一章结尾的情绪状态和未完成的动作
- 每个活跃角色的当前动机、冲突和利益
- hooks.yaml 中 pending 和 partial_advance 状态的钩子
- 卷纲中本章的预期定位
- writing-style.yaml 的 genre.pacing_rules（连续动作章上限、连续日常章上限）+ reader_psychology.expectation_management（压制→释放节奏）
- 最近 3 章的情绪类型（避免连续同类型）

**提案格式（展示给作者）：**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第{N}卷第{M}章 — 情节方向提案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【方向 A】{一句话——核心冲突或事件}
  推理链：{为什么是 A——回应了哪个读者的期待？推进了哪个角色的动机？触及了哪个钩子？一句说清}
  读者语境：{上章结尾读者在等什么？本章是回应它，还是制造新期待？}
  {3-4 个节拍，用箭头连接。节拍是具体可写的事件，不是抽象标签}
  核心角色：{1-3 个}
  章尾改变：{本章结束时什么变了}
  章尾钩子：{本章结束后读者会担心/好奇/期待什么}

【方向 B】{一句话}
  推理链：{为什么是 B——和 A 的差异是什么？一句说清}
  读者语境：{...}
  {节拍链}
  核心角色：{...}
  章尾改变：{...}
  章尾钩子：{...}

【方向 C】{一句话}
  推理链：{为什么是 C——和 A、B 的差异是什么？}
  读者语境：{...}
  {节拍链}
  核心角色：{...}
  章尾改变：{...}
  章尾钩子：{...}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
选一个方向，或说"A和B结合""我自己来"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**提案要求：**
- 3-4 个方向，方向之间必须有明显差异——至少覆盖 3 种差异化维度
- 差异化维度：冲突类型（外部冲突/内部挣扎/关系冲突/信息揭露）、节奏（高压推进/日常过渡/情绪落地）、主角（推进主角线/切换配角视角/群像场景）
- 每个方向 ≤150 字——不给作者制造阅读负担
- 每个方向必须有明确的章尾改变——如果一章写完什么都没变，方向本身不合格
- 至少一个方向推进了卷纲主线，至少一个方向推进了 pending 钩子
- 至少一个方向兑现了读者的既有期待——读者等了三章的事，不能每次都只给新悬念。如果全部方向都是"制造新期待"，方向本身不合格
- 禁止抽象方向：不能是"角色内心成长""关系深化"，必须是"XX做了YY，导致ZZ"

**STOP：** 作者选择或指定方向后，进入 a 填充 outline。

**兜底处理：**
- 作者说"都不好，我想写 XXX"→ 按作者描述的方向走 a
- 作者说"没想法"→ Agent 推荐最自然的方向（选推进卷纲主线 + 有 pending hook 可兑现的那个），告知推荐理由（一句），确认后走 a

#### a. Outline（Agent 填充，作者确认）

根据作者选择的方向，Agent 自动填充 outline 字段（含 narrative_pov）：
- 将方向的 3-4 个节拍展开为具体的 key_points（用具体事件描述，不用类型标签）
- summary 用 2-3 句话概括本章剧情
- characters 列出本章出场角色
- location 和 time 填具体地点和时间跨度
- narrative_pov 沿用上一章设定，若作者未指定则询问确认

**STOP：作者确认 outline**。此步不讨论——Agent 已给出选项、作者已选定方向，outline 是方向的展开，不是新一轮讨论。

#### a2. 禁止清单
**必须询问：** "本章有没有明确禁止出现的场景、元素或情节？" 记录到 memo.prohibitions。作者说"没有"也要确认一次。

#### b. Chapter Memo（7 段，Agent 填充）

从选定方向自动填充，不逐段讨论。填充后展示完整 memo 表格供作者确认——作者可逐段调整，确认后写入。

**填充方法：**
- current_task：从方向的"章尾改变"反推——要达成这个改变，本章必须完成什么
- reader_expectation：从方向的"读者语境"和"章尾钩子"直接继承——state 取上一章结尾读者情绪，strategy 从方向的"读者语境"判断（兑现旧缺口/制造新缺口/延迟/双重），detail 填方向的"章尾钩子"内容
- payoff_plan：扫描 hooks.yaml，标记本章触及的钩子（must_resolve/must_hold/partial_advance）
- downtime_functions：如有过渡场景，标注其承载的功能（埋伏笔/推进关系/建立反差/信息铺垫/情绪缓冲）
- key_choices：方向的核心冲突中涉及的角色选择
- required_changes：提取方向的"章尾改变"（1-3 条，标注类型：信息/关系/物理/权力改变）
- prohibitions：从 a2 来

#### c. Emotional Design
primary_mood、mood_progression、intensity_peak、emotional_hook、intensity_level
**诊断：** "这一章结尾，读者最担心什么？最有好奇心的问题是什么？"（参考「钩子设计」和「期待值管理」）

#### d. 钩子操作
upsert（埋新）→ 归档时从正文提取 seed_text / mention（推进）/ resolve（收束）→ Phase 4 视角转换时注入 seed_text
**诊断（参考「四种开头钩」）：** "本章开头的钩子是什么类型——硬悬疑、思路式、人设式还是目标式？"

#### e. 更新 hooks.yaml

#### f. 微兑现设计
从以下 7 种中选择 ≥1 种，过渡章也至少 1 个。无微兑现 → 警告作者：
- **信息兑现**：读者知道了之前不知道的事（新线索/新信息）
- **关系兑现**：角色间的距离发生了变化（推进/接触）
- **情绪兑现**：读者感到爽、暖、释然或被触动
- **线索兑现**：之前的伏笔在本章有进展
- **能力兑现**：角色变强或获得新工具
- **资源兑现**：角色得到物品/资源
- **认可兑现**：角色被他人承认或尊重

### 填写 current-focus.md
填入 1-3 章聚焦：当前优先级、需推进支线、需提及钩子、节奏意图、限制约束。

### AI味自检（章纲完成后强制执行）

章纲确认前，扫描以下字段的 AI 味。命中即修，不把分析腔留给 Phase 4。

#### 检测清单

| # | 检测项 | 扫描字段 | 命中模式 |
|---|--------|---------|---------|
| 1 | 上帝视角概述 | outline、memo.current_task | "本章讲述了""主角经历了""本章将展示""故事发展到" |
| 2 | 抽象心理总结 | outline、memo.reader_expectation、memo.key_choices | "他感到""他意识到""他内心挣扎""经历了复杂的心理变化" |
| 3 | 分析性结论 | memo.reader_expectation、memo.required_changes | "这表明""这意味着""整件事的性质""这标志着关系转折" |
| 4 | 文学评论腔 | outline、memo.current_task | "通过X展现了Y""完成角色弧光转折""实现情感升华" |
| 5 | "不是而是"句式 | 所有字段 | "不是X，而是Y""不是…是…"——章纲阶段就禁止此句式 |
| 6 | 抽象标签替代具体事件 | memo.current_task、memo.required_changes | "关系递进""情感爆发""冲突升级"——应写具体事件，不写类型标签 |

#### 输出

```
## 章纲 AI 味自检

| 字段 | 命中项 | 处理 |
|------|--------|------|
| outline | 通过 | — |
| memo.current_task | 通过 | — |
| memo.reader_expectation | #3 分析性结论："这表明凶手早有预谋" | 已改为"凶手的准备比警方早三天——这一点在本章末尾会浮现" |
| memo.required_changes | #6 抽象标签 | 已改为具体事件 |

通过率：5/7 字段一轮通过。2 字段经修复后通过。
```

全部通过后，**STOP：作者确认。**
