# Novel Review — 章节深度评审

> 摘要：读正文 + 全部设定文件 → 10 维 60+ 细项逐条诊断 → 出报告。不改正文，不阻断归档。

## Overview

在归档前对章正文进行系统性评审，逐条对照项目中的所有设定文件，给出结构化诊断报告。与写作流程的 15 项 AI 味检测互补——AI 味检测是硬性门禁（AI 腔、模板化），本评审是软性的内容质量诊断（情节、人设、风格、逻辑）。

**定位**：正文生成后 → 作者可选触发本评审 → 归档。也可随时对已归档章节进行回顾评审。

**When NOT to use:** 正文未写入 archives/、章纲不存在、只是想检测 AI 味（→ 写作流程质量门禁）。

**Announce at start:** "我来评审第{N}卷第{M}章《{title}》。将逐条对照项目设定文件进行 10 维诊断。"

## 工具契约

| 工具 | 用途 | 限制 |
|------|------|------|
| Read | 读 10+ 文件（正文/章纲/卷纲/设定/角色/风格/hooks/前文/提示词） | — |
| Write/Edit | 不修改任何文件（诊断工具，只出报告） | 不改正文不改文件 |
| Agent | 可选：开独立检查助手执行逐维度评审 | 助手只用 Read，不可写文件 |

## 执行前提

评审是诊断工具，不自动修改正文。不阻断归档——作者可"已知悉，继续归档"。

## 进入门禁

| 检查项 | 操作 |
|--------|------|
| `archives/vol-{N}-ch-{M}-*.md` 存在？ | 不存在 → **STOP**，正文尚未生成 |
| `chapters/vol-{N}-ch-{M}.md` 存在且 outline 完整？ | 不存在/不完整 → **STOP**，无评审依据 |
| `volumes/volume-{N}.md` 存在？ | 不存在 → 标注"无卷纲，跳过卷级评审" |
| `settings/writing-style.md` 存在？ | 不存在 → 标注"无风格指南，跳过风格评审" |
| `settings/world-setting.md` 存在？ | 不存在 → 标注"无世界设定，跳过世界观评审" |

## 评审流程

### Step 1: 确定评审目标

若作者指定了卷号和章号 → 评审指定章节。若作者只说"评审这章" → 读取 `story.md` 找到 `current_volume` 和 `current_chapter`。

### Step 2: 收集全部上下文

读取以下所有文件，作为评审依据：

| 文件 | 评审用途 |
|------|---------|
| `archives/vol-{N}-ch-{M}-*.md` | **评审对象**——章正文全文 |
| `chapters/vol-{N}-ch-{M}.md` | 章纲——outline、memo（7段）、emotional_design、micro_payoffs |
| `volumes/volume-{N}.md` | 卷纲——核心冲突、章节摘要、主线走向 |
| `settings/world-setting.md` | 世界设定——geography/politics/culture/history/rules/physics/biology/sociology |
| `settings/character-setting/*.md` | 角色设定——世界观/自我定位/价值观/能力/技能/环境/关系/外貌/背景/故事角色/状态历史 |
| `settings/writing-style.md` | 写作风格——core_principles/possible_mistakes/depiction_techniques/genre/skill_layers/reader_psychology/desire_engine/immersion_pillars/emotional_pacing/creative_constitution/character_psychology_method/pov_consistency/natural_expression |
| `settings/hooks.md` | 伏笔/钩子——hooks 列表、状态、payoff_timing、hook_health_rules |
| `settings/anti-ai.md`（若存在） | 原有 AI 味规则（评审时参考，不重复质量门禁检查）。新项目不再生成此文件——AI 味检测由下方 checklist 执行 |
| `archives/vol-{N}-ch-{M-1}-*.md`（上一章正文结尾 200 字） | 前文延续——章节衔接是否无缝 |
| `prompts/vol-{N}-ch-{M}-prompt.md`（若存在） | 写作指令——对照 subagent 收到的完整指令，检查执行度 |

若有上下文文件缺失，标注"该维度无对照依据"，不阻断评审流程。

### Step 3: 主 Agent 执行 10 维评审

**主 Agent 直接执行评审，不调 subagent。** 本评审被调用时，主 Agent 已持有全部上下文（正文全文、章纲、角色文件等），无需重新加载。

按以下 **10 维 60+ 细项**逐条诊断，不得跳过、不得笼统概括：

1. 逐维逐项对照对应的设定文件，检查正文是否合规
2. 每项给出 ✅/⚠️/❌/N/A 判定 + 具体说明（引用正文原文佐证）
3. 全部维度完成后，汇总关键问题 / 改进建议 / 亮点
4. 按指定格式输出评审报告

## 评审维度与细项

### 维度 1：章纲执行度（对照 chapters/vol-{N}-ch-{M}.md）

逐条检查章正文是否兑现了章纲中的每一项规划。

| # | 细项 | 对照字段 | 评审方法 |
|---|------|---------|---------|
| 1.1 | 剧情概要么执行 | `outline.summary` | 正文是否完整执行了 summary 描述的剧情？逐句对照 summary，检查正文是否有对应段落。偏离/跳过/自行发挥 → 记录偏离点 |
| 1.2 | 关键情节点覆盖 | `outline.key_points[]` | 逐条检查每个 key_point 在正文中是否有对应的叙事段落。缺失 → 标记缺失项。正文写出了 key_points 之外的重要情节 → 标记为"自行发挥" |
| 1.3 | 出场角色兑现 | `outline.characters[]` | 列出的角色是否全部出场？正文中出现但未列在 characters 中的角色 → 标记为"未规划出场" |
| 1.4 | 地点设定遵守 | `outline.location` | 正文中的场景地点是否与 location 一致？多出/遗漏地点 → 标记 |
| 1.5 | 时间设定遵守 | `outline.time` | 正文的时间跨度是否与 time 一致？偏差 → 标记 |
| 1.6 | 叙事视角遵守 | `outline.narrative_pov` | 正文是否严格遵循指定的叙事视角（第三人称有限/第一人称/全知）？视角跳跃/违规 → 摘出违规句 |
| 1.7 | 当前任务完成 | `memo.current_task` | 本章必须完成的具体动作是否达成？未完成/部分完成 → 标记 |
| 1.8 | 读者期待管理 | `memo.reader_expectation` | 对照 state（上一章结束时的情绪状态）、strategy（制造新缺口/延迟兑现/兑现旧缺口/双重）、detail（读者在等什么）——正文是否按策略管理了读者期待？落空/错位 → 标记 |
| 1.9 | 伏笔兑现执行 | `memo.payoff_plan.must_resolve[]` | 列出的必须兑现伏笔是否在正文中收束？遗漏 → 标记 |
| 1.10 | 伏笔压制执行 | `memo.payoff_plan.must_hold[]` | 列出的必须压住的伏笔是否被意外泄露/提前揭露？泄露 → 标记 |
| 1.11 | 伏笔推进执行 | `memo.payoff_plan.partial_advance[]` | 列出的需推进伏笔是否有推进段落？推进方式是否符合规划？遗漏 → 标记 |
| 1.12 | 过渡段功能映射 | `memo.downtime_functions[]` | 每个日常/过渡段落是否完成了规划的功能（埋伏笔/推进关系/建立反差/信息铺垫/情绪缓冲）？空转段落 → 标记 |
| 1.13 | 关键抉择检验 | `memo.key_choices[]` | 角色在本章的重要选择是否通过了三连问检验（利益驱动？符合人设？读者不突兀？）？正文中的抉择与规划是否一致？ |
| 1.14 | 章尾改变兑现 | `memo.required_changes[]` | 列出的 1-3 条章尾改变是否在正文中实际发生？类型是否匹配（信息/关系/物理/权力改变）？遗漏 → 标记 |
| 1.15 | 禁止清单遵守 | `memo.prohibitions[]` | 正文是否触犯了任何一条 prohibitions？触发 → 标记违规点和具体内容 |
| 1.16 | 主情绪达成 | `emotional_design.primary_mood` | 正文整体情绪基调是否与 primary_mood 一致？偏差 → 描述实际情绪和预设的差距 |
| 1.17 | 情绪走向执行 | `emotional_design.mood_progression` | 正文的情绪变化轨迹是否符合 mood_progression（从X→经Y→以Z结束）？ |
| 1.18 | 情绪峰值场景 | `emotional_design.intensity_peak` | 情绪最强烈的场景是否出现在规划的位置？峰值的强度和位置是否合理？ |
| 1.19 | 爽点/情绪回报 | `emotional_design.satisfaction_beat` | 本章的小爽点/情绪回报是否兑现？读者是否能感受到？落空 → 标记 |
| 1.20 | 章尾情绪钩子 | `emotional_design.emotional_hook` | 章尾是否按规划留下了对应类型的情绪钩子（悬念/共情/期待/愤怒/好奇/选择/渴望）？缺钩子/类型不符 → 标记 |
| 1.21 | 情绪强度匹配 | `emotional_design.intensity_level` | 正文的情绪强度是否接近规划的 1-10 级别？偏差超过 2 级 → 标记 |
| 1.22 | 微兑现交付 | `emotional_design.micro_payoffs[]` | 逐条检查每个 micro_payoff 的 type 是否在正文中兑现、location 是否正确。过渡章至少 1 个微兑现。全部缺失 → 标记 |

### 维度 6：前文延续性（对照上一章 archives + 提示词中的前章锚定）

|---|------|---------|---------|

## 评分标准

每个细项按以下标准评判：

| 评级 | 符号 | 含义 |
|------|------|------|
| 完全通过 | ✅ | 正文完全符合该细项要求 |
| 基本通过 | ⚠️ | 正文大部分符合，有轻微偏差但不影响整体 |
| 未通过 | ❌ | 正文明显不符合该细项要求 |
| 不适用 | N/A | 该细项因上下文缺失而无法评估（如无卷纲、无角色文件等） |

**维度评分**：每个维度按通过率计算 1-10 分。
- 10 分：该维度所有适用细项全部 ✅
- 7-9 分：大部分 ✅，少数 ⚠️
- 4-6 分：部分 ❌，或较多 ⚠️
- 1-3 分：大量 ❌

**总评分** = 10 个维度评分的加权平均（维度 1 章纲执行度权重 2.0，其余维度权重 1.0）。
