# 设定阶段自检清单

设定是整本小说的地基。世界观、角色、风格三者互相锚定——世界观决定角色可能的行动，角色推动世界观展现，风格决定读者体验。**任何一个残缺，后面的写作循环都会出问题。**

> **两条底线**（任一不过，禁止进入 Phase 2 卷纲）：
>
> 1. **角色可诊断**——每个主要角色（protagonist / antagonist）必须通过"生气时怎么表现""有什么缺点"两个诊断。填了 yaml 字段不等于角色活了。
> 2. **世界观可感知**——设定不是说明书。必须能用"角色经历的场景"来呈现，而不是罗列条目。core 字段（geography / politics / rules）缺任何一项视为未完成。

---

## 最高原则：设定服务于故事

> 下面所有规则之前，先问一句：**"读者会在正文里感受到这个设定吗？"**

如果一条设定只能在设定文档里看到、正文里完全无法体现——它就不属于小说设定，属于作者笔记。**不要写进 world-setting.yaml。**

**判定标准：**
- 每条设定反问："哪类场景/对话能展示这一点？" 答不出的 → 删或合并
- world-setting 的全部内容应该能通过角色经历自然浮现，不需要旁白解释
- 角色 yaml 里的 worldview / values 应该和 world-setting 有可追溯的对应关系

---

## 一、世界观（world-setting.yaml）

### 1. core 字段（必填，缺一不可）

| 字段 | 填写标准 | 常见问题 |
|------|---------|---------|
| geography | 故事发生的地理空间。至少：主要场景的位置关系（城/村/秘境）、气候基调、地理限制（高山/水域/边界） | 只写"东方大陆"四个字——等于没写 |
| politics | 权力结构。至少：统治形式、主要势力/阵营、力量平衡、社会分层 | 只写"朝廷统治"——没有具体衙门、派系、权力关系 |
| rules | 这个世界的特殊规则。至少：力量体系（如有）、社会潜规则、禁忌、技术/魔法限制条件 | 有力量体系但没说限制和代价——主角将无敌 |

**每个 core 字段写完，反问：** "这个设定如果删掉，故事还能不能讲？" 删掉不受影响 → 要么它不重要，删掉；要么你的故事其实依赖它但你没写出来。

### 2. extended 字段（可选，按需追加）

| 字段 | 什么时候需要 |
|------|-------------|
| culture | 当不同区域角色的行为差异来自文化背景时 |
| history | 当世界里有历史事件影响当前剧情时 |
| physics | 当世界的物理规则与现实不同时 |
| biology | 当非人种族/特殊生物是重要设定时 |
| sociology | 当社会组织形式影响角色行动时 |

**原则：** 写到该字段直接影响角色决策了才填。**不要预设性填满——跑了再补比先填后删好。** 正文写作中随时可追加。

## 二、角色（character-setting/<id>.yaml）

### 1. 角色字段填写标准

| 字段 | 合格标准 | 不合格例子 |
|------|---------|-----------|
| cognition | 角色如何理解和判断事物。有认知偏差或习惯性思维模式 | "聪明机智"（不是字段，是评价） |
| worldview | 角色相信什么、不相信什么。可与其他角色冲突 | "相信正义"（太空，缺乏具体场景） |
| self_identity | 角色如何看待自己。可以有自我认知偏差 | "是个好人"（无信息量） |
| values | 什么事会让 ta 破例？什么不可以做？ | "重视朋友"（写成万能用语） |
| abilities | 实际能力，不是头衔。说"能用剑劈开落下的树叶"不说"剑术高超" | "武功很强"（无具体表现） |
| skills | 可习得/可传授的技术。有来源和局限 | "会炼丹"（没有体系支撑） |
| environment | 出身环境+当前处境。决定了 ta 的行为基线 | "穷苦出身"（可以补但不构成完整处境） |

### 2. 角色诊断（必须通过）

每个主要角色（protagonist / antagonist / 重要 supporting）逐人过：

- [ ] **TA 生气时怎么表现？和其他角色有什么不同？**——A 角生气砸东西，B 角生气不说话，C 角生气笑着说反话。如果所有角色生气都"愤怒地拍桌子"，角色还没活。
- [ ] **TA 有什么缺点？**——缺点必须是实质性缺陷（傲慢/懦弱/轻信/自私），不是"太正直了""太努力了"这种伪装优点。
- [ ] **配角除了服务剧情，自己有什么目的？**——配角如果只在这个场景出现、只负责送信息、之后再不出现，ta 就不是角色而是道具。

**额外诊断**（protagonist / antagonist 必须做）：
- [ ] **主角的初始状态和终局状态之间，差了什么？**——没想清楚这个，整本书就没有弧线。
- [ ] **反派的"我有道理"是什么？**——反派不觉得自己是反派。你能不能写出反派的辩护词？

### 3. 角色档案完整性

- [ ] story_role 已分配（protagonist / antagonist / supporting / minor）
- [ ] 文件名使用拼音 id（如 `chen-xiaoming.yaml`），不用中文
- [ ] 所有必填字段有值（cognition / worldview / self_identity / values / abilities / skills / environment）

## 三、写作风格（writing-style.yaml）

### 1. 风格确认

- [ ] 非模板默认值——至少有一个字段经过了讨论和调整
- [ ] role 字段已填写（AI 在写作中扮演的角色定位，不填 = subagent 自行猜测）
- [ ] core_principles 已填写（写作的核心原则，subagent 必须遵守的约束）
- [ ] possible_mistakes 已填写（AI 易犯的错误列表，subagent 知道要避开什么）
- [ ] depiction_techniques 已填写（推荐的描写技法参考）

### 2. 类型档案（genre）

- [ ] 已选择类型档案（genre_profile 有值）或确认"暂不选择"
- [ ] 已选择类型 → genre_config 已合并到 writing-style.yaml
- [ ] anti-ai.yaml 已追加 genre_fatigue_words（若有）
- [ ] 未选择类型 → 题材配置已从零讨论并入 genre 字段

### 3. 诊断

- [ ] **"书名和标签写的是什么题材？正文第一章读者能立刻感受到这个题材的核心味道吗？"**
- [ ] 类型禁忌已确认——作者知道这个类型不该写什么

---

## 四、author-intent.md

- [ ] 核心主题已填写（一句话说出这本书在讲什么）
- [ ] 写作信条已填写（你在写作上绝不妥协的事）
- [ ] 终局方向已填写（可选，但有更好）

## 设定阶段自检

走完上面全部规则后，逐项过：

- [ ] **世界观 core 三项全部有值**（geography / politics / rules）且通过了"删掉能不能讲"反问
- [ ] **每个 core 字段能用角色经历的场景呈现**（不是说明书式罗列）
- [ ] **所有主要角色 yaml 已创建，story_role 已分配**
- [ ] **每角色必填字段非空**（cognition / worldview / self_identity / values / abilities / skills / environment）
- [ ] **角色诊断已通过**（生气表现 + 缺点 + 配角目的）
- [ ] **主角有弧线方向**（初始状态→终局状态的差距）
- [ ] **反派有自洽逻辑**（"我也有道理"测试通过）
- [ ] **writing-style 四字段有值**（role / core_principles / possible_mistakes / depiction_techniques）
- [ ] **genre_profile 已配置**（已选类型或暂不选择+手动配置）
- [ ] **genre_config 已合并**（爽点/节奏/反套路等）
- [ ] **author-intent 核心主题和写作信条已填写**
- [ ] **诊断验收：书名/标签的题材味道能在第一章体现？**

### 验证方法

**优先级：Agent Teams > subAgent > 自检**

1. **Agent Teams（最优）**：开独立 reviewer agent，传入 `world-setting.yaml` + 各角色 yaml + `writing-style.yaml` + 本清单，逐项核查并汇报结论
2. **subAgent（次优）**：同上，用 subagent
3. **自检（兜底）**：严格逐项过——特别注意角色诊断和"设定可感知"反问，这两项最容易目测放行

**规则：拿到结论后先按 fail 项改完，再进入 Phase 2。不得带着 fail 项进入卷纲讨论。**
