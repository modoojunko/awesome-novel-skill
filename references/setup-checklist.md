# 设定阶段指南

> novel-setup 子技能的方法论参考。每个环节给出：输入什么、产出什么、怎么引导、最高原则、合格判断。

---

## 1. 开工三问

开始讨论前摸清作者起点。

**输入：** 作者说"写小说"
**产出：** 确定三条分支——新建/导入、清楚/模糊题材、有/无风格偏好
**怎么引导：** 依次问三个问题
1. 新建还是导入已有草稿？
2. 清楚要写什么题材吗？
3. 有风格偏好还是从零探索？
**最高原则：** 这三个决定后面所有步骤的路径，不要跳过直接进 YAML。

---

## 2. 题材诊断

选定题材后核查三项。

**输入：** 作者已选定题材或正在选定
**产出：** 三项诊断通过——不脱节、有辨识度、混搭有逻辑
**怎么引导：**

| 诊断 | 提问 |
|------|------|
| 脱节？ | "书名和标签是什么题材？第一章读者能感受到这个题材的核心味道吗？" |
| 无辨识度？ | "删掉书名和标签，读者能通过内容判断题材吗？"——检查要素清单 |
| 混搭无逻辑？ | "两个题材靠什么逻辑共存？"——新手无50万字经验不混搭 |

**各题材不能缺的核心要素：**

| 题材 | 缺了就脱节 |
|------|-----------|
| 宅斗 | 后宅权力斗争、家族利益权衡、符合时代的对话行为 |
| 系统 | 系统贯穿主线、核心功能推动剧情 |
| 年代 | 票证制度、集体劳动、时代标语 |
| 职场 | 专业能力突破、职业高光 |
| 言情 | 情感互动细节、误会-化解拉扯 |

**最高原则：** 题材不是标签，是读者在正文里每一页都能感受到的味道。
**合格判断：** 第一章场景/对话/环境能让读者判断这是什么题材；该题材的核心要素不缺。

---

## 3. 人设诊断

角色 yaml 写完后逐项排查。

**输入：** 角色 yaml 初稿
**产出：** 五项排查通过——不扁平、不前后矛盾、立得住、配角有目的、不太完美
**怎么引导：** 逐角色问以下问题

| 问题 | 排查项 | 触发后处理 |
|------|--------|-----------|
| "TA 生气时怎么表现？和其他角色有什么不同？" | 扁平化（全拍桌子） | 要求作者找差异化表现 |
| "回顾最近三章（如有），TA 的决定还符合初始人设吗？" | 前后不统一 | 要么补大纲，要么承认人设变了 |
| "删掉所有外貌描写和自我介绍，读者还能认出 TA 吗？" | 立不住（靠台词堆） | 改为通过行动立人——冲突中做什么决定了是谁 |
| "这个配角除了服务剧情，他自己的目的是什么？" | 工具人 | 给配角一条自驱动线 |
| "TA 的实质性缺点是什么？" | 太完美 | 加实质性缺陷（傲慢/懦弱/轻信，不是"太正直"） |

**最高原则：** 通过行动立人设，不通过台词贴标签。
**合格判断：** 同场景下每角色反应不同；角色三章内行为一致；配角有自驱动目的；主角有实质性缺点。

---

## 4. 世界观构建

**输入：** 世界类型已确定
**产出：** world-setting.yaml core 三项有值
**怎么引导：**
1. **正反体系对应：** 先立正派势力体系 → 推导反派体系 → 两者对应才有张力
2. **展示非灌输：** 每条设定反问"哪类场景能展示这个？" 答不出的不写
3. **按顺序逐项讨论：** geography（主要场景位置关系）→ politics（权力结构/势力/力量平衡）→ rules（特殊规则/限制/代价）→ extended（按需）
**最高原则：** 设定是故事的空间/权力/规则约束，不是说明书。读者在正文里感受不到的不属于小说设定。
**合格判断：** 每项通过"删掉能不能讲"反转测试；每条设定能对应一个具体场景。

---

## 5. 体系设定（如故事有等级体系）

**输入：** 有修炼/职场/段位等级
**产出：** 等级名称 + 晋级条件 + 每级量化差距
**怎么引导：** 三问——等级是什么？晋级条件？每级差距多大？
**合格判断：** 读者看完一场战斗能判断主角在同级中的水平。

---

## 6. 金手指设计（如主角有特殊能力）

**输入：** 金手指方案
**产出：** 确认金手指不盖过主角
**怎么引导：** "没有金手指，主角会混得很差吗？" 会→金手指太强，削弱。
**最高原则：** 金手指是辅助。主角靠智慧/努力/人格魅力推动故事。

---

## 7. 人称选择（如作者还没决定）

**输入：** 字数预期 + 视角结构
**产出：** 确定人称
**怎么引导：** 短篇/强代入感→推荐第一人称（60-70万字后乏力）；长篇/群像→推荐第三人称。拿不准→各写500字开头对比代入感。
**最高原则：** 确定后全程统一，不混入。

---

## 8. 写作风格

**输入：** 风格讨论阶段
**产出：** writing-style 四字段有值
**怎么引导：** "你最喜欢的三本小说/电影是什么？最吸引你的是什么？"——从作者偏好中提炼四字段
**最高原则：** 你的阅历和对事情的看法是别人学不走的东西。不盲目追榜，不故意秀文笔。
**合格判断：** 至少一个字段经过了和作者的讨论调整。

---

## 9. 类型档案选择

**输入：** 准备选类型
**产出：** genre_profile + genre_config 确定
**怎么引导：**
1. 读 index.yaml，扁平列出 24 种
2. 展示摘要（文风+氛围+要素+禁忌）
3. 作者确认或微调
**最高原则：** 选了就要让读者在第一页感受到。
**合格判断：** 类型要素在第一章正文中可感知。

---

## 10. 硬规则

- **`world-setting.yaml#core` 缺任意一项，禁止进入 Phase 2 卷纲。**
- **每主要角色必须通过"生气表现+缺点"诊断，否则禁止进入 Phase 3。**
- **writing-style 四字段必须全部有值并在提示词中注入，缺失任一 subagent 会在最关键的地方放飞。**
- **文件名拼音 id，不用中文。**
- **写入任何 YAML 前先读：** 非模板内容→追加不覆盖，展示变更摘要等作者确认。

---

## 11. 完工自检

全过才能进入卷纲：

- [ ] 世界观 core 三项有值 + 通过反转测试
- [ ] 主要角色 yaml 存在，必填字段非空 + 通过"生气+缺点"诊断
- [ ] 主角弧线方向明确
- [ ] 反派逻辑自洽
- [ ] writing-style 四字段有值
- [ ] genre_profile 已配置或"暂不选择"已确认
- [ ] genre_config 已合并
- [ ] author-intent 核心主题 + 写作信条已填写
- [ ] 题材诊断通过：第一章读者能感受到题材味道
