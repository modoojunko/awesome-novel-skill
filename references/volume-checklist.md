# 卷纲自检清单

卷纲是整卷写作的导航系统。一个靠谱的卷纲让写作循环有据可依——每章知道自己服务于哪个冲突、推动哪条弧线、兑现哪个钩子。**卷纲模糊，章纲必然跑偏；章纲跑偏，正文必然失控。**

> **两条底线**（任一不过，禁止进入 Phase 3 逐章写作）：
>
> 1. **核心冲突可一句说完**——卷的主线冲突必须能用一句话表述，不需要"因为所以""与此同时""另一方面"。说不清楚就是没想清楚。一卷可以有支线，但主线只能有一根。
> 2. **每章有可写的冲突载体**——chapters_summary 的每一条 summary 必须包含一个可写的事件冲突。只有"建立人物""世界观展示""过渡铺垫""关系递进"而无具体冲突事件的章纲等于没写。冲突载体就是"这一章读者看完后什么东西变了"。

---

## 最高原则：卷纲服务于写作循环

> 下面所有规则之前，先问一句：**"拿着这个卷纲，能直接写出每章的方向提案吗？"**

卷纲不是给读者看的概要，不是给作者看的笔记——是给写作循环的定向指令。它的唯一用途是：当 Agent 打开 chapter-loop，读到 volume-{N}.yaml 时，能立刻推理出本章的情节方向提案。

**判定标准：**
- chapters_summary 里每条 summary 不是"发生了什么"的梗概，而是"推动什么冲突、改变什么状态"的指令
- 卷纲的核心冲突必须能逐章追溯——每一章都可以反问："这章服务于核心冲突的哪一步？"
- 卷纲读完后，即使不翻 chapter.yaml，也应该清楚地知道"这一卷的主线是什么、每章做什么、哪些钩子在跑"

**允许的：**

| 行为 | 说明 |
|------|------|
| 章纲留白（非首卷时，后续章只写标题和一句方向） | 首卷必须全部有冲突载体，卷 N+1 允许近端 3 章详细、远端留白，但留白章也必须写一句冲突方向 |
| 卷方向随写作调整核心冲突的表述 | 但不得推翻已归档章节的事实基础 |

**禁止的：**

| 行为 | 后果 |
|------|------|
| 写"主题式"章纲（如"这一章讲信任"）——没有具体冲突事件 | 写作循环无从下手，方向提案变猜谜 |
| 一章承载多个不相关的主线冲突 | 章纲分裂，正文东拉西扯 |
| 首卷跳过方向确定（无论是模板还是角色发声）直接写章纲 | 结构塌方——写了 3 章发现方向不对，前面全废 |
| 卷 N+1 不做角色发声，直接套首卷模板 | 角色状态被忽略，后续行为与历史断裂 |

---

## 一、卷方向确定

卷纲的第一步，不是写章纲，是定方向。方向错了，后面全部白写。首卷走故事结构模板，卷 N+1 走角色发声。

### 1. 首卷方向

| 字段 | 填写标准 | 常见问题 |
|------|---------|---------|
| 结构模板选择 | 从三幕式/起承転結/悬疑递进/人物弧线中选一，或从 references/genre-example 取匹配模板。必须有明确的阶段划分，不是"随便写写" | 只说"先写几章看看"——等同于没有方向 |
| 各阶段功能 | 每个阶段有明确的一句话功能（如"setup：建立主角处境和核心悬念"），不能用"发展""展开"这样的空词 | "第二阶段：故事展开"——展开什么？谁和谁发生了什么事？ |
| 章节数分配 | 每阶段分配具体章节数，总数匹配预期卷长。允许作者调整，但必须有数 | "大概 10-15 章"——没有分配，无法判断比例是否合理 |
| 与题材类型匹配 | 如果选了 genre_profile，模板应与类型匹配（悬疑类用悬疑递进模板，修仙类用升级弧线模板） | 都市恋爱题材用了三幕式但第一幕 8 章——读者等不到冲突就走了 |

**卷 1 方向诊断：**

- [ ] **结构模板已选定**（四个模板之一或 corpus 中指定模板），作者已确认
- [ ] **各阶段的"核心推进"写出来了**——不是阶段名，是这一阶段要完成什么冲突推进
- [ ] **章节数已分配到各阶段**，比例大致合理（不是 setup 1 章 confrontation 8 章）
- [ ] **模板选型与 genre_profile 匹配**（如有）

**示例（起承転結，悬疑题材）：**

```
✅ 合格：
起（2章）：建立失踪案悬念 + 主角调查能力展现。第1章接案，第2章勘查现场发现矛盾。
承（3章）：线索展开 + 第一重反转。第3章发现旧案关联，第4章遭遇阻碍，第5章获得关键突破。
転（2章）：真相逼近 + 主角个人危机浮现。第6章发现更大阴谋，第7章主角被迫选择。
結（1章）：小案收束，大局揭开。找到失踪者，但主角知道背后有更大的东西。
```

```
❌ 不合格：
起（3章）：介绍世界观和人物关系。
承（4章）：故事逐渐展开，冲突慢慢升级。
転（2章）：故事出现转折。
結（1章）：故事结束。
```
> 不合格原因：全是空词——"展开""升级""转折""结束"。写不了方向提案。

### 2. 卷 N+1 方向（角色发声）

| 字段 | 填写标准 | 常见问题 |
|------|---------|---------|
| 活跃角色筛选 | 读各角色 state_history + 最近 3 章 chapter.yaml 的 hooks 字段，找出有未完成目标/未收束钩子/状态变化的角色。不是所有角色都发言，只选下一卷有戏份的 | 列了全部 12 个角色发声——作者根本看不过来 |
| 角色处境描述 | 必须具体到"卷末落位"：在哪儿、和谁在一起、掌握了什么信息。不能是"上一卷经历了很多" | "陆征继续调查"——没有落位，没说他知道了什么 |
| 角色意志冲突 | 多个角色的发声应自然形成冲突或交集。"A 想查真相+B 想掩盖真相+C 想拉拢 A"好于"A 想查真相+B 也想查真相" | 所有角色方向一致——没有冲突就没有主线 |
| 作者确认 | 角色发声展示后，作者必须明确确认卷方向。不得从发声直接跳到写章纲 | Agent 自认为"作者应该同意"直接开工 |

**卷 N+1 方向诊断：**

- [ ] **活跃角色已筛选**（有未完成目标/未收束钩子的角色），不是全角色列表
- [ ] **每个发声角色有具体"卷末落位"**（位置/关系/已知信息）
- [ ] **角色意志之间存在冲突或交集**——至少有两个角色的方向产生摩擦
- [ ] **作者已确认卷方向**（从发声到方向有明确的选择记录）

**示例（都市悬疑，钟声卷 1→卷 2）：**

```
✅ 合格：
【陆征的处境】卷末在办公室，发现被翻动。掌握了三件事：苏棠是自愿躲藏的、三年前案子被压了、灰短袖男人在跟踪他。手里有方岩给的旧案卷宗照片。
【陆征现在想做的事】查出三年前到底发生了什么——谁压了案子、为什么压。苏棠的失踪只是引子，背后的东西才是真正的目标。
【方岩的处境】在警队内部，知道有人在盯陆征。给了陆征卷宗但被上级警告。
【方岩现在想做的事】暗中帮陆征但不能暴露。他想知道为什么三年前的案子被压——这和他的职业尊严有关。
```

```
❌ 不合格：
【陆征的处境】经历了第一卷的冒险，破获了失踪案。
【陆征现在想做的事】继续当侦探，接下一个案子。
```
> 不合格原因：没有落位信息，没有具体的已知/未知，没有残留动机。"继续当侦探"等于没有方向。

## 二、核心冲突

一卷一根主线，所有章节服务于同一个核心冲突。支线可以有，但不能和主线抢资源或方向。

| 字段 | 填写标准 | 常见问题 |
|------|---------|---------|
| 冲突表述 | 一句话："谁 + 想做什么 + 被什么阻碍"。必须有对抗力量，不能是单向愿望 | "主角想变强"——没有阻碍方，冲突不存在 |
| 对抗力量 | 阻碍必须是具体的角色/势力/处境，不能是抽象概念 | "对抗命运""对抗社会"——太虚，写不了具体情节 |
| 与章节的追溯关系 | 每章的 summary 至少含有一个动词指向核心冲突的推进（发现/遭遇/选择/失败/推进/阻碍） | 第 3 章 summary 是"主角去市场买东西"——与主线冲突无关，是填充章节 |
| 单线 vs 多线 | 一卷只能有一个主线冲突。如果有支线冲突，必须能在 2 句话内说清支线和主线的关系 | 一卷三条线各自独立，每章在三条线间跳——读者跟不住，章纲必然分裂 |

**核心冲突诊断：**

- [ ] **冲突可一句说完**（谁做啥被啥阻碍），不需要"同时""另一方面""此外"
- [ ] **对抗力量是具体的**——角色/势力/处境，不是抽象概念
- [ ] **每章的 summary 可以追溯到核心冲突的某一步**（对着 chapters_summary 逐条反问"这章和主线冲突有什么关系？"）
- [ ] **如果有支线，支线和主线的关系可两句话说清**

**示例（核心冲突一句话）：**

```
✅ 合格：
"陆征想查出三年前谁压了案子，但警队内部有人要阻止他翻旧账。"
——有谁（陆征）、想做什么（查清三年前压案真相）、被什么阻碍（警队内部势力阻止）

"陈潇想在大赛中证明自己，但师兄处处设局让他出丑。"
——清晰地对抗关系

❌ 不合格：
"陆征在都市中调查各种案件，同时面对自己过去的阴影。"
——"各种案件"不是一根主线。"过去的阴影"没有具体对抗力量。"调查案件"是动作不是冲突

"主角在异世界中逐步成长，最终成为强者。"
——没有对抗力量，是单向叙述
```

## 三、章节列表（chapters_summary）

chapters_summary 是卷纲最核心的产出——它直接决定了每章的章纲能不能写出来。

### 1. 章节结构

| 字段 | 填写标准 | 常见问题 |
|------|---------|---------|
| id | 格式 "卷号-章号"（如 1-1, 1-2）。连续不跳号 | 1-1, 1-3（跳了 1-2）——写作循环会断 |
| title | 提供信息量。不用"第X章"或纯功能名。标题本身应该让读者（或 Agent）猜到本章的核心事件 | "第一章"（等于没写）、"过渡章"（暴露了没想清楚内容） |
| summary | 一句话，含三个要素：**谁做了什么 + 发生了什么事（冲突） + 本章结束时什么变了**。缺一个要素就是不合格 | "主角继续调查"——谁调查？调查什么？发现了什么？什么都没变？ |

### 2. 冲突载体（每章必检）

每章 summary 必须包含一个可写的事件冲突。下面是四种不合格的章纲类型和对应的修改方法：

| 不合格类型 | 典型句式 | 为什么写不了 | 修改方向 |
|-----------|---------|-------------|---------|
| 主题式 | "本章讲信任""探讨自由与责任" | 没有事件，只有概念。Agent 无法推理出情节 | 把概念变成具体事件：谁在什么事上信任了谁？结果是？ |
| 功能式 | "建立人物关系""展示世界观" | 功能不是故事。建立人物关系可以写成"两个角色在某件事上发生分歧" | 把功能折入冲突事件：通过某件事来展示/建立 |
| 梗概式 | "主角经历了一天的冒险" | 没有具体冲突点。一天里有无数件事——Agent 不知道该写哪件 | 指认这一天里最核心的一件事 |
| 结果式 | "主角赢了比赛" | 只有结果没有冲突过程。赢了谁？怎么赢的？过程中遇到什么阻碍？ | 写出阻碍和对抗："主角在决赛中面对 XXX，通过 YYY 赢得比赛" |

**章节列表诊断：**

- [ ] **每章 id 连续不跳号**
- [ ] **每章 title 有信息量**——不是"第X章""过渡""日常"
- [ ] **每章 summary 含三个要素**（谁做啥 + 冲突事件 + 章末变化）
- [ ] **每章有可写的冲突载体**——对着 summary 反问"这章能直接写方向提案吗？"
- [ ] **每章可追溯到核心冲突**——反问"这章服务于主线冲突的哪一步？"
- [ ] **各章之间有因果递进**——前一章的章末变化是后一章的起点或动机，不是各章独立无关联

**示例：**

```
✅ 合格（悬疑题材，卷 1 前三章）：
1-1  委托人   苏沫来访委托陆征调查姐姐失踪。陆征发现警方忽略的关键细节，决定接案。
              → 冲突载体: 陆征 vs 已被警方判定为"主动离开"的案子（职业本能 vs 官方结论）
1-2  出租屋   陆征勘查现场发现与"主动离开"矛盾的证据，离开时遭人跟踪拍照。
              → 冲突载体: 陆征 vs 掩盖真相的人（证据指向非主动离开，但有人要他停下）
1-3  旧案     线索指向三年前悬案，找方岩调卷宗遭警告。回办公室发现被侵入。
              → 冲突载体: 陆征 vs 压案势力（调查升维，从失踪案变为旧案追查）
```

```
❌ 不合格：
1-1  开端    主角登场，展示世界观。
             → 没有冲突事件，没有谁做啥，没有章末变化
1-2  发展    主角开始调查，遇到各种阻碍。
             → "各种阻碍"不是具体事件。Agent 不知道"哪种"阻碍
1-3  转折    主角获得关键线索，知道了真相。
             → 什么线索？什么真相？冲突在哪里？只有结果没有过程
1-4  高潮    主角与反派决斗。
             → 谁和谁？为什么打？赌注是什么？
1-5  结局    主角获胜，故事结束。
             → 赢了什么？付出了什么代价？什么都没变？
```

**章节间因果链示例：**

```
✅ 合格（因果链清晰）：
1-1  接案 → 1-2  勘查发现矛盾（因为接了案才去勘查）→ 1-3  深入调查遭遇阻碍（因为发现矛盾才追查旧案）
每一章是前一章的结果、后一章的原因。

❌ 不合格（各章独立）：
1-1  主角去A地点 → 1-2  主角见B角色 → 1-3  主角获得C物品
三件事没有因果递进——只是主角去了不同地方做了不同事，不是故事在推进。
```

## 四、新角色与设定追加

卷纲讨论中可能发现需要新增角色或补充世界观设定。这些追加必须在进入写作循环前落实。

### 1. 新角色

| 检查项 | 标准 |
|--------|------|
| 角色 yaml 已创建 | `settings/character-setting/{id}.yaml` 存在，字段填写完整度不低于设定阶段标准 |
| story_role 已分配 | 不能留空或写"待定" |
| 必填字段有值 | worldview / self_identity / values / abilities / skills / environment 至少填写了其中与角色功能相关的字段 |
| 角色诊断（如果本卷戏份 > 1 章） | 生气表现 + 缺点 + 自己目的（参照设定阶段角色诊断标准） |
| 配角目的明确 | 如果配角只服务于一场戏（送信息/引路/被救），必须标注"一次性角色"或展开为多场景角色 |

### 2. 新设定追加

| 检查项 | 标准 |
|--------|------|
| 新设定已写入 world-setting.yaml | 相关字段下追加内容，不是口头讨论完就忘了 |
| 新设定通过"可感知"反问 | "读者会在正文里感受到这个设定吗？"——答不出就不要加 |
| 新设定不影响已有章节的一致性 | 如果追加的设定与已归档章节矛盾，必须标注冲突并给出处理方案（修改已归档 / 限制新设定的生效范围 / 放弃追加） |

**新角色/设定追加诊断：**

- [ ] **所有新增角色已有 yaml 文件**，story_role 已分配
- [ ] **所有新增设定已写入 world-setting.yaml**，不是只存在于对话记录中
- [ ] **新设定通过"可感知"反问**（答不出 → 删或改为作者笔记不写入设定文件）
- [ ] **新设定与已归档章节一致**（如有矛盾，已有明确处理方案）

**示例：**

```
✅ 合格：
卷纲讨论中决定"本案需要一位法医角色提供专业线索" →
  创建 fa-yi-li-ming.yaml
  story_role: supporting
  abilities: 法医专业，能发现常人忽略的尸检细节
  environment: 市局法医科，与方岩有工作往来
  角色目的：提供关键线索+作为方岩的"内部信息源"

❌ 不合格：
卷纲讨论中觉得"这个世界需要一个地下交易市场" →
  口头确认了但没有写入 world-setting.yaml
  三天后开始写第 5 章，Agent 不知道地下市场的规则，自行编造了一个
  → 与作者意图冲突，第 5 章重写
```

## 卷纲阶段自检

走完上面全部规则后，逐项过：

- [ ] **核心冲突可一句说完**（谁 + 做什么 + 被什么阻碍），不对着 yaml 念，张口能说
- [ ] **卷方向已确定**（首卷有结构模板、卷 N+1 有角色发声+作者确认方向）
- [ ] **首卷：结构模板选定 + 阶段功能写清 + 章节数已分配**
- [ ] **卷 N+1：活跃角色已筛选、卷末落位具体、角色意志有冲突、作者已确认方向**

### 快速嗅探（15 秒内完成）

如果不想逐条过，先做这三个快速嗅探——三个全过，大概率合格：

1. **能不看文件说出核心冲突一句话吗？** → 说不出来 → 没想清楚
2. **随便指一章 summary，能说出"这章结束后什么东西变了"吗？** → 说不出来 → 章纲没有冲突载体
3. **随便指两章，能说出"为什么第一章之后是第二章、不是直接跳到第三章"吗？** → 说不出来 → 没有因果链

### 验证方法

**优先级：Agent Teams > subAgent > 自检**

1. **Agent Teams（最优）**：开独立 reviewer agent，传入 `volume-{N}.yaml` + 活跃角色 yaml + 最近 3 章 chapter.yaml + 本清单。reviewer agent 逐项核查，对每项结论给出"通过/不通过+证据"。不通过项附具体位置（哪个字段、哪句话不达标）。
2. **subAgent（次优）**：同上，使用 subagent 执行审查。
3. **自检（兜底）**：严格按本清单逐项过。特别注意"章节可追溯到核心冲突"和"章节间因果递进"——这两项最容易目测放行。建议自检时把 chapters_summary 打印出来，拿铅笔逐条划"冲突载体""与主线的连接""与前一章的因果"。

**规则：拿到结论后先按 fail 项改完，再进入 Phase 3。不得带着 fail 项进入逐章写作。**
