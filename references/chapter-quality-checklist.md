# 正文验收标准

**定位：** subagent 正文的追溯诊断工具集——当正文不符合作者预期时，定位问题出在哪个环节（prompt / 章纲 / 风格参考），以便针对性修正。所有验收项不是终审法官，是诊断探针。

subagent 生成正文后，主 Agent 对正文进行逐项检查。但这些检查不是为了"拦住不合格正文"——**正文是否合格，作者说了算。** 检查的目的是：当作者说"不对"时，能快速定位问题源头。

---

## 最高原则

> **正文评价标准就一个：是否符合作者预期。**

下面所有规则是辅助判断的工具，不是终审法官。规则全部通过但作者觉得不对 → 以作者判断为准。规则部分未过但作者觉得没问题 → 尊重作者判断。

规则的价值在于提供结构化的检查维度，降低遗漏。但每一章的最终裁决者是作者本人，不是规则本身。

## 一、AI 味检测——疲劳词

疲劳词本身不是错词，但某些词在一章内高频出现（3 次以上）即形成"AI 腔"。以下按类别列出阈值和替代策略。

### 1. 副词类

| 疲劳词 | 单章阈值 | 替代策略 |
|--------|---------|---------|
| 忽然、突然 | 3 次 | 大多数"突然"是多余的——事件本身的意外性已经传达了"突然"。删掉"突然"，直接写事件 |
| 竟然、居然 | 2 次 | 让读者自己判断"竟然"，不要替读者惊讶。删掉"竟然"，事实本身够意外就够了 |
| 默默、微微 | 3 次 | 用具体动作替代。"默默地站起来" → "他站起来，椅子腿蹭过地面，声音比平时轻" |
| 渐渐、逐渐 | 2 次 | 删掉副词，用变化的细节来体现渐进。"天色渐渐暗下来" → "窗外梧桐的影子从地面爬上了墙" |
| 依旧、仍然 | 2 次 | 用状态描写替代。"他依旧坐在那里" → "他的姿势和一小时前完全一样" |

```
✅ 合格（疲劳词→具体描写）：
"他站起来，椅子腿蹭过地面，声音比平时轻。"
→ 没有"默默地"，但读者知道他是静悄悄地起来的

"窗外梧桐的影子从地面爬上了墙。"
→ 没有"渐渐"，但读者能感受到天色在变

"他的姿势没变过。一小时。"
→ 没有"依旧"，但信息完整，节奏更好

❌ 不合格（疲劳词密集）：
"他突然站了起来，竟然发现天已经渐渐黑了。他依旧坐在那里，默默想着刚才的事。"
→ 一句话 5 个疲劳词。"突然""竟然""渐渐""依旧""默默"——读者会觉得在喝一碗味精水
```

### 2. 动词类

| 疲劳词 | 单章阈值 | 替代策略 |
|--------|---------|---------|
| 露出（笑容/表情） | 3 次 | "露出微笑" → 写笑的具体方式。"她笑起来" 或 "嘴角动了一下" |
| 勾起（回忆/嘴角） | 2 次 | 写触发回忆的具体事物，不是"勾起"这个动作 |
| 泛起（涟漪/暖意/酸楚） | 2 次 | 用身体感受替代。"心里泛起暖意" → "他把那碗粥推回来，又加了一勺糖" |
| 浮现（画面/记忆） | 2 次 | 直接写回忆内容或触发细节，不写"浮现"这个中间层 |
| 陷入（沉默/沉思/回忆） | 2 次 | 写沉默之后的具体行为或状态 |
| 涌起（情绪/回忆） | 2 次 | 用身体反应替代。"愤怒涌起" → "他的手指用力扣进掌心" |
| 充斥 | 1 次 | 几乎总是可以删掉或用具体描写替代 |

```
✅ 合格：
"他把那碗粥推回来，又加了一勺糖。"
→ 没有"泛起暖意"，但读者感受到了

"她没说话。第三次翻到同一页。"
→ 没有"陷入沉思"，但沉默的状态很具体

"手指用力扣进掌心。"
→ 没有"涌起愤怒"，但身体的紧张传达了愤怒

❌ 不合格：
"他露出了一丝苦笑，心头泛起复杂的情绪，过去的回忆不断在脑海中浮现。"
→ "露出""泛起""浮现"三个动词都是疲劳词——等于什么都没写。读者只看到了"角色在感慨"，没看到感慨什么、为什么感慨、感慨之后做了什么
```

### 3. 形容词/抽象词类

| 疲劳词 | 单章阈值 | 替代策略 |
|--------|---------|---------|
| 复杂、微妙 | 各 3 次 | 写具体矛盾。"情绪复杂" → 写两种对立的冲动同时存在 |
| 难以言喻、莫名 | 各 1 次 | 既然"难以言喻"，就不要用这个词来逃避——写出那个难以言喻的东西的具体表现 |
| 异样、浓重 | 各 2 次 | 写具体的异样在哪、浓重的是什么 |

```
✅ 合格：
"她想笑，但眼眶在发酸。"
→ 没有"情绪复杂"，但两种对立的身体反应给出了复杂感

"他闻到了铁锈味。不是错觉——手指摸过窗台，指尖暗红。"
→ 没有"异样的感觉"，但给出了具体的异样

❌ 不合格：
"空气中弥漫着一种难以言喻的紧张感，气氛微妙而复杂。"
→ "难以言喻""微妙""复杂"三个词叠在一起——但读者什么也没看到。什么叫"难以言喻的紧张"？谁紧张？谁表现出了紧张？
```

### 4. 连接词类

| 疲劳词 | 单章阈值 | 替代策略 |
|--------|---------|---------|
| 然而、不过 | 4 次 | 大多数转折不需要"然而"——前后句的内容本身就是转折。删除连接词，让句子直接碰撞 |
| 于是 | 3 次 | "于是"是 LLM 的条件反射——前后两件事有因果，"于是"就是多余的胶水。删除后因果更自然 |
| 随即、紧接着 | 2 次 | 删掉。动作的自然顺序不需要时间副词标注 |
| 与此同时 | 1 次 | "与此同时" = 我不会写转场。直接用新场景开头的画面来转场 |

```
✅ 合格：
"陆征推门进去。方岩在办公桌后面，没抬头。
'坐。'
他没坐。"
→ 无连接词。三句话本身的关系读者能理解

❌ 不合格：
"陆征推门进去，然而方岩并没有抬头看他。于是陆征站在门口等了一会儿。与此同时，走廊里传来了脚步声。"
→ "然而""于是""与此同时"——三句话被三个连接词粘成一个球。拆掉连接词，意思照样通，节奏好得多
```

### 5. 身体反应类（疲劳模板）

| 疲劳词 | 单章阈值 | 替代策略 |
|--------|---------|---------|
| 瞳孔一缩、身体一僵 | 2 次 | 这两个是 LLM 最顽固的癖好——恐惧是瞳孔一缩、震惊是身体一僵。换成具体场景：恐惧可以是手抖到按不了打火机，震惊可以是杯子倒了忘了扶 |
| 倒吸一口凉气 | 1 次 | 网文频率太高，读者已经麻木。换成具体的呼吸变化或其他身体反应 |
| 呼吸一滞 | 1 次 | 同上——已经被用滥了的生理反应 |

```
✅ 合格：
"他盯着屏幕。打火机按了三次没点着。"
→ 恐惧和紧张不在"瞳孔一缩"里，在打火机按不着的细节里

"她的咖啡杯倒了，她没扶。棕色的液体淌到桌沿，一滴滴往下掉。"
→ 震惊不在"身体一僵"里，在忘记扶杯子的停顿里

❌ 不合格：
"他看到了那条消息，瞳孔猛地一缩，身体一僵，倒吸了一口凉气。"
→ 三条身体反应模板排队出场，像在走受惊的流水线。"瞳孔一缩""身体一僵""倒吸一口凉气"——读者能猜到下一句是"呼吸一滞"
```

### 6. 元叙事类和学术腔

| 疲劳词/句式 | 处理方法 |
|-------------|---------|
| 本章讲述了、接下来、在这一章中、故事还在继续 | **全部删除**。这些是章纲里的定位语句，被 subagent 抄进了正文。AI 味检测最快速的一条：正文中不允许出现"本章""下一章""故事""读者" |
| 综上所述、总而言之、从某种意义上说、不可否认、显而易见 | 删除。正文不是议论文 |
| 似乎、也许、大概、仿佛 | 每 500 字不超过 2 个。用确定性描写替代模糊词——不是"他似乎很生气"，是"他把烟掐了，搓了三次手指" |

### 7. 英文高危疲劳词

英文题材（LitRPG / Progression Fantasy 等）额外检查：`delve` / `tapestry` / `testament` / `embodiment` / `harbinger` / `veritable` / `inexorable` / `nonetheless` / `furthermore` / `however` / `seemed to` / `as if`。中文题材可跳过此项。

### 疲劳词扫描方法

1. 将正文粘贴到纯文本编辑器
2. Ctrl+F 逐个搜索上述疲劳词
3. 每命中一次记录行号
4. 按类别汇总次数
5. 超阈值的词 → 逐个替换（不是批量删除——每个词都要判断是否有叙事功能）

## 二、AI 味检测——句式违规

8 种结构性句式癖好和 6 种句式规则。逐项扫描，超阈值标记。

### 1. 结构性句式癖好（8 种）

参考 `tic-patterns.yaml` 的定义。每种模式有单章阈值，超过触发警告。

#### 1.1 "不是……而是"句式

```
单章阈值: 3 次 | 严重度: high

✅ 合格（直接陈述）：
"他在愤怒。"
→ 不需要"他不是害怕，而是愤怒"——直接说愤怒

"她说这话不是要安慰他，是想让他知道她看见了。"
→ 这个"不是……而是"有叙事功能——区分了她的动机和表象。保留。

❌ 不合格（对比癖好）：
"她不是害怕黑暗，而是害怕黑暗中的孤独。他不是犹豫，而是不知道该选哪边。这顿饭不是为了吃，而是为了说那些话。"
→ 一章出现 4 次"不是而是"。这是 LLM 最顽固的统计癖好——它学会了"用对比来制造深度感"。修复：删除"不是"分句，只保留"而是"后的内容，或改为两个独立句子
```

#### 1.2 "没有……只是/只不过"句式

```
单章阈值: 3 次 | 严重度: medium

✅ 合格：
"她在等。不是等车，是等一个人离开。"
→ 偶尔使用可以制造节奏，但一章内不要超过 3 次

❌ 不合格：
"没有风，只是空气在流动。没有人说话，只是沉默在膨胀。没有离开，只是在原地腐烂。"
→ "没有X，只是Y"三连发。这是"不是而是"的变体，同样的问题——用否定-纠正来制造伪深度。
```

#### 1.3 否定-纠正句式

```
单章阈值: 2 次 | 严重度: high
命中模式: "不，不是这样""其实并非如此""更准确地说"

✅ 合格：
"墙上的钟停了。不，不是停了——是被摘下来了。"
→ 有叙事功能：角色发现过程的两步纠正。保留。

❌ 不合格：
"他的选择看起来是对的。不，不是这样。其实并非如此。更准确地说，他从一开始就错了。"
→ 三连否定-纠正，LLM 在和自己辩论。直接删掉前两句否定，从"他从一开始就错了"开始。
```

#### 1.4 破折号解释句式

```
单章阈值: 5 次 | 严重度: medium
命中模式: "——他/她/它/这/那"（破折号后接代词展开解释）

适度使用是风格，超过 5 次是癖好。每一处反问："没有这个破折号解释，意思会缺失吗？"——不会就删。
```

#### 1.5 公式化副词

```
单章阈值: 4 次 | 严重度: high
命中模式: "不由得""不禁""忍不住""不由自主地""下意识地""鬼使神差地"

✅ 合格：
"他后退了一步。"
→ 没有"不由得"，动作干净

"他笑了一声，然后又笑了一声。"
→ 没有"不禁"，但"又"传达了情不自禁

❌ 不合格：
"他不禁后退了一步，忍不住想起了昨天的事，不由自主地握紧了拳头。"
→ "不禁""忍不住""不由自主"三连——一个动作配一个副词。全部删掉，句子立刻干净："他后退一步。昨天的事。拳头收紧。"
```

#### 1.6 "一边……一边"句式

```
单章阈值: 2 次 | 严重度: medium

✅ 合格：
"他一边煮面一边接电话，水开了，烫到手，电话掉进锅里。"
→ 一次"一边"服务于后面的意外。保留。

❌ 不合格：
"他一边走路一边看手机一边吃东西一边听歌。"
→ 四个"一边"——动作描写扁平化。拆成独立动作或突出一个主动作
```

#### 1.7 "就在这时/正在这时/突然"句式

```
单章阈值: 5 次 | 严重度: medium

这是常见转场手法，阈值较高，但每 500 字出现 2 次以上暴露模板感。
修复：删掉过渡词直接写事件。"就在这时，门开了" → "门开了。"
```

#### 1.8 身体部位情绪模板

```
单章阈值: 6 次 | 严重度: high
命中模式: "眼神/目光/瞳孔/心头/心里/心中/心底/手心/指尖/喉咙" + "一/微/暗/沉/紧/颤/抖/缩/松/暖/寒/凉/热/酸"

这是 LLM 最顽固的另一个癖好——把所有情绪都诉诸固定的身体部位反应模板。写了三章读者已经能预测你的身体反应了。

✅ 合格：
"他把打火机按了三次才点着。"
→ 恐惧不在"喉咙发紧"里，在打火机点不着的细节里

"她把那碗粥推回来，又加了一勺糖。"
→ 温情不在"心里一暖"里，在"加了勺糖"的动作里

❌ 不合格：
"他看到消息，心里一沉。她转身走了，他心里一酸。孩子喊了一声爸爸，他心里一暖。"
→ "心里一沉""心里一酸""心里一暖"——三连身体情绪模板。每个情绪都被打包进同一个公式。
```

### 2. 其他句式规则

#### 2.1 "随着"开头

"随着脚步声靠近，他的心跳加速" → "脚步声靠近。心跳加速。" 删掉"随着"，把两件事拆成两个句子，因果自现。

#### 2.2 "让/令/使"句式

"这一幕让他想起了昨天" → "他想起了昨天。" 删掉"让"，让主语直接行动。"让"是作者在操纵角色，角色自己会行动。

#### 2.3 "当"句式

"当他走进房间时，所有人看着他" → "走进房间。所有人看着他。" 删掉"当……时"，把从句变成独立的动作句。

#### 2.4 "了"的滥用

| 规则 | 阈值 |
|------|------|
| 每段"了"的数量 | ≤ 3 个 |

过度使用：↓
> "他放下了杯子，走到了窗前，打开了窗户，看了看外面的街道，然后叹了口气。" ← 5 个"了"连成一条

修复：删掉一半"了"，换成短句——"他放下杯子。杯底磕在桌上。起身走到窗前。"

#### 2.5 感知词掩盖

"看到/听到/闻到/感到"后接抽象描述 = 用感知词逃避具体描写。

```
✅ 合格：
"门缝里透出油烟味和电视声。"
→ 没有"他闻到""他听到"，但感官信息直接打在读者脸上

❌ 不合格：
"他看到房间里一片狼藉，感到一阵强烈的不安，听到远处传来了奇怪的声音。"
→ "看到""感到""听到"三个感知词——每个都在告诉读者"这是角色的感官"，而不是让读者直接体验。修复：删掉感知词，直接写狼藉、不安的身体反应、奇怪的声音
```

#### 2.6 连续同结构开头

```
规则: 禁止连续 4 句以上以相同主语或连词开头
重点检查: 他 / 她 / 这时 / 接着 / 然后 / 于是 / 便 / 就 / 却 / 但

✅ 合格（句式混合）：
"他放下酒杯。方岩没接。
'你认真的？'
手指在桌面上敲了三下。他等到敲击声停了才开口。"
→ 主语在变（他→方岩→对话→手指→他），句式在变

❌ 不合格（连续同主语）：
"他走到窗前。他看了看外面。他转过身。他叹了口气。他坐了下来。"
→ 连续 5 句"他……"开头。修复：变换主语或句式——"窗外很安静。转身的时候，膝盖撞到了茶几。坐下来。叹气声比预期的长。"
```

### 句式扫描方法

句式违规不像疲劳词可以 Ctrl+F（部分可以），建议：

1. 打印正文或换字体阅读（换字体可以打破脱敏）
2. 用笔逐段标注句式开头（连续 4 句同主语画圈）
3. 搜索正则：`不是.*(而是|是)`、`没有.*(只是|而是)`、`不由得|不禁|忍不住|不由自主|下意识`
4. "了"扫描：每段数"了"，超 3 个标记

## 三、对话规则

### 1. 对话标签

```
规则1: 对话标签以"说/道/问"为主，占对话标签的 80% 以上
规则2: 禁止副词修饰对话标签——"他愤怒地说"是 AI 味经典标志

✅ 合格：
"'别碰那个。'陆征说。
方岩停在柜子前面，手悬在半空。'你这里面有东西？'
陆征没回答。他从椅子上站起来，走到柜子前，拿走了方岩手底下的文件夹。
'有意思。'方岩收回手，笑了。"
→ 对话标签只有"说"，但情绪全在动作里。"方岩收回手，笑了"比"他意味深长地说"有力得多

❌ 不合格：
"'别碰那个。'陆征严肃地说。
'你这里面有东西？'方岩好奇地问道。
'没什么。'陆征冷淡地回答，继而补充道，'只是旧文件。'"
→ 三个对话标签三个副词。"严肃地""好奇地""冷淡地"——每个都在告诉读者应该怎么感受，而不是让读者自己感受
```

### 2. 对话间隔

```
规则: 对话之间必须有动作、环境或内心活动作为节奏间隔
连续 8 句纯对话（无任何间隔）= AI 味高发区

✅ 合格：
"'你真的要去？'
'嗯。'
'什么时候回来？'
他没看她。杯子里的水喝完了，他还端着空杯子。
'不一定。'
她等了一会儿。然后拿起外套。"
→ 对话之间穿插了动作（端着空杯子）、停顿（她等了一会儿）、动作（拿起外套）

❌ 不合格：
"'你去哪？'
'出去。'
'什么时候回来？'
'不知道。'
'你到底怎么了？'
'没事。'
'你每次说没事的时候都有事。'
'我说了没事。'"
→ 8 句纯对话，像读剧本。读者不知道人物在哪、在干什么、表情是什么
```

### 3. 感叹号和省略号

```
规则: 不能用 "!" 来替代情绪描写；每段最多 1 个感叹号

✅ 合格：
"'放下那把刀。'
他说得很平静。但她听到了。她把手里的东西放下来，动作很慢，像在水里移动。
→ 紧张感在节奏和细节里，不在感叹号里

❌ 不合格：
"'放下那把刀！！！'
'我不放！你走开！！'
→ 用感叹号替代情绪描写——读者只看到标点，看不到人
```

### 4. 说明书式对话

```
规则: 角色不能互相告知已知信息

✅ 合格：
"'苏局那个案子，你当时在场？'
方岩盯着他看了两秒。'你想问什么。'
→ 两个角色都懂彼此在说什么。对话有信息差和试探

❌ 不合格：
"'你还记得三年前那个案子吗？就是张明被指控受贿的那个。当时你在场，是副队长，我和你一起出过现场，后来案子被压下来了。'
'是的，我记得。那个案子确实很可疑。'
→ 两个角色在互相告知对方已经知道的信息——这是写给读者看的，不是角色在对话
```

### 5. 对话意图

```
规则: 每段对话至少有一个意图——说服、试探、隐藏、拒绝、转移话题、获取信息、施加压力

无意图的对话（闲聊式、问候式）= 填充，应删除或改为叙事
```

## 四、设定合规校验

**参照基准：** `settings/world-setting.yaml`、角色人设文档、题材规则（genre-taboos）

正文的设定合规性无法从正文自身判断——必须绑定前置设定文件做对标校验。

### 1. 世界观规则遵守

正文中出现的能力、现象、事件是否符合世界观设定的核心规则？

- 能力边界——角色的修为/魔法/科技能力未超出设定范围
- 规则一致性——世界的基本规则（社会等级、物理法则、力量体系）未被违反
- 时空逻辑——地理、时间线与设定一致，无矛盾

### 2. 角色一致性（OOC 检测）

角色的言行、决策、情绪反应与其人设文档一致：

- 核心性格——本章中的性格表现与人物设定一致（内向不会突然主动搭讪）
- 能力边界——角色不做超出设定能力的事
- 知识边界——角色不知道其不可能知道的信息
- 关系状态——角色间关系与人设相符

### 3. 题材禁忌

对照题材的 taboos 列表逐条确认正文未触犯。

### 4. 信息连续性

与前置章节无矛盾：
- 前章已破坏的物品/建筑在本章未无故复原
- 上章结尾的角色身体/情绪状态与本章衔接合理

## 五、章纲兑现校验

章纲 memo 承诺的事情，正文做到了吗？

### 1. required_changes 逐条核对

章纲 memo 的 `required_changes` 是本章最核心的承诺。正文写完后，逐条过：

| 检查项 | 方法 |
|--------|------|
| 每条"从什么变成什么"在正文中实际发生了吗？ | 在正文中找出对应段落。找不出来 = 未兑现 |
| 改变的类型匹配吗？ | 章纲写的是"信息改变"，正文不能只是一句"他知道了很多" |
| 读者能感受到改变吗？ | 不是作者知道变了——是纸上写出来的东西让读者知道变了 |

```
✅ 合格：
章纲 required_changes:
  - "信息改变：确认苏棠非主动离开——房间状态有 3 个矛盾细节"

正文：
"床没铺。被子和枕头卷成一团堆在床脚，不像有人准备出门。床头柜上手机充电器还插在墙上，白色的线垂下来，末端的接口在床和柜子之间的缝里晃。
他去拉冰箱。冷冻室第一格，半盒速冻饺子，表面结了霜，不是最近放进去的。冷藏室三个格子都是满的。"
→ 三处矛盾细节（床/充电器/冰箱）都写出来了。读者能自己得出结论"不像主动离开"

❌ 不合格：
章纲 required_changes:
  - "信息改变：确认苏棠非主动离开"

正文：
"陆征检查完房间，发现了一些证据，意识到苏棠的失踪没有那么简单。"
→ "发现了一些证据"——什么证据？"没有那么简单"——怎么不简单？正文没写出来，只是用总结句带过了
```

### 2. payoff_plan 逐条核对

| 检查项 | 方法 |
|--------|------|
| must_resolve 每条钩子都在正文中收束了吗？ | 找到收束的具体段落。找不到 = 漏写 |
| must_hold 每条钩子没有被意外泄露？ | 正文没有透露被压住的信息 |
| partial_advance 每条钩子推进到位了吗？ | 对比章纲的 advance 描述和正文的推进程度——推进不足或推进过头都是偏离 |

### 3. micro_payoffs 兑现

章纲的 micro_payoffs 至少 1 个必须兑现。在正文中确认：
- type 匹配（info/relationship/emotion/clue/ability/resource/recognition）
- location 匹配（该在的位置是否放了微兑现）
- 读者能在该位置感知到"有收获"

### 4. key_choices 兑现

章纲中列出的角色关键抉择，正文是否写出来了？抉择的动机与章纲分析的一致吗？

### 5. prohibitions 未触犯

逐条对照 prohibitions，确认正文没有触犯。如果有触犯 → 立即修改。

## 六、钩子兑现检查

钩子管理不在全局 hooks.yaml 中——真相源在各章 chapter.yaml 的 hooks 字段和 memo.payoff_plan。正文检查时确认：

- [ ] **本章新埋的钩子**在正文中确实出现了（有具体场景，不是一句暗示）
- [ ] **本章收束的钩子**在正文中确实完成了闭环（收束有明确的段落，读者不会错过）
- [ ] **未发现"忘了写"的章纲承诺**——章纲 payoff_plan 列了但正文里找不到对应内容的
- [ ] **钩子收束不是"一句话带过"**——如果铺垫了 3 章的钩子用一句话收束（"他明白了真相"），等于没写

```
✅ 合格：
铺垫 3 章的钩子 hook-003（"快递盒里到底是谁的遗物"）本章收束：
正文中有一段完整的开盒场景——盒子里是什么、怎么打开的、角色看到后的反应、对后续的影响。
→ 收束有分量，读者不会觉得"就这？"

❌ 不合格：
铺垫 3 章的钩子本章收束：
正文："他打开快递盒，里面是一只能证明对方身份的旧怀表。他知道，该去找那个人了。"
→ 三章铺垫用两句话收束——读者会觉得被耍了
```

## 七、阅读体验校验

**参照基准：** 章纲 爽点设计、情绪曲线、钩子规划

### 1. 爽点落地

章纲设计的爽点类型和侧重，在正文中是否产生了预期的阅读效果：

- 爽点有具体落地场景——不是"主角变强了"的概括，是读者能看到的变强过程/结果
- 爽点情绪释放与章纲定位一致——暧昧爽有心动的画面，打脸爽有解气的时刻
- 爽点有足够的细节积累，不是"写到了但没感觉"

### 2. 情绪曲线匹配

正文实际的情绪起伏与章纲设计的曲线是否一致：

- 抑→扬→收三段与章纲对齐——该压抑的段落没有写得太松弛
- 转折点的情绪渲染到位——该紧张的地方紧张了，该温情的地方温情了
- 无意外跑偏——章纲"压抑收尾"，正文不能写成"阳光结局"

### 3. 章尾牵引力

章末是否让读者有继续读的动力：

- 有情绪缺口（未解决/未挑明/未收束的线）
- 没有"安全着陆"（所有冲突在本章内完美解决）
- 读者读完有"然后呢？"的感觉

### 4. 阅读获得感

读者在本章至少有一项"有收获"：

- 获得了新信息（剧情推进）
- 获得了情绪体验（笑了/紧张了/触动了）
- 获得了关系进展（角色关系变化或变化预告）

## 八、技术质量

### 1. 视角一致

```
规则: 全章使用单一视角（除非有明确的双视角设计且章纲已标注）
第三人称有限视角：只能写视角角色的所见所闻所感，不能跳到其他角色的内心

✅ 合格（第三人称有限，陆征视角）：
"方岩在办公桌后面，没抬头。陆征不知道他在看什么，桌上那叠文件挡住了他的视线。"
→ 始终在陆征的认知范围内——"不知道"和"挡住了视线"是从陆征的眼睛看的

❌ 不合格（视角跳跃）：
"陆征走进办公室。方岩在办公桌后面，心想这人怎么又来了。陆征不知道方岩刚才被领导骂了。"
→ "方岩心想" = 跳到了方岩的内心。第三人称有限视角下，陆征不可能知道方岩在想什么，也不可能知道方岩刚才被领导骂了
```

### 2. 时态/人称一致

```
规则: 全章时态一致、人称一致。不允许中间跳变
中文通常默认现在/过去混合叙述，但同一场景内不能来回跳

✅ 合格：
"雨下了一夜。早上还在下。"
→ 时间线清晰

❌ 不合格：
"雨下了一夜。早上还在下着。他正在往窗外看。后来他会明白这场雨是什么意思。"
→ "正在"和"后来他会"时态冲突——到底是正在发生的还是回望的叙事？
```

### 3. 段落长度

```
规则:
- 单段不超过 200 字（手机屏幕一屏左右）
- 单段不低于 20 字（除非是有意用的单句段制造节奏）
- 单句段占比建议 25%-45%（用于节奏打断）
- 4 个逗号以上必须考虑拆段

✅ 合格：
段落 A（180 字，描写场景）
段落 B（12 字，节奏打断——"他等了很久。"）
段落 C（120 字，对话+动作）
段落 D（8 字，节奏打断——"没人来。")
→ 长短交替，有呼吸感

❌ 不合格：
一段 400 字无分段——手机上需要翻两页才能读完。主 Agent 读到第 200 字已经忘记开头在写什么。
或者：连续 5 段单句段——节奏碎成了渣，读者的注意力无法落地。
```

## 九、内容质量

### 1. 上帝视角摘要和元叙述

```
规则: 正文中不得出现"本章讲述了""故事还在继续""与此同时（总结性使用）""在这一章中"等元叙事语句
正文不得出现画外音式的总结和评价

✅ 合格：
"走出公寓的时候，他回头看了一眼。楼道的灯灭了。"
→ 只写事件，不写"这一章就此结束"

❌ 不合格：
"这一章就这样结束了。陆征还不知道，更大的风暴正在等着他。故事，还在继续。"
→ 典型的 LLM 章末收尾——元叙事+"故事还在继续"。全部删掉
```

### 2. 预设读者反应

```
规则: 正文不应指导读者应该怎么感受
不写"让人感到""令人不禁""读者看到这里一定……"

✅ 合格：
"她把那封信折了又拆，拆了又折。"
→ 写动作，不解释"这让人感到她的犹豫"

❌ 不合格：
"这一幕让人忍不住为她感到心酸。读者看到这里一定也会觉得她的选择令人动容。"
→ 预设读者反应 + 替读者感受。全部删掉，只保留事件和动作
```

### 3. 无夹带非正文内容

检查正文中不应存在的内容：
- [ ] 引导语（"下面开始写正文""第X章正式开始"）
- [ ] 文末总结（"以上就是第X章的内容""这章主要写了"）
- [ ] 括号注释（"（此处可以写得更好）""（这一段可以考虑删掉）"）
- [ ] subagent 的自我评价（"我觉得这段描写不错""这里呼应了前面的"）

### 4. 信息准确

- [ ] 角色名拼写一致（不能陆征/陆正/陆铮混用）
- [ ] 时间线不矛盾（昨天说的"一周前"到今天不能变成"三天前"）
- [ ] 物品状态不矛盾（上章碎了的花瓶本章不能完好在桌上）
- [ ] 人物关系不矛盾（上章"第一次见面"本章不能"多年前就认识"——除非这是设计好的伏笔）

## 十、风格一致性

### 1. 与写作风格对齐

正文写完后，对照写作风格指南 `references/writing-style.md` 的四字段：

| 字段 | 自检方法 |
|------|---------|
| 叙事身份 | 写作身份是否在正文中体现？如果身份是"克制白描"，正文是否出现了煽情长段？ |
| 核心原则 | 逐条检查正文是否遵守了核心原则。某条原则违反了 → 标记 |
| 常见错误 | 这些都是"AI 易犯的错误"——正文中有没有踩？踩了 → 标记并修改 |
| 描写技法 | 推荐的描写技法有没有在正文中出现？不是全部都要用，但至少要能看出风格方向 |

### 2. 题材味道

如果项目配置了 `genre_profile`，检查正文是否传递了该题材的核心味道：
- 悬疑：读者是否感受到了"想知道真相"的驱动力？
- 修仙：是否有世界观特有的力量体系细节？
- 都市：日常细节是否具有真实感（地名、物价、时间感）？
- 历史：是否避免了现代用语穿越？

题材禁忌（genre_taboos）逐条确认未触犯。

### 3. 行文节奏

- [ ] 叙事段（描写/内心）和场景段（对话/动作）交替出现。不允许连续 1000 字无对话或连续 500 字纯对话
- [ ] 情绪强弱有波动——同一章内不能全程高能或全程平静
- [ ] 情绪最高点不在章末——章末是钩子，高潮在章末前 1/4 处附近（读者需要余味消化）

## 正文验收总表

走完上面全部规则后，逐项过：

### AI 味——疲劳词
- [ ] **副词类**（忽然/突然/竟然/居然/默默/微微/渐渐/逐渐/依旧/仍然）各自未超阈值
- [ ] **动词类**（露出/勾起/泛起/浮现/陷入/涌起/充斥）各自未超阈值
- [ ] **形容词/抽象词类**（复杂/微妙/难以言喻/莫名/异样/浓重）各自未超阈值
- [ ] **连接词类**（然而/不过/于是/随即/紧接着/与此同时）各自未超阈值
- [ ] **身体反应类**（瞳孔一缩/身体一僵/倒吸一口凉气/呼吸一滞）各自未超阈值

- [ ] **英文疲劳词**（如适用）已检查

### AI 味——句式违规
- [ ] **"不是而是"句式** 不超过 3 次
- [ ] **"没有X只是Y"句式** 不超过 3 次
- [ ] **否定-纠正句式** 不超过 2 次
- [ ] **破折号解释句式** 不超过 5 次
- [ ] **公式化副词**（不由得/不禁/忍不住/不由自主/下意识/鬼使神差）不超过 4 次
- [ ] **"一边一边"句式** 不超过 2 次
- [ ] **"就在这时"句式** 不超过 5 次
- [ ] **身体部位情绪模板** 不超过 6 次
- [ ] **"随着"开头** 已清理
- [ ] **"让/令/使"句式** 已清理
- [ ] **"当"句式** 已清理
- [ ] **"了"** 每段不超过 3 个
- [ ] **感知词掩盖**（看到/听到/闻到/感到后接抽象描述）已清理

### 对话规则
- [ ] **对话标签** 以"说/道/问"为主（>80%），无副词修饰对话标签

- [ ] **感叹号** 每段不超过 1 个，无"!"替代情绪描写
- [ ] **无说明书式对话**——角色不互相告知已知信息
- [ ] **对话有意图**（说服/试探/隐藏/拒绝/转移话题等），无填充式闲聊

### 设定合规校验
- [ ] **世界观规则遵守**——能力/规则/时空逻辑未违反世界设定
- [ ] **角色 OOC 检测**——言行、决策、情绪与人设文档一致

### 快速嗅探（Agent 秒级执行）

先做这四项，全过再走完整检查：

1. **通读正文**——标记读起来拗口、像 AI 写的句段
2. **遮住章纲**——判断本章结束时什么东西变了（说不出来 → 章纲兑现有问题）
3. **搜索高频词**——"突然""竟然""于是""然而"各超 4 次 → 疲劳词警报
4. **看最后 300 字**——章末有牵引力（想知道后续）还是安全着陆（故事讲完了）

### 执行方式

**优先级：Agent Teams > subAgent > 自检**

1. **Agent Teams（最优）**：开独立 reviewer agent，传入本章正文 + 本章章纲（chapter.yaml）+ `references/writing-style.md` + `anti-ai.yaml`（如存在）+ 本清单。reviewer agent 逐项核查，对每项给出"通过/不通过 + 证据（引用正文原文）"。疲劳词扫描和句式扫描让 agent 自动做（正则匹配），人工只做确认。
2. **subAgent（次优）**：同上，使用 subagent 执行审查。subagent 对"内容质量"和"风格一致性"的判断力弱于完整 agent，建议 AI 味检测（疲劳词/句式规则）用 subagent，内容质量部分主 Agent 亲自过。
3. **自检（兜底）**：严格按本清单逐项过。特别注意三个高危盲区：
   - **疲劳词脱敏**：写完正文 30 分钟后自检——刚写完时人对自己的文字完全脱敏，看不出问题。换字体、换设备（从电脑换到手机）、或打印出来看——任何一种方式都能打破脱敏。
   - **memo 承诺自欺**：最容易对自己说"虽然没有明写，但暗示了"。检查方法是：指着正文的具体段落，能不能说"这句话兑现了章纲的这条承诺"？指不出来就是没兑现。
   - **对话节奏盲区**：连续的大段对话最容易在自检时被跳过——因为读起来"挺快的"。检查方法是：把所有对话单独提取出来，看中间隔了几次动作/环境。连续 8 句纯对话以上 = 必有问题。

**拿到结论后怎么做：** 诊断结果指向问题环节——AI 味重 → 查 prompt 模块 5 的约束是否到位；memo 未兑现 → 查种子核心事件是否覆盖；视角跳跃 → 查 prompt 视角锁定字段。修完对应环节再生成。**但最终正文是否归档，由作者判断，不是由清单判定。**
