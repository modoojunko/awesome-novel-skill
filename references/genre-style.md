# 题材选择参考

> 在设定阶段的第一步选题材。题材决定世界观的基础方向、角色的定位基准、写作风格的默认值。选完题材再进世界观讨论。

---

## 原则 1：题材是读者拿到书时的第一判断轴

**规则：** 题材不是贴标签——同一题材可以写出完全不同的故事。题材定义的是"读者读这本书在等什么"和"多快给到读者"。

```
题材 → 世界观（世界观提供题材满足类型的空间）
     → 角色（题材决定角色类型定位）
     → 写作风格（题材自带 style_blueprint 作为起点）
```

✅ 正确：
> 选仙侠，读者期待境界突破、秘境夺宝、打脸反杀。
> 选悬疑，读者期待线索推进、反转、真相揭露。
> → 题材告诉你该给读者看什么。

❌ 错误：
> "我的故事什么题材都沾一点，不分那么细。"
> → 不分题材就没法管理读者预期。可以叠加，但得有一个主类型。

**判断标准：** 题材选完后，能不能说清"读者翻开第一章时心里在等什么"？说不出来 → 没选透，继续聊。

---

## 原则 2：先选题材，再定其他

**规则：** 题材是 Phase 2 第一步。世界观、角色、写作风格都贴着题材聊。

AI 直接说：
> "我们先把题材定了，再聊世界观和角色。题材定了，后面的讨论才有方向。"

如果作者说"我不知道选什么"：
> "没关系，说说你脑子里大概是什么方向的故事？——古代修仙、现代都市、未来科幻、古代权谋、悬疑破案、还是末世生存？"

如果作者说"我想写个融合的"：
> "可以。先选主类型，后续可以叠加。主类型定框架，叠加类型调参数。"

**判断标准：** 作者是否理解"题材先定"是流程的第一步，不是可选的？不清楚 → 用上面的话术说明为什么。

---

## 原则 3：选主类型，不纠结完美匹配

**规则：** 24 种预置类型在 `genre-corpus/index.yaml` 中注册。没有完全匹配的类型 → 选最接近的基类。

| 作者说了 | 怎么处理 |
|---------|---------|
| "就是仙侠" | 选 xianxia |
| "仙侠+悬疑" | 选 xianxia，后续可手动叠加悬疑要素 |
| "我也不知道算什么" | 读 index.yaml 的清单，让作者挑第一印象最接近的 |
| "不想选" | "暂不选择"是有效选择，后续慢慢明确 |

✅ 正确：
> "我想写一个在修仙世界里破案的故事。"
> → 选 xianxia（世界观是仙侠），悬疑要素可以叠加到 genre_config。

❌ 错误：
> "没有完全匹配的类型，那先不选了，以后再定。"
> → 以后不会定的。选最接近的基类，比不选强。

**判断标准：** 24 种里能不能挑出一个最接近的？能 → 就它。不能 → "暂不选择"也是答案。

---

## 原则 4：题材自带三套配置，选定即注入

**规则：** 选好类型后，三套配置直接合并进 `settings/genre-setting.yaml`。作者确认即可，不需要从零讨论。

| 配置 | 内容 | 作用 |
|------|------|------|
| **style_blueprint** | voice、atmosphere、genre_elements、pov_description | 写作风格讨论的起点（不是约束） |
| **genre_config** | satisfaction_types、pacing_rules、anti_cliches | 方向提案和正文写作时校验读者预期 |
| **genre_taboos** | 该类型忌讳的写法 | 章纲和正文绕道 |

展示给作者的确认格式：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  已选题材：{genre.label}

  style_blueprint（写作风格起点）：
  {voice 摘要}

  genre_config（节奏和满足类型）：
  · 满足类型：{satisfaction_types 列表}
  · 节奏模式：{pacing_rules 摘要}
  · 避免套路：{anti_cliches 列表}

  类型禁忌：
  {genre_taboos 列表}

  确认使用这个配置？还是需要调整。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

作者可能的选择：
- "确认" → 写入选定到 genre-setting.yaml
- "XX 不想用" → 去掉对应项
- "加 XX 类型的要素" → 从另一个类型的 corpus 取对应配置追加

---

## 写进文件

```yaml
genre_profile: xianxia  # 选定的类型 id，或 "暂不选择"
config:
  satisfaction_types:
    - <满足类型1>
    - <满足类型2>
  pacing_rules:
    chapter_types:
      action: 40%
      setup: 25%
      development: 20%
      payoff: 15%
    tension_cycle: <描述>
    lull_limit: <描述>
  anti_cliches:
    - <套路1>
    - <套路2>
```

## 案例

### 案例一：仙侠玄幻

```yaml
genre:
  genre_profile: xianxia
  config:
    satisfaction_types:
      - 境界突破（硬实力跃升）
      - 秘境夺宝（资源争夺战）
      - 打脸反杀（实力碾压后反转评价）
      - 大道争锋（理念冲突 vs 实力对决）
    pacing_rules:
      chapter_types:
        - action: 40%     # 战斗/冲突/高潮
        - setup: 25%      # 铺垫/探索/信息
        - development: 20% # 修炼/准备/养成
        - payoff: 15%     # 收获/转折/揭示
      tension_cycle: 3-5章一个小高潮，10章一个大高潮
      lull_limit: 最多连续2章铺垫章，第3章必须有进展
    anti_cliches:
      - 主角跳崖必得奇遇
      - 拍卖会必装逼打脸
      - 入门测试必被看不起
      - 境界越高越要隐藏实力然后在关键时刻震惊全场
```

### 案例二：都市异能

```yaml
genre:
  genre_profile: urban
  config:
    satisfaction_types:
      - 能力觉醒（获得力量/解锁新能力）
      - 隐藏身份（普通人不知道异能世界，主角在两者间周旋）
      - 体制 vs 地下（特管处和暗网的灰色地带博弈）
      - 日常融合（超能力用在日常场景里的反差趣味）
    pacing_rules:
      chapter_types:
        - action: 25%     # 异能对抗/追逃/冲突
        - setup: 30%      # 信息收集/世界探索
        - development: 25% # 日常/关系/缓慢成长
        - payoff: 20%     # 反转/阴谋揭露/对决
      tension_cycle: 4-6章一个小高潮，15章一个大高潮
      lull_limit: 最多连续3章日常，第4章必须有事件推进
    anti_cliches:
      - 主角觉醒就是S级
      - 特管处全是官僚废物
      - 异能者穷得吃不起饭（都有超能力了还付不起房租？）
      - 女人负责受伤和被拯救
```
