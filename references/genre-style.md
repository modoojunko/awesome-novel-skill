# 题材设定参考

> 题材设定的要求、方法论和案例。agent 读完知道"题材设定要做到什么样、怎么引导作者选、填完后怎么检查"。

---

## 题材配置是什么

题材是**读者拿到书时的第一判断轴**——"这是什么类型的故事"。读者选仙侠期待的是境界突破、打脸反杀、秘境夺宝；选悬疑期待的是线索推进、反转、真相揭露。题材配置决定了两件事：**给读者看什么**（爽点/满足类型）和**多快给到读者**（节奏）。

这不是"贴标签"——同一种题材可以写出完全不同的故事。题材配置是让 agent 知道"这个题材的读者在等什么"，然后在该给的时候给到。题材不是限制创意的框架，是管理读者预期的工具。

## 作用（下游为什么需要它）

- **提示词注入**——genre 的 prompt_segment 在 Phase 3.3 注入提示词，约束 subagent 的叙事选择
- **节奏基准**——pacing_rules 决定章节类型分布和张力周期，方向提案和章纲阶段用来校验节奏是否合理
- **爽点/满足类型**——satisfaction_types 告诉 agent "读者读这本书是为了什么"，方向提案时用来检验提案是否踩在读者期待上
- **反套路**——anti_cliches 标记读者已经厌倦的模式，避免被评价"太套路"

## 理解方法——三步选型法

### 第一步：确定题材类型

24 种预置类型在 `genre-corpus/index.yaml` 中注册。引导作者选择：

**没有完全匹配的类型怎么办？** 选最接近的基类 + 手动叠加变体。例如"仙侠+悬疑"= 选 xianxia，然后在 genre_config 中追加悬疑要素的 satisfaction_types 和 pacing_rules。

**不想选怎么办？** 没问题——"暂不选择"也是有效选择。后续写作中慢慢明确。

### 第二步：配置题材参数

选好类型后，从类型档案合并 genre_config。配置项包括：

| 配置项 | 问题 | 设计要点 |
|--------|------|---------|
| **satisfaction_types** | 读者读这本书的爽点是什么？ | 3-5 种给读者提供的情绪满足。方向提案时对照——如果提案不在任何一个满足类型上，它可能跑偏了 |
| **pacing_rules** | 故事的节奏是什么样？ | 包括章节类型比例（action/setup/development/payoff）、张力周期（几章一个小高潮）、铺垫上限（连续几章铺垫必须给进展） |
| **anti_cliches** | 哪些套路读者已经烦了？ | 列出该类型常见的狗血桥段，正文中遇到类似情境时绕道走 |

### 第三步：和世界观/风格锚定

题材和世界观/写作风格必须自洽：

- 题材的 satisfaction_types → 世界观提供这些满足的空间？（比如"秘境夺宝"需要地理上有秘境设定）
- 题材的 pacing_rules → 写作风格的 core_principles 不冲突？（比如"最多连续 2 章铺垫"和"每章必须有冲突推进"应该一致）
- 题材的 anti_cliches → 写正文时遇到对应桥段，知道要改写

## 必须包含的内容

- genre_profile — 选定类型（或"暂不选择"）
- satisfaction_types — 3-5 种读者满足类型（每种一句话描述）
- pacing_rules — 章节类型比例 + 张力周期 + 铺垫上限
- anti_cliches — 至少 3 条该类型常见套路（遇到时绕道或翻新）

## 输出去哪

`settings/genre-setting.yaml`。

## 案例

### 案例一：仙侠玄幻

```yaml
genre:
  genre_profile: xianxia
  config:
    satisfaction_types:
      - 境界突破（硬实力跃升）
      - 秘境夺宝（资源争夺战）
      - 打脸反杀（实力碾压后反转评价）
      - 大道争锋（理念冲突 vs 实力对决）
    pacing_rules:
      chapter_types:
        - action: 40%     # 战斗/冲突/高潮
        - setup: 25%      # 铺垫/探索/信息
        - development: 20% # 修炼/准备/养成
        - payoff: 15%     # 收获/转折/揭示
      tension_cycle: 3-5章一个小高潮，10章一个大高潮
      lull_limit: 最多连续2章铺垫章，第3章必须有进展
    anti_cliches:
      - 主角跳崖必得奇遇
      - 拍卖会必装逼打脸
      - 入门测试必被看不起
      - 境界越高越要隐藏实力然后在关键时刻震惊全场
```

### 案例二：都市异能

```yaml
genre:
  genre_profile: urban
  config:
    satisfaction_types:
      - 能力觉醒（获得力量/解锁新能力）
      - 隐藏身份（普通人不知道异能世界，主角在两者间周旋）
      - 体制 vs 地下（特管处和暗网的灰色地带博弈）
      - 日常融合（超能力用在日常场景里的反差趣味）
    pacing_rules:
      chapter_types:
        - action: 25%     # 异能对抗/追逃/冲突
        - setup: 30%      # 信息收集/世界探索
        - development: 25% # 日常/关系/缓慢成长
        - payoff: 20%     # 反转/阴谋揭露/对决
      tension_cycle: 4-6章一个小高潮，15章一个大高潮
      lull_limit: 最多连续3章日常，第4章必须有事件推进
    anti_cliches:
      - 主角觉醒就是S级
      - 特管处全是官僚废物
      - 异能者穷得吃不起饭（都有超能力了还付不起房租？）
      - 女人负责受伤和被拯救
```

## 合格检查

| 检查项 | 方法 | 不通过处理 |
|--------|------|-----------|
| genre_profile 已选定 | 已从 24 种预置类型中选择，或确认"暂不选择" | 未选→退回讨论 |
| genre_config 已合并 | 类型档案的默认配置已合并到 genre-setting.yaml | 未合并→执行合并 |
| satisfaction_types 具体 | 读一条满足类型，能想到对应场景 | 太抽象（"精彩的故事"）→退回改可感知 |
| pacing_rules 可执行 | 张力周期和铺垫上限能用来校验方向提案 | 节奏规则是空话→退回改可操作 |
| anti_cliches 有用 | 遇到清单中的桥段时 agent 知道要改写 | 套路不常见→退回补充"真会遇到的坑" |
| 自洽 | 题材和世界观/写作风格不冲突 | 冲突→退回调整 |
