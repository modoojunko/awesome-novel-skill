# 设定阶段参考

> 设定阶段每个板块的要求、输出格式和案例。agent 读完知道"设定要做到什么样、写到哪里、长什么样"。

---

## 世界观设定

### 要求

**必须架空，不得与真实世界（地理/历史/政治）有任何重合。** 真实地名（北京、长安、巴黎）、真实朝代名（唐、宋、明）、真实历史人物一律禁止。世界观越具体越有辨识度，"架空"意味着作者可以自由定义一切。

core 三项必填。每条设定必须能用角色经历的场景呈现，不是说明书式罗列。

**geography** — 主要场景的位置关系、气候基调、地理限制
**politics** — 统治形式、主要势力/阵营、力量平衡、社会分层
**rules** — 特殊规则、力量体系、社会潜规则、限制和代价

extended（culture / history / physics / biology / sociology）按需追加——正文写着发现需要了再来补，不预设性填满。

### 输出去哪

`settings/world-setting.md`，Markdown 格式。## 二级标题分节。

### 案例

```markdown
## geography

青州城坐落于苍梧山脉东麓，三面环山一面临水。城分内外：内城是官署和富户聚居区，外城是平民市井。城外东南有青江渡口，是商旅往来的命脉。冬季湿冷多雾，夏季闷热多雨。

翻过苍梧山向西，是妖兽横行的荒原——没有人活着穿越过。

## politics

朝廷设青州府，知府陈伯远掌军政大权。城中三股势力：
- 官府——维持秩序，但力量有限。府兵五百，装备精良但士气不高
- 青帮——控制码头和市井生意，与官府表面合作暗中角力。帮主洪四海，手下三千
- 玄天宗外门——修行者势力，不干涉俗务但拥有超然地位。驻守长老一位，筑基弟子十人

三足鼎立，任何两方联合都会打破平衡。

## rules

修炼体系：引气→筑基→金丹→元婴。每级晋升需天材地宝+功法+机缘。

限制：
- 没有功法无法自主修炼
- 强行突破会走火入魔
- 金丹以上修士受天道誓约约束，不得对凡人出手

代价：每一次晋升都在消耗寿元——修行是逆天而行。
```

### 合格检查

| 检查项 | 方法 | 不通过处理 |
|--------|------|-----------|
| 架空 | 扫一遍地名/朝代名/人物名有无真实世界重合 | 发现真实地名→退回重写 |
| 完整性 | 检查 ## geography / ## politics / ## rules 三个节都存在 | 缺节→退回补全 |
| 可感知 | 读每条设定，想一个具体场景呈现它 | 想不到场景→太抽象，退回 |
| 规则代价 | rules 节中有无限制条件或代价 | 主角可以随心所欲→退回加限制 |
| core 够用 | 仅凭 core 三项，第一章是否已经可以动笔 | 不行→补 core，不要急着填 extended |

## 角色设定

### 要求

每个主要角色（protagonist / antagonist / supporting）独立文件。七字段填到能推断行为。

**cognition** — 如何理解和判断事物（有认知偏差或思维模式）
**worldview** — 相信什么、不相信什么（可与其他角色冲突）
**self_identity** — 如何看待自己（可以有认知偏差）
**values** — 什么事会破例、什么不可做
**abilities** — 实际能力（有具体表现，不是头衔）
**skills** — 可习得技术（有来源和局限）
**environment** — 出身+当前处境（决定行为基线）

### 输出去哪

`settings/character-setting/<拼音id>.yaml`，每角色一文件。

### 案例

```yaml
id: shen-mu
name: 沈牧
story_role: protagonist

cognition: 习惯从利益角度分析一切——任何人做事都有动机，找不到动机就说明信息不够。不信任"善意"这种解释。

worldview: 强者制定规则，弱者遵守规则。想改变处境就要变强，抱怨没有用。

self_identity: 觉得自己是个冷静理性的谋定后动者——但实际遇到亲友遇险时第一反应是冲上去，理性是事后找补的。

values: 不背叛兄弟。不欺压平民。可以利用对手但不能侮辱对手的尊严。

abilities: 能在十步内察觉杀气来源；单手拉八十斤弓，三箭连发不脱靶；过目不忘，但需要时间回忆。

skills: 青帮底层混出来的市井生存术（听脚步辨人数、看衣着断身份）；跟账房先生学过珠算和记账。

environment: 苍梧山脚猎户之子，父母死于妖兽袭村，被青帮老掌柜收养。在码头帮工长大，见过三教九流。
```

### 合格检查

- [ ] 每角色通过"生气表现+缺点"诊断
- [ ] cognition/worldview 推得出场景行为
- [ ] abilities 有具体表现和边界
- [ ] 配角有自驱动目的（不为主角存在）
- [ ] 主角有实质性缺点（不是"太正直"）
- [ ] 文件名拼音 id，不用中文

## 写作风格

### 要求

四字段互相锚定，缺一不可。默认值只是起点，至少一个字段经讨论调整。

**role** — AI 在写作中扮演的叙事身份（冷静旁观？亲密叙述？带讽刺的观察者？）
**core_principles** — 不可违背的信条（节奏底线、禁忌、风格承诺）
**possible_mistakes** — AI 易犯的错误列表（已知会出问题的模式）
**depiction_techniques** — 推荐的描写手法和层次

### 输出去哪

`settings/writing-style.yaml`，四字段写入顶层。

### 案例

```yaml
role: 一个冷静但不冷漠的叙事者——客观描述角色行动和环境，不替角色感受。
  让对话和动作传递情绪，不做心理分析旁白。

core_principles:
  - 每章结尾必须有钩子，不能平铺直叙收尾
  - 不在对话中插入旁白解释角色情感——让读者从语气和用词判断
  - 对话为主、描写为辅，一段叙事后尽快回到对话推进
  - "突然""猛然""瞬间"每章不超过3次
  - 不写无意义的日常——每个场景必须有冲突或信息推进

possible_mistakes:
  - 过度使用面部表情描写（"他皱了皱眉""她嘴角上扬"）
  - 心理活动用"他感到""他意识到""他心想"开头
  - 战斗场面写成一招一式的慢动作
  - 环境描写独立成段脱离剧情——环境应该和角色的行动/情绪交织
  - 每段开头"他/她"连续超过3次

depiction_techniques:
  - 情绪通过动作表达：紧张时抠指甲、愤怒时反而轻声细语
  - 环境感知融入角色行动：不是"街道很吵"，是"他得扯着嗓子对方才能听见"
  - 对话用语气和用词差异区分角色——不靠"他说""她说"标签
  - 时间流逝用环境变化暗示：阳光角度、影子长度、温度变化
  - 战斗用结果和后果描写代替过程："等声音停下来，他的左臂已经不自然地垂在身侧"
```

### 合格检查

- [ ] 四字段全部有值
- [ ] 至少一个字段经讨论调整
- [ ] possible_mistakes 有具体反模式（不是"不要写太AI"）
- [ ] core_principles 可执行（3-5条硬规则）

## 题材配置

### 要求

已选类型档案 → genre_config 合并。未选类型 → 手动配置题材要素。

### 输出去哪

`writing-style.yaml#genre` 字段。

### 案例

```yaml
genre:
  genre_profile: xianxia
  config:
    satisfaction_types:
      - 境界突破（硬实力跃升）
      - 秘境夺宝（资源争夺战）
      - 打脸反杀（实力碾压后反转评价）
    pacing_rules:
      chapter_types:
        - action: 40%     # 战斗/冲突/高潮
        - setup: 25%      # 铺垫/探索/信息
        - development: 20% # 修炼/准备/养成
        - payoff: 15%     # 收获/转折/揭示
      tension_cycle: 3-5章一个小高潮，10章一个大高潮
      lull_limit: 最多连续2章铺垫章，第3章必须有进展
    anti_cliches:
      - 主角跳崖必得奇遇
      - 拍卖会必装逼打脸
      - 入门测试必被看不起
      - 境界越高越要隐藏实力然后在关键时刻震惊全场
```

### 合格检查

- [ ] genre_profile 已选定或"暂不选择"确认
- [ ] genre_config 已合并
- [ ] 第一章读者能感知到题材味道

## author-intent

### 要求

核心主题和写作信条必填。终局方向可选。

**核心主题** — 一句话说出这本书在讲什么（不是情节概括，是主题）
**写作信条** — 写作上绝不妥协的事（可执行的规则）
**终局方向** — 故事最终想去的地方（指南针）

### 输出去哪

`author-intent.md`，Markdown 格式。

### 案例

```markdown
## 核心主题
一个被命运抛弃的人如何重新找回活着的意义——不是通过复仇，而是通过重新与人建立联结。

## 写作信条
- 不写无意义的甜宠桥段——每个情感互动必须有剧情功能
- 不机械降神——主角的每次破局必须在前文有铺垫
- 配角不是工具人——每个重要配角有自己的完整弧线

## 终局方向（可选）
沈牧最终没有成为天下第一——但他找到了一群值得守护的人，和一个值得活下去的世界。
```

### 合格检查

- [ ] 核心主题已填写（是一句话主题，不是情节概括）
- [ ] 写作信条已填写（可执行原则，不是空话）
