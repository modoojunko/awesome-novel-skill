# 设定阶段参考

> 设定阶段每个板块的要求、输出格式和案例。agent 读完知道"设定要做到什么样、写到哪里、长什么样"。

---

## 世界观设定

### 世界观是什么

世界观是**角色活动和故事发生的舞台和规则集合**。它回答三个问题：故事发生在什么样的空间里（地理）、这个空间由谁控制怎么运行（政治）、这个空间有什么特殊规则（规则）。三者加起来，决定了角色能去哪里、受谁管、能做什么不能做什么。

好的世界观不需要读者读说明书——角色走在街上、和人说话、遵守或打破规则时，读者自然感受到世界的轮廓。

### 要求

**必须架空，不得与真实世界（地理/历史/政治）有任何重合。** 真实地名（北京、长安、巴黎）、真实朝代名（唐、宋、明）、真实历史人物一律禁止。世界观越具体越有辨识度，"架空"意味着作者可以自由定义一切。

core 三项必填。每条设定必须能用角色经历的场景呈现，不是说明书式罗列。

#### 地理（geography）

**理解方式：** 地理是作者给读者画的一张"空间地图"。读者不需要地图，但作者和 agent 需要知道——角色移动时经过什么、世界有多大、不同区域之间是什么关系。

**思考尺度：** 从故事最需要的层级开始，不需要所有层级都填满。

```
星系 → 星球 → 大陆 → 国家 → 省 → 市 → 县 → 区 → 街道 → 小区 → 栋 → 楼 → 房间
```

引导作者确认故事在哪个层级展开：
- 星际史诗 → 从星系/星球开始
- 仙侠/玄幻 → 从大陆/国家开始
- 古风权谋 → 从省/市开始
- 都市/年代 → 从市/区/街道开始

确定层级后，向下展开 2-3 层就够了。比如都市文选"市"层级，展开到街道就够，不需要写星系。

**必须包含的内容：**
- 主要场景的名称、位置关系（城/村/秘境之间多远、怎么走）
- 气候和季节特征（四季变化、极端天气）
- 地理限制（山川/水域/边界——什么阻隔了不同区域）
- 区域间的通行方式和难度

#### 政治（politics）

**理解方式：** 政治是**权力在谁手里、怎么用、谁限制谁**的规则。它回答：角色在这个世界能自由到什么程度？得罪了谁会被抓？谁保护弱者？弱者和强者之间有没有规则以外的相处方式？

政治不是只有"朝廷"才叫政治——宗门内部的长老排名是政治，家族里的嫡庶之争是政治，帮派码头的势力划分也是政治。**有人的地方就有权力关系，有权力关系的地方就有政治。**

**作用（下游为什么需要它）：**
- 方向提案时判断角色和哪方势力冲突/合作
- 角色决策的约束条件——出身决定能做什么不能做什么
- 冲突来源——势力之间的对抗是最稳定的戏剧动力
- 提示词注入派系关系，让角色互动有阵营依据

**思考方法——三角分析法：**

每个政治体系由三个角色构成：

| 角色 | 问题 | 案例 |
|------|------|------|
| **统治者** | 谁说了算？权力集中还是分散？ | 皇帝/长老会/帮主/宗主 |
| **制衡者** | 什么力量限制统治者？ | 另一势力/法律/天道誓约/民意 |
| **服从者** | 普通人为什么服从？不服从的代价？ | 怕死/怕被逐出/无力反抗 |

故事的政治张力来自三角失衡——统治者和制衡者之间的拉锯，或者服从者开始质疑"为什么要服从"。

**势力范围递进：** 从故事核心涉及的政治层级开始

```
星际联盟 → 国家/王朝 → 地方官府 → 宗门/帮派 → 家族 → 个人
```

引导作者确认故事在哪个层级展开：
- 星际/帝国权谋 → 从星际联盟/国家开始，延伸到地方
- 仙侠宗门 → 从宗门层级开始，延伸到修行界格局
- 古风权谋 → 从朝廷/地方开始，延伸到家族
- 都市帮派 → 从帮派/家族开始，延伸到个人
- 后宫/宅斗 → 从家族开始，延伸到个人

确定层级后，向上延伸一层（更大的背景压力），向下延伸一层（具体影响角色的组织）。

**必须包含的内容：**
- 统治形式（中央集权/分封/宗门/城邦？谁说了算？是否受限制？）
- 主要势力名单（至少 2-3 个，每个一句话说清立场和诉求）
- 势力之间的力量平衡（谁强谁弱？合作关系还是对抗？）
- 社会分层（平民/贵族/修行者/奴隶？阶层间怎么流动？）
- 不服从的代价（违反规则会怎样？谁执行惩罚？）

#### 规则（rules）

**理解方式：** 规则是**让角色做有趣选择的东西**。没有规则的世界，角色想做什么就做什么——没有冲突、没有代价、没有张力。好的规则让角色必须在"两个坏选项"或"好但代价高"之间做选择。

规则不只是"力量体系"——社会潜规则（青帮的地盘不能踩）、制度约束（官员不能随意调动军队）、自然法则（冬天封山过不去）都是规则。**限制在哪里，戏剧张力就在哪里。**

**作用（下游为什么需要它）：**
- 方向提案时判断角色能做什么不能做什么——限制催生创意
- 章纲阶段决定冲突的解决方式——角色是否冒着代价用规则
- 正文阶段约束角色能力范围——不让角色无所不能
- 读者期待——规则的边界一旦确立，读者会期待角色挑战它

**思考方法——规则三层法：**

规则分三个层级，每层影响范围和例外程度不同：

| 层级 | 范围 | 例子 | 例外 |
|------|------|------|------|
| **世界级** | 对所有人都生效 | 天道法则、物理规则、魔法本源 | 几乎没有，有就是世界观的漏洞 |
| **社会级** | 对特定群体生效 | 法律、宗门规矩、行规、禁忌 | 有——权力大的人可以绕过 |
| **个人级** | 只对特定角色生效 | 血脉诅咒、功法限制、契约誓言 | 有——找到破解方法就是剧情 |

引导作者按层级确认：
1. 世界级规则是什么（力量体系/天道/自然法则）——先定这个，后两者不能与其矛盾
2. 社会级规则有哪些（法律/规矩/行业禁忌）——从政治部分的势力推导
3. 个人级规则涉及谁（主角的血脉/反派的诅咒/盟友的誓言）——从角色设定推导

**每条规则的三个要素，缺一不可：**

| 要素 | 问题 | 不写会怎样 |
|------|------|-----------|
| 能做什么 | 这条规则赋予什么能力或权限？ | 读者不知道规则有什么用 |
| 不能做什么 | 限制条件是什么？什么情况下不成立？ | 主角无所不能 |
| 代价 | 使用规则要付出什么？ | 读者觉得"那就用啊，为什么不用？"——没有张力 |

**必须包含的内容：**
- 世界级规则——力量体系/天道/魔法/科技的核心描述
- 社会级规则——法律/禁忌/行业规矩（至少 2 条具体规则）
- 个人级规则——影响角色的特殊限制（如有）
- 每条规则的三个要素：能做什么、不能做什么、代价
- 规则的例外：有例外吗？谁可以例外？为什么？

#### 扩展（extended）

culture / history / physics / biology / sociology 按需追加——正文写着发现需要了再来补，不预设性填满。

### 输出去哪

`settings/world-setting.md`，Markdown 格式。## 二级标题分节。

### 案例

```markdown
## 地理

青州城坐落于苍梧山脉东麓，三面环山一面临水。城分内外：内城是官署和富户聚居区，城墙高三丈，设四门，夜闭晨开；外城是平民市井，无城墙，沿江而建，街巷纵横如蛛网。

城外东南十五里有青江渡口，是商旅往来的命脉。沿江下行可至江南诸州，上行通往苍梧山深处。渡口每日卯时开渡，酉时收渡，夜不行船。

苍梧山脉横贯南北，是青州与西境的天然屏障。山中有三条可通行的古道，但冬季大雪封山，只有猎户熟知的小路能走。翻过苍梧山向西，是妖兽横行的荒原——没有人活着穿越过。

气候：冬季湿冷多雾，青江水面晨雾弥漫，至午方散。夏季闷热多雨，六月至八月是雨季，青江水位暴涨时渡口停运三至五日。

## 政治

统治形式：朝廷设州府制，青州府归江南道管辖。知府陈伯远掌军政大权，受朝廷吏部考核。

三股势力：
- **官府**——维持秩序，但力量有限。府兵五百，装备精良但士气不高。知府受限于朝廷法度，不能随意调动资源
- **青帮**——控制码头和市井生意，与官府表面合作暗中角力。帮主洪四海，手下三千帮众，渗透了府衙底层吏员。青帮不敢公然对抗官府，但能通过"信息差"左右局势
- **玄天宗外门**——修行者势力，不干涉俗务但拥有超然地位。驻守长老一位，筑基弟子十人。朝廷对修行者无管辖权——玄天宗弟子犯法由宗门自行处置

三足鼎立，任何两方联合都会打破平衡。官府和青帮互相制衡，玄天宗是两者都不敢招惹的变量。

社会分层：
- 修行者（超然地位，不受世俗法律约束）
- 官员/士绅（有特权，但受朝廷法度管）
- 平民（纳税服役，无特权。得罪青帮可能被打，得罪官府可能坐牢）
- 贱籍（倡优皂隶，后代不得科举，不得与平民通婚）

不服从的代价：平民抗税→打板子+坐牢。帮众背叛→三刀六洞。官员贪腐被查→抄家流放。对玄天宗不敬→没人能帮你，官府也管不了。

## 规则

### 世界级：修炼体系

修炼路径：引气（感应天地灵气入体）→ 筑基（灵气淬体，打通经脉）→ 金丹（凝聚本命金丹）→ 元婴（金丹化婴，神识外放）。每级晋升需天材地宝+功法+机缘，缺一不可。

**能做什么：** 引气期可强身健体，百病不生。筑基期可御物飞行，火球术等低阶法术。金丹期可御剑飞行，神识探查百里。元婴期可元神出窍，瞬息百里。

**不能做什么：** 无功法无法自主修炼（引气期是极限）。任何强行突破都会走火入魔——轻则经脉尽断修为全废，重则当场暴毙。金丹以上修士受天道誓约约束，不得对凡人出手——违者天劫加身，九死一生。

**代价：** 每一次大境界晋升都在消耗寿元——筑基减十年，金丹减三十年，元婴减五十年。修行是逆天而行，每向上一步都在燃烧生命。寿元耗尽则神魂俱灭，不入轮回。

### 社会级：青州城潜规则

- **凡人不得插手修行者争端。** 看到了就绕道走，报官也没用——知府管不了修行者的事
- **青帮地盘外人不得做买卖。** 码头、赌场、当铺三条线归青帮管，在青帮地盘上做生意要交"平安钱"。不交→店被砸、人被扔出城
- **玄天宗弟子在城内不得施展法术。** 这是朝廷和玄天宗的默契——修行者可以在青州城驻留，但不能在城内动武。违反者由宗门自行召回处置

### 个人级：主角沈牧的限制

沈牧幼年被妖兽抓伤，体内残留一丝妖气。修炼到筑基后期后妖气开始反噬——月圆之夜灵力失控，情绪激动时瞳孔变竖瞳。若不找到压制之法，金丹劫时妖气会与灵气冲突，必死无疑。
```

### 合格检查

| 检查项 | 方法 | 不通过处理 |
|--------|------|-----------|
| 架空 | 扫一遍地名/朝代名/人物名有无真实世界重合 | 发现真实地名→退回重写 |
| 完整性 | 检查 ## 地理 / ## 政治 / ## 规则 三个节都存在 | 缺节→退回补全 |
| 政治三角 | 统治/制衡/服从三个角色都能找到对应描述 | 缺一方→退回补，否则冲突缺乏制衡力量 |
| 可感知 | 读每条设定，想一个具体场景呈现它 | 想不到场景→太抽象，退回 |
| 规则三要素 | 每条规则都有"能做什么/不能做什么/代价"三项 | 缺项→退回补全，否则角色行为无约束 |
| 规则层级 | 世界级/社会级/个人级至少覆盖两级 | 只有力量体系没有社会规则→退回补社会级 |
| core 够用 | 仅凭 core 三项，第一章是否已经可以动笔 | 不行→补 core，不要急着填 extended |

## 角色设定

### 要求

每个主要角色（protagonist / antagonist / supporting）独立文件。七字段填到能推断行为。

**cognition** — 如何理解和判断事物（有认知偏差或思维模式）
**worldview** — 相信什么、不相信什么（可与其他角色冲突）
**self_identity** — 如何看待自己（可以有认知偏差）
**values** — 什么事会破例、什么不可做
**abilities** — 实际能力（有具体表现，不是头衔）
**skills** — 可习得技术（有来源和局限）
**environment** — 出身+当前处境（决定行为基线）

### 输出去哪

`settings/character-setting/<拼音id>.yaml`，每角色一文件。

### 案例

```yaml
id: shen-mu
name: 沈牧
story_role: protagonist

cognition: 习惯从利益角度分析一切——任何人做事都有动机，找不到动机就说明信息不够。不信任"善意"这种解释。

worldview: 强者制定规则，弱者遵守规则。想改变处境就要变强，抱怨没有用。

self_identity: 觉得自己是个冷静理性的谋定后动者——但实际遇到亲友遇险时第一反应是冲上去，理性是事后找补的。

values: 不背叛兄弟。不欺压平民。可以利用对手但不能侮辱对手的尊严。

abilities: 能在十步内察觉杀气来源；单手拉八十斤弓，三箭连发不脱靶；过目不忘，但需要时间回忆。

skills: 青帮底层混出来的市井生存术（听脚步辨人数、看衣着断身份）；跟账房先生学过珠算和记账。

environment: 苍梧山脚猎户之子，父母死于妖兽袭村，被青帮老掌柜收养。在码头帮工长大，见过三教九流。
```

### 合格检查

- [ ] 每角色通过"生气表现+缺点"诊断
- [ ] cognition/worldview 推得出场景行为
- [ ] abilities 有具体表现和边界
- [ ] 配角有自驱动目的（不为主角存在）
- [ ] 主角有实质性缺点（不是"太正直"）
- [ ] 文件名拼音 id，不用中文

## 写作风格

### 要求

四字段互相锚定，缺一不可。默认值只是起点，至少一个字段经讨论调整。

**role** — AI 在写作中扮演的叙事身份（冷静旁观？亲密叙述？带讽刺的观察者？）
**core_principles** — 不可违背的信条（节奏底线、禁忌、风格承诺）
**possible_mistakes** — AI 易犯的错误列表（已知会出问题的模式）
**depiction_techniques** — 推荐的描写手法和层次

### 输出去哪

`settings/writing-style.yaml`，四字段写入顶层。

### 案例

```yaml
role: 一个冷静但不冷漠的叙事者——客观描述角色行动和环境，不替角色感受。
  让对话和动作传递情绪，不做心理分析旁白。

core_principles:
  - 每章结尾必须有钩子，不能平铺直叙收尾
  - 不在对话中插入旁白解释角色情感——让读者从语气和用词判断
  - 对话为主、描写为辅，一段叙事后尽快回到对话推进
  - "突然""猛然""瞬间"每章不超过3次
  - 不写无意义的日常——每个场景必须有冲突或信息推进

possible_mistakes:
  - 过度使用面部表情描写（"他皱了皱眉""她嘴角上扬"）
  - 心理活动用"他感到""他意识到""他心想"开头
  - 战斗场面写成一招一式的慢动作
  - 环境描写独立成段脱离剧情——环境应该和角色的行动/情绪交织
  - 每段开头"他/她"连续超过3次

depiction_techniques:
  - 情绪通过动作表达：紧张时抠指甲、愤怒时反而轻声细语
  - 环境感知融入角色行动：不是"街道很吵"，是"他得扯着嗓子对方才能听见"
  - 对话用语气和用词差异区分角色——不靠"他说""她说"标签
  - 时间流逝用环境变化暗示：阳光角度、影子长度、温度变化
  - 战斗用结果和后果描写代替过程："等声音停下来，他的左臂已经不自然地垂在身侧"
```

### 合格检查

- [ ] 四字段全部有值
- [ ] 至少一个字段经讨论调整
- [ ] possible_mistakes 有具体反模式（不是"不要写太AI"）
- [ ] core_principles 可执行（3-5条硬规则）

## 题材配置

### 要求

已选类型档案 → genre_config 合并。未选类型 → 手动配置题材要素。

### 输出去哪

`writing-style.yaml#genre` 字段。

### 案例


## author-intent

### 要求

核心主题和写作信条必填。终局方向可选。

**核心主题** — 一句话说出这本书在讲什么（不是情节概括，是主题）
**写作信条** — 写作上绝不妥协的事（可执行的规则）
**终局方向** — 故事最终想去的地方（指南针）

### 输出去哪

`author-intent.md`，Markdown 格式。

### 案例

```markdown
## 核心主题
一个被命运抛弃的人如何重新找回活着的意义——不是通过复仇，而是通过重新与人建立联结。

## 写作信条
- 不写无意义的甜宠桥段——每个情感互动必须有剧情功能
- 不机械降神——主角的每次破局必须在前文有铺垫
- 配角不是工具人——每个重要配角有自己的完整弧线

## 终局方向（可选）
沈牧最终没有成为天下第一——但他找到了一群值得守护的人，和一个值得活下去的世界。
```

### 合格检查

- [ ] 核心主题已填写（是一句话主题，不是情节概括）
- [ ] 写作信条已填写（可执行原则，不是空话）
