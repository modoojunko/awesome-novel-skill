# 提示词自检清单

章纲写好、叙事段落拆分确认后，Agent 自动生成写作提示词（prompt.md）。这份提示词是 subagent 的唯一输入——subagent 不会质疑提示词、不回看前文、不追问意图，它只执行提示词里的每一个字。提示词里写糊的，正文里一定跑偏；提示词里带上的分析腔，正文里一定复刻。**提示词是 subagent 的宪法——它看不懂的事，你写再多字也白搭。**

> **两条底线**（任一不过，禁止进入正文生成阶段（3.4））：
>
> 1. **视角已锁定且不含上帝视角残留**——提示词的每个叙事段落的 what_to_write 中，不得出现"主角感到""主角意识到""读者应该感受到""这标志着""这意味着"等上帝视角解构语。全篇提示词可以通过当前 POV 角色的感官写出来。
> 2. **what_to_write 全部可执行且无分析腔**——subagent 读完每个段落的 what_to_write 后，能直接写出正文。不含"核心心理动作""A 和 B 的区别在于""本章做的是前者"等教学语言。每个 what_to_write 不超过 200 字。

---

## 最高原则：提示词写给 subagent，不写给你自己

> 下面所有规则之前，先问一句：**"一个没有直觉、没读过前文的 subagent，拿着这份提示词，能写出正确的正文吗？"**

你知道"这段要写出悬念感"是什么意思，但 subagent 看到的是四个字。你知道"情绪基调：偏激"背后的三层心理状态，但 subagent 会逐字照抄"偏激"然后编一个最标准的愤怒场景。你知道"不要写得太拖沓"是希望节奏紧凑，但 subagent 不知道什么叫"太拖沓"——它写的每一句都自认为不拖沓。

**判定标准：**
- 提示词的每个约束，替换成"subagent 能用它来判断某句话该不该写吗？"——判断不了的约束等于没有约束
- 每个 what_to_write，替换成"subagent 读完能用它写出这段正文吗？"——写不出来的就是太模糊或太分析腔
- 整份提示词读完后，可以在脑子里过一遍本章的完整正文——过不下来就说明有缺口或矛盾

**允许的：**

| 行为 | 说明 |
|------|------|
| what_to_write 写具体可感知的节拍（"天还没亮，陆征拨通老邱的电话"） | 好——subagent 知道写什么 |
| 约束用大白话写具体禁止行为（"不要直接写'他很愤怒'——写动作和身体反应"） | 好——subagent 知道什么不能做、怎么做替代 |
| 部分约束留空（如本章无 dialogue_push 段，dialogue_intent 可略） | 留空必须有理由，不是忘了 |

**禁止的：**

| 行为 | 后果 |
|------|------|
| what_to_write 里写心理机制分析（"核心心理动作：三次失败→偏激"） | subagent 在正文里分析角色心理 |
| what_to_write 里写概念对比（"A 和 B 的区别在于…"） | subagent 在正文里做对比论证 |
| what_to_write 里写教学语言（"不通过 X 来写，通过 Y"） | subagent 连教学方法一起写进正文 |
| what_to_write 超过 200 字 | subagent 只剩下扩写，不再是创作——正文会像缩写版的复读 |
| 约束写成抽象空词（"保持紧张感""不要太拖沓"） | subagent 无法执行——它不懂"多紧张算紧张" |
| 约束用否定式引导（"不要写成 XXX"） | subagent 在脑子里想一遍 XXX，然后更可能写出它 |

## 一、视角转换（核心）

章纲是上帝视角写的（"主角发现 X""读者感受到 Y"），提示词必须锁定单一视角。这是提示词生成的第一步，也是最不能出错的一步——视角没锁住，整章报废。

### 1. 视角锁定规则

| 规则 | 说明 | 常见错误 |
|------|------|---------|
| 锁定单一 POV | 整章只能通过一个角色的眼睛看世界。只写这个角色看到的、听到的、闻到的、触碰到的、想到的 | 同一段内出现两个角色的内心活动——读者会眩晕 |
| 禁止心理断言 | 不写"主角感到""主角意识到""主角判断出""主角明白"。心理活动通过可观察的身体反应和动作暗示 | 写"他感到不安"——读者看到了旁白，不是角色 |
| 禁止读者指向 | 不写"读者应该感受到""读者会注意到""让读者看到"。读者感受来自正文本身，不需要提示词指导 | 写"让读者感受到紧张"——subagent 会在正文里帮你制造"紧张"两个字 |
| 角色不知道的不写 | 不在 what_to_write 中暗示当前 POV 角色不知道的信息——subagent 会不自觉让角色"恰好知道" | 写"陆征不知道楼下的男人在跟踪他"——subagent 可能在正文里让陆征回头看一眼 |
| 用直接陈述代替解释性过渡 | 写"门开了"，不写"他注意到门开了"；写"雨打在玻璃上"，不写"他能听到雨打在玻璃上的声音" | 解释性过渡就是旁白——第三人称有限视角下不该出现 |

### 2. 章纲→提示词视角转换对照

章纲是上帝视角的事件描述，提示词的 what_to_write 必须转为角色视角的感官记录。

```
✅ 合格转换：
章纲 outline: "主角走进废墟，发现了一封信。读者应该感受到不安。"
提示词 what_to_write: "走进废墟。脚下是烧焦的木屑，踩上去没有声音。门框斜挂着，铰链断了一颗。桌上放着一封信——信封角落有三道暗红色的痕，拇指宽。窗外的光线刚好照在'苏棠'两个字上。"
→ 读者通过主角的感官发现信，不安来自木屑无声、门框倾斜、血迹——不靠"应该感受到不安"

章纲 outline: "陆征发现苏棠不是主动离开的。出租屋的状态与离家出走的特征不符。"
提示词 what_to_write: "床没铺。被窝保持掀开的状态，枕头上有头压过的凹痕。充电器还插在床头墙上，线垂到枕头边。冰箱打开——半盒剩饭、两瓶矿泉水。不是'收拾好行李离开'的样子。"
→ 不写"他意识到""他发现"，只写他看到了什么。结论让读者自己下

❌ 不合格转换：
章纲 outline: "主角走进废墟，发现了一封信。读者应该感受到不安。"
提示词 what_to_write: "主角走进废墟，内心感到强烈的不安。他在桌上发现了一封信，这让他意识到事情远比想象的复杂。"
→ "内心感到""意识到" = 上帝视角旁白，"远比想象的复杂" = 替读者下了结论

章纲 outline: "陆征发现苏棠不是主动离开的。"
提示词 what_to_write: "陆征环顾房间，意识到苏棠不是主动离开的——床铺状态、充电器和冰箱剩饭都与离家出走的特征不符。"
→ "意识到" = 解释性过渡，"与……不符" = 分析性结论。应该只写他看到了什么具体物品，不写他"意识到"了什么
```

### 3. 多角色视角确认

如果章纲中存在多角色的信息线（如 A 角色做了一件事，B 角色在另一场景有自己的线索），提示词生成前必须确认本章锁定的单一视角。

| 检查项 | 标准 |
|--------|------|
| narrative_pov 字段 | chapter.yaml 的 narrative_pov 必须有值（第三人称有限/第一人称/全知）。空 → 必须先确认 |
| 单一视角确认 | 本章写谁的视角？这个角色在所有场景中都在场吗？不在场的场景如何通过他的视角获取信息？ |
| 信息不对等处理 | 如果当前 POV 角色不知道某信息，提示词中不暗示该信息的存在——不写"虽然他当时并不知道"等补丁 |

```
✅ 合格：
本章锁定陆征视角。苏沫的内心不写。方岩打电话时的表情只写陆征看到的——"方岩点了根烟，没有说话，把打火机翻了两面"。苏沫那边警察来的场景本章不写——只写陆征接到苏沫电话后听到的结果。

❌ 不合格：
本章以陆征视角写调查过程，同时在 what_to_write 中写"苏沫在警局等待时越来越焦虑，她开始怀疑自己找陆征是不是一个错误"——苏沫不在陆征的视野内，这个焦虑状态陆征看不到。
```

## 二、叙事段（segment）功能分配

每个 segment 有明确的叙事功能，功能之间合理分布，过渡自然。一整章全是 action_beat = 读者累死；全是 atmosphere = 读者困死；没有一个 emotional_landing = 读者不知道章末该感受什么。

### 1. 功能类型分配

| function | 作用 | 分配原则 | 常见错误 |
|----------|------|---------|---------|
| atmosphere | 氛围建立——环境、五感、世界基调 | 首段常用，让读者先站稳再推剧情 | 一章内 atmospheres 过多——每段开头都写天气 |
| character_beat | 角色状态——习惯/情绪/内心，建立读者连接 | 首段或重大事件后 | 放到 action_beat 之后却没有情绪反应——事件发生了但没有"人"在感受 |
| dialogue_push | 对话推进——有意图的对话，推动信息或关系变化 | 每个涉及重要对话的 key_point | 对话没有意图字段（dialogue_intent 为空）——subagent 写无目的闲聊 |
| action_beat | 动作事件——冲突、动作、关键事件 | 每个涉及动作/事件的 key_point | 连续 3 段以上 action_beat——读者神经绷断 |
| revelation | 信息揭露——新线索、真相、伏笔推进 | 每个涉及信息释放的 key_point | 信息没铺垫就直接揭露——读者觉得"怎么就突然知道了" |
| emotional_landing | 情绪落地——事件余波、角色反应、章末钩子 | **末段必须** | 结尾没有 emotional_landing——章末像切歌一样戛然而止 |
| transition | 过渡桥接——时间/地点/情绪转换 | 仅在必要时使用，不滥用 | 每两段之间都来一个 transition——节奏被切断成碎块 |

### 2. 功能分布自检

| 检查项 | 标准 |
|--------|------|
| 首段是 atmosphere 或 character_beat | 读者需要先知道"在哪""谁在场"再进入事件。首段不应该是 action_beat——读者还没站稳就开打，会一头雾水 |
| 末段是 emotional_landing | 必须。不能以 revelation 或 action_beat 结尾——信息揭露后必须有情绪反应，动作打完后必须有余波 |
| emotional_landing 留下了钩子 | 章末的 emotional_landing 必须呼应 emotional_design.emotional_hook——读者合上本章时带着一个未闭合的情绪缺口 |
| 同一 function 不连续出现 3 段以上 | 尤其是 action_beat 和 atmosphere——连续 3 段同类型是功能分配失败 |
| transition 只用于必要的位置转换 | 每 5-7 段中 transition 不超过 1 个。如果每两段之间都有一个 transition → 说明 key_points 之间的逻辑没有写通，靠过渡段落补洞 |
| segment 数量 5-7 段 | 过少（<5 段）= 章节扁平，没有节奏变化；过多（>7 段）= 可能在 key_points 之外再拆了不必要的段落 |

### 3. 段间过渡自检

| 检查项 | 标准 |
|--------|------|
| 每段的 ends_with 存在且具体 | ends_with 必须是一个可感知的画面或状态——"他推门出去，走廊的声控灯亮了一半"，不是"完成了调查" |
| 下一段起始场景锚定正确 | 段落 2+ 的提示词中必须写"上一段结束于——{prev.ends_with}。从此画面无缝接续……" |
| 过渡不生硬 | 不出现"与此同时""镜头转向""画面一转"等影视化过渡词——小说不是分镜脚本 |

```
✅ 合格：
段落 1 ends_with: "陆征推开事务所的门，走廊的声控灯亮了一半，另一半黑着。"
段落 2 起始场景: "上一段结束于——陆征推开事务所的门，走廊的声控灯亮了一半。从此画面无缝接续：他穿过走廊，电梯门开着，里面的灯一直在闪。"

→ 直接从"出事务所门"接到"穿过走廊进电梯"——不跳跃，不重复描述

❌ 不合格：
段落 1 ends_with: "陆征完成了初步调查。"
段落 2 起始场景: "上一段结束于——陆征完成了初步调查。画面一转，第二天早上，陆征来到物流公司。"

→ ends_with 是抽象结论（"完成了"），无法对接具体画面。"画面一转"是影视术语，不该出现在小说提示词里
```

## 三、writing-style 四字段注入

writing-style.yaml 的四个核心字段必须全部注入提示词。缺任何一个字段，subagent 在该维度的行为就是随机——可能写出正确的，也可能完全放飞。

### 1. 四字段注入标准

| 字段 | 注入位置 | 注入方式 | 缺失后果 |
|------|---------|---------|---------|
| role | "角色定位"段 | 用 genre-corpus 的 role_override（如有），否则回退到 writing-style.yaml 的 role + personality。告诉 subagent"你是谁" | subagent 使用默认 AI 角色——没有风格身份，写出来是标准 AI 公文 |
| core_principles | "写作原则与禁忌"段 + "写作要求"段 | 提炼 8-15 条大白话约束，不分类不编号。从 global_rules / natural_expression / description_vs_depiction / character_building / pov_consistency 中取最关键的 | subagent 不知道写作底线——可能跳视角、堆形容词、做道德化说教 |
| possible_mistakes | "写作原则与禁忌"段 | 与 core_principles 合并提炼。按致命程度排序，最不能犯的排最前面 | subagent 不知道什么不能做——会在正文里插元评论、简化为刻板形象、拒绝直面冲突 |
| depiction_techniques | "写作要求"段 | 将推荐的描写技法转化为大白话操作指引（"用对话展示"→"每一句对话都是角色的微型动作"），作为 subagent 的技法工具箱 | subagent 只用最标准的描写方法——全是"他很生气""她很伤心" |

### 2. 提炼质量自检

| 检查项 | 标准 |
|--------|------|
| 约束是大白话 | "不要让角色做不符合他性格的事" → 合格。"禁止角色行为偏离人设基线" → 不合格（书面化）|
| 每条约束可执行 | subagent 读完能判断"这句话违规了吗" |
| 约束不分类不编号 | 不用 🔴🟡🟢 或 1/2/3 编号——subagent 会对"红色规则"和"绿色规则"区别对待 |
| 约束不解释"为什么" | 只说"别做什么"，不说"因为读者会出戏""因为这样破坏了代入感"——解释占用 token，且 subagent 不需要知道理论 |
| 约束按致命程度排序 | 视角跳跃 > 直述情绪 > 段尾总结升华 > 句式规整 > 形容词堆砌。最不能犯的排最前面 |
| 数量 8-15 条 | 少于 8 条 = 关键约束遗漏；多于 15 条 = subagent 记不住，后半截约束形同虚设 |

```
✅ 合格提炼：
写作原则与禁忌：
- 只写陆征能看到、听到、碰到的东西。不在场的场景不写，别人心里的想法不写。
- 不写"他很愤怒"——写他捏碎杯子、声音压低、指节发白。想写感受的时候，写身体反应。
- 每一句对话都有目的——试探、拒绝、转移、暗示或攻击。不写为交代信息而交代信息的对话。
- 环境描写单段不超过2行。顺手带过，不当成重点来渲染。
- 不写"通过这件事，陆征意识到……"——只写发生了什么事，结论让读者自己下。
- 相邻四句不以"他"开头。单段不超过5行。每三句中有一句短句（1-5字）做节奏切断。
- 章末不留"下章预告"——只留情绪缺口。读者带着"想知道接下来会怎样"的心情合上这一章。
- 禁止诗词腔：不用排比古风句、不堆四字成语、不用空洞意境虚词（浮生/流年/凡尘/寂寥等）。

→ 8条，大白话，每条可执行，不解释"为什么"，不分类不编号。最致命的（视角 + 直述情绪）排最前。

❌ 不合格提炼：
写作原则与禁忌：
🔴 视角一致性：必须严格保持第三人称有限视角，禁止上帝视角
🟡 情感表达：通过Show, Don't Tell的方式展现情感
🟢 语言风格：保持自然松弛的写作风格

→ 分类标记干扰 subagent。每条都是空词——"严格的第三人称有限视角"是什么？subagent 不知道怎么才算严格。"自然松弛"是什么？subagent 没办法判断。
```

### 3. 四字段全部注入验证

```
✅ 合格提示词（四字段全部可追溯）：
角色定位 = role 字段注入："你是悬疑刑侦题材的职业写作者。写作时像陆征本人一样——对细节偏执、不急于下判断、相信实物证据多于口头描述。"
写作原则与禁忌 = core_principles + possible_mistakes 提炼的 10 条大白话约束
写作要求中"用暗示替换直述"类的操作指引 = depiction_techniques 转化（"动作描写"→"不写感受写身体动作"，"对话展示"→"对话是微型动作"）

❌ 不合格提示词（字段缺失）：
role 字段是模板默认值（"你是全球闻名的小说作家"）——没有类型的专属身份，subagent 以通用 AI 角色写作
possible_mistakes 未注入——提示词中没有"行文逻辑不可断裂""拒绝简化复杂角色"等警告，subagent 可能在章末提前做下章预告
depiction_techniques 未注入——subagent 只用最标准的描写方式，全是直述情绪
```

## 四、类型提示段注入

如果项目已选择 genre_profile，对应 genre-corpus 的 prompt_segment 必须原样注入提示词的"类型专属指引"段。prompt_segment 包含该类型的特定写作规则、爽点结构、禁忌等——不注入 = subagent 以通用模式写作，丢失题材味道。

### 1. 注入检查

| 检查项 | 标准 |
|--------|------|
| genre_profile 已配置 | writing-style.yaml 的 genre_profile 字段非空。若空 → 是作者选择"暂不选择"还是漏填？确认后决定是否跳过此段 |
| prompt_segment 已读取 | 从 genre-corpus/index.yaml 找到 genre_profile 对应的 corpus 文件，读取 prompt_segment |
| prompt_segment 结构保留 | 原样注入，保持其段落结构和表述方式。不压缩、不改写、不"提炼"——prompt_segment 是类型专家的输出，Agent 二次改写可能丢失关键信息 |
| genre_taboos 已作为约束写入 | genre-corpus 中如有 genre_taboos 字段，必须合并到"写作原则与禁忌"段 |
| role_override 已覆盖默认 role | "角色定位"段优先使用 genre-corpus 的 role_override.role + role_override.personality，而非 writing-style.yaml 的模板默认值 |

### 2. 注入验证


## 五、章纲→提示词完整性

章纲的所有关键信息必须转化为提示词中的具体指引，但转化不是粘贴——章纲是给作者看的笔记，提示词是给 subagent 的指令。转化过程中丢失的每一项信息，都会成为正文里的空位。

### 1. 完整性映射

| 章纲来源 | 提示词去向 | 转化标准 |
|---------|-----------|---------|
| outline.summary | "故事背景"段的本章上下文 | 不是粘贴 summary——转为一两句话的场景起点："本章开始时，陆征刚从出租屋回到事务所。桌上放着一张便利贴，上面只有一个电话号码。" |
| outline.key_points | 每个 key_point → 1-2 个 segment 的 what_to_write | key_point 不是复制到 what_to_write——必须视角转换 + 双轮净化 |
| memo.current_task | "写作要求"段的本章目标 | 转化为 subagent 知道"本章航向"的一句话。不写"本章讲述了"，写"本章完成的具体动作" |
| memo.reader_expectation | emotional_landing 段的情绪钩子指引 | state/strategy/detail 转化为章末的情绪落点指令 |
| memo.payoff_plan | "故事背景"段的活跃钩子速览 + segment 级别的微兑现锚定 | must_resolve / must_hold / partial_advance 分别转化为"本章必须收的钩子""本章绝对不能碰的底牌""本章推进到哪个程度" |
| memo.prohibitions | "写作要求"段的禁止事项（如非空） | 章纲的 prohibitions 直接追加到提示词的禁止事项中 |
| memo.required_changes | emotional_landing 段必须覆盖 | 章末必须让读者感知到的改变——不写"信息改变了"，写"读者知道了 X" |
| emotional_design | 各 segment 的 emotional_tone 序列 + emotional_landing 段的钩子 | mood_progression 必须与 segment 的 emotional_tone 序列一致；emotional_hook 必须在末段落地 |
| 前章结尾状态 | "故事背景"段的前章锚定 | 告诉 subagent 上一章结束时的画面和情绪——subagent 不从零开始，是从上一章的结尾接着写 |
| 本章字数目标 | "写作要求"段 | 全章总字数 = 各 segment word_target 之和。不少于目标字数的 80% |

### 2. 常见转化遗漏

| 遗漏项 | 后果 | 检查方法 |
|--------|------|---------|
| memo.prohibitions 未写入提示词 | subagent 不知道本章的红线——可能写作者禁止的内容 | 扫描提示词"禁止事项"段，逐条对照 memo.prohibitions |
| memo.required_changes 未体现在 emotional_landing 段 | 章末没有让读者感知到改变——读者看完觉得"什么都没发生" | 读末段 what_to_write + emotional_tone，反推"章末什么东西变了" |
| 前章结尾状态缺失 | subagent 不知道上一章结束时是什么场景、什么情绪——可能写出与上章矛盾的起始状态 | 提示词的"故事背景"段中是否有"上一章结束于……"的描述 |
| mood_progression 与 segment emotional_tone 序列不一致 | subagent 在各段之间情绪切换混乱——前一段是"好奇"、后一段突然变成"紧张"，中间没有过渡 | 列出 segment emotional_tone 序列，与 chapter.yaml 的 mood_progression 逐节点对比 |
| payoff_plan 的 must_resolve / partial_advance 未注入 | 该收的钩子没收，该推进的埋伏没有推进——读者会在后续章节发现"这个事之前没讲啊" | 扫描提示词全文，搜索每个钩子 ID——确保每条需操作的钩子有对应指引 |

```
✅ 合格：
章纲 memo.prohibitions:
  - "不要揭示陆征三年前离开警队的详细原因——只通过细节暗示他以前是刑警"
  - "不要让楼下的男人和陆征对话"

提示词"写作要求"段：
本章禁止：
- 不要写陆征三年前离开警队的原因。可以通过同事的对话或他的习惯暗示他以前是刑警，但不说破具体事件。
- 楼下的男人不跟陆征说话。只写他的存在——站在路灯下、烟头明灭、眼睛没有离开过陆征的方向。

→ 每条约 prohibitions 都有对应的提示词禁止项，且有具体的执行描述。

章纲 memo.required_changes:
  - "信息改变：确认苏棠非主动离开——房间状态3个矛盾细节"

提示词末段（emotional_landing）what_to_write：
"陆征把钥匙还给房东。下楼时他在楼梯间站了一会儿——手机里有今晚拍的六张照片。床没铺。充电器还在墙上。冰箱里的剩饭还没馊。他给方岩发了条消息：这个案子不是主动离开。发完之后把手机翻过去，看着楼梯间窗户外面对面楼的灯。"

→ required_changes 的"确认非主动离开"在末段落地。subagent 写这段时知道：要写陆征确认了非主动离开，要带出三个细节，要作为章末收束。

❌ 不合格：
memo.prohibitions 写了 4 条，提示词中"禁止事项"段只写了"不要太拖沓"和"保持节奏"——两条抽象空词替代了全部具体红线
memo.payoff_plan 写了 partial_advance hook-001，提示词全文搜索不到 "hook-001"——该推进的钩子完
全没有指引
mood_progression: "日常松弛→好奇→紧张→悬念收束"，但 segment emotional_tone 序列是"松弛→紧张→紧张→悬念"——从松弛直接跳到紧张，漏了"好奇"这个过渡节点
```

## 六、禁止用否定句式引导

提示词只写你想要的。不写"不要写成 XXX"——subagent 读到"不要写成热血战斗"时，它的注意力已经被引导到"热血战斗"上，正文里反而更容易出现热血战斗的影子。

### 1. 否定句式替换规则

| 禁止写法（否定引导） | 替换为（正面指引） | 原理 |
|----------------------|-------------------|------|
| "不要写成热血战斗" | "冲突发生在日常场景中——出租屋、物流公司、街边大排档。武器是信息和观察力，不是拳头。" | 用场景限制自然避开热血战斗 |
| "不要写得拖沓" | "每个环境细节顺手带过，不超过 2 句。对话每轮 3-5 句收住，不延伸成闲聊。" | 用具体字数/句数限制控制节奏 |
| "不要让主角太完美" | "陆征有个习惯——调查到一半会停下来抽根烟，对着墙发呆。他也会在笔记本边上画小人。" | 用具体行为塑造不完美 |
| "不要使用诗词腔" | "叙事用大白话。环境只写三样东西——光、声音、温度。不写'晚风微凉，暮色沉沉'，写'路灯亮了一排，风从巷口灌进来'" | 用正面范例替代禁止清单 |
| "不要跳视角" | "全章只写陆征的眼睛——他看到什么、听到什么、碰到什么。其他人脸上的表情，只有陆征看到的时候才写。" | 用感官边界自然锁住视角 |

### 2. 否定句式扫描

| 检测项 | 命中模式 | 修正方法 |
|--------|---------|---------|
| 否定引导式约束 | "不要写成……""避免……""别让……""切忌……" | 全部替换为正面描述：写"应该是什么样"，不写"不该是什么样" |
| 隐含否定的"不是……而是……" | "不是 X，而是 Y"——subagent 的注意力先到 X，再到 Y。正文里 X 和 Y 都可能出现 | 直接写 Y。如果需要对比，写 Y 的正面描述，"X"这个字不出现 |
| "不要只写……" | "不要只写动作，也要写心理"——subagent 可能理解为"少写动作" | 写"动作写完之后，让角色停下来——发呆、看窗外、翻手机。这两个节奏交替。" |

```
✅ 合格：
写作原则与禁忌中包含：
- 冲突场景控制在日常环境中——出租屋、街边大排档、物流公司仓库。不写追逐、打斗、爆炸。
- 每段对话 3-5 句后收住。不延伸成闲聊或辩论。
- 环境描写单段不超 2 行。顺手带过——写光、声音、温度，不写风景。

→ 全是正面指引。subagent 知道"应该写成什么样"，不需要先在脑子里否定错误版本。

❌ 不合格：
写作原则与禁忌中包含：
- 不要写成热血战斗文
- 不要写得太拖沓
- 不要用诗词腔
- 不要跳视角
- 不要写得太平淡

→ 全是负面词汇。subagent 不知道"热血战斗""拖沓""诗词腔""跳视角""平淡"的具体边界在哪——它只能猜。而且每读一个否定词，注意力就在那个方向上停一下。"不要写得太拖沓"→ subagent 在脑子里过了一遍"什么是拖沓"→ 然后在正文里更可能写出它理解的"不拖沓"——可能是写得太赶。
```

## 七、what_to_write 双轮净化

提示词写完后，每个 segment 的 what_to_write 必须经过两轮净化——第一轮去掉分析腔（结构层），第二轮去掉分析词（词句层）。两轮净化漏掉的内容，会原封不动地出现在正文里。

### 第一轮（结构层）：禁止 7 种 AI 味模式

| # | 命中模式 | 示例 | 修正 |
|---|---------|------|------|
| 1 | 心理机制分析 | "核心心理动作：三次失败→偏激。不是'他不怕死了'——是他把命押上去了。" | 不分析。写"今天和前几次不一样——不是试方案，是收命。" |
| 2 | 教 subagent "怎么写" | "这个认识不通过'他想起'来写。通过身体：后背、后脑、肋骨——上次死的时候被捅穿的位置。" | 不教。直接写"后背、后脑、肋骨——上次被捅穿的位置，在早上醒来的时候凉了一下。" |
| 3 | 概念对比论证 | "A 和 B 的区别在于：推演是'这样能行吗'，确认是'这样就行'。林默今天的脑子在做的是后者。" | 不区分。写"方案在心里跑了一遍——不是推演能不能行，是确认每个步骤。" |
| 4 | 代词指代分析结论 | "本章林默在后者。他不需要说服自己——方案已经是确定的了。" | 不指代。写"方案已经是确定的了。他在脑子里把它跑一遍。" |
| 5 | 分层心理指引 bullet list | "心理刻画指引：\n- 核心心理动作：……\n- 认识通过身体来写\n- 方案在心里跑的时候……" | 合并为连贯的场景叙述。不列条目不编号。 |
| 6 | 情绪标签化描述 | "情绪基调：偏激。是三次死亡堆出来的那种冷。" | 去标签。写"情绪：冷。不是平静——是火烧完之后压成密度更大的东西的那种冷。" |

### 第二轮（词句层）：删除 5 类分析词

| # | 删除目标 | 命中词 | 替换为 |
|---|---------|--------|--------|
| 1 | 分析性动词 | "做出决定""意识到""发现""察觉""明白""不再""开始试图""判断出" | 具体的身体动作或外部事件 |
| 2 | 总结性判断 | "这表明""这意味着""整件事的性质变了""碎片开始拼成""图景开始浮现" | 删除。读者自己品，不给结论 |
| 3 | 动机分析 | "因为""不是……而是……""之所以""背后的动机是" | 只写发生了什么，不写为什么发生 |
| 4 | 心理标签 | "犹豫了""在思考""在权衡""内心挣扎" | 身体反应——手指停在笔杆上、没接话、看了两秒窗外 |
| 5 | 硬上限 | what_to_write 超过 200 字 | 继续压缩。每个 beat 一句话，不连成段落 |

### 净化前后对照

```
章纲 what_to_write（净化前）：
陆征做出决定：不再等下一个威胁找上门。他给老邱打电话，老邱犹豫了一下——不是不想帮，是知道陆征不在队里了。最后还是查了。挂电话前低声说证词档案被人调走过。陆征意识到事情比他想象的严重——整件事的性质正在改变。

→ 命中项：
  - "做出决定" = 分析性动词
  - "犹豫了一下" = 心理标签
  - "不是……而是……" = 动机分析
  - "意识到" = 分析性动词
  - "整件事的性质正在改变" = 总结性判断

what_to_write（净化后）：
天还没亮，陆征拨通老邱的电话，报老马的号码请他查最后一次活跃位置。老邱在电话那头顿了两秒才应声。键盘响了一阵。挂电话前压低了声音——证词档案被人调走过，调档记录没留名字。

→ 92 字。全是具体动作和信息。没有解释、没有分析、没有结论。subagent 拿到就能写。
```

## 提示词自检

走完上面全部规则后，逐项过：

### 视角转换
- [ ] narrative_pov 已锁定，全章单一视角
- [ ] 所有 what_to_write 不含"主角感到""主角意识到""读者应该感受到"

### 快速嗅探（60 秒内完成）

如果不想逐条过，先做这四个快速嗅探——四个全过，大概率合格：

1. **抽一个 what_to_write，遮住字段名，能说出"这一段正文会发生什么"吗？** → 说不出来 → what_to_write 太模糊或分析腔太重
2. **扫描全篇提示词，找到一句"他感到""他意识到""这表明""这意味着"吗？** → 找到了 → 视角转换或词句净化漏网
3. **列出所有 segment 的 emotional_tone 序列，和 chapter.yaml 的 mood_progression 对比，节点数和方向一致吗？** → 不一致 → 段间情绪设计有断裂
4. **对着 memo.prohibitions 逐条，能在提示词的禁止事项中找到对应的可执行描述吗？** → 找不到 → 章纲红线未注入提示词

### 验证方法

**优先级：Agent Teams > subAgent > 自检**

1. **Agent Teams（最优）**：开独立 reviewer agent，传入 `prompts/vol-{N}-ch-{M}-prompt.md` + `chapter.yaml` + `writing-style.yaml` + `anti-ai.yaml` + 本清单。reviewer agent 逐项核查，对每项结论给出"通过/不通过 + 证据 + 具体位置（提示词第几行）"。
2. **subAgent（次优）**：同上，使用 subagent 执行审查。subagent 对"分析腔"的敏感度低于完整 agent——可能在 what_to_write 的分析腔前脱敏（因为 subagent 自己也是 AI）。
3. **自检（兜底）**：严格按本清单逐项过。特别注意三个高危盲区：
   - **what_to_write 的分析腔**：自检时自己最容易脱敏——你写的分析腔你自己读着很正常。建议把所有 what_to_write 复制出来，换一种字体、换一个阅读环境再看。或者倒着读（从末段看到首段）——打破阅读惯性。
   - **否定句式的隐蔽形态**："让 subagent 避免写成热血战斗"看起来像是正面指引，但核心动词是"避免"——这仍然是负面引导。扫描时关注"避免""防止""别让""切忌""注意不要"等隐蔽否定词。
   - **约束的可执行性**：一条约束你读着"很对"，但 subagent 读着"很空"——这是自检最容易放行的地方。每条约束问自己："subagent 拿着这条，能在写到某句话时停下来判断'这句话违规了'吗？"——判断不了的约束，删掉或重写。

**规则：拿到结论后先按 fail 项改完，再进入正文生成。不得带着 fail 项进入 Phase 5。**

### 自检实战示例

以下是同一章提示词"自检前"和"自检后"的对比——展示本清单的实际作用。

**自检前（有 5 个致命问题）：**

```
## 叙事段落 3：character_beat — 陆征独自整理线索

- 本段场景：
  陆征意识到这个案件并不简单。他在心里梳理着所有发现的线索，碎片开始拼成一副不祥的图景。
  不是单纯的失踪——背后有更复杂的势力在运作。他做出了一个决定：不再被动等待，
  要主动出击。核心心理动作：从被动反应转为主动追查。这标志着他心态的重要转折。

  → 问题1: "意识到" = 视角转换失败（上帝视角旁白）
  → 问题2: "碎片开始拼成图景" = 总结性判断
  → 问题3: "不是……而是……" = 否定句式 + 动机分析
  → 问题4: "核心心理动作" = 心理机制分析
  → 问题5: "这标志着" = 分析性结论
  → 问题6: 超过 200 字，全是分析腔，没有一个具体可写的动作

## 写作原则与禁忌：
  - 不要写成热血战斗文
  - 不要太拖沓
  - 不要用诗词腔

  → 三条全是否定引导——subagent 脑子过一遍"热血战斗""拖沓""诗词腔"，然后不知道怎么写
```

**自检后（修正为可执行的版本）：**

```
## 叙事段落 3：character_beat — 陆征独自整理信息

- 本段场景：
  回事务所。把便利贴贴在白板上。旁边是苏棠的照片——从苏沫那里拿的，一个短发女孩，
  笑起来有点歪嘴。便利贴上写了四个时间：最后通话 / 朋友圈地址查无此址 / 充电器还在墙上 /
  同事说"她去外地了"。第三个时间上画了一个圈。给方岩发消息：帮我查一个车牌。
  发完靠在椅背上，看着白板。外面有人在骂街，他没动。

  → 78字。全是可感知的动作和画面。没有分析、没有结论。subagent 拿到就能写。

## 写作原则与禁忌：
  - 冲突场景控制在日常环境中——出租屋、街边大排档、物流公司仓库。不写追逐和打斗。
  - 每个环境细节顺手带过，不超过 2 句。对话每轮 3-5 句收住。
  - 叙事用大白话。环境只写三样东西——光、声音、温度。不写风景。

  → 三条全是正面指引。subagent 知道"应该写成什么样"，不需要在脑子里先否定错误版本。
```

每个 fail 项都对应一个修正——修正后的提示词，subagent 拿到就能写。
