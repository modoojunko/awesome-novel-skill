# 主线拆纲指南

> Phase 2 第一步——定全书骨架再逐卷展开。拆完后写入 `story.md` 的 `story_arc` 段 + `volumes` 列表。
> 共 4 步：总主线 → 多卷结构选型 → 逐卷拆解 → 写入文件。首次进 Phase 2 走全流程，之后跳过。

---

## 总主线

**写什么：** 全书核心冲突 — 一句话说清全书核心冲突。

标准：**不做完所有卷答不出来的那种。** 每卷揭开一层，直到最后一卷才看到全貌。

| ✅ 合格 | ❌ 不合格 |
|---------|---------|
| "陆征追查失踪案发现背后横跨二十年的阴谋——每卷揭开一层，最后一卷才看到全貌" | "主角在异世界变强"——没有冲突、没有对抗、一卷就能写完 |
| "一个选错专业的程序员用代码重建崩溃的文明"——有对抗（崩溃的世界）、有路径（代码重建）、有终点（文明恢复） | "陆征在都市中调查各种案件"——"各种案件"不是一根主线 |

**判断标准：** 这句话包含了"谁 + 追求什么 + 对抗什么/克服什么"，且做完一卷答不完、需要多卷才能讲完。

### 怎么聊

不要让作者直接填字段。让作者说故事，你来提取：

> "设定都完成了，现在来规划整个故事的结构。用一句话说——你的主角从头到尾最核心的目标是什么？他要对抗什么才能达成这个目标？"

从作者的话里提取总主线，浓缩为一句话展示确认：

> "我概括一下全书主线：**{一句话}**——大致对吗？"

条件分支：

| 作者说 | 怎么接 |
|--------|--------|
| "对，就是这个" | 进入多卷结构选型 |
| "不太对，应该是……" | 按作者修正重写 | 
| "我还不知道全貌" | "没关系，先聊聊开头——主角最初的动机是什么？这个故事大概需要几卷来写完？" |
| "就一个案子，一卷搞定" | 单卷结构，跳过选型，直入逐卷拆解 |

## 多卷结构选型

**写什么：** 结构类型 + 总卷数。

不是让作者选分类，是根据总主线的体量和题材推荐：

| 结构 | 卷数 | 特点 | 适合 |
|------|------|------|------|
| **短篇/单卷** | 1 | 起承转合在一卷内完成 | 独立短篇、试水作、命题征文 |
| **三卷式** | 3 | 起(引入)→承(展开)→转合(高潮+结局) | 中篇悬疑、都市、仙侠升级 |
| **五卷式** | 5 | 起→承1(深化)→承2(转折前奏)→转(高潮)→合(余韵) | 长篇史诗、多势力博弈 |
| **多卷连载** | 3+ | 每卷自闭环 + 大主线推进 | 超长篇网文、开放式世界观 |

| ✅ 合格 | ❌ 不合格 |
|---------|---------|
| "三卷式——中篇悬疑，第一卷建立谜团，第二卷深入对抗，第三卷收束真相。" Agent 推荐+作者确认 | 不问三七二十一直接按"三卷式"写——没有判断过程 |
| "单卷——一个案子一卷搞定，不是长篇。" 作者自己确认的体量 | 选了"五卷式"但总主线一句话就讲完了——注水 |

### 怎么聊

> "你觉得这个故事大概需要几卷来写完？——不用精确，感觉是短篇一卷完，还是能展开写个三五卷？"

Agent 根据题材+总主线体量直接推荐（**不问"你想要哪种"**）：

> "我推荐 **{推荐结构}**——{一句理由}。你觉得这个体量合适吗？"

条件分支：

| 作者说 | 怎么接 |
|--------|--------|
| "合适" | 确认，进入逐卷拆解 |
| "我觉得多了/少了" | 按作者调整 |
| "我也不确定" | "那就先定第一卷，边写边感知整体体量。" → `template = "待定"` |

## 逐卷拆解

**写什么：** 分卷规划下每卷的标题 / 弧线位置 / 核心冲突 / 预估章节。

逐卷讨论，一卷一卷定：

| 字段 | 填写标准 | ✅ 合格 | ❌ 不合格 |
|------|---------|---------|---------|
| title | 卷标题，有信息量，不是"第一卷" | "失踪""深水""破局" | "第一卷" |
| arc_position | 在全篇弧线中的位置+本卷功能 | "起—引入谜团""转合—真相与代价" | "发展阶段"（空泛、没有功能描述） |
| core_conflict | 本卷核心冲突——"谁+做什么+被什么阻碍" | "追查苏棠失踪案，发现三年前旧案被压" | "主角继续调查"（没有对抗力量） |
| estimated_chapters | 预估章节数，允许±30%浮动 | 10, 12, 8 | 0 或空（没想清楚体量） |

### 怎么聊

先从第一卷开始：

> "第一卷你大概想写什么？从哪开始、到哪结束？主角在这卷里要达成什么？"

每卷确认后总结：

> "第一卷《失踪》：追查苏棠失踪案，发现三年前旧案被压。大约 10 章。对？"

一卷聊完聊下一卷，直到覆盖所有卷。

**注意：** 作者可能只知道第一卷细节，后面的只有一个模糊方向。允许后卷只写一句话 core_conflict + 浮动章节数，后续写到那卷时再细化。**拆纲是定骨架，不是定死。**

### 写入 story.md 后的完整示例

```markdown
# 钟声

> 项目索引——记录全书元信息和分卷规划。

- **作者：** 测试作者
- **创建时间：** 2026-04-30

## 引用路径

| 设定 | 路径 | 概要 |
|------|------|------|
| 世界观 | `settings/world-setting.yaml` | 当代虚构城市江城，现实主义都市悬疑 |
| 角色 | `settings/character-setting/` | 陆征（主角/私家侦探）、苏沫（委托人）、方岩（刑警） |
| 写作风格 | `settings/writing-style.yaml` | 冷峻写实风格，第三人称有限视角 |

## 故事主线

- **结构类型：** 三卷式
- **总卷数：** 3
- **全书核心冲突：** 陆征追查失踪案发现背后横跨二十年的阴谋——每卷揭开一层，最后一卷才看到全貌

## 分卷规划

### 卷1：失踪

- **弧线位置：** 起—引入谜团
- **核心冲突：** 追查苏棠失踪案，发现三年前旧案被压
- **预估章节：** 10

### 卷2：深水

- **弧线位置：** 承—深入对抗
- **核心冲突：** 陆征触及警队内部保护伞，每一步都有人抢先一步
- **预估章节：** 12

### 卷3：破局

- **弧线位置：** 转合—真相与代价
- **核心冲突：** 与幕后黑手正面交锋，所有线索汇聚到二十年前那件事
- **预估章节：** 8
```

## 各字段怎么被下游使用

| 字段 | 谁用 | 怎么用 |
|------|------|--------|
| 全书核心冲突（故事主线） | Phase 2.1 卷方向确定 + Phase 2.2 卷纲讨论 | 每卷规划时反问"这卷推进了总主线的哪一步？"——如果某卷核心冲突和总主线无关，就是注水卷 |
| 弧线位置（分卷规划） | Phase 2.1 模板选择/角色发声 | 首卷：弧线位置影响结构模板的阶段侧重（"起"阶段以引入为主）；卷 N+1：弧线位置提示本卷在全篇中的紧张度——"转"卷节奏应加快 |
| 核心冲突（分卷规划） | Phase 2.2 卷纲讨论 | 作为本卷 volume.yaml 的核心冲突来源，直接写入 |
| 预估章节（分卷规划） | Phase 2.2 章节数分配 | 作为 chapters_summary 各阶段分配章节数的总量参考 |
| 结构类型（故事主线） | 跨卷节奏感知 | 知道总共有几卷，帮助判断每卷在整体中的节奏位置——三卷式的第二卷不能拖太长 |

## 验收：从下游使用反推

### 总主线

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | 全书核心冲突非空 |
| 该不该 | 是作者确认过的，不是 Agent 猜的 |
| 对不对 | **后续每卷规划时**要反问"这卷推进了总主线的哪一步？"如果某个卷的核心冲突跟总主线对不上——或者总主线太宽泛导致"什么都能往里塞"——说明总主线不合格。→ 验证：指着总主线问"第三卷的内容能追溯到总主线吗？"能 → 合格 |

### 多卷结构选型

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | `template` 有值或"待定" |
| 该不该 | 是 Agent 推荐+作者确认的，不是默认值 |
| 对不对 | **跨卷节奏判断**要用 `template`+`total_volumes` 来感知每卷在整体中的节奏位置。如果 `total_volumes=5` 但实际只写了 3 卷的内容，剩下的 2 卷会在后续阶段变成注水。→ 验证：读完所有卷的 core_conflict，问"这些冲突够撑 {total_volumes} 卷吗？" |

### 逐卷拆解

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | 每卷的 title / arc_position / core_conflict / estimated_chapters 均有值 |
| 该不该 | 每卷核心冲突是"谁+做什么+被什么阻碍"格式，不是孤立的愿望 |
| 对不对 | **Phase 2.2 卷纲讨论**要从卷的核心冲突推本卷的章节分配。如果核心冲突太宽泛（"主角调查"），Agent 在做章节分配时不知道哪些事件属于本卷。→ 验证：挑一卷的核心冲突，问"这卷大概会经历哪几个阶段才能解决这个冲突？"能答上来 → 合格 |

## 文件层面验收

读取 `story.md` 确认：

- [ ] 故事主线章节有"全书核心冲突"，含"谁+追求+对抗"三要素
- [ ] "结构类型"有值（或标记"待定"）
- [ ] "总卷数" > 0
- [ ] 分卷规划下每卷有标题（非"第X卷"）、弧线位置（含功能描述）、核心冲突（含对抗）、预估章节（> 0）
- [ ] volumes 的 arc_position 合起来覆盖了从开头到结局的完整弧线（没有缺少"结尾"段）
- [ ] 所有字段是作者确认过的，不是 Agent 代填的
