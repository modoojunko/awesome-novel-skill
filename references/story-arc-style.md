# 主线拆纲指南

> Phase 2 第一步——定全书骨架再逐卷展开。拆完后写入 `story.md` 的 `story_arc` 段 + `volumes` 列表。
> 共 3 步：总主线 → 从结局倒推找断点 → 定每卷冲突。首次进 Phase 2 走全流程，之后跳过。

---

## 总主线

**写什么：** 全书核心冲突 — 一句话说清全书核心冲突。

标准：**不做完所有卷答不出来的那种。** 每卷揭开一层，直到最后一卷才看到全貌。

| ✅ 合格 | ❌ 不合格 |
|---------|---------|
| "陆征追查失踪案发现背后横跨二十年的阴谋——每卷揭开一层，最后一卷才看到全貌" | "主角在异世界变强"——没有冲突、没有对抗、一卷就能写完 |
| "一个选错专业的程序员用代码重建崩溃的文明"——有对抗（崩溃的世界）、有路径（代码重建）、有终点（文明恢复） | "陆征在都市中调查各种案件"——"各种案件"不是一根主线 |

**判断标准：** 这句话包含了"谁 + 追求什么 + 对抗什么/克服什么"，且做完一卷答不完、需要多卷才能讲完。

### 怎么聊

不要让作者直接填字段。让作者说故事，你来提取：

> "设定都完成了，现在来规划整个故事的结构。用一句话说——你的主角从头到尾最核心的目标是什么？他要对抗什么才能达成这个目标？"

从作者的话里提取总主线，浓缩为一句话展示确认：

> "我概括一下全书主线：**{一句话}**——大致对吗？"

条件分支：

| 作者说 | 怎么接 |
|--------|--------|
| "对，就是这个" | 进入定终点，从结局倒推 |
| "不太对，应该是……" | 按作者修正重写 | 
| "我还不知道全貌" | "没关系，先聊聊结局——你想象中最后一幕是什么画面？从那里倒推回来就好拆了。" |
| "就一个案子，一卷搞定" | 单卷，跳过拆纲直入卷纲讨论 |

---

## 从结局倒推

不要问"几卷"，不要问"结构模板"。问终点。

### 第一步：定终点

先让作者说出结局——最后一卷结束时主角在哪儿、世界什么样、冲突怎么收场的。

> "你想象的故事最后一幕是什么画面？主角最终站在什么位置？"

**判断标准：** 能说清起点和终点，中间的路自然浮现。

| ✅ 说得清 | ❌ 说不清 |
|---------|----------|
| "主角揭开二十年前的真相，但选择不公开——有些秘密带进坟墓比公之于众更好。他继续当他的私家侦探，只是心里有了一个永远不会说的答案。" | "主角变强了"——什么程度？在哪里？和谁？ |
| "仙尊转世的主角最终回到仙界，坐上了当年被赶下来的位置，但发现坐上这个位置意味着要承担他不想要的代价。" | "打败最终boss"——打败之后呢？做了boss的位置？世界变好了？ |

**不用一步到位：** 作者不一定想好了结局。如果连大方向都没有，聊这几种可能性帮 ta 定调子：

> "你是更倾向于 happy ending，还是悲剧结局，或者开放式留白？"

### 第二步：找自然断点

从起点到终点之间，故事会经历几个大的阶段。**不是凭空想象，是从"怎么从起点走到终点"推理。**

> "从开头到结局，你觉得故事会经历几个大的转折？比如'主角第一次发现真相→遭遇重大失败→反击→最终对决'这样。"

Agent 从作者的描述中提取断点，不要问分类。

| ✅ 正确引导 | ❌ 错误引导 |
|-----------|-----------|
| "听起来你的故事从接案到揭开真相，中间会经历'发现旧案被压→深入调查遭遇警告→触及保护伞→最后对决'这几个阶段？" | "你选三卷式还是五卷式？"——分类没有帮助 |
| "主角从废柴到成仙，中间有几个坎要过？比如'入门被看不起→秘境夺宝→宗门大战→渡劫飞升'" | "第一卷引入，第二卷展开，第三卷高潮"——套模板，不是作者的故事 |

**规律（仅供参考，不套模板）：**

| 故事体量 | 典型断点数 | 对应卷数 |
|---------|-----------|---------|
| 短篇 | 0-1 个断点 | 1 卷 |
| 中篇 | 2 个断点 | 3 卷（起·承·转合） |
| 长篇 | 3-4 个断点 | 4-5 卷（起·承·转·合或更多） |

**兜底：** 作者完全说不清 → "那就先定第一卷的内容，边写边感知体量。写完了第一卷自然知道第二卷该往哪走。"

### 第三步：定每卷冲突

每个断点之间就是一卷。逐卷确认：

| 字段 | 填写标准 | ✅ 合格 | ❌ 不合格 |
|------|---------|---------|---------|
| 标题 | 有信息量，不是"第一卷" | "失踪""深水""破局" | "第一卷" |
| 弧线位置 | 在全篇弧线中的位置+本卷功能 | "起—引入谜团""转合—真相与代价" | "发展阶段"（空泛） |
| 核心冲突 | "谁+做什么+被什么阻碍" | "追查苏棠失踪案，发现三年前旧案被压" | "主角继续调查"（没有对抗） |
| 预估章节 | 允许±30%浮动 | 10, 12, 8 | 0 或空 |

**每卷确认方式：**

> "第一卷《失踪》：从接案到发现旧案被压，核心是追查苏棠下落但遭遇阻力。大约 10 章。对？"

一卷确认完再聊下一卷。

**允许后卷模糊：** 作者可能只知道第一卷细节，后面的只有一个方向。后卷只写一句话核心冲突 + 浮动章数即可，写到那卷时再细化。

**结构类型标签在最后标注：** 所有卷拆完后，回头看几卷，标记结构类型（三卷式/五卷式/单卷）。这是归纳标签，不是起点。

### 写入 story.md 后的完整示例

```markdown
# 钟声

> 项目索引——记录全书元信息和分卷规划。

- **作者：** 测试作者
- **创建时间：** 2026-04-30

## 引用路径

| 设定 | 路径 | 概要 |
|------|------|------|
| 世界观 | `settings/world-setting.yaml` | 当代虚构城市江城，现实主义都市悬疑 |
| 角色 | `settings/character-setting/` | 陆征（主角/私家侦探）、苏沫（委托人）、方岩（刑警） |
| 写作风格 | `settings/writing-style.yaml` | 冷峻写实风格，第三人称有限视角 |

## 故事主线

- **结构类型：** 三卷式
- **总卷数：** 3
- **全书核心冲突：** 陆征追查失踪案发现背后横跨二十年的阴谋——每卷揭开一层，最后一卷才看到全貌

## 分卷规划

### 卷1：失踪

- **弧线位置：** 起—引入谜团
- **核心冲突：** 追查苏棠失踪案，发现三年前旧案被压
- **预估章节：** 10

### 卷2：深水

- **弧线位置：** 承—深入对抗
- **核心冲突：** 陆征触及警队内部保护伞，每一步都有人抢先一步
- **预估章节：** 12

### 卷3：破局

- **弧线位置：** 转合—真相与代价
- **核心冲突：** 与幕后黑手正面交锋，所有线索汇聚到二十年前那件事
- **预估章节：** 8
```

## 各字段怎么被下游使用

| 字段 | 谁用 | 怎么用 |
|------|------|--------|
| 全书核心冲突（故事主线） | Phase 2.1 卷方向确定 + Phase 2.2 卷纲讨论 | 每卷规划时反问"这卷推进了总主线的哪一步？"——如果某卷核心冲突和总主线无关，就是注水卷 |
| 弧线位置（分卷规划） | Phase 2.1 模板选择/角色发声 | 首卷：弧线位置影响结构模板的阶段侧重（"起"阶段以引入为主）；卷 N+1：弧线位置提示本卷在全篇中的紧张度——"转"卷节奏应加快 |
| 核心冲突（分卷规划） | Phase 2.2 卷纲讨论 | 作为本卷 volume.yaml 的核心冲突来源，直接写入 |
| 预估章节（分卷规划） | Phase 2.2 章节数分配 | 作为 chapters_summary 各阶段分配章节数的总量参考 |
| 结构类型（故事主线） | 跨卷节奏感知（仅参考） | 事后归纳的标签，不是规划依据。写第二卷时不需要因为"这是三卷式的第二卷所以节奏要快"，而是因为"这卷的内容是深入对抗所以节奏要快" |

## 验收：从下游使用反推

### 总主线

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | 全书核心冲突非空 |
| 该不该 | 是作者确认过的，不是 Agent 猜的 |
| 对不对 | **后续每卷规划时**要反问"这卷推进了总主线的哪一步？"如果某个卷的核心冲突跟总主线对不上——或者总主线太宽泛导致"什么都能往里塞"——说明总主线不合格。→ 验证：指着总主线问"第三卷的内容能追溯到总主线吗？"能 → 合格 |

### 定终点

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | 作者说过结局方向（最终画面 / BEorHE / 主角最终状态），非空 |
| 该不该 | 是作者自己描述的，不是 Agent 编的 |
| 对不对 | **后续每卷规划时**要看每卷的弧线位置是否在朝这个结局推进。如果第三卷的内容和结局方向不符——比如作者说"悲剧结局"但第三卷写得像合家欢——说明方向没定好。→ 验证：指着终点的描述问"第三卷的内容在往这个方向走吗？" |

### 找自然断点

| 验收方式 | 怎么验 |
|--------|--------|
| 有没有 | 作者说了几个阶段/转折，不是 Agent 硬塞的分类 |
| 该不该 | 断点是从"怎么从起点走到终点"推理出来的，不是套模板 |
| 对不对 | **跨卷节奏判断**要看断点之间的距离是否合理。第一个断点在开头后不久，最后一个断点在结局前不远——中间分布均匀。如果前三章内容密集、后两卷空空，说明断点没找对。→ 验证：读完所有卷的核心冲突，问"这些冲突是从起点到终点的自然路径吗？" |

### 定每卷冲突

| 验收方式 | 怎么验 |
|---------|--------|
| 有没有 | 每卷的标题 / 弧线位置 / 核心冲突 / 预估章节均有值 |
| 该不该 | 每卷核心冲突是"谁+做什么+被什么阻碍"格式，不是孤立的愿望 |
| 对不对 | **Phase 2.2 卷纲讨论**要从卷的核心冲突推本卷的章节分配。如果核心冲突太宽泛（"主角调查"），Agent 在做章节分配时不知道哪些事件属于本卷。→ 验证：挑一卷的核心冲突，问"这卷大概会经历哪几个阶段才能解决这个冲突？"能答上来 → 合格 |

## 文件层面验收

读取 `story.md` 确认：

- [ ] 故事主线章节有"全书核心冲突"，含"谁+追求+对抗"三要素
- [ ] 故事有终点方向（最后一幕 / 主角最终状态 / BEorHE，任一即可，不是空谈）
- [ ] 分卷数有依据（从起点到终点的自然断点，不是硬凑的数字）
- [ ] "结构类型"有值（或标记"待定"）——这是事后归纳，不是出发点
- [ ] 每卷有标题（非"第X卷"）、弧线位置（含功能描述）、核心冲突（含对抗）、预估章节（> 0）
- [ ] 所有卷合起来覆盖了从起点到终点的完整路径（没有跳过的弧线段）
- [ ] 所有字段是作者确认过的，不是 Agent 代填的
