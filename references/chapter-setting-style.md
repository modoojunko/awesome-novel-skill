# 章纲设定指南

> 第三阶段第二步——给选定的一章填充内容。包括方向提案确认、备忘录 7 段、情绪设计。产出写入 `chapters/vol-{N}-ch-{M}.yaml`。

**前置条件：** 方向提案已确认（第三阶段第一步 硬节点通过）。volume-N.yaml 的 chapters_summary 已有本章的定位。

---

## 核心原则：章纲是写给 subagent 的需求文档

> 下面所有规则之前，先问一句：**"一个没有直觉、没读过前文的 subagent，拿着这份章纲和提示词，能写出正确的正文吗？"**

章纲不是给作者看的笔记，不是给读者看的预告——是给subagent的指令。卷纲定"这章在卷的哪一步"，章纲定"这一段的正文怎么写"。

**判定标准：**
- 每段备忘录换成"这件事发生在正文的哪个位置？以什么方式呈现？"——答不出来就是没写清楚
- 大纲的每条关键点换成"subagent 能用它写出正文的一个段落吗？"——写不出来就是太模糊
- 整份章纲读完后，能在脑子里过一遍本章的完整正文——过不下来就是有缺口

**两条底线**（任一不过，禁止进入提示词生成）：
1. **每条备忘录可执行**——subagent 读完后能说出"我要在正文的第 X 段写 Y 这件事"。"这一章要有悬念感"不算可执行。
2. **章纲不含上帝视角解构**——大纲和备忘录中不得出现"本章讲述了""主角经历了""通过 X 展现了 Y"。这些被 subagent 抄进正文的概率极高，一旦抄进去整章报废。

---

## 一、方向确认

方向提案确定了"这章写什么方向"，章纲要把方向转成"这章怎么写"。

**怎么聊：** 读 volume-N.yaml 中本章的 chapters_summary，问作者：

> "这章的概要（summary）是'陆征勘查现场发现矛盾'。从这出发，大刚来说这章有几个段落（场景）？比如'进入出租屋→发现床没铺→发现充电器→找到便利贴→离开时被盯上'这样的分段。"

把作者说出的段落写入大纲关键点（outline.key_points）。每段对应正文的一个可写段落。**关键点的粒度是"subagent 看到就能写一段"**。

**常见问题与处理：**

| 作者说 | 问题 | 接法 |
|--------|------|------|
| "就一个场景，勘查现场" | 太粗，subagent 没法在一个段落里写出勘查全过程 | "勘查现场本身有几个步骤？进门看到什么→发现了什么具体东西→这些让主角得出什么判断？" |
| "先写勘查再写被盯上" | 只有两步，中间缺过渡 | "勘查完了怎么离开的？离开时有没有异常？" |
| "大概 3-4 段吧" | 模糊 | "具体哪几段？给每段取个名字或说一句'这段写什么'" |

**产出：** 大纲概要（outline.summary，1 句概括）+ 大纲关键点（outline.key_points，3-6 条，每条对应一段正文）+ 大纲出场角色（outline.characters）+ 大纲场景地点（outline.location）+ 大纲时间（outline.time）。

---

## 二、备忘录 7 段

备忘录是章纲的核心——7 段分别对应 subagent 写作时需要的 7 种指令。每段回答不同的"怎么写"。

**通用判断：** 每段写完后反问——"subagent 看到这句知道写哪段正文吗？答得出具体场景/动作/事件吗？"

---

### 1. 当前任务（current_task）

subagent 需要知道"这一章的目标是什么"——不是主题，不是感受，是完成什么具体动作。

**写什么：** 一句话：本章要完成的具体动作 + 与哪条故事线的什么关系。

**怎么聊：** 从大纲提炼。大纲拆出了段落，当前任务是这些段落的共同目标。

> "这章几个段落下来，核心要完成什么事？用一句话说。"

如果作者说"建立悬念"或"推进主线"，追一句：

> "具体通过什么事来建立悬念/推进主线？把事写进当前任务（current_task），把感受留给读者期待（reader_expectation）。"

```
✅ 合格：
"建立陆征的人物形象和日常工作状态，通过苏沫的委托引入主线案件"
→ subagent 知道：写日常状态 + 写委托场景 + 引入案件

"通过现场勘查推进案件调查，线索逐渐指向苏棠不是主动离开的"
→ subagent 知道：写勘查 + 推翻警方结论

❌ 不合格：
"本章建立悬念"
→ 建立什么悬念？通过什么事？subagent 只能猜

"主角继续调查案件"
→ "继续"是什么意思？调查什么？发现了什么？等于没写
```

---

### 2. 读者期待（reader_expectation）

这是整份备忘录的核心——决定了本章的情绪走向。subagent 需要知道：读者在上一章结束时处于什么状态，本章该怎么对待这个状态。

**写什么：** 三个子字段——情绪状态（state，读者当前情绪）、策略（strategy，本章的处理策略）、具体说明（detail，读者在等什么）。

**怎么聊：** 读上一章章纲文件的情绪设计（emotional_design）。看上一章落的是什么情绪钩子，读者带着什么情绪缺口进入本章。

> "上一章结尾，读者处在什么情绪？比如'好奇苏棠去哪了'、'紧张陆征被盯上了'。"

> "那这章怎么对待这个情绪？——给点进展满足一下，还是压着不给，还是先给一点再制造更大的疑问？"

> "具体来说，读者在等什么？等谁出场、等某个真相、等一场对决？说具体的人/事/物。"

```
✅ 合格：
state: "好奇——上一章结尾留下悬念，读者想知道苏棠到底去哪了"
strategy: "制造新缺口"
detail: "读者期待调查有进展——前半章给进展（房间矛盾细节），后半章制造新疑问（楼下的男人是谁？）"
→ subagent 知道：前半章给，后半章要

❌ 不合格：
state: "期待"
strategy: "维持悬念"
detail: "读者期待下一章的内容"
→ 三行全空——subagent 不知道读者在等什么、该给什么
```

---

### 3. 兑现计划（payoff_plan，旧事管理）

告诉 subagent 本章该碰哪些旧事、不该碰哪些。管理读者记忆：什么事该提、什么事不该提、什么事只能提一半。

**写什么：** 三个列表——必须了结（must_resolve）、必须压住（must_hold）、推进不收束（partial_advance）。不用编号，用大白话描述。

**怎么聊：** 从前面几章的章纲读过来——哪些事还在悬着、哪些事该收了、哪些底牌还太早不能掀。

> "前面埋了哪些还没收的线？这章有没有到了该收的？"

> "哪些事现在绝对不能碰——掀了就收不回来的？"

> "哪些事可以推一把但不能全亮？推多少？——是'让读者知道有联系'还是'让主角隐约感觉不对'？"

```
✅ 合格：
must_resolve: []  # 第 1 章，没有要收的旧事
must_hold: ["不能揭示陆征离开警队的原因"]
partial_advance:
  - "种下主线悬念——苏棠失踪，警方判定主动离开，但细节有矛盾"

❌ 不合格：
partial_advance:
  - "推进主线"
→ subagent 不知道"推进"什么意思——可能推一毫米，也可能直接收束

must_resolve: ["揭示真相"]
must_hold: ["揭示真相"]
→ 同一件事既要收又要压——逻辑矛盾
```

---

### 4. 过渡功能（downtime_functions）

非冲突段落不是"休息"——每段过渡场景必须有明确的叙事功能。没功能的段落不该存在。

**写什么：** 每条 `[段落位置/场景] → [承担功能]`。功能必须是具体动作。

**怎么聊：** 从大纲中拎出非冲突段落，逐段反问：

> "这段去掉会影响什么？"

不影响就删掉。如果作者说"调节奏""过渡一下"，追一句：

> "调节奏的同时这段能不能做点别的事——比如埋伏笔、展示角色性格、补充信息？"

```
✅ 合格：
- "开头办公室场景 → 建立陆征的日常状态和性格底色"
- "路上环境描写 → 南岸新区与老城区的对比，暗示城市的两面性"

❌ 不合格：
- "过渡段落" → 没有功能映射，subagent 会写无意义填充
- "日常描写" → subagent 可能写一页吃饭睡觉
```

---

### 5. 关键抉择（key_choices）

本章中角色的重要选择必须经过检验——为什么做、符合人设吗、读者会觉得突兀吗。

**写什么：** 每条包含角色做了什么选择 + 为什么 + 为什么符合人设 + 读者感受验证。

**怎么聊：** 读角色文件（世界观/自我定位/价值观），判断选择是否从设定自然生长出来。

> "这章 xxx 要做 YYY 这个决定。为什么？是利益驱动还是情感驱动？"

> "这符合 ta 之前的人设吗？——如果作者说'符合'，追问'具体哪条设定支撑这个选择？'"

> "读者看到这个选择会怎么想？会觉得'他当然会这么做'还是'他怎么突然变了'？"

如果选择只是"情节需要"而不是角色驱动——退回调整。

```
✅ 合格：
- "陆征为什么接下案子？——一半是需要收入维持事务所运转，一半是苏沫说的地址查无此址让他觉得警方草率了。符合他人设：不急于下判断，相信细节。"
→ 不是热血也不是同情，是专业警觉+现实需要

- "方岩为什么私下警告陆征？——他对陆征有愧疚，但也不希望陆征捅出篓子连累自己。矛盾的选择符合人设。"
→ 有愧疚也有自保，不是单纯的"好人帮忙"

❌ 不合格：
- "主角选择了调查"
→ 为什么调查？什么驱动？符合人设吗？全没答——subagent 会默认写成"因为他正义"
```

---

### 6. 章尾改变（required_changes）

本章结束时什么东西必须变了。章末什么都没变 → 这章不该存在。

**写什么：** 1-3 条具体改变，每条标注改变类型（信息/关系/物理/权力）+ 从什么变成什么。

**怎么聊：** 从当前任务（current_task）推。

> "这章要完成 XXX 这个任务。那结束时，什么东西和开头不一样了？——谁知道了什么？谁和谁的关系变了？主角在哪方面不同了？"

如果作者说"调查推进了"或"关系变化了"，追一句：

> "'推进了'具体指什么？从'不知道'变成'知道'了什么？关系从'信任'变成'怀疑'还是从'陌生'变成'合作'？说清'从什么变成什么'。"

```
✅ 合格：
- "信息改变：陆征（和读者）知道了苏棠失踪的基本事实和几个疑点"
- "威胁引入：楼下的男人盯着陆征——调查被人盯上了"

❌ 不合格：
- "调查有进展" → 什么进展？知道了什么？没有新信息 = 没有改变
- "关系变化" → 谁和谁？从什么变成什么？全空
```

---

### 7. 硬约束（prohibitions）

告诉 subagent 绝对不能做什么。不是"最好不要"，是"做了就报废"。

**写什么：** 具体的禁止场景或行为，格式："不要在 X 场景中做 Y 事"或"不要让 X 角色做 Y 事"。必须可验证。

**怎么聊：** 从卷纲到本章的渐进式约束积累。

> "这章有什么绝对不能碰的东西？——哪些设定还不能揭示、哪些角色还不能出场、哪些关系还不能挑明？"

如果作者说"不要太拖沓"或"保持紧张感"，打回：

> "硬约束（prohibitions）是'不要做什么'，不是'要做什么'。说具体——不让谁出场？不揭示什么信息？不让发生什么场景？"

```
✅ 合格：
- "不要揭示陆征三年前离开警队的详细原因——只通过细节暗示他以前是刑警"
→ 正文写完后可以逐句扫描"三年前""离开警队"等关键词，验证

- "不要让楼下的男人和陆征对话——只通过眼神和氛围制造压力"
→ 肢体可写、氛围可写，但对话是红线

❌ 不合格：
- "不要太拖沓" → 怎么算拖沓？subagent 没有判断标准
- "保持紧张感" → 这不是禁止，是期望
```

## 三、情绪设计（emotional_design）

情绪设计是给 subagent 的情绪操作指令：从哪个情绪开始、经过什么情绪转折、以什么情绪结束。

**怎么聊：** 从读者期待（reader_expectation）的情绪状态（state）和策略（strategy）推导。

> "读者开头是什么情绪？——就是情绪状态（state）描述的那个。"

> "中间经历什么转折？因为什么事，情绪从 A 转向 B？"

> "结尾落在什么情绪？情绪钩子（emotional_hook）是哪个类型？具体谁/什么事让读者产生这个情绪？"

### 字段填写指南

| 字段 | 怎么聊 | 填写标准 | 常见错误 |
|------|--------|---------|---------|
| 主情绪（primary_mood） | "这章的主色调是什么情绪？——选一个最精准的词" | 一个精确情绪词（好奇/紧张/压抑/爽快/温情/悲伤/愤怒/释然） | "复杂""混合"——subagent 不知道主色调 |
| 情绪走向（mood_progression） | "开头什么情绪→中间发生了什么→以什么情绪结束？至少三步，每步跟一个具体事件" | A 情绪 → 因 XX 事件 → B 情绪 → 因 YY 事件 → C 情绪 | "先抑后扬"——只有两个节点没有中间过渡 |
| 强度峰值（intensity_peak） | "哪一段情绪最强烈？——不是'结尾'，是哪一段里的哪个瞬间" | 具体到关键点或场景 | "结尾"——subagent 不知道具体哪段 |
| 强度等级（intensity_level） | "和上一章比，这章强度高还是低？相邻章差不超过 2" | 1-10 数字，与相邻章差 ≤ 2 | 每章都填 7 或者每章都填 3——没有变化 |
| 情绪钩子（emotional_hook） | "章末留在读者心里的情绪是什么？——不是类型词，是具体谁/什么事让读者睡不着" | 情绪类型+具体内容 | "悬念"——什么悬念？读者对什么产生悬念？ |

### 微兑现（micro_payoffs）

每章至少 1 个微兑现——读者在本章能获得的小收获。过渡章也必须至少 1 个。

**怎么聊：**

> "这章读者看完能带走什么？——一个新信息、一段关系推进、还是一个情绪释放？放在哪一段？"

如果作者说"这章主要是过渡，没什么好给的"，打回：

> "过渡章更要有微兑现——不然读者看完觉得'什么都没发生'。哪怕只是'确认了主角的某个性格特征'也算信息型兑现。"

| 字段 | 填写标准 |
|------|---------|
| 类型（type） | 信息（info，新线索/新信息）/ 关系（relationship，关系推进）/ 情绪（emotion，情绪释放）/ 线索（clue，伏笔回收）/ 能力（ability，能力展现）/ 资源（resource，获得物品）/ 认可（recognition，获得认可） |
| 描述（description） | 具体事件：读者获得了什么，不是"获得了信息" |
| 位置（location） | 前段/中段/后段——不要放在章末，别跟情绪钩子（emotional_hook）抢位置 |

## 四、AI 味自检

章纲阶段的 AI 味会直接污染提示词。章纲里的每一句"本章讲述了""主角经历了""通过 X 展现了 Y"都会变成正文里的上帝视角解构。**写完章纲后，逐段扫描以下模式：**

### 6 种要命模式

| # | 模式 | 扫描位置 | 命中例句 | 修正后 |
|---|------|---------|---------|--------|
| 1 | 上帝视角概述 | 大纲、备忘录当前任务 | "本章讲述了陆征进入出租屋勘查" | "陆征进入出租屋勘查。推开门的瞬间闻到一股剩饭味" |
| 2 | 抽象心理总结 | 备忘录读者期待、关键抉择 | "他感到不安" | "他合上笔记本，推门出去" |
| 3 | 分析性结论 | 备忘录读者期待、章尾改变 | "这表明案件比想象中复杂" | 只写"什么变了"，不写"这意味着什么" |
| 4 | 文学评论腔 | 备忘录当前任务、过渡功能 | "通过细致入微的观察展现了主角专业能力" | "陆征注意到床没铺、充电器还在墙上、冰箱有剩饭" |
| 5 | "不是而是"句式 | 所有备忘录字段 | "他不是愤怒，而是失望" | 直接写"他失望了"。必须用对比就拆两句 |
| 6 | 抽象标签 | 备忘录当前任务、章尾改变 | "关系递进""冲突升级""性格成长" | "陆征和方岩的关系从互不信任变成交换信息" |

### 空洞形容词速查

| 类别 | 命中词 | 改法 |
|------|--------|------|
| 空洞形容词 | 精彩、紧张、有趣、不错、震撼、感人、深刻 | 删掉，改可验证描述——"紧张"→"他在后视镜里看那人的手在口袋里" |
| 抽象副词 | 深刻地、充分地、淋漓尽致地、完美地 | 全部删除 |
| 程度词 | 充分、足够、适当、一定程度 | 改具体数量或范围 |

### 可执行快速验证

| 备忘录段 | 反问 |
|---------|------|
| 当前任务（current_task） | "读完这句，能说出正文要写哪几个场景吗？" |
| 读者期待（reader_expectation） | "读完这三行，知道正文前半章给什么、后半章要什么吗？" |
| 兑现计划（payoff_plan） | "每条推进不收束（partial_advance）读完后，知道这章该碰旧事的哪部分吗？" |
| 章尾改变（required_changes） | "每条的'从什么变成什么'读完后，知道章末什么必须不一样吗？" |
| 硬约束（prohibitions） | "每条读完后，正文写完后能否检查'这条违反了吗'？" |

---

## 验收

### 快速嗅探（30 秒）

1. **遮住章纲，能说出"这一章结束时什么东西变了"吗？** → 说不出 → 章尾改变没写清楚
2. **对着读者期待的具体说明，能说出"正文前半章写什么、后半章要什么"吗？** → 说不出 → 备忘录不可执行
3. **扫描备忘录，找到"本章讲述了""主角经历了""通过X展现了Y"吗？** → 找到了 → AI 味污染，立即修

### 逐项检查

- [ ] **大纲关键点（outline.key_points）** 每条对应一段可写正文，粒度是"subagent 看到就能写一段"
- [ ] **当前任务（current_task）** 不含抽象标签，不含上帝视角概述，可执行
- [ ] **读者期待（reader_expectation）** 的情绪状态/策略/具体说明都有值，具体说明具体到人/事/物
- [ ] **兑现计划（payoff_plan）** 三个列表完整（空列表显式 `[]`），推进不收束写了具体推进方式
- [ ] **过渡功能（downtime_functions）** 每条有 `[场景] → [功能]` 映射
- [ ] **关键抉择（key_choices）** 每条回答了"为什么做 + 符合人设吗 + 读者觉得突兀吗"
- [ ] **章尾改变（required_changes）** 1-3 条，每条有改变类型 + "从什么变成什么"
- [ ] **硬约束（prohibitions）** 每条可验证（正文写完后能对照检查），不含抽象禁忌
- [ ] **主情绪（primary_mood）** 是精确情绪词，不是"复杂""混合"
- [ ] **情绪走向（mood_progression）** 至少三个节点（A→B→C），节点间有因果锚点
- [ ] **强度峰值（intensity_peak）** 具体到场景，不是"结尾"
- [ ] **强度等级（intensity_level）** 与相邻章差不超过 2
- [ ] **情绪钩子（emotional_hook）** 有具体内容（谁/什么事/什么悬念）
- [ ] **微兑现（micro_payoffs）** 至少 1 个，类型/描述/位置都有值
- [ ] **备忘录无上帝视角概述、无分析性结论、无文学评论腔、无"不是而是"句式、无空洞形容词**
