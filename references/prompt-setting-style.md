# 提示词组装指令

将章纲 + writing-style + genre-profile 组装为 subagent 可执行的提示词。产出写入 `prompts/vol-{N}-ch-{M}-prompt.md`。

---

## 一、通用底层骨架

所有题材通用的 7 个必填模块，格式固定，每章填一次。

```
## 1 基础定位
题材：
整体基调：
本章定位：铺垫/推进/冲突/收尾
目标字数：

## 2 前置承接
前情摘要：（50 字以内，上章结尾关键剧情/人物矛盾/停留场景）
本章场景：（本章核心发生地）

## 3 在场人物即时状态
人物A：当下情绪 + 立场 + 小动作习惯
人物B：当下情绪 + 立场 + 隐藏想法

## 4 本章叙事任务
（一句话：这一章要完成什么事、埋哪个伏笔、产生什么人物张力）

## 5 行文硬规则
1. 固定单一 POV 视角，不跳转。
2. 允许每 1000 字 1-2 句心理活动或内心独白——全程外化反而显刻意。
3. 允许 10% 以内的情绪直述（"他心里一紧""她感到一阵酸涩"），关键节点不需强行绕弯子。
4. 允许段落长短自由——该沉浸时放开写，该收时果断停，不一刀切均分字数。
5. 允许角色发呆、走神、注意无关细节——这些"毛边"增加生活感，不是每句话都服务剧情。
6. 无鸡汤升华、无套路旁白、无注水。
7. 对话贴合人设，不书面化、不台词化。

## 6 挂载风格插件
【】悬疑冷峻白描  【】都市细腻日常  【】玄幻恢弘叙事  【】网文快节奏

## 7 附加规则
（按需勾选，可空）
```

## 二、核心方法：叙事种子法

模块 4（本章叙事任务）的填充方法。

### 原理

不把剧情写成写作指令（"这段写男主对女主的了解"），写成 POV 角色的叙事画面（"陆屿站在书架前，手指扫过书脊，准确抽出一本旧画册"）。subagent 读到的是"继续写这段"，不是"去做什么"。

**注意：** 叙事种子是"路标"不是"铁轨"——写够方向即可，不必每句都精准到分毫不差。留一些模糊空间让 subagent 自己发挥，正文反而更自然。种子太精确 → 执行痕迹重 → 像剧本不像小说。

### 三条转化规则

| 规则 | 含义 |
|------|------|
| 问"角色看到/听到/做了什么？" | 不写作者意图（展露在乎/关系拉近），只写具体画面 |
| 写"正在发生的事" | 不写将来时（本章将/接下来会），写进行时（她走到巷口） |
| 用角色感官写 | 不写分析（他意识到/读者感受到），写画面声音气味 |

### 示例

```
叙事任务填法：

☐ "陆屿帮苏晚整理书架，过程中展露他对她的在乎"
→ 写作指令，subagent 会在正文里"执行展露"

☑ "陆屿站在书架前，手指扫过书脊——然后准确抽出一本旧画册：
'这本你不是一直在找？'——写到这里。两个人同时伸手时指尖差一点碰到。"
→ 叙事画面，subagent 以相同语感继续
```

## 三、风格插件

主骨架不变，只替换模块 6（行文硬规则）中的约束集。

### 插件A：悬疑刑侦·冷峻白描

**适用：** 悬疑、刑侦、冷峻现实主义、社会派

```
行文硬规则（悬疑冷峻）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、环境外化情绪为主——但允许每 1000 字 1-2 句内心推理（"他脑子里过了一遍现场的画面"）。
3. 环境细节不超过 2 行/段，顺手带过，不渲染。
4. 对话每轮 3-5 句收住，不闲聊。
5. 允许偶尔的冗余细节——一个不相关的观察、一段沉默、一个没来由的回忆闪念。
6. 段落长短自由——推理沉浸处可展开，对话处果断收，不卡死行数。
7. 不写"他意识到""他发现"——但允许关键情绪节点直述（"他心里一沉""后背一阵发凉"），10% 以内。
8. 不写"这意味着""这表明"——结论让读者自己下。
9. 不用诗词腔、不堆四字成语、不用浮生/流年/寂寥等虚词。
10. 章末不留下章预告——留情绪缺口（一个画面/一个未说完的动作）。
11. 禁止升华总结——不在章末替读者总结"这一章说明了什么"。
```

### 插件B：都市言情·细腻日常

**适用：** 现代/古代言情、治愈系、日常向

```
行文硬规则（都市言情）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、细节外化情绪为主——但允许每 1000 字 1-2 句直接心理（"她心想""她在那一瞬间承认了"）。
3. 环境描写用于烘托氛围，不限行数，以沉浸感为准。
4. 对话自然放松，允许日常闲聊感，不必每句话都有信息量。
5. 允许角色走神、发呆、注意无关细节——飞过的鸽子、墙上的水渍、隔壁的收音机声。
6. 段落长短自由——氛围沉浸时可放开写，对话时果断收。
7. 不写"他/她意识到"——但允许关键情感节点直接写（"她感到委屈""他心里发酸"）。
8. 不写鸡汤升华，不替读者总结情感结论。
9. 章末留余韵——一个画面、一个念头、一个没说出口的动作。
10. 允许适当留白——不必填满每段的信息量。
```

### 插件C：玄幻仙侠·叙事恢弘

**适用：** 玄幻、仙侠、奇幻、史诗向

```
行文硬规则（玄幻仙侠）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、环境氛围外化情绪为主——但允许每 1000 字 1-2 句内心活动（"他默念心法""他脑中闪过师尊的话"）。
3. 场景描写可展开，不受行数限制，以画面完整为准。
4. 战斗场景注重节奏——蓄势→爆发→余韵，不拖不赶。
5. 修炼/境界提升通过感官变化体现（灵气流动/经脉反应/天地异象）。
6. 不写"他意识到"——但允许关键节点直接写情绪（"他心中骇然""一股寒意从脊背升起"）。
7. 对话可带古风感，但不过度文言——自然中带文气。
8. 允许环境氛围渲染烘托战斗/修炼的张力。
9. 章末可留悬念钩子（发现/遭遇/变故），不必强行收住。
10. 允许偶尔的闲笔——一段风景描写不为叙事服务，只为意境。
```

### 插件D：网文快节奏·重爽点

**适用：** 系统文、赘婿文、战神文、快节奏爽文

```
行文硬规则（网文快节奏）：
1. 固定单一 POV 视角，不跳转。
2. 开头 200 字内出冲突或信息差，不慢热铺垫。
3. 对话推进剧情为主，减少大段场景描写。
4. 每 500-800 字设一个小爽点或反转。
5. 情绪直给——允许适度直述情绪（震惊/狂喜/冰冷），不绕弯子。
6. 不写长句排比，不用文学性修辞。
7. 章末必须有钩子（发现/打脸预告/身份揭露），勾着读者看下一章。
8. 禁用文艺腔、禁用景物抒情、禁用哲学感慨。
9. 允许角色偶尔的碎碎念或内心 OS——增加阅读代入感。
```

## 四、附加规则（可选）

按题材/个人偏好勾选，不勾不影响正文生成。

```
☐ 禁止俗套比喻（如"时间仿佛凝固了""空气像凝固了一样"）
☐ 禁止副词滥修饰（"他冷冷地说""她轻轻地笑了"）
☐ 禁止章末升华（"这一夜，他明白了……"）
☐ 禁止对话标签堆砌（"他无奈地笑着摇了摇头说"）
☐ 不写角色外貌/穿着——除非与当前场景直接相关
```

## 五、使用流程

1. **定叙事任务** — 从章纲 key_points 提取本章要推进的剧情，填模块 4（叙事种子法）。
2. **摘前情** — 上章结尾关键画面/矛盾/停留场景，50 字填模块 2。
3. **写人物快照** — 本章在场角色当下状态，每人一句填模块 3。
4. **选风格插件** — 按题材选模块 6 对应的约束集，直接替换。
5. **填剩余字段** — 模块 1（基础定位）、模块 5（硬规则默认通用版）、模块 7（按需）。
6. **发送生成** — 完整提示词发给 subagent。

## 六、验收清单

生成前快速过：

- [ ] 叙事任务是画面（不是写作指令），每条以角色感官写
- [ ] 前情摘要 ≤ 50 字，只说上章结尾的关键状态
- [ ] 人物快照不复制百科人设——写的是"这一章 TA 的状态"
- [ ] POV 单一，全章不跳视角
- [ ] 叙事任务不含"本章将""接下来会""写到XXX时"等将来时
- [ ] 叙事任务不以分析语言写（"展露在乎""关系拉近"等意图解读）
- [ ] 风格插件选定了，约束集不是通用默认而是匹配题材
