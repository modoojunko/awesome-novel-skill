# 提示词组装指令

你的任务：将章纲 + writing-style + genre-profile 组装为 subagent 可执行的提示词。产出写入 `prompts/vol-{N}-ch-{M}-prompt.md`。

**前置条件：** 章纲已通过验收（作者确认 + 检查清单通过 + 快速嗅探 + AI味自检）。

**最高要求：** subagent 产出的正文符合作者的预期。每一步操作都以这个为判断标准——这个字段写进去能让正文更符合作者预期吗？这句话放在提示词里能帮助产出预期效果吗？与这个目标无关的都是多余。

---

### 输出格式（默认模板）

`prompts/vol-{N}-ch-{M}-prompt.md` 固定 6 部分，按以下顺序排列：

| # | 部分 | 写什么 | 来源 |
|---|------|--------|------|
| 1 | **核心基础要求** | 小说类型、核心基调、目标受众、**单章定位** | writing-style + genre-profile |
| 2 | **世界观与背景设定** | 时代背景、世界观规则、**本章涉及场景**、前章结尾状态 | world-setting + 本卷概览 |
| 3 | **人物设定** | 主角1（含**本章人物状态**）、主角2（含**本章人物状态**）、关键配角 | character 设定 |
| 4 | **剧情核心要求** | **本章目标**、全局叙事锚点、人物情境锚、**叙事种子**（逐条）、节奏、禁止内容 | 章纲 key_points（叙事种子化）+ memo |
| 5 | **文字与细节要求** | 视角要求、字数目标、**写作原则与禁忌**（四字段注入） | writing-style + genre-example |
| 6 | **补充说明** | 特殊要求、参考风格 | 按需 |

部分 1-3、5-6 从全局设定填充（一次写好每章复用），部分 4 从当前章纲生成（每章独立）。

---

## 一、方法论：叙事种子法

### 目的

作者对每章正文有预期——情节进展、人物状态、情绪氛围、风格调性。提示词的作用是把这些预期无损地传给 subagent。

方法：**不把预期翻译成写作指令，直接把预期写成叙事画面。** subagent 读到的不是"去做什么"，而是"继续写这段"。

### 从章纲到提示词的完整流程

章纲（`chapters/vol-{N}-ch-{M}.md`）包含这些字段，每个字段对应提示词的一个处理动作：

```
章纲字段                  →  处理动作                  →  提示词去向
─────────────────────────────────────────────────────────────
narrative_pov             →  锁定全章视角               →  约束部分 5
outline.summary           →  提取场景地点               →  部分 2 本章涉及场景
outline.key_points[]      →  逐条→叙事种子 (核心)       →  部分 4
memo.current_task         →  一句话本章目标             →  部分 4 开头
memo.reader_expectation   →  影响节奏和详略分配          →  部分 4
memo.payoff_plan          →  检查哪些钩子要收/要留      →  对应叙事种子中体现
memo.prohibitions         →  直接抄入                   →  部分 4 禁止内容
memo.required_changes     →  末条叙事种子自然呈现        →  部分 4 末节点
chapter_word_count        →  字数倒推算段数             →  决定分段策略
前章结尾状态              →  写入本章起始语境           →  部分 2
```

---

### 第一步：读取章纲，提取关键信息

拿到 `chapters/vol-{N}-ch-{M}.md`，逐个读取字段：

**1. narrative_pov** — 确定本章写谁的视角。第三人称有限/第一人称。锁定后所有叙事种子只能从这个角色的感官出发。

**2. outline.summary** — 本章概况。提取两个信息：
- 本章发生在哪些场景（写入部分 2 本章涉及场景）
- 本章在全卷中的定位（铺垫章/推进章/过渡章/高潮章/收尾章→写入部分 1 单章定位）

**3. outline.key_points[]** — 正文素材，核心处理对象。每条 key_point 对应一个叙事种子。后文详述。

**4. memo 区字段：**
- `current_task` → 写入部分 4 开头，一句话说明本章目标
- `reader_expectation` → 决定哪些节点写细、哪些点到为止
- `payoff_plan` → 扫描 must_resolve/partial_advance 的钩子 ID，在对应叙事种子中覆盖
- `prohibitions` → 直接写入部分 4 的"禁止出现的内容"
- `required_changes` → 确保末条叙事种子自然呈现这个变化（不是"他知道了"式的直述）

**5. chapter_word_count** → 字数倒推：总字数 ÷ 1500 向上取整 = 段数。段数决定分段策略。

---

### 第二步：填充全局设定字段（部分 1、2、3、5、6）

这一步是机械填充，从项目已有文件中直接读取：

| 提示词部分 | 填充内容 | 来源文件 |
|-----------|---------|---------|
| 1 核心基础要求 | 小说类型、核心基调、目标受众、**单章定位**（上一步提取） | writing-style + genre-profile |
| 2 世界观与背景设定 | 时代背景、世界观规则、**本章涉及场景**（上一步提取）、前章结尾状态 | world-setting + 本卷概览 |
| 3 人物设定 | 主角1（含**本章人物状态**）、主角2（含**本章人物状态**）、配角 | character 设定 |
| 5 文字与细节要求 | 视角要求 + 四字段 | writing-style + genre-example |
| 6 补充说明 | 特殊要求、参考风格 | 按需 |

**本章人物状态怎么确定：** 从 character 设定中的"成长弧"或当前卷进展推断。不是复制角色百科的"性格描述"，而是"这一章里这个角色处于什么情绪/心理位置"。

**四字段注入：** 从 `references/genre-example/` 的 corpus 文件读取：
- 叙事身份（role）→ 写入部分 1
- 核心原则 + 常见错误 → 合并为 8-15 条大白话，写入部分 5
- 描写技法 → 写入部分 5

---

### 第三步：字数倒推，确定分段

```
chapter_word_count ÷ 1500 = 段数（向上取整）
```

| 目标字数 | 段数 | 单段最大 | 场景分配示例 |
|---------|------|---------|------------|
| 1500-2000 | 1 段 | 2000 | 早餐铺一个场景写完 |
| 2000-3500 | 2 段 | 各 1750 | 早餐铺 + 书店 |
| 3500-5000 | 3 段 | 各 1700 | 早餐铺 + 书店 + 黄昏收尾 |
| 5000-7000 | 4-5 段 | 各 1400 | 更多场景或长场景细分 |

段数 ≥ 2 时，为每段分配场景和字数。段数 = 1 时跳过第四步。

---

### 第四步：写叙事锚点（仅段数 ≥ 2）

**全局叙事锚点（200-300 字）** — 用叙事语言写一段统摄全章语感的开场，不是规则说明。

素材来源：outline.summary 中的场景氛围 + 本章基调。把"这个信息"写成画面：

```
outline.summary → "陆屿回到临海市的第一天，和苏晚在梧桐巷重逢"

→ 转换为叙事锚点：
"梧桐巷的清晨，是从张奶奶的油条锅开始的。
苏晚站在书店二楼的窗前，看着楼下巷口的那个背影。
——全章主视角：苏晚，第三人称限知
——不看过去心理，不看未来走向，只看此时此刻她看到的、听到的、感受到的
——每段开头写画面，结尾落到她心里那个没说出口的念头"
```

**人物情境锚（100-150 字）** — 把角色设定的百科条目转化为情境化动作。subagent 拿到"身高 165cm、温柔内敛"会原文照搬。

素材来源：character 设定中的性格/习惯字段。提取 2-3 个可动作化的特点，改写为"当 XXX 时，TA 会做 YYY"：

```
character 设定 → "苏晚：温柔内敛，心思细腻，紧张时会摩挲书页"

→ 转换为情境锚：
"苏晚看陆屿时，目光先落在他左腕的手链上；紧张时会摩挲书页、把桌上的东西摆整齐。
她不是'温柔内敛'，她是话到嘴边又咽回去。"
```

---

### 第五步：章纲 key_points → 叙事种子（核心）

这是最核心的步骤。逐条处理 `outline.key_points[]`。

对**每一条** key_point，按以下三个问题转换：

| 问题 | 含义 | 示例 |
|------|------|------|
| 谁在看？ | 从 narrative_pov 角色出发 | "苏晚看到……" |
| 看到了什么？ | 写具体的画面/声音/气味/触感 | "陆屿坐在早餐铺最靠里的位置，面前两碗豆浆" |
| 正在发生什么？ | 写进行时的动作，不是将来时 | "她走到巷口"，不是"她去巷口" |

**三条禁止：**
- 不写作者意图（"展露在乎""关系拉近""表现了解"）
- 不写分析（"他感到""他意识到""这让读者感受到"）
- 不写将来时（"本章将""接下来会""写到XXX时"）

**产出格式：** 每条约 150-300 字，以叙事画面开头，以 `——写到这里` 引导 subagent 自然延续。

**完整转换示例：**

```
章纲 key_points 原文：
[1] 清晨，陆屿在早餐铺等苏晚。两人久别重逢，气氛既熟悉又有点微妙。
[2] 两人一起去拾光集书店，陆屿主动帮她整理书架，过程中流露出他对她的了解。
[3] 黄昏，陆屿送苏晚回家。一片梧桐叶落在她头发上，他帮她拿掉。
[4] 苏晚独自在房间里，回想今天的一切，意识到自己的心意没有变。

逐条转换过程：

[1] 谁在看？→ 苏晚。看到了什么？→ 陆屿坐在早餐铺里。
    正在发生？→ 她走到巷口，他已经在等了。
    → "苏晚走到巷口，看见陆屿坐在早餐铺最靠里的位置。
       面前两碗豆浆，一碗已经放了糖。张奶奶在厨房里喊：
       '小屿可算回来了，晚晚快来！'
       ——写到这里。下面写陆屿把苏晚碗里的香菜挑到自己碗里，像高中时一样。"

[2] 谁在看？→ 苏晚。看到了什么？→ 陆屿准确找到她找不到的书。
    正在发生？→ 他们一起整理书架。
    → "推开拾光集的门，旧书的味道扑面而来。
       陆屿站在书架前，手指扫过书脊——然后准确抽出一本旧画册：
       '这本你不是一直在找？'
       ——写到这里。两个人同时伸手去够同一本书时，指尖差一点碰到。"

[3] 谁在看？→ 苏晚。看到了什么？→ 陆屿帮她拿掉落叶。
    正在发生？→ 他送她回家。
    → "店里的光线从金黄变成了橘红。
       陆屿说'我送你'，不是一个问句。
       梧桐叶落在苏晚头发上，他抬手帮她拿掉——动作太自然了，
       好像他们从未分开过这三年。
       ——写到这里。苏晚走在他影子边上，步子慢了下来。"

[4] 谁在看？→ 苏晚。看到了什么？→ 窗外的背影。
    正在发生？→ 她独处回想。
    → "苏晚上楼后站在窗边，看着陆屿的背影走远。
       她坐下来，拿出画笔，开始画今天看到的第一个画面。
       ——结尾。不写'她明白了自己的心意'——读者已经看到了。"
```

**检查每条约 three 禁止：**
- [ ] 有没有作者意图语言？（"展露在乎""关系拉近"等）
- [ ] 有没有分析语言？（"他感到""他意识到""读者感受到"等）
- [ ] 有没有将来时？（"本章将""接下来会"等）

---

### 第六步：写分段叙事入口（仅段数 ≥ 2）

段数 ≥ 2 时，每段写一个独立入口（200-300 字），由画面重入 + 节奏指引组成。

**画面重入**：一句话锚定场景+情绪，让 subagent 重新进入状态。素材来源：该段对应的 key_point 的场景和氛围。

**节奏指引**：列出本段 4-5 个节奏点，赋字数。素材来源：key_point 的细节拆分 + memo.reader_expectation 决定的详略。

```
场景一｜早餐铺（~500字）

苏晚走到巷口时已经磨蹭了快十分钟。她在紧张什么，自己也不愿意想清楚。
然后她看见了陆屿——他坐在张奶奶早餐铺最靠里的位置，面前两碗豆浆。

——这一段写四个节奏：
1. 张奶奶的对话打破沉默（~150字）
2. 陆屿挑香菜——不用解释，这就是他的问候（~150字）
3. 张奶奶念叨两人小时候的事，制造轻松笑点（~100字）
4. 苏晚走出早餐铺时，他自然地走在她左边（~100字）

只用苏晚的感官。不说"她感到温暖"，写她把手指贴在豆浆碗壁上。
```

段间用 `---` 物理隔开。收尾段要有情绪落点——一个画面、一个念头、一个没说完的动作。

---

### 第七步：写入禁止内容 + 完整性检查

**禁止内容：** 从 memo.prohibitions 直接复制到部分 4 的"禁止出现的内容"。每条约 prohibitions 必须转化为可执行描述（不写"不要太拖沓"，写"环境描写单段不超过 2 行"）。

**完整性检查清单（逐一确认）：**
- [ ] Chapter_word_count → 字数倒推已做，段数确定
- [ ] outline.summary → 已提取本章涉及场景 + 单章定位
- [ ] memo.current_task → 已写入部分 4 本章目标
- [ ] memo.payoff_plan → must_resolve/partial_advance 已体现在对应叙事种子中
- [ ] memo.prohibitions → 全部写入部分 4 禁止内容
- [ ] memo.required_changes → 末条叙事种子已自然呈现
- [ ] 前章结尾状态 → 已写入部分 2
- [ ] 四字段 → 已从 genre-example 注入

## 二、验收

全部通过后才进入正文生成。

### 叙事种子验收
- [ ] 每条是叙事画面，不是写作指令——读起来像正文片段
- [ ] 不是"这段要写什么"的口吻——没有作者意图分析
- [ ] 不是将来时——没有"本章将""接下来会""写到XXX时"
- [ ] 每条以 POV 角色的感官写
- [ ] 不含元指令动词——没有"明确""需""要""注意""体现"
- [ ] 每条都经过了 三步检查：无作者意图 / 无分析语言 / 无将来时

### 分段结构验收（字数 ≥ 2000）
- [ ] 字数倒推已做：总字数 ÷ 1500 确定段数，每段 ≤ 1500 字
- [ ] 有全局叙事锚点（叙事语言开场，不是规则说明）
- [ ] 有人物情境锚（具体动作，不是百科条目）
- [ ] 每段有独立画面重入，段间用 `---` 物理隔开
- [ ] 段落间有场景/时间跳跃，不用过渡句硬连
- [ ] 收尾段有情绪落点——画面、念头、未说完的动作
- [ ] 各段字数相加 ≈ 目标总字数

### 视角与 POV 验收
- [ ] 全章单一视角，narrative_pov 已锁定
- [ ] 没有"他感到""他意识到""读者应该感受到"
- [ ] 没有"他注意到""他能听到""他发现"
- [ ] 当前 POV 角色不在场的场景未出现

### 四字段完整性验收
- [ ] 叙事身份已写入部分 1
- [ ] 核心原则 + 常见错误已合并写入部分 5（8-15 条，大白话，不分类不编号）
- [ ] 描写技法已写入部分 5
- [ ] 每条约束 subagent 能判断"某句话违规了吗"

### 章纲完整性验收
- [ ] outline.summary → 已提取场景和定位
- [ ] memo.current_task → 已写入部分 4
- [ ] memo.payoff_plan → 已体现在叙事种子中
- [ ] memo.prohibitions → 全部写入部分 4
- [ ] 前章结尾状态 → 已写入部分 2
- [ ] 字数目标已设定，各段分配合理

### 快速嗅探（60 秒）

1. 抽部分 4 一条叙事种子，遮住字段名——能说出"这一段正文会发生什么"？
2. 扫描全篇，找到"他感到""他意识到""这表明""这意味着"？
3. 对着 memo.prohibitions 逐条，在部分 4"禁止内容"中找到对应描述？

全部通过 → 进入正文生成。任一 fail → 先修再写。
