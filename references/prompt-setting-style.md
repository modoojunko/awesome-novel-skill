# Prompt 生产 SOP + Prompt 质检标准

**定位：** 将章纲 + writing-style + genre-profile 生产为 subagent 可执行的 prompt.md（`prompts/vol-{N}-ch-{M}-prompt.md`），并对产物做合格判定。只对 prompt 负责，不涉及正文写作。

**全局约束：**
- writing-style 四字段必须全部注入 prompt——role 定叙事身份，core_principles 定不可违背的写作信条，possible_mistakes 定 AI 易犯错误列表，depiction_techniques 定描写层次和手法。缺任何一个 subagent 都会在最关键的地方放飞。
- 正文字数目标、写作模型（writing_model）、情绪目标等执行参数从 `settings/writing-style.md` 和 `chapters/vol-{N}-ch-{M}.md` 读取，不在 prompts/ 中重复定义——一处修改全局生效。

## 一、通用底层骨架

所有题材通用的 6 个模块，格式固定，每章填一次。

```
## 1. 基础定位
- 题材：______
- 整体基调：______
- 本章定位：______（铺垫/推进/情绪升温）
- 目标字数：______（允许 ±200 字浮动）
- 本章爽点类型：______（打脸/成长/暧昧/解谜/绝境翻盘/权谋等）
- 本章情绪基调：______（压抑/紧绷/温情/悲壮/释然）

## 2. 前置承接
- 前情摘要（50-80 字，真人式回忆）：______
- 本章核心场景（3 个以内，自然递进）：______
- 本章核心情绪线（不刻意隐藏）：______

## 3. 人物即时状态
- 主角1（情绪起点→转折→落点，含小习惯）：______
- 主角2（情绪起点→转折→落点，含小犹豫）：______
- 关键配角（1-2 个，生活化）：______

## 4. 本章叙事任务

### 场景分解（按场景分段，字数随情绪浓淡自然分布）

场景1：______（______基调，______字区间）
- 前置状态（开场画面+人物位置+情绪底色）：______
- 核心事件（本场景必须发生的关键动作/对话，不可删减）：______
- 收尾落点（结束时状态/留白节点）：______
- 衔接→场景2：______

场景2：______（______基调，______字区间）
- 前置状态：______
- 核心事件（不可删减）：______
- 收尾落点：______
- 衔接→场景3：______

场景3：______（______基调，______字区间）
- 前置状态：______
- 核心事件（不可删减）：______
- 收尾落点：______
- 章尾余韵/钩子：______

*场景数 2-4 个均可，按叙事单元分段。每场景字数浮动 ±300 字。*

### 约束与边界（必填）
- 不可删减约束（关键情节/台词/动作，禁止省略、改写、弱化）：______
- 禁止自由发挥（不允许加支线人物/额外冲突/擅自改动机）：______

### 核心情绪落点
- 整体情绪落点：______
- 可保留的真人冗余（必填）：______

### 爽点与跌宕设计
- 本章核心爽点（类型+侧重）：______（暧昧爽/扮猪吃虎/反转震撼/情感释放/身份揭露/爽点组合等）
- 情绪跌宕曲线（抑→扬→收）：______
- 关键小反转（1-2个，自然而不突兀）：______
- 情绪卡点（有克制才有释放）：______
- 读者钩子（章尾预埋，不卖关子）：______

> **爽点守则：** ① 不牺牲人物真实性（爽点须从角色性格和情境自然生长） ② 反转需有铺垫（回头看恍然大悟，不是空降） ③ 卡点是克制的，不是编剧式强按头 ④ 钩子是余味不是卖关子（"欲知后事"是反面教材） ⑤ 情绪跌宕不能违背 §3（人物状态）的情绪弧光

## 5. 文字与细节要求
- 视角：单一 POV（允许 10% 心理直述）
- 描写要求：
  1. 环境描写：有烟火气，允许无用细节
  2. 动作描写：有笨拙感、犹豫感
  3. 情绪描写：允许每 1000 字 1-2 句直述
- 节奏：有张有弛，允许慢下来
- 禁止项（每条配正反例）：
  1. ❌ "他意识到/她感到"直述情绪 → ✅ 用动作、环境、对话外化
  2. ❌ AI套话、强行升华（"这一夜他明白了…"） → ✅ 停在一个画面/动作上
  3. ❌ OOC → ✅ 每句台词合角色性格
  4. ❌ 过度刻意的暧昧/冲突设计 → ✅ 所有戏剧性从日常细节自然生长

## 6. 真人感补充
- 无用但有温度的细节：______
- 情绪波动：______
- 语言风格：有停顿、有重复、有词穷
```

## 二、核心填充方法

每个模块有独立的填充方法，定义输入源、填充原则、产出要求。

**全局原则：** 全部字段填完后全局通读一次，掩住字段名看每个种子读起来是叙事画面（✅）还是写作指令（❌）。

### 模块1：基础定位

从全局设定 + 章纲直接提取，每字段一行：

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 题材 | genre-profile | 照抄，每章不变 |
| 整体基调 | writing-style | 概括句（1 句），不抄细分列表 |
| 本章定位 | 卷纲→章纲 | 三选一：铺垫/推进/情绪升温。取本章在全卷中的功能角色 |
| 目标字数 | 章纲 字数 | 直接取 |
| 本章爽点类型 | 章纲 爽点设计 | 取核心爽点类型，1-2 个 |
| 本章情绪基调 | 章纲 情绪曲线 | 取本章整体情绪调性，一个词 |

### 模块2：前置承接

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 前情摘要 | 上章章尾 + 章纲前情承接 | 50-80 字画面回忆——写"上章结束时发生了什么/角色停在什么状态"。不是剧情概括 |
| 本章核心场景 | 章纲 场景分解 | 直接取场景名列表（2-4 个），→ 连接，自然递进 |
| 本章核心情绪线 | 章纲 情绪曲线 | 一句话概括："从 XX 到 XX，最终 XX"。不加分析 |

### 模块3：人物状态

每角色一句，写"这章开始时 TA 是什么状态→经历什么转折→结束时什么状态"。不是角色百科。

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 主角1 | 章纲 人物状态 + 角色设定 | 情绪起点→转折→落点。含一个微动作习惯 |
| 主角2 | 章纲 人物状态 + 角色设定 | 同上。含一个微犹豫/词穷时的习惯动作 |
| 关键配角 | 章纲 | 只在本章出场时填。不出场不填 |

### 模块4：本章叙事任务

#### 场景分解

场景是 prompt 的基本叙事单元。场景切分依据情绪线拐点——每个基调转折都是一个边界：

> 例："客气生疏 → 被念叨打破 → 自然互动升温 → 暧昧收尾" → 3 个场景（抑/扬/收）

边界信号（满足任意一个即切分）：

| 信号 | 判定 | 例 |
|------|------|----|
| 换地点 | 角色走到另一个地方 | 早餐铺 → 书店 |
| 跳时间 | 有不可见的时间流逝 | 上午 → 黄昏 |
| 情绪转折 | 基调变了 | 尴尬 → 自然 |

字数估算参照：

| 情绪浓度 | 典型字数 |
|---------|---------|
| 浓（高潮/冲突/深度对话） | 1500-2500 |
| 中（日常互动/推进剧情） | 800-1500 |
| 淡（过渡/铺垫/气氛渲染） | 500-1200 |

典型场景分布（以 4500 字为例，只参考不照搬）：

| 场景数 | 字数分布示例 | 适用 |
|--------|-------------|------|
| 2 个 | 2000 + 2500 | 双场景章（前因→后果/铺垫→爆发） |
| 3 个 | 800 + 2000 + 1700 | 最常见，抑→扬→收各一场 |
| 4 个 | 600 + 1200 + 1500 + 1200 | 需要穿插过渡/伏笔场景 |

#### 种子三字段结构

每个场景种子固定三个字段。先定场景结构再逐场景填种子。

| 字段 | 写什么 | 控制力 |
|------|--------|--------|
| 前置状态 | 角色位置、正在做什么、情绪底色——用环境/动作暗示 | 定画面基调 |
| 核心事件 | 本场景**必须发生的**关键动作/对话——不可删减、不可替换 | 保剧情不跑偏 |
| 收尾落点 | 场景结束时必须停在的状态或留白节点 | 控节奏不拖尾 |

```
场景1｜梧桐巷早餐铺（抑，800-1000 字）
前置状态：苏晚提前到了，坐在靠窗老位置，手指来回摩挲菜单边缘。陆屿推门进来，在她对面坐下。
核心事件：两人客气打招呼→张奶奶端豆浆过来念叨糗事→苏晚低头笑了，陆屿松了表情。
收尾落点：尴尬消散，气氛自然松懈。
```

#### 约束与边界

**不可删减约束：** 挑 2-4 个最关键的情节节点/台词/动作细节，subagent 以这些为铁轨，其余自由发挥。

> 例：张奶奶必须念叨小时候糗事；旧画册必须出现且两人指尖差一点碰到；章尾梧桐叶画面不可省略

**禁止自由发挥边界：** 写"不能做什么"，不写"应该做什么"。

> 例：禁止加额外人物（不出场同学/路人甲乙）；禁止陆屿本场景告白或挑明心意

> **硬性规定：** 核心事件未完成（遗漏、弱化不可删减节点），视为 prompt 不合格，必须重填，不可跳过。

#### 禁止配替代

每条禁止配一条"可以怎么写"。统一 ❌/✅ 格式：

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| 直述情绪结论 | 用动作/神态/环境外化 |
| 章末升华总结 | 停在画面/动作上 |
| 副词修饰对话 | 去掉副词，用动作暗示 |

#### 爽点与跌宕设计题材参考

| 题材 | 爽点侧重 | 情绪曲线特点 | 反转/钩子写法 |
|------|---------|-------------|--------------|
| 玄幻仙侠 | 突破晋级、打脸反杀、秘境夺宝 | 压抑→爆发→余韵 | 伪善暴露、隐藏机缘 |
| 都市言情 | 暧昧拉扯、误会解开、心意流露 | 压抑→上扬→回落未挑明 | 旧物回忆、欲言又止 |
| 古风权谋 | 博弈翻盘、沉冤得雪、智斗碾压 | 压抑→上扬→回落 | 盟友反水、身份揭露 |
| 悬疑刑侦 | 解谜突破、真凶浮现、证据反转 | 全程偏压抑 | 真凶另有其人、线索反转 |
| 科幻星际 | 危机解除、技术突破、文明发现 | 压抑→上扬→新威胁 | 内部背叛、外星信号 |

#### 分段叙事入口

每段一条，放在该段种子前面。固定三要素：

| 要素 | 写什么 | 不要写 |
|------|--------|--------|
| 画面重入（1 句） | 承接上一场结尾 | "接下来写…" |
| 情绪导向（1 句） | 本场情绪基调 | 抽象描述 |
| 节奏点（3-5 个） | 画面 + 字数 | 精确到每句 |

> **格式说明：** 提示词中的 `---` 仅为分段标记（区分不同叙事段落的内部格式），不写入正文。正文纯小说格式，不含任何 Markdown 标记。

### 模块5：文字与细节要求

根据题材从 `references/genre-example/` 中选取对应的约束集替换模板通用版。genre-example 中的 `prompt_segment` 段直接对位替换模块 5 的全部内容（视角 + 描写要求 + 节奏 + 禁止项）。如果无题材特定约束，使用模板中的通用版。

### 模块6：真人感补充

| 字段 | 输入源 | 填充原则 |
|------|--------|---------|
| 无用细节 | 章纲 memo + 作者脑补 | 1-2 个与主线无关但有温度的细节画面。不硬凑 |
| 情绪波动 | 章纲 人物情绪弧 | 角色在本章的情绪变化路线（从→到→最终），50 字以内 |
| 语言风格 | writing-style | 照搬模板提示（"有停顿、有重复、有词穷"），每章不变 |

## 三、验收

### 快速嗅探

读模块 4（场景分解）场景 1 的**核心事件**——遮住字段名，读起来是"发生了什么事"（✅）还是"去写什么"（❌）？前者合格，后者是写作指令，重填种子。

### 六条硬校验

通过这 6 条 = prompt 合格。每条可人工打勾 ✅/❌：

| # | 硬校验项 | 通过标准 | 检测方式 |
|---|---------|---------|---------|
| 1 | 结构完整 | 6 个模块齐全，无遗漏字段 | 模板对照 |
| 2 | 三字段完整 | 每个场景有前置状态/核心事件/收尾落点，无缺位 | 逐场景检查 |
| 3 | 核心事件可执行 | 核心事件写的是叙事画面（"角色做了什么/说了什么"），非写作指令（"去写…"） | 嗅探法 |
| 4 | 约束双侧闭合 | 不可删减约束有 2-4 个具体节点；禁止自由发挥写"不做什么" | 字段检查 |
| 5 | 推演忠实 | 场景结构、情绪线、人物状态均与章纲一致，无偏离 | 章纲对照 |
| 6 | 题材匹配 | 模块 5 的约束集与题材类型对应正确 | 目录对照 |

**判定：** 6 项全部 ✅ → prompt 合格。1-2 项 ❌ → 针对性修改。3 项及以上 ❌ → 整份重填。

### 逐模块检查

填充阶段用，写正文前拦截：

| 模块 | 合格标准 | 不合格信号 |
|------|---------|-----------|
| 1 基础定位 | 一读就知道"这章讲什么、什么调性" | 定位和基调对不上，或模板级空话 |
| 2 前置承接 | 前情像正常回忆，不是 bullet list | 50 字写成了剧情梗概，没有画面感 |
| 3 人物状态 | 有情绪起点→转折→落点，写的是"这章 TA 的状态" | 复制了角色百科的性格描述，或只有状态没有变化 |
| 4 场景分解 | 每场景有独立情绪基调，种子是叙事画面非指令，衔接自然不硬跳 | 场景之间没有情绪过渡，或种子是"这场景写…"写作指令 |
| 4a 爽点跌宕 | 有曲线（抑→扬→收）、有反转/卡点、有钩子 | 爽点类型不匹配题材，或曲线平铺直叙没有起伏 |
| 4b 约束与边界 | 不可删减约束标明 2-4 个关键节点；禁止自由发挥写"不做什么" | 约束空泛（"禁止跑偏"之类），或边界写成了"应该做什么" |
| 5 文字要求 | 视角锁定，与题材类型匹配 | 禁止项是空话（"禁止注水"无具体定义）|
| 6 真人感补充 | 至少有 1 个"无用细节"和"情绪波动" | 空着或只写了"无" |

全部通过 → 进入正文生成。任一不合格 → 先修再写。
