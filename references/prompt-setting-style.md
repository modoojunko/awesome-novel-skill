# 提示词组装及验收指南

将章纲 + writing-style + genre-profile 组装为 subagent 可执行的提示词。产出写入 `prompts/vol-{N}-ch-{M}-prompt.md`。

---

## 一、通用底层骨架

所有题材通用的 6 个模块，格式固定，每章填一次。

```
## 1. 基础定位
- 题材：______
- 整体基调：______
- 本章定位：______（铺垫/推进/情绪升温）
- 目标字数：______（允许 ±200 字浮动）
- 本章爽点类型：______（打脸/成长/暧昧/解谜/绝境翻盘/权谋等）
- 本章情绪基调：______（压抑/紧绷/温情/悲壮/释然）

## 2. 前置承接
- 前情摘要（50-80 字，真人式回忆）：______
- 本章核心场景（3 个以内，自然递进）：______
- 本章核心情绪线（不刻意隐藏）：______

## 3. 人物即时状态
- 主角1（情绪起点→转折→落点，含小习惯）：______
- 主角2（情绪起点→转折→落点，含小犹豫）：______
- 关键配角（1-2 个，生活化）：______

## 4. 本章叙事任务

### 场景分解（按场景分段，字数随情绪浓淡自然分布）

场景1：______（______基调，______字区间）
- 叙事种子：______
- 衔接→场景2：______

场景2：______（______基调，______字区间）
- 叙事种子：______
- 衔接→场景3：______

场景3：______（______基调，______字区间）
- 叙事种子：______
- 章尾余韵/钩子：______

*场景数 2-4 个均可，按情绪推进自然分段，不机械均分。每场景字数浮动 ±300 字。*

### 核心情绪落点
- 整体情绪落点：______
- 可保留的真人冗余（必填）：______

### 爽点与跌宕设计
- 本章核心爽点（类型+侧重）：______（暧昧爽/扮猪吃虎/反转震撼/情感释放/身份揭露/爽点组合等）
- 情绪跌宕曲线（抑→扬→收）：______
- 关键小反转（1-2个，自然而不突兀）：______
- 情绪卡点（有克制才有释放）：______
- 读者钩子（章尾预埋，不卖关子）：______

> **爽点守则：** ① 不牺牲人物真实性（爽点须从角色性格和情境自然生长） ② 反转需有铺垫（回头看恍然大悟，不是空降） ③ 卡点是克制的，不是编剧式强按头 ④ 钩子是余味不是卖关子（"欲知后事"是反面教材） ⑤ 情绪跌宕不能违背 §3（人物状态）的情绪弧光

## 5. 文字与细节要求
- 视角：单一 POV（允许 10% 心理直述）
- 描写要求：
  1. 环境描写：有烟火气，允许无用细节
  2. 动作描写：有笨拙感、犹豫感
  3. 情绪描写：允许每 1000 字 1-2 句直述
- 节奏：有张有弛，允许慢下来
- 禁止项：
  1. 禁止 AI 口水话、套话、强行升华
  2. 禁止 OOC
  3. 禁止过度刻意的暧昧设计

## 6. 真人感补充
- 无用但有温度的细节：______
- 情绪波动：______
- 语言风格：有停顿、有重复、有词穷
```

## 二、核心方法：叙事种子法

模块 4（本章叙事任务）的填充方法。

### 前提：场景先行

叙事种子写得好不好，前提是场景分得好不好。不要一上来就写种子——先定本章几个场景、每个场景的情绪基调、大致字数范围。

**场景分解三步法：**

**第一步：读情绪线，找拐点。** 本章情绪线（模块 2 已写）的每一次转折就是一个场景边界。

> 例："客气生疏 → 被念叨打破 → 自然互动升温 → 暧昧收尾"
> 拐点在"被念叨打破"和"暧昧收尾"两处 → 3 个场景（抑/扬/收）

**第二步：问三个边界信号。** 在两个情绪拐点之间，问三个问题——任意一个答"是"，就切分：

| 信号 | 问法 | 例 |
|------|------|----|
| 换地点 | 角色从这里走到另一个地方了？ | 早餐铺 → 书店 |
| 跳时间 | 中间有不可见的时间流逝？ | 上午 → 黄昏 |
| 情绪转折 | 基调变了（紧张→放松/压抑→爆发）？ | 尴尬 → 自然 |

**第三步：估字数区间。** 每场问"这场情绪够写多少字"，不是反过来的：

| 情绪浓度 | 典型字数 | 说明 |
|---------|---------|------|
| 浓（高潮/冲突/深度对话） | 1500-2500 | 展开写，不赶 |
| 中（日常互动/推进剧情） | 800-1500 | 正常节奏 |
| 淡（过渡/铺垫/气氛渲染） | 500-1200 | 收住不拖 |

写完每个场景后，用一句话验证："在 **（地点）**，TA **（核心动作）**，基调 **（情绪）**"。说不清 = 边界没切对。

**典型场景分布（以 4500 字为例，只参考不照搬）：**

| 场景数 | 字数分布示例 | 适用 |
|--------|-------------|------|
| 2 个 | 2000 + 2500 | 双场景章（前因→后果/铺垫→爆发） |
| 3 个 | 800 + 2000 + 1700 | 最常见，抑→扬→收各一场 |
| 4 个 | 600 + 1200 + 1500 + 1200 | 需要穿插过渡/伏笔场景 |

### 衔接：种子之间的呼吸

种子之间不能硬跳。场景切换处必须有**衔接画面**——一个让读者"跟着走"的过渡：

```
场景1 末尾：两人吃完却没人先开口说走。
衔接：张奶奶忙去了，苏晚低头拿包，陆屿瞥见她钥匙串上的书店钥匙。
场景2 开头："你开店了？"他问。苏晚抬头，愣了一下。
```

衔接是"情绪呼吸"——给读者几百毫秒的缓冲，然后自然过渡到下一场。不是过渡段，是一两个画面或一句对话。没有衔接 = 碎片拼接。

### 原理

不把剧情写成写作指令（"这段写男主对女主的了解"），写成 POV 角色的叙事画面（"陆屿站在书架前，手指扫过书脊，准确抽出一本旧画册"）。subagent 读到的是"继续写这段"，不是"去做什么"。

**注意：** 叙事种子是"路标"不是"铁轨"——写够方向即可，不必每句都精准到分毫不差。留一些模糊空间让 subagent 自己发挥，正文反而更自然。种子太精确 → 执行痕迹重 → 像剧本不像小说。

### 三条转化规则

| 规则 | 含义 |
|------|------|
| 问"角色看到/听到/做了什么？" | 不写作者意图（展露在乎/关系拉近），只写具体画面 |
| 写"正在发生的事" | 不写将来时（本章将/接下来会），写进行时（她走到巷口） |
| 用角色感官写 | 不写分析（他意识到/读者感受到），写画面声音气味 |

### 种子情绪倾向

叙事种子不只是画面——它还告诉 AI"这场是什么情绪"。同一个动作可以有不同的情绪倾向：

```
☐ "陆屿帮苏晚整理书架，两人在书架间走动"
→ 纯动作，AI 不知道是紧张还是暧昧还是例行公事

☑ "陆屿站在书架前，手指扫过书脊——然后准确抽出一本旧画册。
两人同时伸手时指尖差一点碰到。"
→ 动作里带着小心试探和暧昧
```

**写法：种子里的动作选词就暗示情绪**——"抽"比"拿"有力、"差一点碰到"比"没碰到"更有暧昧感。不以形容词定调（"紧张地"），靠画面选词定调。

### 示例

```
叙事任务填法：

☐ "陆屿帮苏晚整理书架，过程中展露他对她的在乎"
→ 写作指令，subagent 会在正文里"执行展露"

☑ "陆屿站在书架前，手指扫过书脊——然后准确抽出一本旧画册：
'这本你不是一直在找？'——写到这里。两个人同时伸手时指尖差一点碰到。"
→ 叙事画面，subagent 以相同语感继续
```

### 禁止配替代

每条禁止都要配一条"可以怎么写"，不让 AI 束手束脚：

| 不要 | 替代写法 |
|------|---------|
| "他意识到她喜欢这本书" | "他注意到她翻到某一页时停了一下" |
| "他感到愤怒" | "他握紧杯子，指节发白" |
| "这一夜，他明白了……"（升华总结） | "他关灯，在黑暗里睁着眼" |
| "气氛变得暧昧起来" | "两人同时伸手，指尖差一点碰到" |

**原则：堵一个写法，就要开一个窗口。** 禁止项必须紧跟在它后面的"可替代写法"。

## 三、风格插件

主骨架不变，风格插件替换模块 5（文字与细节要求）中的描写要求和禁止项。

### 插件A：悬疑刑侦·冷峻白描

**适用：** 悬疑、刑侦、冷峻现实主义、社会派

```
文字与细节要求（悬疑冷峻）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、环境外化情绪为主——但允许每 1000 字 1-2 句内心推理（"他脑子里过了一遍现场的画面"）。
3. 环境细节服务氛围，不硬限制行数。
4. 对话每轮 3-5 句收住，不闲聊。
5. 允许偶尔的冗余细节——一个不相关的观察、一段沉默、一个没来由的回忆闪念。
6. 段落长短自由——推理沉浸处可展开，对话处果断收，不卡死行数。
7. 不写"他意识到""他发现"——但允许关键情绪节点直述（"他心里一沉""后背一阵发凉"），10% 以内。
8. 不写"这意味着""这表明"——结论让读者自己下。
9. 不用诗词腔、不堆四字成语、不用浮生/流年/寂寥等虚词。
10. 章末不留下章预告——留情绪缺口（一个画面/一个未说完的动作）。
11. 禁止升华总结——不在章末替读者总结"这一章说明了什么"。
```

### 插件B：都市言情·细腻日常

**适用：** 现代/古代言情、治愈系、日常向

```
文字与细节要求（都市言情）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、细节外化情绪为主——但允许每 1000 字 1-2 句直接心理（"她心想""她在那一瞬间承认了"）。
3. 环境描写用于烘托氛围，不限行数，以沉浸感为准。
4. 对话自然放松，允许日常闲聊感，不必每句话都有信息量。
5. 允许角色走神、发呆、注意无关细节——飞过的鸽子、墙上的水渍、隔壁的收音机声。
6. 段落长短自由——氛围沉浸时可放开写，对话时果断收。
7. 不写"他/她意识到"——但允许关键情感节点直接写（"她感到委屈""他心里发酸"）。
8. 不写鸡汤升华，不替读者总结情感结论。
9. 章末留余韵——一个画面、一个念头、一个没说出口的动作。
10. 允许适当留白——不必填满每段的信息量。
```

### 插件C：玄幻仙侠·叙事恢弘

**适用：** 玄幻、仙侠、奇幻、史诗向

```
文字与细节要求（玄幻仙侠）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、环境氛围外化情绪为主——但允许每 1000 字 1-2 句内心活动（"他默念心法""他脑中闪过师尊的话"）。
3. 场景描写可展开，不受行数限制，以画面完整为准。
4. 战斗场景注重节奏——蓄势→爆发→余韵，不拖不赶。
5. 修炼/境界提升通过感官变化体现（灵气流动/经脉反应/天地异象）。
6. 不写"他意识到"——但允许关键节点直接写情绪（"他心中骇然""一股寒意从脊背升起"）。
7. 对话可带古风感，但不过度文言——自然中带文气。
8. 允许环境氛围渲染烘托战斗/修炼的张力。
9. 章末可留悬念钩子（发现/遭遇/变故），不必强行收住。
10. 允许偶尔的闲笔——一段风景描写不为叙事服务，只为意境。
```

### 插件D：网文快节奏·重爽点

**适用：** 系统文、赘婿文、战神文、快节奏爽文

```
文字与细节要求（网文快节奏）：
1. 固定单一 POV 视角，不跳转。
2. 开头 200 字内出冲突或信息差，不慢热铺垫。
3. 对话推进剧情为主，减少大段场景描写。
4. 每 500-800 字设一个小爽点或反转。
5. 情绪直给——允许适度直述情绪（震惊/狂喜/冰冷），不绕弯子。
6. 不写长句排比，不用文学性修辞。
7. 章末必须有钩子（发现/打脸预告/身份揭露），勾着读者看下一章。
8. 禁用文艺腔、禁用景物抒情、禁用哲学感慨。
9. 允许角色偶尔的碎碎念或内心 OS——增加阅读代入感。
```

## 四、附加规则（可选，填模块 6 或跳过）

按题材/个人偏好选加，不选不影响正文生成。直接填入模块 6 的"无用但有温度的细节"。

```
☐ 禁止俗套比喻（如"时间仿佛凝固了""空气像凝固了一样"）
☐ 禁止副词滥修饰（"他冷冷地说""她轻轻地笑了"）
☐ 禁止章末升华（"这一夜，他明白了……"）
☐ 禁止对话标签堆砌（"他无奈地笑着摇了摇头说"）
☐ 不写角色外貌/穿着——除非与当前场景直接相关
```

### 爽点守则（选加，控制爽点不生硬）

上述 5 项选填至模块 6"无用但有温度的细节"位。如果使用模块 4 的爽点与跌宕设计，建议至少选加 2-3 项：

```
☐ 不牺牲人物真实性——爽点从角色性格和情境自然生长
☐ 反转需有铺垫——回头看恍然大悟，不是空降
☐ 卡点克制，不编剧式强按头
☐ 章尾钩子是余味，不是"欲知后事如何"
☐ 情绪跌宕不违背人物即时状态（§3）的情绪弧光
```

## 五、使用流程

1. **定叙事任务** — 先按场景分解（情绪线→场景数→每场景基调+字数），再逐场景填叙事种子和场景衔接。填模块 4 场景分解子模块。
2. **设爽点跌宕** — 定爽点类型+情绪曲线+反转卡点+钩子，填模块 4 爽点与跌宕设计子模块。
3. **摘前情** — 上章结尾关键画面/矛盾/停留场景，50-80 字填模块 2。
4. **写人物快照** — 本章在场角色当下状态，每人一句填模块 3。
5. **选风格插件** — 按题材选对应的约束集，替换模块 5 的描写要求和禁止项。
6. **填剩余字段** — 模块 1（基础定位）、模块 6（真人感补充）。
7. **发送生成** — 完整提示词发给 subagent。

## 六、验收

### 30 秒快速嗅探

读模块 4（场景分解）场景1的叙事种子，遮住字段名——读起来像正文片段（✅）还是像写作指令（❌）？如果是后者，整章提示词要重新按场景分解再填种子。

### 逐模块检查

| 模块 | 合格标准 | 不合格信号 |
|------|---------|-----------|
| 1 基础定位 | 一读就知道"这章讲什么、什么调性" | 定位和基调对不上，或模板级空话 |
| 2 前置承接 | 前情像正常回忆，不是 bullet list | 50 字写成了剧情梗概，没有画面感 |
| 3 人物状态 | 有情绪起点→转折→落点，写的是"这章 TA 的状态" | 复制了角色百科的性格描述，或只有状态没有变化 |
| 4 场景分解 | 每场景有独立情绪基调，种子是叙事画面非指令，衔接自然不硬跳 | 场景之间没有情绪过渡，或种子是"这场景写…"写作指令 |
| 4a 爽点跌宕 | 有曲线（抑→扬→收）、有反转/卡点、有钩子 | 爽点类型不匹配题材，或曲线平铺直叙没有起伏 |
| 5 文字要求 | 视角锁定，描写要求具体可感 | 禁止项是空话（"禁止注水"无具体定义）|
| 6 真人感补充 | 至少有 1 个"无用细节"和"情绪波动" | 空着或只写了"无" |

全部通过 → 进入正文生成。任一不合格 → 先修再写。
