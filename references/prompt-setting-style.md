# Prompt 生产 SOP + Prompt 质检标准

**定位：** AI 用的标准作业流程——将章纲 + writing-style + genre-profile 生产为 subagent 可执行的 prompt.md（`prompts/vol-{N}-ch-{M}-prompt.md`），并做产出质量验收。

---

## 一、通用底层骨架

所有题材通用的 6 个模块，格式固定，每章填一次。

```
## 1. 基础定位
- 题材：______
- 整体基调：______
- 本章定位：______（铺垫/推进/情绪升温）
- 目标字数：______（允许 ±200 字浮动）
- 本章爽点类型：______（打脸/成长/暧昧/解谜/绝境翻盘/权谋等）
- 本章情绪基调：______（压抑/紧绷/温情/悲壮/释然）

## 2. 前置承接
- 前情摘要（50-80 字，真人式回忆）：______
- 本章核心场景（3 个以内，自然递进）：______
- 本章核心情绪线（不刻意隐藏）：______

## 3. 人物即时状态
- 主角1（情绪起点→转折→落点，含小习惯）：______
- 主角2（情绪起点→转折→落点，含小犹豫）：______
- 关键配角（1-2 个，生活化）：______

## 4. 本章叙事任务

### 场景分解（按场景分段，字数随情绪浓淡自然分布）

场景1：______（______基调，______字区间）
- 前置状态（开场画面+人物位置+情绪底色）：______
- 核心事件（本场景必须发生的关键动作/对话，不可删减）：______
- 收尾落点（结束时状态/留白节点）：______
- 衔接→场景2：______

场景2：______（______基调，______字区间）
- 前置状态：______
- 核心事件（不可删减）：______
- 收尾落点：______
- 衔接→场景3：______

场景3：______（______基调，______字区间）
- 前置状态：______
- 核心事件（不可删减）：______
- 收尾落点：______
- 章尾余韵/钩子：______

*场景数 2-4 个均可，按叙事单元完整性分段，不机械均分。每场景字数浮动 ±300 字。*

### 约束与边界（必填）
- 不可删减约束（关键情节/台词/动作，禁止省略、改写、弱化）：______
- 禁止自由发挥（不允许加支线人物/额外冲突/擅自改动机）：______

### 核心情绪落点
- 整体情绪落点：______
- 可保留的真人冗余（必填）：______

### 爽点与跌宕设计
- 本章核心爽点（类型+侧重）：______（暧昧爽/扮猪吃虎/反转震撼/情感释放/身份揭露/爽点组合等）
- 情绪跌宕曲线（抑→扬→收）：______
- 关键小反转（1-2个，自然而不突兀）：______
- 情绪卡点（有克制才有释放）：______
- 读者钩子（章尾预埋，不卖关子）：______

> **爽点守则：** ① 不牺牲人物真实性（爽点须从角色性格和情境自然生长） ② 反转需有铺垫（回头看恍然大悟，不是空降） ③ 卡点是克制的，不是编剧式强按头 ④ 钩子是余味不是卖关子（"欲知后事"是反面教材） ⑤ 情绪跌宕不能违背 §3（人物状态）的情绪弧光

## 5. 文字与细节要求
- 视角：单一 POV（允许 10% 心理直述）
- 描写要求：
  1. 环境描写：有烟火气，允许无用细节
  2. 动作描写：有笨拙感、犹豫感
  3. 情绪描写：允许每 1000 字 1-2 句直述
- 节奏：有张有弛，允许慢下来
- 禁止项（每条配正反例）：
  1. ❌ "他意识到/她感到"直述情绪 → ✅ 用动作、环境、对话外化
  2. ❌ AI套话、强行升华（"这一夜他明白了…"） → ✅ 停在一个画面/动作上
  3. ❌ OOC → ✅ 每句台词合角色性格
  4. ❌ 过度刻意的暧昧/冲突设计 → ✅ 所有戏剧性从日常细节自然生长

## 6. 真人感补充
- 无用但有温度的细节：______
- 情绪波动：______
- 语言风格：有停顿、有重复、有词穷
```

## 二、核心方法：叙事种子法

模块 4（本章叙事任务）的填充方法。

### 前提：场景先行

先定本章几个场景、每个场景的情绪基调、大致字数范围，再填种子。

**场景分解：**

从情绪线拐点切分场景，每个转折就是一个边界：

> 例："客气生疏 → 被念叨打破 → 自然互动升温 → 暧昧收尾"
> 拐点两处 → 3 个场景（抑/扬/收）

边界信号（满足任意一个即切分）：

| 信号 | 判定 | 例 |
|------|------|----|
| 换地点 | 角色走到另一个地方 | 早餐铺 → 书店 |
| 跳时间 | 有不可见的时间流逝 | 上午 → 黄昏 |
| 情绪转折 | 基调变了 | 尴尬 → 自然 |

按情绪浓度估算字数：

| 情绪浓度 | 典型字数 |
|---------|---------|
| 浓（高潮/冲突/深度对话） | 1500-2500 |
| 中（日常互动/推进剧情） | 800-1500 |
| 淡（过渡/铺垫/气氛渲染） | 500-1200 |

**典型场景分布（以 4500 字为例，只参考不照搬）：**

| 场景数 | 字数分布示例 | 适用 |
|--------|-------------|------|
| 2 个 | 2000 + 2500 | 双场景章（前因→后果/铺垫→爆发） |
| 3 个 | 800 + 2000 + 1700 | 最常见，抑→扬→收各一场 |
| 4 个 | 600 + 1200 + 1500 + 1200 | 需要穿插过渡/伏笔场景 |

**叙事单元完整性分段（替代字数机械分段）：**

字数不决定分段——叙事完整性决定。核心原则：

1. **镜头决定段落**：一个完整镜头场景为一个自然段落。场景切换、时间跳转、视角切换才允许换行分段。
2. **禁止中间切段**：同一场景内动作连贯、对话连贯——不为凑字数强行切断。
3. **字数后置参考**：字数只做估算不做前置拆分依据——先按镜头写，写完了再调总量。
4. **单段硬上限**：日常段 ≤1500 字，高潮/冲突段 ≤2000 字。超上限则在内部分镜头，不强行切断剧情线。

### 衔接

场景切换处必须写衔接画面——承接上一场末尾，引入下一场开头：

```
场景1 末尾：两人吃完却没人先开口说走。
衔接：张奶奶忙去了，苏晚低头拿包，陆屿瞥见她钥匙串上的书店钥匙。
场景2 开头："你开店了？"他问。苏晚抬头，愣了一下。
```

衔接是一两个画面或一句对话，不是过渡段。

### 种子三字段结构

单一叙事种子容易跑偏——subagent 可能忽略关键节点或自由发挥过头。每个场景种子固定三个字段，从"在哪干啥"到"必须发生什么"再到"停在哪儿"：

| 字段 | 写什么 | 控制力 |
|------|--------|--------|
| 前置状态 | 角色位置、正在做什么、情绪底色——用环境/动作暗示，不直写心理 | 定画面基调 |
| 核心事件 | 本场景**必须发生的**关键动作/对话——不可删减、不可替换 | 保剧情不跑偏 |
| 收尾落点 | 场景结束时必须停在的状态或留白节点 | 控节奏不拖尾 |

填充示例：

```
场景1｜梧桐巷早餐铺（抑，800-1000 字）
前置状态：苏晚提前到了，坐在靠窗老位置，手指来回摩挲菜单边缘。陆屿推门进来，在她对面坐下。
核心事件：两人客气打招呼→张奶奶端豆浆过来念叨糗事→苏晚低头笑了，陆屿松了表情。
收尾落点：尴尬消散，气氛自然松懈。
```

#### 最小不可删减约束

列出本章**哪些节点绝对不能省略、改写或弱化**——subagent 以这些为"铁轨"，其余可自由发挥。

写法：挑 2-4 个最关键的情节节点/台词/动作细节。太多会失去重点。

> 例：张奶奶必须念叨小时候糗事；旧画册必须出现且两人指尖差一点碰到；章尾梧桐叶画面不可省略

#### 禁止自由发挥边界

列出本章**不允许的操作**——subagent 在这些边界内自由发挥。

写法：写"不能做什么"，不写"应该做什么"。

> 例：禁止加额外人物（不出场同学/路人甲乙）；禁止陆屿本场景告白或挑明心意；禁止私自加外部冲突（电话/突发事件）

> **硬性规定：** 核心事件未完成（遗漏关键动作、删除关键台词、弱化不可删减节点），视为本章生成不合格，必须重写，不可跳过。

### 规则优先级排序

约束冲突时按此优先级裁决：**叙事完整性 ＞ 对话自然度 ＞ 句式代词细则 ＞ 单段行数限制**。先保剧情和人物，再抠文笔细节，最后卡排版。

### 三条转化规则

| 规则 | 含义 |
|------|------|
| 问"角色看到/听到/做了什么？" | 不写作者意图（展露在乎/关系拉近），只写具体画面 |
| 写"正在发生的事" | 不写将来时（本章将/接下来会），写进行时（她走到巷口） |
| 用角色感官写 | 不写分析（他意识到/读者感受到），写画面声音气味 |

### 场景例外放宽规则

不同叙事场景的约束执行力度不同，避免一套规则卡死所有段落。明确三类场景可适度放宽：

| 场景类型 | 放宽项 | 仍需遵守 |
|---------|-------|--------|
| 冲突高潮段 | 单段放宽至 2000 字；允许连续短句制造紧张感；代词重复容错适度放宽 | 禁止俗套比喻、禁止升华总结 |
| 密集对话段 | 句式约束松绑；代词不重复规则放宽；行数限制不做硬性要求 | 对话不加副词修饰、禁止 OOC |
| 环境铺陈/转场段 | 允许稍长白描，不受 1500 字上限限制 | 禁止空泛抒情、禁止诗词腔 |

其余日常叙事、人物静态刻画、情绪铺垫段——**严格执行原规则**。

### 章节终止条件：硬性停笔边界

完成所有叙事种子必达节点即收尾，禁止以下续写行为：

1. **禁止加戏**：完成核心事件后，不得新增支线人物、额外冲突、多余心理补叙
2. **禁止拖尾**：场景结束即切，不在余韵处反复渲染、过度铺陈
3. **禁止升华**：章末禁止升华总结、上帝视角感慨、强行留白煽情

**完成所有必达节点 = 章节自然结束。没有写完的，下一章再写。**

### 示例

```
三字段种子 + 约束与边界 填充示例：

前置状态：苏晚说开了个小书店，陆屿说正好没事跟着去看看。书店不大，堆着刚到的旧书。
核心事件：陆屿站在书架前，手指扫过书脊——准确抽出一本旧画册："这本你不是一直在找？"
             两人同时伸手，指尖差一点碰到。
收尾落点：苏晚心跳加速，两人都顿了片刻，暧昧在沉默里蔓延。

约束与边界：
不可删减约束：旧画册必须出现且两人指尖差一点碰到；陆屿不能在此表白
禁止自由发挥：禁止加额外人物；禁止设计"突然停电/书掉地上"等俗套桥段
```

### 禁止配替代

每条禁止都要配一条"可以怎么写"，不让 AI 束手束脚。统一用 ❌/✅ 格式，便于 subagent 理解 + 自动化校验：

| ❌ 禁止（违规） | ✅ 替代（合规） |
|------|---------|
| "他意识到她喜欢这本书" | "他注意到她翻到某一页时停了一下" |
| "他感到愤怒" | "他握紧杯子，指节发白" |
| "这一夜，他明白了……"（升华总结） | "他关灯，在黑暗里睁着眼" |
| "气氛变得暧昧起来" | "两人同时伸手，指尖差一点碰到" |

禁止项必须紧跟在它后面的可替代写法。

### 规则正反例标准化

所有核心约束统一标注 ✅ 合规示例 / ❌ 违规示例，避免 subagent 猜规则。固定格式：**规则一句话 → ❌ 违规 → ✅ 合规**。

| 规则 | ❌ 违规示例 | ✅ 合规示例 |
|------|-----------|-----------|
| 情绪不直写心理（允许 10% 以内） | "她感到一阵温暖""他意识到自己动心了" | "她低头看豆浆冒出的白气，眼睛有点发酸" |
| 禁止俗套比喻/空泛抒情 | "时间仿佛凝固了""空气像凝固了一样" | "他坐着没动，过了很久才眨了一下眼" |
| 对话不加副词修饰 | "他冷冷地说""她轻轻地笑了" | "他说，没抬头" / "她笑了，没说话" |
| 相邻四句不重复主语代词 | "他拿起书。他翻了两页。他皱了皱眉。他放下书。" | "他拿起书，翻了两页，皱了皱眉，放下。" |
| 单段行数 | 日常段 2000+ 字无分镜 | 1500 字左右自然切换分镜 |
| 章末不升华总结 | "这一夜，他明白了……" | 停在画面："他关灯，在黑暗里睁着眼" |

### 分段叙事入口

每段一条，放在该段种子前面。固定格式：

```
场景二｜拾光集书店（~1800 字）
画面重入：陆屿拿着旧画册，目光落在苏晚发红的耳尖上，语气不自觉放软。
情绪导向：从第一段的疏离压抑转为暧昧上扬，释放核心暧昧爽点。
节奏点：
1. 聊画册，陆屿说出苏晚的喜好（~300 字）
2. 整理时指尖相触，放大暧昧（~400 字，重点写微动作）
3. 反转：明信片出现，打破疏离（~400 字，爽点核心）
4. 对话拉扯，苏晚心境松动（~400 字）
5. 情绪卡点：话到嘴边又咽回去（~300 字）
```

入口三要素：

| 要素 | 写什么 | 不要写 |
|------|--------|--------|
| 画面重入（1 句） | 承接上一场结尾 | "接下来写…" |
| 情绪导向（1 句） | 本场情绪基调 | 抽象描述 |
| 节奏点（3-5 个） | 画面 + 字数 | 精确到每句 |

段间用 `---` 隔开，不写过渡句。

## 三、风格插件

主骨架不变，风格插件替换模块 5（文字与细节要求）中的描写要求和禁止项。

### 插件A：悬疑刑侦·冷峻白描

**适用：** 悬疑、刑侦、冷峻现实主义、社会派

```
文字与细节要求（悬疑冷峻）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、环境外化情绪为主——但允许每 1000 字 1-2 句内心推理（"他脑子里过了一遍现场的画面"）。
3. 环境细节服务氛围，不硬限制行数。
4. 对话每轮 3-5 句收住，不闲聊。
5. 允许偶尔的冗余细节——一个不相关的观察、一段沉默、一个没来由的回忆闪念。
6. 段落长短自由——推理沉浸处可展开，对话处果断收，不卡死行数。
7. ❌ "他意识到""他发现" → ✅ 动作/神态/环境外化；允许关键节点直述（"他心里一沉""后背一阵发凉"），10% 以内
8. ❌ "这意味着""这表明" → ✅ 只写画面，结论让读者自己下
9. ❌ 诗词腔、四字成语堆砌、浮生/流年/寂寥等虚词 → ✅ 白描叙事
10. ❌ 章末留下章预告 → ✅ 留情绪缺口（画面/未说完的动作）
11. ❌ 升华总结（"这一章他明白了"） → ✅ 章末停在画面/动作上
```

### 插件B：都市言情·细腻日常

**适用：** 现代/古代言情、治愈系、日常向

```
文字与细节要求（都市言情）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、神态、细节外化情绪为主——但允许每 1000 字 1-2 句直接心理（"她心想""她在那一瞬间承认了"）。
3. 环境描写用于烘托氛围，不限行数，以沉浸感为准。
4. 对话自然放松，允许日常闲聊感，不必每句话都有信息量。
5. 允许角色走神、发呆、注意无关细节——飞过的鸽子、墙上的水渍、隔壁的收音机声。
6. 段落长短自由——氛围沉浸时可放开写，对话时果断收。
7. ❌ "他/她意识到" → ✅ 关键情感节点允许直述（"她感到委屈""他心里发酸"）
8. ❌ 鸡汤升华、替读者总结情感结论 → ✅ 章末留余韵（画面/念头/没说出口的动作）
9. 允许适当留白——不必填满每段的信息量。
```

### 插件C：玄幻仙侠·叙事恢弘

**适用：** 玄幻、仙侠、奇幻、史诗向

```
文字与细节要求（玄幻仙侠）：
1. 固定单一 POV 视角，不跳转。
2. 以动作、环境氛围外化情绪为主——但允许每 1000 字 1-2 句内心活动（"他默念心法""他脑中闪过师尊的话"）。
3. 场景描写可展开，不受行数限制，以画面完整为准。
4. 战斗场景注重节奏——蓄势→爆发→余韵，不拖不赶。
5. 修炼/境界提升通过感官变化体现（灵气流动/经脉反应/天地异象）。
6. ❌ "他意识到" → ✅ 关键节点直接写情绪（"他心中骇然""一股寒意从脊背升起"）
7. 对话可带古风感，但不过度文言——自然中带文气。
8. 允许环境氛围渲染烘托战斗/修炼的张力。
9. 章末可留悬念钩子（发现/遭遇/变故），不必强行收住。
10. 允许偶尔的闲笔——一段风景描写不为叙事服务，只为意境。
```

### 插件D：网文快节奏·重爽点

**适用：** 系统文、赘婿文、战神文、快节奏爽文

```
文字与细节要求（网文快节奏）：
1. 固定单一 POV 视角，不跳转。
2. 开头 200 字内出冲突或信息差，不慢热铺垫。
3. 对话推进剧情为主，减少大段场景描写。
4. 每 500-800 字设一个小爽点或反转。
5. 情绪直给——允许适度直述情绪（震惊/狂喜/冰冷），不绕弯子。
6. ❌ 长句排比、文学性修辞 → ✅ 短句推进，简洁有力
7. 章末必须有钩子（发现/打脸预告/身份揭露），勾着读者看下一章。
8. ❌ 文艺腔、景物抒情、哲学感慨 → ✅ 情绪直给，不绕弯子
9. 允许角色偶尔的碎碎念或内心 OS——增加阅读代入感。
```

## 四、附加规则（可选，填模块 6 或跳过）

按题材/个人偏好选加，不选不影响正文生成。直接填入模块 6 的"无用但有温度的细节"。

```
❌ "时间仿佛凝固了""空气像凝固了一样"（俗套比喻） → ✅ "他坐着没动，过了很久才眨了一下眼"
❌ "他冷冷地说""她轻轻地笑了"（副词修饰对话） → ✅ 去掉副词，用动作/语气暗示（"他说，没抬头"）
❌ "这一夜，他明白了……"（章末升华） → ✅ 停在一个画面/动作上
❌ "他无奈地笑着摇了摇头说"（堆砌对话标签） → ✅ "他笑了，没说话" 或 "他说：'没事。'"
❌ 写角色外貌/穿着（除非与场景直接相关） → ✅ 只在与场景相关时才写穿着
```

### 写作约束总则：范式锁死，不依赖词表枚举（选加，覆盖模块 5 禁止项）

放弃无限枚举禁用词/AI腔词表的管控方式（无法穷尽新增变体），改为**限定合法叙事范式 + 句法语义约束**，根上封堵，无需维护黑名单。

```
1. 合法叙事白名单（全文仅允许三类写法）
   ① 客观环境、场景、物件白描
   ② 人物肢体动作、微表情、行为链条刻画
   ③ 人物原生对话 + 对话附着动作神态

2. 永久禁止叙事范式（句法级封禁）
   ① 禁止直述心理、下情绪结论——只用动作/神态/环境侧面外化
   ② 禁止上帝视角旁白、作者评述、空泛抒情、套路比喻、价值升华、心境解读
   ③ 禁止抽象形容词概括人物状态——必须用具体行为落地呈现
   ④ 禁止插入叙事者主观解读、脑补人物动机、增设支线

3. 验收判定（语义优先，不靠关键词匹配）
   ① 跳过动作细节直接给情绪结论 → 违规
   ② 出现作者旁白、上帝视角解读、强行升华 → 违规
   ③ 抽象定语概括状态无动作支撑 → 违规
   ④ 超出三类合法范式私自插入内心独白/主观评述 → 违规

4. 规则兼容
   范式约束优先级高于零散词汇禁令。句法合规则个别边缘词不拦截；落入禁止范式则无论用词多优美一律重写。
```

### 爽点守则（选加，控制爽点不生硬）

上述 5 项选填至模块 6"无用但有温度的细节"位。如果使用模块 4 的爽点与跌宕设计，建议至少选加 2-3 项：

```
☐ 不牺牲人物真实性——爽点从角色性格和情境自然生长
☐ 反转需有铺垫——回头看恍然大悟，不是空降
☐ 卡点克制，不编剧式强按头
☐ 章尾钩子是余味，不是"欲知后事如何"
☐ 情绪跌宕不违背人物即时状态（§3）的情绪弧光
```

## 五、使用流程

1. **定叙事任务** — 先按场景分解（情绪线→场景数→每场景基调+字数），再逐场景填三字段种子（前置状态/核心事件/收尾落点）和场景衔接。最后填约束与边界（不可删减+禁止自由发挥）。填模块 4 场景分解子模块。
2. **写分段入口** — 每段写画面重入+情绪导向+节奏点，放在该段种子前面。段与段用 `---` 隔开。
3. **设爽点跌宕** — 定爽点类型+情绪曲线+反转卡点+钩子，填模块 4 爽点与跌宕设计子模块。
4. **摘前情** — 上章结尾关键画面/矛盾/停留场景，50-80 字填模块 2。
5. **写人物快照** — 本章在场角色当下状态，每人一句填模块 3。
6. **选风格插件** — 按题材选对应的约束集，替换模块 5 的描写要求和禁止项。
7. **填剩余字段** — 模块 1（基础定位）、模块 6（真人感补充）。
8. **发送生成** — 完整提示词发给 subagent。

## 六、验收

### 六条硬校验

通过这 6 条 = 正文合格。每条可人工打勾 ✅/❌，也可对接脚本检测：

| # | 硬校验项 | 通过标准 | 检测方式 |
|---|---------|---------|---------|
| 1 | 叙事范式合规 | 全文仅使用环境白描、动作神态、附着对话三类范式 | 人工通读 |
| 2 | 无直述心理情绪 | 无"她感到/他意识到/她心想"等情绪结论与内心解读 | 关键词搜索 + 人工 |
| 3 | 无上帝视角旁白 | 无作者评述、空泛抒情、套路比喻、价值升华 | 关键词搜索 + 人工 |
| 4 | 叙事单元完整 | 按镜头自然分段；场景转换有衔接画面 | 人工判断 |
| 5 | 核心事件无遗漏 | 所有标注"不可删减"的必达节点完整出现 | 模板对照 |
| 6 | 视角与行文合规 | 单一 POV 无跳转；日常 ≤1500 字 / 高潮 ≤2000 字；相邻四句无重复主语代词 | 脚本统计 + 人工 |

**判定：** 6 项全部 ✅ → 正文合格。1-2 项 ❌ → 针对性修改。3 项及以上 ❌ → 整章重写。

### 30 秒快速嗅探

读模块 4（场景分解）场景1的**核心事件**——遮住字段名，读起来是"发生了什么事"（✅）还是"去写什么"（❌）？前者合格，后者是写作指令，整章要重新填种子。

### 提示词逐模块检查（填充阶段用，写正文前拦）

| 模块 | 合格标准 | 不合格信号 |
|------|---------|-----------|
| 1 基础定位 | 一读就知道"这章讲什么、什么调性" | 定位和基调对不上，或模板级空话 |
| 2 前置承接 | 前情像正常回忆，不是 bullet list | 50 字写成了剧情梗概，没有画面感 |
| 3 人物状态 | 有情绪起点→转折→落点，写的是"这章 TA 的状态" | 复制了角色百科的性格描述，或只有状态没有变化 |
| 4 场景分解 | 每场景有独立情绪基调，种子是叙事画面非指令，衔接自然不硬跳 | 场景之间没有情绪过渡，或种子是"这场景写…"写作指令 |
| 4a 爽点跌宕 | 有曲线（抑→扬→收）、有反转/卡点、有钩子 | 爽点类型不匹配题材，或曲线平铺直叙没有起伏 |
| 4b 约束与边界 | 不可删减约束标明 2-4 个关键节点；禁止自由发挥写"不做什么" | 约束空泛（"禁止跑偏"之类），或边界写成了"应该做什么" |
| 5 文字要求 | 视角锁定，描写要求具体可感 | 禁止项是空话（"禁止注水"无具体定义）|
| 6 真人感补充 | 至少有 1 个"无用细节"和"情绪波动" | 空着或只写了"无" |

全部通过 → 进入正文生成。任一不合格 → 先修再写。

### 进阶验收四项（正文生成后，爽点/情绪/反转/钩子）

| # | 验收项 | 合格标准 | 不合格信号 |
|---|-------|---------|-----------|
| 7 | 爽点 | 有明确的爽点类型，站位合理，细节具体不抽象 | 爽点无细节（只写"主角打脸"没写怎么打），或占比过低 |
| 8 | 情绪 | 严格遵循抑→扬→收三幕曲线，变化自然，人物有完整弧光 | 全篇平调没有起伏，或转折生硬没有铺垫 |
| 9 | 反转与卡点 | 有 1 处轻反转（情理之中意料之外）+ 1 处情绪卡点（停在最勾人处） | 反转空降无铺垫，卡点强行中断不自然 |
| 10 | 钩子 | 结尾有明确悬念/隐情/新危机，不把剧情讲透，有追更欲 | 结尾圆满了没有下文，或钩子强行堆砌与本章无关 |

## 七、题材差异化适配（AI 直接参考，不增加字段）

各题材的爽点类型、情绪曲线侧重、反转写法不同，以下供填充模块 4 时参考，不用额外改结构：

| 题材 | 爽点侧重 | 情绪曲线特点 | 反转/钩子写法 |
|------|---------|-------------|--------------|
| 玄幻仙侠 | 突破晋级、打脸反杀、秘境夺宝 | 压抑（瓶颈/轻视）→爆发（顿悟/突破）→余韵（境界稳固/新发现） | 伪善反派暴露、隐藏机缘出现、高人在侧 |
| 都市言情 | 暧昧拉扯、误会解开、心意流露 | 压抑（疏离/误会）→上扬（互动升温/心动瞬间）→回落（余韵/未挑明） | 隐藏心意意外流露、旧物引出的回忆、欲言又止 |
| 古风权谋 | 博弈翻盘、沉冤得雪、智斗碾压 | 压抑（构陷/困境）→上扬（布局破解/反击）→回落（余波/新棋子） | 盟友反水、隐藏身份揭露、螳螂捕蝉黄雀在后 |
| 悬疑刑侦 | 解谜突破、真凶浮现、证据反转 | 全程偏压抑，上扬在关键线索/突破口出现时，不真正放松 | 真凶另有其人、线索指向错误方向、证人隐瞒关键信息 |
| 科幻星际 | 危机解除、技术突破、文明发现 | 压抑（故障/孤立）→上扬（抢修成功/援军到）→回落（新威胁暗涌） | 内部背叛、外星文明信号、系统被入侵 |
