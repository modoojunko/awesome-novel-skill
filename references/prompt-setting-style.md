# 提示词组装指令

你的任务：章纲通过验收后，将章纲 + writing-style + genre-profile 组装为 subagent 可执行的提示词。产出写入 `prompts/vol-{N}-ch-{M}-prompt.md`。

**前置条件：** 章纲已通过验收（作者确认 + 检查清单通过 + 快速嗅探 + AI味自检）。

**两条底线**（任一不过，禁止进入正文生成）：
1. **视角已锁定且不含上帝视角残留**——提示词中不得出现"主角感到""主角意识到""读者应该感受到""这标志着""这意味着"等上帝视角解构语。
2. **what_to_write 全部可执行且无分析腔**——subagent 读完每个段落的 what_to_write 后，能直接写出正文。不含"核心心理动作""A 和 B 的区别在于""本章做的是前者"等教学语言。每个 what_to_write 不超过 200 字。

## 最高原则：提示词是写给 AI 来执行写作者想要的正文

提示词的每一个字，subagent 都会逐字照做。它不会质疑、不会补充上下文、不会追问意图。**写进去的它照抄，没写的它自己编。**

## 输出格式

产出文件 `prompts/vol-{N}-ch-{M}-prompt.md` 固定 6 部分，按以下顺序排列：

| # | 部分 | 写什么 | 来源 |
|---|------|--------|------|
| 1 | **核心基础要求** | 小说类型、核心基调、目标受众、**单章定位** | writing-style + genre-profile |
| 2 | **世界观与背景设定** | 时代背景、世界观规则、**本章涉及场景** | world-setting + 本卷概览 |
| 3 | **人物设定** | 主角1（含**本章人物状态**）、主角2（含**本章人物状态**）、关键配角 | character 设定 |
| 4 | **剧情核心要求** | **本章核心内容、本章核心亮点**、节奏、禁止内容 | 章纲 key_points + memo（经视角转换） |
| 5 | **文字与细节要求** | 视角、章节要求 | writing-style + genre-example |
| 6 | **补充说明** | 特殊要求、参考风格 | 按需

## 组装流程

按以下顺序填充输出文件的各部分。其中部分 1-3、5-6 从项目全局设定填充（每次写作复用），部分 4 从当前章纲填充（每章独立）。

### 第一步：视角转换（核心）

章纲是上帝视角写的（"主角发现 X""读者感受到 Y"），提示词必须锁定单一视角。这是第一步，也是最不能出错的一步——视角没锁住，整章报废。视角转换后的内容写入**部分 4（剧情核心要求）**。

#### 1. 视角锁定规则

| 规则 | 说明 | 常见错误 |
|------|------|---------|
| 锁定单一 POV | 整章只能通过一个角色的眼睛看世界。只写这个角色看到的、听到的、闻到的、触碰到的、想到的 | 同一段内出现两个角色的内心活动——读者会眩晕 |
| 禁止心理断言 | 不写"主角感到""主角意识到""主角判断出""主角明白"。心理活动通过可观察的身体反应和动作暗示 | 写"他感到不安"——读者看到了旁白，不是角色 |
| 禁止读者指向 | 不写"读者应该感受到""读者会注意到""让读者看到"。读者感受来自正文本身，不需要提示词指导 | 写"让读者感受到紧张"——subagent 会在正文里帮你制造"紧张"两个字 |
| 角色不知道的不写 | 不在 what_to_write 中暗示当前 POV 角色不知道的信息——subagent 会不自觉让角色"恰好知道" | 写"陆征不知道楼下的男人在跟踪他"——subagent 可能在正文里让陆征回头看一眼 |
| 用直接陈述代替解释性过渡 | 写"门开了"，不写"他注意到门开了"；写"雨打在玻璃上"，不写"他能听到雨打在玻璃上的声音" | 解释性过渡就是旁白——第三人称有限视角下不该出现 |

#### 2. 章纲→提示词视角转换对照

```
✅ 合格转换：
章纲 outline: "主角走进废墟，发现了一封信。读者应该感受到不安。"
提示词 what_to_write: "走进废墟。脚下是烧焦的木屑，踩上去没有声音。门框斜挂着，铰链断了一颗。桌上放着一封信——信封角落有三道暗红色的痕，拇指宽。窗外的光线刚好照在'苏棠'两个字上。"
→ 读者通过主角的感官发现信，不安来自木屑无声、门框倾斜、血迹——不靠"应该感受到不安"

章纲 outline: "陆征发现苏棠不是主动离开的。出租屋的状态与离家出走的特征不符。"
提示词 what_to_write: "床没铺。被窝保持掀开的状态，枕头上有头压过的凹痕。充电器还插在床头墙上，线垂到枕头边。冰箱打开——半盒剩饭、两瓶矿泉水。不是'收拾好行李离开'的样子。"
→ 不写"他意识到""他发现"，只写他看到了什么。结论让读者自己下

❌ 不合格转换：
章纲 outline: "主角走进废墟，发现了一封信。读者应该感受到不安。"
提示词 what_to_write: "主角走进废墟，内心感到强烈的不安。他在桌上发现了一封信，这让他意识到事情远比想象的复杂。"
→ "内心感到""意识到" = 上帝视角旁白，"远比想象的复杂" = 替读者下了结论

章纲 outline: "陆征发现苏棠不是主动离开的。"
提示词 what_to_write: "陆征环顾房间，意识到苏棠不是主动离开的——床铺状态、充电器和冰箱剩饭都与离家出走的特征不符。"
→ "意识到" = 解释性过渡，"与……不符" = 分析性结论。应该只写他看到了什么具体物品，不写他"意识到"了什么
```

#### 3. 多角色视角确认

如果章纲中存在多角色的信息线（如 A 角色做了一件事，B 角色在另一场景有自己的线索），提示词生成前必须确认本章锁定的单一视角。

| 检查项 | 标准 |
|--------|------|
| narrative_pov 字段 | chapter.yaml 的 narrative_pov 必须有值（第三人称有限/第一人称/全知）。空 → 必须先确认 |
| 单一视角确认 | 本章写谁的视角？这个角色在所有场景中都在场吗？不在场的场景如何通过他的视角获取信息？ |
| 信息不对等处理 | 如果当前 POV 角色不知道某信息，提示词中不暗示该信息的存在——不写"虽然他当时并不知道"等补丁 |

```
✅ 合格：
本章锁定陆征视角。苏沫的内心不写。方岩打电话时的表情只写陆征看到的——"方岩点了根烟，没有说话，把打火机翻了两面"。苏沫那边警察来的场景本章不写——只写陆征接到苏沫电话后听到的结果。

❌ 不合格：
本章以陆征视角写调查过程，同时在 what_to_write 中写"苏沫在警局等待时越来越焦虑，她开始怀疑自己找陆征是不是一个错误"——苏沫不在陆征的视野内，这个焦虑状态陆征看不到。
```

### 第二步：设定 subagent 写作身份

subagent 需要知道"我是谁"才能进入写作状态。选定题材后，从 `references/genre-example/` 的 corpus 文件中匹配四字段，写入**部分 1（核心基础要求）**和**部分 5（文字与细节要求）**：

- **叙事身份（role）** → 写入部分 1。告诉 subagent 它以什么身份写正文。从 genre-example 的"叙事者角色"+"人格"节读取，或使用 role_override（如有自定义）。
- **核心原则（core_principles）** → 写入部分 5。从文风蓝图 + 类型要素中提炼 8-15 条大白话约束，不分类不编号。
- **常见错误（possible_mistakes）** → 写入部分 5。从类型禁忌 + 文风要求中提炼，与核心原则合并，按致命程度排序。
- **描写技法（depiction_techniques）** → 写入部分 5。从文风蓝图转化可操作的大白话指引。

缺任何一个字段，subagent 在该维度的行为就是随机——可能写对，也可能完全放飞。

> 如有自定义需求，参考 `references/writing-style.md` 的方法论。使用 genre-example 默认值时无需查阅。

#### 2. 约束提炼质量规则

| 规则 | 标准 |
|------|------|
| 约束是大白话 | "不要让角色做不符合他性格的事" → 合格。"禁止角色行为偏离人设基线" → 不合格（书面化）|
| 每条约束可执行 | subagent 读完能判断"这句话违规了吗" |
| 约束不分类不编号 | 不用 🔴🟡🟢 或 1/2/3 编号——subagent 会对"红色规则"和"绿色规则"区别对待 |
| 约束不解释"为什么" | 只说"别做什么"，不说"因为读者会出戏""因为这样破坏了代入感"——解释占用 token，且 subagent 不需要知道理论 |
| 约束按致命程度排序 | 视角跳跃 > 直述情绪 > 段尾总结升华 > 句式规整 > 形容词堆砌。最不能犯的排最前面 |
| 数量 8-15 条 | 少于 8 条 = 关键约束遗漏；多于 15 条 = subagent 记不住，后半截约束形同虚设 |

```
✅ 合格提炼：
写作原则与禁忌：
- 只写陆征能看到、听到、碰到的东西。不在场的场景不写，别人心里的想法不写。
- 不写"他很愤怒"——写他捏碎杯子、声音压低、指节发白。想写感受的时候，写身体反应。
- 每一句对话都有目的——试探、拒绝、转移、暗示或攻击。不写为交代信息而交代信息的对话。
- 环境描写单段不超过2行。顺手带过，不当成重点来渲染。
- 不写"通过这件事，陆征意识到……"——只写发生了什么事，结论让读者自己下。
- 相邻四句不以"他"开头。单段不超过5行。每三句中有一句短句（1-5字）做节奏切断。
- 章末不留"下章预告"——只留情绪缺口。读者带着"想知道接下来会怎样"的心情合上这一章。
- 禁止诗词腔：不用排比古风句、不堆四字成语、不用空洞意境虚词（浮生/流年/凡尘/寂寥等）。

→ 8条，大白话，每条可执行，不解释"为什么"，不分类不编号。最致命的（视角 + 直述情绪）排最前。

❌ 不合格提炼：
写作原则与禁忌：
🔴 视角一致性：必须严格保持第三人称有限视角，禁止上帝视角
🟡 情感表达：通过Show, Don't Tell的方式展现情感
🟢 语言风格：保持自然松弛的写作风格

→ 分类标记干扰 subagent。每条都是空词——"严格的第三人称有限视角"是什么？subagent 不知道怎么才算严格。"自然松弛"是什么？subagent 没办法判断。
```

#### 3. 四字段完整性验证

```
✅ 合格（四字段完整）：
部分 1（核心基础要求）中叙事身份已正确设定："你是悬疑刑侦题材的职业写作者。写作时像陆征本人一样——对细节偏执、不急于下判断、相信实物证据多于口头描述。"
部分 5（文字与细节要求）包含 10 条大白话约束 + 描写技法转化

❌ 不合格（字段缺失）：
叙事身份是模板默认值（"你是全球闻名的小说作家"）——没有类型专属身份，subagent 以通用 AI 角色写作
常见错误缺失——提示词中没有警告，subagent 可能在章末做下章预告
描写技法缺失——subagent 只用最标准描写方式，全是直述情绪
```

### 第三步：写入类型专属指引

如果项目已选择 genre_profile，从 `references/genre-example/` 的 corpus 文件中读取 prompt_segment，原样写入**部分 1（核心基础要求）**和**部分 5（文字与细节要求）**。prompt_segment 包含该类型的特定写作规则、爽点结构、禁忌等——不写 = subagent 以通用模式写作，丢失题材味道。

#### 检查清单

| 检查项 | 标准 |
|--------|------|
| genre_profile 已配置 | 项目的 genre_profile 非空。若空 → 是作者选择"暂不选择"还是漏填？确认后决定是否跳过此段 |
| prompt_segment 已读取 | 从 references/genre-example/index.md 找到 genre_profile 对应的 corpus 文件，读取 prompt_segment |
| prompt_segment 结构保留 | 原样注入，保持其段落结构和表述方式。不压缩、不改写、不"提炼"——prompt_segment 是类型专家的输出，Agent 二次改写可能丢失关键信息 |
| genre_taboos 已作为约束写入 | references/genre-example 中如有 genre_taboos 字段，必须合并到"写作原则与禁忌"段 |
| role_override 已覆盖默认叙事身份 | "角色定位"段优先使用 references/genre-example 的 role_override，而非 genre-example 文件自身的默认叙事者角色 |

#### 验证示例

```
✅ 合格注入：
类型专属指引 → 部分 1 + 部分 5：
[来自 references/genre-example/suspense-crime.md 的 prompt_segment 原文——保持其段落结构]
- 信息释放策略：每章至少给读者一个"原来是这样"的时刻……
- 红鲱鱼规则：每卷至少一条有说服力的伪线索……
- 侦探视角下的日常场景：每个环境细节可能是有用的……

叙事身份 → 部分 1（核心基础要求）：
"你是悬疑刑侦题材的职业级作家。用陆征的视角写作——对细节偏执，不急于下判断，相信实物证据多于口头描述。行文冷峻克制，像案件档案的笔触——客观、精确、不加形容词。"

→ prompt_segment 原样保留，role_override 覆盖了默认叙事身份。

❌ 不合格注入：
genre_profile 为 "suspense-crime"，但部分 1+5 缺少类型专属指引——prompt_segment 未被读取或未写入
部分 1 的叙事身份仍引用 genre-example 默认值——role_override 未被读取
prompt_segment 被 Agent "提炼"为几条要点——丢失了类型专家原始表述中的微妙约束
```

### 第四步：映射章纲到提示词

章纲的所有关键信息必须转化为提示词中的具体指引，但转化不是粘贴——章纲是给作者看的笔记，提示词是给 subagent 的指令。转化结果写入**部分 4（剧情核心要求）**。转化过程中丢失的每一项信息，都会成为正文里的空位。

#### 1. 叙事种子法（通向 100 分的关键）

当前流程（80 分上限）的路径是：章纲 key_point → 视角转换 → 映射为写作指令 → 双轮净化去掉分析词。这条路径的产出是"干净的写作指令"——subagent 逐字执行，产出干净的正文，但干净不等于像人写的。

**100 分的路径不同：不写指令，写叙事种子。**

对每条 key_point，不做"这段正文应该写成什么样"的分析，而是直接写一段**POV 角色此刻的叙事画面**。subagent 读到的不是"去做什么"，而是"继续写这段"。两到四句足够。

```
章纲 key_point
  ↓
80分：写写作指令 → 双轮净化去掉分析腔
    → "整理书架，过程中流露心动细节。用动作暗示，不写内心独白。"
    → subagent 执行：按指示写了一段"整理书架"戏，指令逐条照做，但节奏生硬

  ↓
100分：写 POV 角色的叙事画面
    → "陆屿站在书架前，手指扫过书脊，然后停在一本旧书边上。
       苏晚在吧台后面擦杯子，余光里是他左腕上那条磨得发亮的手链。
       两个人都不说话，但不是尴尬的那种安静。"
    → subagent 继续写：同一段叙事语言的自然延伸，没有"执行指令"的痕迹
```

**三条转化规则：**

**规则一：问"POV 角色看到/听到/做了什么？"，不问"这段要表达什么？"**

key_point 里写的是作者意图（"表现男主对女主的了解"），但叙事种子只写画面（"他记得每本书的位置，手指精准地抽出一本旧书"）。

```
章纲 key_point："陆屿帮苏晚整理书架，过程中展露他对她的在乎，两人的关系在安静中拉近"

❌ 80分：抓住"展露在乎""关系拉近"做文章
→ "整理书架时，陆屿用行动表明他很在乎她。两人不说话，但距离在拉近。
   注意：不写内心独白，通过动作表现。"

✅ 100分：只写 POV 角色看到的画面，不提解读性的"在乎""拉近"
→ "陆屿站在书架前，手指从一排书脊上滑过去，然后准确抽出那本
   苏晚上周找不到的旧画册。他递给她时说：'这本你不是一直在找？'
   苏晚接过来，书脊上还有他指尖的温度。"
```

**规则二：写"正在发生的事"，不写"会发生的事"**

```
❌ "本章开始时陆屿在早餐铺等苏晚，两人一起吃了早餐"
→ 将来时态 → subagent 按步骤执行，流水账

✅ "苏晚走到巷口，一眼就看见陆屿坐在早餐铺最靠里的位置。
    面前两碗豆浆，一碗已经放了糖。"
→ 正在进行时态 → subagent 以相同语感继续推进
```

**规则三：用 POV 角色的感官写，不用分析写**

```
❌ "这段的重点是让读者感受到梧桐巷的烟火气"
→ 分析语言 → subagent 在正文里"营造烟火气"

✅ "油锅滋滋响，张奶奶的围裙上沾着面粉，隔壁中药铺的收音机放着粤曲。"
→ 感官语言 → subagent 在正文里自然地延续环境描写
```

**整章 key_points → 叙事种子的完整转换：**

```
章纲 key_points：
[1] 清晨，陆屿在早餐铺等苏晚。两人久别重逢，气氛既熟悉又有点微妙。
[2] 两人一起去拾光集书店，陆屿主动帮她整理书架，过程中流露出他对她的了解。
[3] 黄昏，陆屿送苏晚回家。一片梧桐叶落在她头发上，他帮她拿掉。
[4]（结尾）苏晚独自在房间里，回想今天的一切，意识到自己的心意没有变。

❌ 80分转换（写作指令 → 双轮净化 → 干净但生硬）：
1. 清晨早餐铺。陆屿先到，苏晚后到，张奶奶打招呼。
2. 书店整理书架。陆屿准确找到苏晚找不到的书。
3. 陆屿送苏晚回家，梧桐叶落在她头发上。
4. 苏晚独处，回想今天的事。

✅ 100分转换（叙事种子 → subagent 延续正文）：
1. 苏晚走到巷口，看见陆屿坐在早餐铺最靠里的位置。
   面前两碗豆浆，一碗已经放了糖。张奶奶在厨房里喊：
   "小屿可算回来了，晚晚快来！"
   ——写到这里。下面写陆屿把苏晚碗里的香菜挑到自己碗里，像高中时一样。

2. 推开拾光集的门，旧书的味道扑面而来。
   陆屿站在书架前，手指扫过书脊——然后准确抽出一本旧画册：
   "这本你不是一直在找？"
   ——写到这里。两个人同时伸手去够同一本书时，指尖差一点碰到。

3. 店里的光线从金黄变成了橘红。
   陆屿说"我送你"，不是一个问句。
   梧桐叶落在苏晚头发上，他抬手帮她拿掉——动作太自然了，
   好像他们从未分开过这三年。
   ——写到这里。苏晚走在他影子边上，步子慢了下来。

4. 苏晚上楼后站在窗边，看着陆屿的背影走远。
   她坐下来，拿出画笔，开始画今天看到的第一个画面。
   ——结尾。不写"她明白了自己的心意"——读者已经看到了。
```

**叙事种子法的完成标准：** 每条约 150-300 字，以叙事画面开头，以"——写到这里"引导 subagent 自然延续。读起来像正文的片段，不像写作指令。subagent 不会觉得"我在执行指令"，而是觉得"我在继续写这段文字"。

**与关键步骤的关系：**

| 步骤 | 在叙事种子法中扮演的角色 |
|------|------------------------|
| 视角转换 | 叙事种子的前提——每句话都是 POV 角色的感官，不是上帝视角 |
| 完整性映射 | 确认每条 key_point 是否都有对应的叙事种子，没有的补上 |
| 长章分段 | 叙事种子太长时，按场景拆为多个段落，每段独立叙事重入 |
| 否定句式替换 | 叙事种子本身不含否定句式——它已是正面叙事，无需替换步骤 |
| 双轮净化 | 检查叙事种子是否混入了分析语言——如果写对了叙事种子，这一步几乎不需要 |

**叙事种子法 vs 旧流程的耗时对比：**

```
旧流程（每条 key_point）：
上帝视角 key_point → 视角转换（改到 POV） → 映射为写作指令 → 否定句式替换（改到正面）
    → 双轮净化第一轮（去分析腔） → 双轮净化第二轮（去分析词） → 产出是"干净的写作指令"
    → subagent 按指令执行 → 产出干净的正文

叙事种子法（每条 key_point）：
上帝视角 key_point → 直接写 POV 叙事画面 → 检查是否混入分析语言 → 产出是"正文种子"
    → subagent 延续正文 → 产出是同一语感的正文
```

叙事种子法绕过"写作指令"这个中介层。省掉视角转换、否定句式替换、双轮净化三个步骤——不是通过省略检查，而是通过从一开始就写对了。

#### 2. 完整性映射表

| 章纲来源 | 提示词去向（模板部分） | 转化标准 |
|---------|----------------------|---------|
| outline.summary | 部分 2（世界观与背景设定） | 不是粘贴 summary——转为一两句话的场景起点："本章开始时，陆征刚从出租屋回到事务所。桌上放着一张便利贴，上面只有一个电话号码。" |
| outline.key_points | 每条 key_point → 部分 4（剧情核心要求）中的一条剧情节点 | key_point 不是复制——必须视角转换 + 双轮净化 |
| memo.current_task | 部分 4（剧情核心要求）的本章目标 | 转化为 subagent 知道"本章航向"的一句话。不写"本章讲述了"，写"本章完成的具体动作" |
| memo.reader_expectation | 融入部分 4 的剧情节奏 | state/strategy/detail 决定每条什么写细、什么留白——需要满足期待的部分写具体，制造缺口的部分点到为止 |
| memo.payoff_plan | 融入部分 4 对应节点 | must_resolve 体现在本条写到什么程度；must_hold 体现在本条不碰什么；partial_advance 体现在本条推到哪一步 |
| memo.prohibitions | 部分 4 的"禁止出现的内容"（如非空） | 章纲的 prohibitions 直接追加到禁止内容中 |
| memo.required_changes | 部分 4 末节点体现改变 | 章末改变不需要单独注明——通过末条的场景和动作自然呈现。不写"陆征知道了"，写"他把钥匙还给房东" |
| 前章结尾状态 | 部分 2（世界观与背景设定）的前章锚定 | 告诉 subagent 上一章结束时的画面和情绪——subagent 不从零开始，是从上一章的结尾接着写 |
| 本章字数目标 | 部分 5（文字与细节要求）的章节要求 | 全章总字数。各节点写对应的篇幅指引 |

#### 2. 长章特殊处理（字数 ≥ 2000 时必做）

当本章字数目标 ≥ 2000 字时，直接填充 6 部分表单会产生**中部漂移问题**——subagent 写前 1000 字时还受叙事身份和写作原则牵引，写到 2000 字时语感已经散掉，开始出现重复措辞、节奏失调、角色走形。

**解决方案：三段式长章架构**

把 6 部分表单中的"部分 4（剧情核心要求）"从一条平铺的剧情线，拆为"全局锚点 + N 段独立叙事入口"。N 由字数倒推：总字数 ÷ 1500 向上取整（4500 字 ÷ 1500 = 3 段），每段 1500 字以内。

```
4500 字章的结构：
┌─────────────────────────────────────┐
│ 全局叙事锚点（POV + 语感 + 人物锚）    │  ← 写一次，影响全章
├─────────────────────────────────────┤
│ 分段入口 1/3（画面重入 + 本段指引）      │  ← 每段独立，重新锚定
│ 分段入口 2/3（画面重入 + 本段指引）      │
│ 分段入口 3/3（画面重入 + 本段指引 + 收尾) │
└─────────────────────────────────────┘
```

**全局叙事锚点（200-300 字，写在部分 4 开头）**

用叙事语言写一段统摄全章语感的开场，不是规则说明。subagent 读完这一段就"进入状态"，后续每段都能延续这个调性。

```
✅ 写作格式（叙事语言）：
梧桐巷的清晨，是从张奶奶的油条锅开始的。
苏晚站在书店二楼的窗前，看着楼下巷口的那个背影。
——全章主视角：苏晚，第三人称限知
——不看过去心理，不看未来走向，只看此时此刻她看到的、听到的、感受到的
——每段开头写画面，结尾落到她心里那个没说出口的念头
```

```
❌ 不合格格式（规则说明，不是叙事）：
本章以苏晚的视角展开。注意保持第三人称限知视角，避免上帝视角。
不要写心理断言，不要跳过当前场景。
```

**人物情境锚（100-150 字，紧接全局锚点）**

人物设定不是百科条目——subagent 拿到"身高 165cm、性格温柔内敛"会原文照搬到正文。用**具体情境 + 具体动作**代替抽象标签：

```
✅ 合格人物锚（情境化）：
苏晚看陆屿时：
- 目光先落在他左腕的手链上（她编的那条），然后再看他的脸
- 紧张时会做具体的事：摩挲书页、把桌上的东西摆整齐、假装在看别处
- 她不是"温柔内敛"，她是"话到嘴边又咽回去"

陆屿看苏晚时：
- 他不说太多，但动作都到位——挑香菜、帮拿落叶、记得每一件小事
- 他看她的眼神很专注，但不逼人
```

```
❌ 不合格人物锚（百科条目，会被 copy 到正文）：
苏晚，165cm，纤细。性格温柔内敛，心思细腻。紧张时会摩挲书页。
陆屿，188cm，外冷内热。不擅长说情话，但行动力强。
```

**分段叙事入口（每段 200-300 字）**

每段开头写一段**画面重入**——一句话锚定场景+情绪，让 subagent 重新进入状态。然后给出本段的具体指引。

```
✅ 分段入口格式：
场景一｜早餐铺（~500字）

苏晚走到巷口时已经磨蹭了快十分钟。她在紧张什么，自己也不愿意想清楚。
然后她看见了陆屿——他坐在张奶奶早餐铺最靠里的位置，面前两碗豆浆。

——这一段写四个节奏：
1. 张奶奶的对话打破沉默，自然引出话题
2. 陆屿挑香菜——不用解释，这就是他的问候
3. 穿插张奶奶念叨两人小时候的事，制造轻松笑点
4. 结尾落在：她走出早餐铺时，他自然地走在她左边

只用苏晚的感官。不说"她感到温暖"，写她把手指贴在豆浆碗壁上。
```

**分段规则**

| 规则 | 说明 |
|------|------|
| 每段 1200-1500 字 | subagent 在 1500 字内可以保持语感稳定。超过 2000 字需要再细分 |
| 逐段执行 | subagent 写完一段后停下来，读完下一段入口，再写下一段。不是一次输出全部 |
| 物理空行分隔 | 段与段之间用 `---` 隔开，subagent 会在段落间隙自然停顿 |
| 段落间有场景/时间跳跃 | 早餐铺 → 书店 → 黄昏——每组场景有自然的时间间隙，不用过渡句硬连 |
| 收尾段有情绪落点 | 不写"本章完"式的总结，但最后一段要有安静的余韵——一个画面、一个念头、一个没说完的动作 |

**字数倒推法**

从目标字数反向拆段：

| 目标字数 | 段数 | 每段字数 | 章节容量 |
|---------|------|---------|---------|
| 1500-2000 | 1 段 | 单段完整 | 一个场景完整展开 |
| 2000-3500 | 2 段 | 1000-1750 | 两个场景 + 过渡 |
| 3500-5000 | 3 段 | 1200-1700 | 起承转合完整 |
| 5000-7000 | 4-5 段 | 各 1400 | 多线叙事或长场景扩展 |

```
✅ 合格字数规划：
目标 4500 字 → 3 段 × 1500 字
场景一：早餐铺重逢（1500 字）
场景二：书店日常（1500 字）
场景三：黄昏收尾（1500 字）

❌ 不合格字数规划：
目标 4500 字 → "写 4500 字左右"
（没有分段、没有字数分配，subagent 写到 2000 字开始漂移）
```

**全局锚点 + 人物锚 + 分段入口 三者缺一不可**——缺全局锚点则每段语感各自为政；缺人物锚则角色出场仍是百科体；缺分段入口则中部漂移。

```
✅ 合格长章（三段结构完整）：
┌─ 全局叙事锚点（梧桐巷清晨画面 + POV + 语感规则）─────┐
│　人物情境锚（苏晚的紧张动作、陆屿看她时的眼神）　　　　│
├─ 分段 1/3 入口（早餐铺·画面重入 + 4 个节奏 + 字数 ）──┤
│　分段 2/3 入口（书店·画面重入 + 5 个节奏 + 字数 ）──┤
│　分段 3/3 入口（黄昏·画面重入 + 5 个节奏 + 字数 + 收尾）┤
└──────────────────────────────────────────────┘

❌ 不合格长章：
"本章写 4500 字。陆屿和苏晚在早餐铺相遇，然后去书店，
最后苏晚回家。保持温馨细腻的风格。"
→ 没有分段、没有叙事入口、字数没有分配、人物用百科体写
```

#### 3. 常见转化遗漏

| 遗漏项 | 后果 | 检查方法 |
|--------|------|---------|
| memo.prohibitions 未写入提示词 | subagent 不知道本章的红线——可能写作者禁止的内容 | 扫描部分 4"禁止出现的内容"，逐条对照 memo.prohibitions |
| memo.required_changes 未体现在末条 what_to_write | 章末没有让读者感知到改变——读者看完觉得"什么都没发生" | 读部分 4 末节点，看最后一句是否呈现了变化 |
| 前章结尾状态缺失 | subagent 不知道上一章结束时是什么场景、什么情绪——可能写出与上章矛盾的起始状态 | 部分 2 中是否有"上一章结束于……"的描述 |
| payoff_plan 的 must_resolve / partial_advance 未注入 | 该收的钩子没收，该推进的埋伏没有推进——读者会在后续章节发现"这个事之前没讲啊" | 扫描提示词全文，搜索每个钩子 ID——确保每条需操作的钩子有对应指引 |

```
✅ 合格：
章纲 memo.prohibitions:
  - "不要揭示陆征三年前离开警队的详细原因——只通过细节暗示他以前是刑警"
  - "不要让楼下的男人和陆征对话"

提示词部分 4（剧情核心要求）的"禁止出现的内容"：
- 不要写陆征三年前离开警队的原因。可以通过同事的对话或他的习惯暗示他以前是刑警，但不说破具体事件。
- 楼下的男人不跟陆征说话。只写他的存在——站在路灯下、烟头明灭、眼睛没有离开过陆征的方向。

→ 每条约 prohibitions 都有对应的提示词禁止项，且有具体的执行描述。

章纲 memo.required_changes:
  - "信息改变：确认苏棠非主动离开——房间状态3个矛盾细节"

末条 what_to_write：
"陆征把钥匙还给房东。下楼时他在楼梯间站了一会儿——手机里有今晚拍的六张照片。床没铺。充电器还在墙上。冰箱里的剩饭还没馊。他给方岩发了条消息：这个案子不是主动离开。发完之后把手机翻过去，看着楼梯间窗户外面对面楼的灯。"

→ three required_changes details embedded naturally——subagent 不需要被"要求带出三个细节"，写这段场景自然就会写到。

❌ 不合格：
memo.prohibitions 写了 4 条，提示词部分 4 的"禁止出现的内容"只写了"不要太拖沓"和"保持节奏"——两条抽象空词替代了全部具体红线
memo.payoff_plan 写了 partial_advance hook-001，提示词全文搜索不到 "hook-001"——该推进的钩子完全没有指引
```

### 第五步：替换否定句式

提示词只写你想要的。不写"不要写成 XXX"——subagent 读到"不要写成热血战斗"时，它的注意力已经被引导到"热血战斗"上，正文里反而更容易出现热血战斗的影子。

#### 1. 否定句式替换规则

| 禁止写法（否定引导） | 替换为（正面指引） | 原理 |
|----------------------|-------------------|------|
| "不要写成热血战斗" | "冲突发生在日常场景中——出租屋、物流公司、街边大排档。武器是信息和观察力，不是拳头。" | 用场景限制自然避开热血战斗 |
| "不要写得拖沓" | "每个环境细节顺手带过，不超过 2 句。对话每轮 3-5 句收住，不延伸成闲聊。" | 用具体字数/句数限制控制节奏 |
| "不要让主角太完美" | "陆征有个习惯——调查到一半会停下来抽根烟，对着墙发呆。他也会在笔记本边上画小人。" | 用具体行为塑造不完美 |
| "不要使用诗词腔" | "叙事用大白话。环境只写三样东西——光、声音、温度。不写'晚风微凉，暮色沉沉'，写'路灯亮了一排，风从巷口灌进来'" | 用正面范例替代禁止清单 |
| "不要跳视角" | "全章只写陆征的眼睛——他看到什么、听到什么、碰到什么。其他人脸上的表情，只有陆征看到的时候才写。" | 用感官边界自然锁住视角 |

#### 2. 否定句式扫描

| 检测项 | 命中模式 | 修正方法 |
|--------|---------|---------|
| 否定引导式约束 | "不要写成……""避免……""别让……""切忌……" | 全部替换为正面描述：写"应该是什么样"，不写"不该是什么样" |
| 隐含否定的"不是……而是……" | "不是 X，而是 Y"——subagent 的注意力先到 X，再到 Y。正文里 X 和 Y 都可能出现 | 直接写 Y。如果需要对比，写 Y 的正面描述，"X"这个字不出现 |
| "不要只写……" | "不要只写动作，也要写心理"——subagent 可能理解为"少写动作" | 写"动作写完之后，让角色停下来——发呆、看窗外、翻手机。这两个节奏交替。" |

```
✅ 合格：
写作原则与禁忌中包含：
- 冲突场景控制在日常环境中——出租屋、街边大排档、物流公司仓库。不写追逐、打斗、爆炸。
- 每段对话 3-5 句后收住。不延伸成闲聊或辩论。
- 环境描写单段不超 2 行。顺手带过——写光、声音、温度，不写风景。

→ 全是正面指引。subagent 知道"应该写成什么样"，不需要先在脑子里否定错误版本。

❌ 不合格：
写作原则与禁忌中包含：
- 不要写成热血战斗文
- 不要写得太拖沓
- 不要用诗词腔
- 不要跳视角
- 不要写得太平淡

→ 全是负面词汇。subagent 不知道"热血战斗""拖沓""诗词腔""跳视角""平淡"的具体边界在哪——它只能猜。而且每读一个否定词，注意力就在那个方向上停一下。"不要写得太拖沓"→ subagent 在脑子里过了一遍"什么是拖沓"→ 然后在正文里更可能写出它理解的"不拖沓"——可能是写得太赶。
```

---

### 第六步：双轮净化 what_to_write

章纲 key_points 转化为 what_to_write 后，每条必须经过两轮净化——第一轮去掉分析腔（结构层），第二轮去掉分析词（词句层）。两轮净化漏掉的内容，会原封不动地出现在正文里。

#### 第一轮（结构层）：禁止 7 种 AI 味模式

| # | 命中模式 | 示例 | 修正 |
|---|---------|------|------|
| 1 | 心理机制分析 | "核心心理动作：三次失败→偏激。不是'他不怕死了'——是他把命押上去了。" | 不分析。写"今天和前几次不一样——不是试方案，是收命。" |
| 2 | 教 subagent "怎么写" | "这个认识不通过'他想起'来写。通过身体：后背、后脑、肋骨——上次死的时候被捅穿的位置。" | 不教。直接写"后背、后脑、肋骨——上次被捅穿的位置，在早上醒来的时候凉了一下。" |
| 3 | 概念对比论证 | "A 和 B 的区别在于：推演是'这样能行吗'，确认是'这样就行'。林默今天的脑子在做的是后者。" | 不区分。写"方案在心里跑了一遍——不是推演能不能行，是确认每个步骤。" |
| 4 | 代词指代分析结论 | "本章林默在后者。他不需要说服自己——方案已经是确定的了。" | 不指代。写"方案已经是确定的了。他在脑子里把它跑一遍。" |
| 5 | 分层心理指引 bullet list | "心理刻画指引：\n- 核心心理动作：……\n- 认识通过身体来写\n- 方案在心里跑的时候……" | 合并为连贯的场景叙述。不列条目不编号。 |
| 6 | 情绪标签化描述 | "情绪基调：偏激。是三次死亡堆出来的那种冷。" | 去标签。写"情绪：冷。不是平静——是火烧完之后压成密度更大的东西的那种冷。" |

#### 第二轮（词句层）：删除 5 类分析词

| # | 删除目标 | 命中词 | 替换为 |
|---|---------|--------|--------|
| 1 | 分析性动词 | "做出决定""意识到""发现""察觉""明白""不再""开始试图""判断出" | 具体的身体动作或外部事件 |
| 2 | 总结性判断 | "这表明""这意味着""整件事的性质变了""碎片开始拼成""图景开始浮现" | 删除。读者自己品，不给结论 |
| 3 | 动机分析 | "因为""不是……而是……""之所以""背后的动机是" | 只写发生了什么，不写为什么发生 |
| 4 | 心理标签 | "犹豫了""在思考""在权衡""内心挣扎" | 身体反应——手指停在笔杆上、没接话、看了两秒窗外 |
| 5 | 硬上限 | what_to_write 超过 200 字 | 继续压缩。每个 beat 一句话，不连成段落 |

#### 净化前后对照

```
章纲 what_to_write（净化前）：
陆征做出决定：不再等下一个威胁找上门。他给老邱打电话，老邱犹豫了一下——不是不想帮，是知道陆征不在队里了。最后还是查了。挂电话前低声说证词档案被人调走过。陆征意识到事情比他想象的严重——整件事的性质正在改变。

→ 命中项：
  - "做出决定" = 分析性动词
  - "犹豫了一下" = 心理标签
  - "不是……而是……" = 动机分析
  - "意识到" = 分析性动词
  - "整件事的性质正在改变" = 总结性判断

what_to_write（净化后）：
天还没亮，陆征拨通老邱的电话，报老马的号码请他查最后一次活跃位置。老邱在电话那头顿了两秒才应声。键盘响了一阵。挂电话前压低了声音——证词档案被人调走过，调档记录没留名字。

→ 92 字。全是具体动作和信息。没有解释、没有分析、没有结论。subagent 拿到就能写。
```

## 验收流程

6 步组装完成后，按以下步骤验收。全部通过后才进入正文生成。

### 第 1 步：逐项过检查清单

#### 视角转换
- [ ] narrative_pov 已锁定，全章单一视角
- [ ] 所有 what_to_write 不含"主角感到""主角意识到""读者应该感受到"
- [ ] 所有 what_to_write 不含解释性过渡（"他注意到""他能听到""他发现"）
- [ ] 当前 POV 角色不在场的场景没有出现在 what_to_write 中
- [ ] 所有心理活动通过动作/对话/环境暗示，没有直接心理断言

#### 部分 4（剧情核心要求）
- [ ] 每条剧情节点已视角转换——不含"主角感到""主角意识到""读者应该感受到"
- [ ] 每条剧情节点已双轮净化——不含分析性动词、总结性判断、动机分析、心理标签
- [ ] 每条剧情节点不超过 200 字

#### 叙事种子法（如果使用——不适用则跳过）
- [ ] 每条 what_to_write 是叙事画面，不是写作指令——读起来像正文片段，不像"这段要写什么"
- [ ] 每条以 POV 角色的感官写——不出现作者意图分析（"展露在乎""关系拉近"等解读性语言）
- [ ] 每条是正在进行时态——不出现"本章将""接下来会""写到XXX时"等将来时
- [ ] 全部 what_to_write 不含"明确""需""要""注意"等元指令动词，不含"示例：""例如："等标记
- [ ] 人物出场不通过百科条目写——通过动作、对话、环境线索自然呈现
- [ ] 如果使用了叙事种子法，双轮净化清单可跳过——叙事种子在写的时候已经绕过了分析语言

#### 写作风格四字段
- [ ] 叙事身份已写入部分 1（核心基础要求）——告诉 subagent 它以什么身份写正文
- [ ] 核心原则已提炼为 8-15 条大白话约束，写入部分 5（文字与细节要求）
- [ ] 常见错误已合并到约束中，按致命程度排序
- [ ] 描写技法已转化为操作指引，写入部分 5
- [ ] 约束不分类、不编号、不解释"为什么"
- [ ] 每条约束 subagent 能用来判断"某句话违规了吗"

#### 类型专属指引
- [ ] genre_profile 已配置（若空，确认是"暂不选择"而非遗漏）
- [ ] prompt_segment 已从 references/genre-example 读取并原样写入部分 1+5（若有）
- [ ] genre_taboos 已合并到部分 5 的约束中（若有）
- [ ] role_override 已覆盖默认叙事身份（若有）

#### 章纲→提示词完整性
- [ ] outline.summary 已转化为部分 2（世界观与背景设定）的本章上下文（非粘贴）
- [ ] memo.current_task 已转化为部分 4（剧情核心要求）的本章目标
- [ ] memo.payoff_plan 的全部操作（must_resolve / must_hold / partial_advance）已体现在部分 4 的对应节点或部分 2
- [ ] memo.prohibitions 全部写入部分 4 的"禁止出现的内容"
- [ ] 前章结尾状态在部分 2 中有描述
- [ ] 本章字数目标已设定，各节点篇幅指引合理

#### 长章特殊处理（字数 ≥ 2000 时必做——不适用则跳过）
- [ ] 已做分段：总字数 ÷ 1500 向上取整确定段数，每段 ≤ 1500 字
- [ ] 有全局叙事锚点（200-300 字叙事语言开场，不是规则说明）
- [ ] 有人物情境锚（具体动作/场景，不是百科条目——"紧张时会摩挲书页"✅ "性格温柔内敛"❌）
- [ ] 每段有独立叙事重入入口（画面+感官开头，重新锚定语感）
- [ ] 每段有字数分配（各段字数相加 = 目标总字数）
- [ ] 段落间有场景跳跃或时间间隔，不需要过渡句硬连
- [ ] 收尾段有情绪落点——一个画面、一个念头、一个没说完的动作，不是总结

#### 否定句式
- [ ] 部分 5（文字与细节要求）不含"不要写成……""避免……""别让……""切忌……"
- [ ] 所有 what_to_write 不含"不是……而是……"句式
- [ ] 所有约束是正面指引——写"应该是什么样"，不写"不该是什么样"

#### 双轮净化（部分 4 剧情节点）
- [ ] 所有 what_to_write 不含心理机制分析
- [ ] 所有 what_to_write 不含教 subagent "怎么写"的教学语言
- [ ] 所有 what_to_write 不含概念对比论证
- [ ] 所有 what_to_write 不含代词指代分析结论
- [ ] 所有 what_to_write 不含 bullet list 式分层心理指引
- [ ] 所有 what_to_write 不含情绪标签化描述
- [ ] 所有 what_to_write 不含分析性动词（"做出决定""意识到""察觉""明白"等）
- [ ] 所有 what_to_write 不含总结性判断（"这表明""这意味着""性质变了"等）
- [ ] 所有 what_to_write 不含动机分析（"因为""不是……而是……"等因果解释）
- [ ] 所有 what_to_write 不含心理标签（"犹豫了""在思考""内心挣扎"等）
- [ ] 每个 what_to_write 不超过 200 字
- [ ] 所有 what_to_write 无分析腔——读起来像场景描述，不是写作教学
- [ ] AI 味自检报告已完成，所有命中项已修复

### 第 2 步：快速嗅探（60 秒内完成）

如果不想逐条过，先做这四个快速嗅探——四个全过，大概率合格：

1. **抽部分 4 一条剧情节点，遮住字段名，能说出"这一段正文会发生什么"吗？** → 说不出来 → what_to_write 太模糊或分析腔太重
2. **扫描全篇提示词，找到一句"他感到""他意识到""这表明""这意味着"吗？** → 找到了 → 视角转换或词句净化漏网
3. **对着 memo.prohibitions 逐条，能在部分 4 的"禁止出现的内容"中找到对应的可执行描述吗？** → 找不到 → 章纲红线未注入提示词

### 第 3 步：验证方法

按以下优先级验证提示词质量：

1. **自检**：严格按第 1 步清单逐项过。特别注意三个高危盲区：
   - **what_to_write 的分析腔**：自检时自己最容易脱敏——你写的分析腔你自己读着很正常。建议把所有 what_to_write 复制出来，换一种字体、换一个阅读环境再看。或者倒着读（从末段看到首段）——打破阅读惯性。
   - **否定句式的隐蔽形态**："让 subagent 避免写成热血战斗"看起来像是正面指引，但核心动词是"避免"——这仍然是负面引导。扫描时关注"避免""防止""别让""切忌""注意不要"等隐蔽否定词。
   - **约束的可执行性**：一条约束你读着"很对"，但 subagent 读着"很空"——这是自检最容易放行的地方。每条约束问自己："subagent 拿着这条，能在写到某句话时停下来判断'这句话违规了'吗？"——判断不了的约束，删掉或重写。

2. **交叉验证**：如果对分析腔脱敏（自己写的提示词自己读不出问题），把提示词发给另一个 agent 做独立评审。重点关注：what_to_write 是否真的可执行、约束是否写得太空。

**规则：** 拿到结论后先按 fail 项改完，再进入正文生成。不得带着 fail 项进入 Phase 3.4。

### 第 4 步：对照示例验证

以下是同一章提示词"验证前"和"验证后"的对比——展示本清单的实际作用。

**验证前（有 5 个致命问题）：**

```
## 四、剧情核心要求

3. 陆征意识到这个案件并不简单。他在心里梳理着所有发现的线索，碎片开始拼成一副不祥的图景。
   不是单纯的失踪——背后有更复杂的势力在运作。他做出了一个决定：不再被动等待，
   要主动出击。核心心理动作：从被动反应转为主动追查。这标志着他心态的重要转折。

  → 问题1: "意识到" = 视角转换失败（上帝视角旁白）
  → 问题2: "碎片开始拼成图景" = 总结性判断
  → 问题3: "不是……而是……" = 否定句式 + 动机分析
  → 问题4: "核心心理动作" = 心理机制分析
  → 问题5: "这标志着" = 分析性结论
  → 问题6: 超过 200 字，全是分析腔，没有一个具体可写的动作

## 五、文字与细节要求：
  - 不要写成热血战斗文
  - 不要太拖沓
  - 不要用诗词腔

  → 三条全是否定引导——subagent 脑子过一遍"热血战斗""拖沓""诗词腔"，然后不知道怎么写
```

**验证后（修正为可执行的版本）：**

```
## 四、剧情核心要求

3. 回事务所。把便利贴贴在白板上。旁边是苏棠的照片——从苏沫那里拿的，一个短发女孩，
   笑起来有点歪嘴。便利贴上写了四个时间：最后通话 / 朋友圈地址查无此址 / 充电器还在墙上 /
   同事说"她去外地了"。第三个时间上画了一个圈。给方岩发消息：帮我查一个车牌。
   发完靠在椅背上，看着白板。外面有人在骂街，他没动。

  → 78字。全是可感知的动作和画面。没有分析、没有结论。subagent 拿到就能写。

## 五、文字与细节要求：
  - 冲突场景控制在日常环境中——出租屋、街边大排档、物流公司仓库。不写追逐和打斗。
  - 每个环境细节顺手带过，不超过 2 句。对话每轮 3-5 句收住。
  - 叙事用大白话。环境只写三样东西——光、声音、温度。不写风景。

  → 三条全是正面指引。subagent 知道"应该写成什么样"，不需要在脑子里先否定错误版本。
```

每个 fail 项都对应一个修正——修正后的提示词，subagent 拿到就能写。
