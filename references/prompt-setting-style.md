# 提示词组装指令

你的任务：将章纲 + writing-style + genre-profile 组装为 subagent 可执行的提示词。产出写入 `prompts/vol-{N}-ch-{M}-prompt.md`。

**前置条件：** 章纲已通过验收（作者确认 + 检查清单通过 + 快速嗅探 + AI味自检）。

---

## 一、方法：叙事种子法

### 核心理念

传统流程：章纲 key_point → 视角转换 → 映射为写作指令 → 否定句式替换 → 双轮净化 → 干净的指令 → subagent 执行 → 干净的正文。这条路到顶 80 分——产出是"正确的指令"，subagent 逐字执行，产出"正确的正文"，但不是人写的语感。

叙事种子法不同：**不写指令，写叙事画面。** 对每条 key_point，直接写一段 POV 角色此刻的叙事画面，subagent 读到的是"继续写这段"，不是"去做什么"。

```
章纲 key_point
  ↓
80分：写指令 → 双轮净化
  → "整理书架，过程中流露心动细节。用动作暗示，不写内心独白。"
  → subagent 执行，节奏生硬

  ↓
100分：写叙事画面
  → "陆屿站在书架前，手指扫过书脊，然后停在一本旧画册上。
     苏晚在吧台后面擦杯子，余光里是他左腕上那条磨得发亮的手链。
     两个人都不说话，但不是尴尬的那种安静。"
  → subagent 继续写：同一语感的自然延伸，没有"执行指令"的痕迹
```

### 三条转化规则

**规则一：问"POV 角色看到/听到/做了什么？"，不问"这段要表达什么？"**

key_point 写的是作者意图（"展露在乎""关系拉近""表现男主对女主的了解"），叙事种子只写画面（"他记得每本书的位置，手指精准地抽出一本旧书"）。

```
❌ 意图分析：
"整理书架时，陆屿用行动表明他很在乎她。两人不说话，但距离在拉近。"

✅ 叙事画面：
"陆屿站在书架前，手指从一排书脊上滑过去，然后准确抽出那本
苏晚上周找不到的旧画册。他递给她时说：'这本你不是一直在找？'"
```

**规则二：写"正在发生的事"，不写"会发生的事"**

```
❌ "本章开始时陆屿在早餐铺等苏晚，两人一起吃了早餐"
→ 将来时 → subagent 按步骤执行，流水账

✅ "苏晚走到巷口，一眼就看见陆屿坐在早餐铺最靠里的位置。
   面前两碗豆浆，一碗已经放了糖。"
→ 进行时 → subagent 以相同语感继续推进
```

**规则三：用 POV 角色的感官写，不用分析写**

```
❌ "这段的重点是让读者感受到梧桐巷的烟火气"
→ 分析语言 → subagent 在正文里"营造烟火气"

✅ "油锅滋滋响，张奶奶的围裙上沾着面粉，隔壁中药铺的收音机放着粤曲。"
→ 感官语言 → subagent 在正文里自然地延续环境描写
```

### 字数倒推法

从目标字数反向拆段。subagent 在 1500 字内可保持语感稳定，超过 2000 字开始出现措辞重复、节奏失调、角色走形。

| 目标字数 | 段数 | 每段字数 | 适用场景 |
|---------|------|---------|---------|
| 1500-2000 | 1 段 | 单段完整 | 一个场景完整展开 |
| 2000-3500 | 2 段 | 1000-1750 | 两个场景 + 过渡 |
| 3500-5000 | 3 段 | 1200-1700 | 起承转合完整 |
| 5000-7000 | 4-5 段 | 各 1400 | 多线叙事或长场景扩展 |

4500 字 → 3 段 × 1500 字。

### 三段式架构

将"部分 4（剧情核心要求）"从一条平铺的剧情线，拆为"全局锚点 + N 段独立叙事入口"。

```
┌─────────────────────────────────────┐
│ 全局叙事锚点（POV + 语感 + 人物锚）    │  ← 写一次，影响全章
├─────────────────────────────────────┤
│ 分段 1/3 入口（画面重入 + 本段指引）    │  ← 每段独立重入
│ 分段 2/3 入口（画面重入 + 本段指引）    │
│ 分段 3/3 入口（画面重入 + 本段指引 + 收尾) │
└─────────────────────────────────────┘
```

**全局叙事锚点（200-300 字）** — 叙事语言写一段统摄全章语感的开场，不是规则说明。subagent 读完这一段就"进入状态"。

```
✅ 叙事语言开场：
"梧桐巷的清晨，是从张奶奶的油条锅开始的。
苏晚站在书店二楼的窗前，看着楼下巷口的那个背影。
——全章主视角：苏晚，第三人称限知
——不看过去心理，不看未来走向，只看此时此刻她看到的、听到的、感受到的
——每段开头写画面，结尾落到她心里那个没说出口的念头"

❌ 规则说明（不是叙事）：
"本章以苏晚的视角展开。注意保持第三人称限知视角，避免上帝视角。"
```

**人物情境锚（100-150 字）** — 用具体动作代替百科标签。subagent 拿到"身高 165cm、温柔内敛"会原文照搬到正文。

```
✅ "苏晚看陆屿时，目光先落在他左腕的手链上；紧张时会摩挲书页、把桌上的东西摆整齐。
   她不是温柔内敛，她是话到嘴边又咽回去。"

❌ "苏晚，165cm，纤细。性格温柔内敛，心思细腻。紧张时会摩挲书页。"
```

**分段叙事入口（每段 200-300 字）** — 画面重入 + 节奏指引。段间用 `---` 物理隔开。

```
✅ 分段入口格式：
场景一｜早餐铺（~500字）

苏晚走到巷口时已经磨蹭了快十分钟。她在紧张什么，自己也不愿意想清楚。
然后她看见了陆屿——他坐在张奶奶早餐铺最靠里的位置，面前两碗豆浆。

——这一段写四个节奏：
1. 张奶奶的对话打破沉默
2. 陆屿挑香菜——不用解释，这就是他的问候
3. 张奶奶念叨两人小时候的趣事，制造轻松笑点
4. 结尾落在：她走出早餐铺时，他自然地走在她左边

只用苏晚的感官。不说"她感到温暖"，写她把手指贴在豆浆碗壁上。
```

三段缺一不可：缺全局锚点则每段语感各自为政；缺人物锚则角色出场仍是百科体；缺分段入口则写到 2000 字开始漂移。

### 叙事种子法完成标准

每条约 150-300 字，以叙事画面开头，以"——写到这里"引导 subagent 自然延续。读起来像正文片段，不像写作指令。

### 写作风格四字段注入

题裁选定后，从 `references/genre-example/` 的 corpus 文件读取四字段：

| 字段 | 写入位置 | 说明 |
|------|---------|------|
| 叙事身份（role） | 部分 1（核心基础要求） | 告诉 subagent 它以什么身份写正文 |
| 核心原则 | 部分 5（文字与细节要求） | 8-15 条大白话约束，不分类不编号 |
| 常见错误 | 合并到核心原则 | 按致命程度排序（视角跳跃 > 直述情绪 > 段尾总结 > 句式规整） |
| 描写技法 | 部分 5（文字与细节要求） | 可操作指引 |

如有 custom role_override 或 genre_taboos，优先使用。

### 输出格式

`prompts/vol-{N}-ch-{M}-prompt.md` 固定 6 部分：

| # | 部分 | 写什么 | 来源 |
|---|------|--------|------|
| 1 | 核心基础要求 | 类型、基调、受众、**单章定位** | writing-style + genre-profile |
| 2 | 世界观与背景设定 | 时代、世界规则、**本章涉及场景** | world-setting + 本卷概览 |
| 3 | 人物设定 | 主角1（含**本章人物状态**）、主角2（含**本章人物状态**）、配角 | character 设定 |
| 4 | 剧情核心要求 | **叙事种子**（每段 150-300 字）、节奏、禁止内容 | 章纲 key_points（叙事种子化） |
| 5 | 文字与细节要求 | 视角、字数约束 | writing-style + genre-example |
| 6 | 补充说明 | 特殊要求、参考风格 | 按需 |

部分 1-3、5-6 从全局设定填充，部分 4 从当前章纲生成（叙事种子法）。所有叙事种子写入部分 4。

## 二、验收

全部通过后才进入正文生成。

### 叙事种子验收
- [ ] 每条 what_to_write 是叙事画面，不是写作指令——读起来像正文片段
- [ ] 不是"这段要写什么"的口吻——不出现作者意图分析（"展露在乎""关系拉近""表现性格"等）
- [ ] 不是将来时——没有"本章将""接下来会""写到XXX时"等时态标记
- [ ] 每条以 POV 角色的感官写——不是作者视角或上帝视角
- [ ] 不含元指令动词——没有"明确""需""要""注意""体现""突出"等
- [ ] 不含负面表述——没有"不要""切忌""避免"，全部是正面描述

### 分段结构验收（字数 ≥ 2000 时必做）
- [ ] 字数倒推已做：总字数 ÷ 1500 确定段数，每段 ≤ 1500 字
- [ ] 有全局叙事锚点（叙事语言开场，不是规则说明）
- [ ] 有人物情境锚（具体动作，不是百科条目——"紧张时会摩挲书页"✅ "性格温柔内敛"❌）
- [ ] 每段有独立画面重入，段间用 `---` 物理隔开
- [ ] 段落间有场景/时间跳跃，不用过渡句硬连
- [ ] 收尾段有情绪落点——画面、念头、未说完的动作，不是"本章完"式总结
- [ ] 每段字数分配合理，各段相加 ≈ 目标总字数

### 视角与 POV 验收
- [ ] 全章单一视角，narrative_pov 已锁定
- [ ] 所有叙事种子不含"他感到""他意识到""读者应该感受到"等上帝视角旁白
- [ ] 所有叙事种子不含解释性过渡（"他注意到""他能听到""他发现"）
- [ ] 当前 POV 角色不在场的场景未出现

### 写作风格四字段验收
- [ ] 叙事身份已写入部分 1——告诉 subagent 它以什么身份写正文
- [ ] 核心原则 + 常见错误已合并写入部分 5——8-15 条大白话，按致命程度排序
- [ ] 描写技法已转化为可操作指引写入部分 5
- [ ] 约束不分类不编号——subagent 不会对"红色规则"和"绿色规则"区别对待
- [ ] 约束每条可执行——subagent 能判断"某句话违规了吗"
- [ ] 约束不含"不要""避免""切忌"——已转为正面指引

### 完整性验收
- [ ] outline.summary 已转化为部分 2 的本章上下文（非粘贴）
- [ ] memo.current_task 已转化为部分 4 的本章目标
- [ ] memo.payoff_plan（must_resolve / must_hold / partial_advance）已体现在叙事种子中
- [ ] memo.prohibitions 全部写入部分 4 的"禁止出现的内容"
- [ ] 前章结尾状态在部分 2 中有描述
- [ ] 本章字数目标已设定，各段篇幅分配合理

### 快速嗅探（60 秒）

如果不想逐条过，先做这三个快速嗅探——全过则大概率合格：

1. **抽部分 4 一条叙事种子，遮住字段名，能说出"这一段正文会发生什么"吗？** → 说不出来 → 叙事种子仍是分析语言
2. **扫描全篇，找到"他感到""他意识到""这表明""这意味着"吗？** → 找到了 → 视角转换漏网
3. **对着 memo.prohibitions 逐条，在部分 4"禁止内容"中找到对应的可执行描述吗？** → 找不到 → 章纲红线未注入

全部通过 → 进入正文生成。任一 fail → 先修再写。
