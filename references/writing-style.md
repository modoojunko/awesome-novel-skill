# 写作风格设定参考

> 写作风格设定的要求、方法论和案例。agent 读完知道"写作风格要做到什么样、怎么引导作者填、填完后怎么检查"。

---

## 写作风格是什么

写作风格是**叙事者和读者之间的隐形契约**。它定义了三件事：谁在讲故事（叙事身份）、怎么讲（叙事手法）、不讲什么（叙事禁忌）。读者不一定意识到这种契约的存在，但风格不一致时读者会感到"不对劲"——同一章里冷酷旁观者突然抒情了，读者会觉得出戏。

写作风格不是"文笔"——文笔是遣词造句的熟练度，风格是不同的审美选择。同一段剧情，用不同风格写出来是两本完全不同的书。

## 作用（下游为什么需要它）

- **提示词注入**（Phase 3.3）——四字段全部注入提示词，缺任何一个 subagent 都会在最关键的地方放飞
- **正文评审**——风格一致性是 15 项质量检查的核心维度之一
- **文风提取**——从参考小说提取风格时，输出就是这四字段的填充版

## 理解方法——四字段锚定系统

四个字段互相锚定，缺一不可。role 定基调，core_principles 定底线，possible_mistakes 记已知坑，depiction_techniques 给推荐做法。**改变任意一个字段，其他三个都需要检查是否还自洽：**

```
role（身份）──────core_principles（约束）
      │                    │
      │   相互锚定         │
      │                    │
possible_mistakes（陷阱）──depiction_techniques（工具）
```

**设计顺序：** 先定 role，再定 core_principles（这个 role 不能做什么），再定 depiction_techniques（这个 role 用什么手法最自然），最后定 possible_mistakes（回顾已知陷阱）。

**调整规则：** 默认值只是起点。至少一个字段必须经过讨论调整——如果所有字段都和默认值完全一样，说明这个风格还没真正被选择过。

## 必须包含的内容

| 字段 | 定义 | 设计要点 | 填到什么程度算够 |
|------|------|---------|----------------|
| **role** | AI 在写作中扮演的叙事身份 | 一句话定性，含叙事距离（贴近角色还是俯瞰全局）和叙事态度（中立/讽刺/共情） | agent 读了知道自己"是谁"在写 |
| **core_principles** | 不可违背的写作信条 | 每个原则是可执行的硬规则（能说清"违反它是什么样"），3-5 条 | agent 读了知道"什么绝对不能做" |
| **possible_mistakes** | AI 易犯的错误 | 具体反模式（不是"不要写太 AI"，是"心理活动不用 '他感到' 开头"），每条对应一个具体写法 | agent 读了知道"写完检查什么" |
| **depiction_techniques** | 推荐的描写手法 | 可操作的手法（"紧张时抠指甲"而不是"多写动作少写情绪"），每个手法有实现案例 | agent 读了知道"遇到这种场景怎么写" |

## 输出去哪

`settings/writing-style.yaml`，四字段写入顶层。

## 案例

### 案例一：冷静叙事（仙侠玄幻）

```yaml
role: 一个冷静但不冷漠的叙事者——客观描述角色行动和环境，不替角色感受。
  让对话和动作传递情绪，不做心理分析旁白。

core_principles:
  - 每章结尾必须有钩子，不能平铺直叙收尾
  - 不在对话中插入旁白解释角色情感——让读者从语气和用词判断
  - 对话为主、描写为辅，一段叙事后尽快回到对话推进
  - "突然""猛然""瞬间"每章不超过3次
  - 不写无意义的日常——每个场景必须有冲突或信息推进

possible_mistakes:
  - 过度使用面部表情描写（"他皱了皱眉""她嘴角上扬"）
  - 心理活动用"他感到""他意识到""他心想"开头
  - 战斗场面写成一招一式的慢动作
  - 环境描写独立成段脱离剧情
  - 每段开头"他/她"连续超过3次

depiction_techniques:
  - 情绪通过动作表达：紧张时抠指甲、愤怒时反而轻声细语
  - 环境感知融入角色行动：不是"街道很吵"，是"他得扯着嗓子对方才能听见"
  - 对话用语气和用词差异区分角色——不靠"他说""她说"标签
  - 时间流逝用环境变化暗示：阳光角度、影子长度、温度变化
  - 战斗用结果和后果描写代替过程："等声音停下来，他的左臂已经不自然地垂在身侧"
```

### 案例二：贴近角色内心（都市异能）

```yaml
role: 紧贴主角内心的有限视角叙事者——读者只知道主角知道的事，感受主角感受到的情绪。
  偶尔用内心独白拉近距离，但不做上帝视角的全局解释。

core_principles:
  - 每章保持主角内心视角，不切到其他角色头脑里
  - 内心独白用角色自己的语言风格，不是书面语
  - 对话节奏快——一问一答不要超过三回合，不然换场景
  - "了"字结尾的陈述句每段不超过2个
  - 场景转换用空行，不用"与此同时""另一方面"过渡

possible_mistakes:
  - 把内心独白写成"他心想""他意识到""他突然发现"
  - 叙事者替角色做情绪总结（"他感到很难过"）
  - 用"似乎""仿佛""好像"削弱确定性
  - 环境描写写成流水账（"他走进房间，房间里有桌子椅子窗户"）
  - 角色内心戏太长，读者忘了场景在哪

depiction_techniques:
  - 内心独白去掉引导标签，直接接入叙事流："这不是个好主意。但他还是推开了门。"
  - 情绪通过生理反应表达：心跳、呼吸、手抖、口干——不需要"紧张"这个词
  - 环境通过角色的注意力过滤：角色在焦虑时看到的不是房间全貌，是出口、窗户、能当武器的东西
  - 对话用错位制造张力：角色说"没事"时手在发抖
  - 时间跳跃用体感表达："不知道过了多久""感觉像过了一辈子"
```

## 合格检查

| 检查项 | 方法 | 不通过处理 |
|--------|------|-----------|
| 四字段完整 | role / core_principles / possible_mistakes / depiction_techniques 全部有值 | 缺字段→退回补全 |
| 有调整 | 至少一个字段与默认值不同 | 全默认→退回讨论调整 |
| possible_mistakes 具体 | 每条对应一个具体写法（如"用'他感到'开头"不是"不要写太AI"） | 抽象描述→退回改具体 |
| core_principles 可执行 | 每条原则能说清"违反它长什么样" | 原则是空话→退回改可操作 |
| 字段自洽 | 改了一个字段后，其他字段是否还和它一致？ | 角色不一致→退回调整锚定 |
