## 1. 安装与配置

### 1.1 前提条件

- 已安装 Claude Code 或 DeepSeek TUI
- 具备命令行操作能力

### 1.2 安装步骤

**方式一：克隆仓库**

```bash
git clone https://github.com/modoojunko/awesome-novel-skilll.git
cd awesome-novel-skilll
```

**方式二：下载最新 Release**

从 [Releases](https://github.com/modoojunko/awesome-novel-skilll/releases) 页面下载最新版本的 zip 包，解压即可。

### 1.3 配置 Claude Code

在 Claude Code 中加载本 Skill：

1. 打开 Claude Code 设置
2. 找到 Skills 或 Plugins 配置
3. 添加 `awesome-novel-skilll` 的路径

或在项目目录中直接运行：

```bash
claude
```

---

## 2. 启动 Skill

### 2.1 进入项目目录

```bash
cd awesome-novel-skilll
```

### 2.2 启动 Skill

在 Claude Code 中输入：

```
/awesome-novel
```

或直接告诉 AI：

> 帮我写一本修仙小说

### 2.3 系统响应

AI 会检测当前状态，发现是新项目，开始初始化流程：

```
检测到：无 story.md → 新项目
正在读取 skills/setup/SKILL.md...
```

## 3. Phase 1：设定

设定阶段一次性完成，包含：题材选择、写作风格、世界观、角色设定。

### 3.1 题材选择

**AI 会问：**

```
让我们从题材开始。你想写什么类型的小说？

我为你准备了几种常见类型作为参考：
1. 玄幻修仙  2. 都市异能  3. 科幻星际  4. 武侠江湖
5. 悬疑推理  6. 都市言情  7. 奇幻冒险  8. 历史穿越

你想写哪种？或者告诉我你的想法，我来帮你匹配。
```

**用户回答：**

> 我想写一个修仙题材的，主角从凡人开始，一步步修炼成仙的故事。

### 3.2 选择满足类型

**AI 解释：**

```
修仙题材通常满足以下几种"爽点类型"：

A. 压抑 → 逆袭型：主角前期被打压，后期实力碾压
B. 成长型：实力从弱到强，每一章都有进步
C. 复仇型：有仇必报，快意恩仇
D. 探索型：解开世界之谜，追寻大道真相

你更倾向于哪种？或者想要组合？
```

**用户回答：**

> 我想要 A + C 的组合，主角前期被宗门抛弃，后来复仇崛起。

## 5. Phase 3：逐章写作

### 5.1 第1章：灭门之夜（章纲）

**AI 会设计章纲：**

```
【第一章：灭门之夜】

本章定位：
- 视角人物：林逸（9岁）
- 时态：过去时态（回忆杀）
- 核心事件：家族被灭门
- 章末变化：林逸目睹父亲被杀

【8段Memo】

1. current_task：本章要完成什么？
林逸9岁生辰之夜，家族遭遇突袭，父亲为保护他而死。

2. reader_expectation：读者期待看到什么？
惨烈的灭门场景，父亲的牺牲，主角的创伤记忆。

3. payoff_plan：前文铺垫怎么回收？
本章是伏笔，读者会好奇"谁灭了林逸家族"，后续揭示陈风是幕后黑手。

4. downtime_functions：缓和段落？
灭门前一家三口其乐融融的场景，与后面的惨烈形成对比。

5. key_choices：主角必须做什么选择？
林逸在逃跑和留下之间选择，父亲让他先走。

6. knowledge_state：角色知道/不知道什么？
林逸不知道是谁灭门，只记得凶手蒙面。

7. required_changes：本章必须发生什么变化？
父亲死亡，林逸性格从天真转为沉默。

8. prohibitions：什么绝对不能出现？
- 不能出现"他眼中闪过一丝寒芒"等 AI 味描写
- 不能让凶手在本章露出真面目
- 不能写得太长，本章定位是"楔子"

【情绪设计】

primary_mood：悲伤、紧张
mood_progression：
- 开头（温馨生辰）→ 中间（突袭恐惧）→ 结尾（悲伤绝望）
intensity_level：中高
intensity_peak：父亲死亡的那一刻
micro_payoffs：
- 温馨的家庭场景
- 父亲的牺牲
- 小林逸的眼泪

【Hooks】

埋设：
- "灭门凶手使用的是青云宗的剑法"（伏笔）
- "父亲临死前把玉佩塞进林逸手里"（线索伏笔）

章纲确认，开始生成提示词？
```

**用户回答：**

## 7. 常见问题

### Q1: 如何暂停/继续写作？

**暂停：**
直接在对话中说"今天先到这里"，AI 会保存当前进度到 `.agent/status.md`。

**继续：**
下次启动时，输入：
```
继续写小说
```
或触发 Skill：
```
/awesome-novel
```
然后说"继续"。

AI 会自动检测 `.agent/status.md`，恢复到上次进度。

### Q2: 发现前面的章节有问题，怎么修改？

1. 输入"修改第X章"
2. AI 会分发到对应子 skill
3. 讨论修改内容
4. 修改后重新验收
5. 更新角色状态（如有影响）

### Q3: 想增加一个新角色怎么办？

1. 输入"增加一个新角色"
2. 告诉 AI 角色设定
3. AI 在 `settings/character-setting/` 创建新角色文件
4. 在后续章节中自然引入

### Q4: 提示词验收不通过怎么办？

1. AI 会指出不通过的原因
2. 根据意见修改提示词
3. 重新验收
4. 直至通过后再写正文

### Q5: 正文中 AI 味太重怎么办？

正文验收会检测：
- 疲劳词（忽然、竟然、陷入等）
- 句式违规（过度使用某种句式）
- 对话违规（标签过多、感叹号过多）

不合格项会自动修复，修复后重新验收。

### Q6: SOLO 模式是什么？

如果用户说"你全权写"、"solo"、"单机"，AI 会进入 SOLO 模式：
- 不停下来等确认
- 自动选择方向
- 直接执行下一步
- 每章完成后提示"/new 继续下一章"

适合熟悉流程后快速写作。
