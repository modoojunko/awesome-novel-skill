# chapter-planner skill


## 流程概览

```text
PreFlight（4 维观察）→ 作者确认草案 → 展开纲要点 → 禁止清单 → Memo 8段 → 情绪设计 → Hooks → 验收
```

## PreFlight：4 维并行读取

规划新章前，从 4 个维度读取项目文件获得观察报告。**四个维度独立读取后汇总，不分发 subagent——主 agent 自行读取。**

### 1. 情节维度

守住情节逻辑链——上章发生的事如何自然延续。

读取：前一章 chapter.md（outline, key_points）、卷纲（chapters_summary）、相关角色文件（状态历史）

输出观察：上章关键情节 → 本章情节如何衔接

### 2. 架构维度

守住钩子兑现路径——哪些旧钩子必须在这章处理。

读取：所有已归档章节的 hooks 汇总（未兑现项）、主线拆纲

输出观察：必须兑现的旧钩子 → 本章应新增的钩子 → 与主线的关系

### 3. 情绪维度

守住角色情绪弧线——角色此刻的心理状态。

读取：前一章 emotional_design、相关角色文件（状态历史）

输出观察：上章情绪弧线 → 角色当前心理状态 → 本章情绪走向建议

### 4. 读者维度

守住读者期待——读者追到这章想知道什么。

读取：前一章 emotional_hook、所有未兑现钩子清单

输出观察：读者此刻最大的悬念 → 读者期待的情感回报 → 本章应给出的 micro_payoff 位置

### 汇总仲裁

收到四份观察后，综合输出规划草案：

| 优先级 | 规则 |
|--------|------|
| 1 | 架构 > 情节：钩子兑现是硬约束，情节逻辑不得违背钩子路径 |
| 2 | 架构 > 读者：必须兑现的钩子优先于读者期待的 micro_payoff |
| 3 | 情节 > 情绪：角色情绪不能与情节逻辑冲突——先有行动再有情绪 |
| 4 | 超出以上规则的冲突 → 标注【冲突】+ 双方观点 → 提交作者仲裁 |

### 规划草案输出格式

```markdown
## 规划草案
### 本章定位
[一句话，基于四份观察综合]
### 情节方向
[基于情节维度]
### 必须兑现的钩子
[基于架构维度]
### 情绪走向
[基于情绪维度]
### 读者期待回应
[基于读者维度]
```

展示给作者确认后，才进入章纲细化。

## 章纲流程

### a. 展开纲要点

展开 8-12 条 key_points，每条 2-3 句锚点描述 + 功能类型标记（`[推进剧情]`/`[造悬念]`/`[过渡]`），有对话的加 `[对话]`。写一句 summary，列出 characters/location/time，确认 narrative_pov。

**STOP：作者确认 outline。**

### a2. 禁止清单

必须询问："本章有没有明确禁止出现的场景、元素或情节？" 记录到 memo.prohibitions。作者说"没有"也要确认一次。

### b. Chapter Memo（8 段）

从方向纲要自动填充，不逐段讨论。填充后展示完整 memo 供作者确认。

- **current_task：** 要达成本章改变必须完成的具体推进动作
- **reader_expectation：** 读者情绪状态 + 本章策略 + 具体说明
- **payoff_plan：** 三列表——must_resolve（旧钩子了结）/ must_hold（不能碰的悬念）/ partial_advance（推进不收束+新钩子）
- **downtime_functions：** 过渡场景的功能映射，每段 [场景] → [功能]
- **key_choices：** 角色选择 + 为什么 + 符合人设 + 读者感受
- **knowledge_state：** 每个出场角色知道什么/不知道什么 + 信息差 + 对话标记
- **required_changes：** 1-3 条，每条标注改变类型 + "从什么变成什么"
- **prohibitions：** 从 a2 来

### c. Emotional Design

- **primary_mood：** 本章整体情绪基调
- **mood_progression：** 情绪变化轨迹（从X→经Y→以Z结束）
- **intensity_peak：** 情绪峰值场景位置
- **emotional_hook：** 章尾情绪钩子类型（悬念/共情/期待/愤怒/好奇/选择/渴望）
- **intensity_level：** 1-10 级别
- **micro_payoffs：** 每章至少 1 个，类型：info/relationship/emotion/clue/ability/resource/recognition。放在前段/中段

### d. Hooks 操作

写入本章的钩子操作（埋/推进/收束）。不读写全局 hooks.md——真相源在各 chapter.md 的 hooks 字段。

### e. 验收（全部通过才进提示词生成）

**第 1 步：给作者看结构化反馈**

展示格式：
```text
📋 第{N}章 章纲反馈
本章定位：{一句话}
主要内容：{2-3 句}
关键场景：...
冲突进展：{推进了什么}
字数目标：{N} 字
```

作者必须明确说"对"才算合格。"差不多""你看着写"不算通过。

**第 2 步：检查清单自检**

24 项检查（高频必检项）：
- [ ] 字数目标已确认（4000+）
- [ ] 段落分布已确认（推进/过渡/埋伏笔各几段）
- [ ] 每条 key_point 对应一个可写场景，含冲突事件
- [ ] 旧钩子兑现/新钩子埋下已记录
- [ ] 硬约束（prohibitions）每条可验证
- [ ] 情绪走向至少三个节点（A→B→C）

**第 3 步：快速嗅探**

遮住章纲能不能说出：
- 分多少条、每条写什么场景
- 每个角色知道什么、不知道什么
- 兑现了哪些旧钩子、新埋了什么钩子

**第 4 步：AI 味自检**

| # | 检测项 | 说明 |
|---|--------|------|
| 1 | 上帝视角概述 | 不含"本章讲述了""主角经历了""本章将展示" |
| 2 | 抽象心理总结 | 不含"他感到""他意识到""他内心挣扎" |
| 3 | 分析性结论 | 不含"这表明""这意味着""这标志着" |
| 4 | 文学评论腔 | 不含"通过X展现了Y""完成角色弧光转折" |
| 5 | "不是而是"句式 | 所有字段避免"不是X，而是Y" |
| 6 | 抽象标签 | 不含"关系递进""冲突升级"——应写具体事件 |

**作者确认章纲后：** 写入 `chapters/vol-{N}-ch-{M}.md`，status → `outline`
