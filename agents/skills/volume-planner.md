# volume-planner skill

> 来源：skills/outline/SKILL.md。主线拆纲仅首次执行，卷纲规划每卷循环。

## 流程概览

```text
首次：主线拆纲（0.0→0.5）→ 三向核对（0.6）→ 卷方向确定（1.x）→ 卷纲讨论（2.x）
卷N+1：跳过0.x，直入1.1 角色发声 → 2.x
```

## 0. 主线拆纲（仅首次执行）

### 0.0 先判断作者类型

问："你目前对全书的故事走向有完整概念吗——从开头到结局都想好了？还是只有第一卷的思路，后面边走边看？"

- **类型 A：都清楚** → 走 0.1→0.5 全流程
- **类型 B：只知道开头** → 跳过 0.2 定终点和 0.3 找断点；0.1 总主线窄化为"第一卷方向"；0.4 只定第一卷冲突，后续卷标"待定"
- **介于之间** → 走类型 A，后卷冲突标"待定"

### 0.1 总主线

从作者回答中提取三要素（**谁 + 追求什么 + 对抗什么**），浓缩为一句话让作者确认。

- 类型 A：标准——不做完所有卷答不出来的全书核心冲突
- 类型 B：只聊第一卷方向——谁 + 想做什么 + 被什么阻碍

### 0.2 定终点

> 类型 B 跳过。终点和断点标"待定"，直入 0.4。

从结局倒推不问"几卷"，问"最后一幕"。根据作者回答类型（有画面/有调性没画面/完全不知道）走不同分支。

终点确定后核验与总主线的一致性：矛盾可能是张力来源，确认作者意图。

### 0.3 找自然断点

> 类型 B 跳过。

起点到终点之间的根本性变化。Agent 可根据题材+总主线推理提案让作者确认。

### 0.4 定每卷冲突

一卷一卷确认，不要一次列完。

| 字段 | 标准 | 不合格示例 |
|------|------|-----------|
| 标题 | 2-4 字，概括本卷核心事件 | "第一卷"（等于没起） |
| 核心冲突 | **总主线在哪一步 +** 谁 + 做什么 + 被什么阻碍。是主线子集，不是另起炉灶 | "主角继续调查"（无对抗）；"主角卷入国际阴谋"（与主线无关） |
| 预估章节 | 根据题材节奏估算，允许 ±30% | 0 或空 |

后卷处理：
- 类型 A：后卷至少写一句话核心冲突+浮动章数，写到时再细化
- 类型 B：只写第一卷，后续卷全部标"待定"

### 0.5 产出写入

按格式写入 `story.md`（详见旧 outline SKILL.md §0.5）：

类型 A 含全书核心冲突 + 结构类型 + 总卷数 + 每卷核心冲突。
类型 B 核心冲突标"待定"，仅第一卷有内容。

### 0.6 验收：三向核对

三项核对全部通过才可进入卷方向确定：

1. **总主线→逐卷：** 指着总主线，对每卷问"这卷是主线的哪一步？"——能答出✅，答不出或牵强❌
2. **卷序列→完整路径：** 所有卷核心冲突串起来，是一条完整的故事线、没有断裂跳步吗？
3. **卷→反向推断总主线：** 只看卷冲突能猜出全书核心冲突吗？还是各自指向不同方向？

类型 B：核对一只验第一卷，核对二/三自动通过。

**总检：** 指着总主线问"没有这卷，主线走得完吗？"——走不完✅

## 1. 卷方向确定（先于卷纲讨论）

### 1.1 卷 N+1：让角色说话

有前卷存档（`archives/` 非空）→ **直接从角色驱动分析开始**，不推模板。

1. 读前卷未收束的钩子和主角当前状态
2. 读各活跃角色的最新状态历史
3. 对每个活跃角色（含 antagonist/重要配角），生成角色"发声"：

```text
【{角色名}的处境】卷末落位——和谁在一起、掌握了什么信息、目前处于什么局面
【{角色名}未完成的事】上一卷留下的执念/目标/恐惧——什么还没解决
【{角色名}卷间思考】在卷与卷之间的间隙里反复在想什么
【{角色名}现在想做的事】基于当前处境下一卷最可能推动什么
```

4. 展示给作者。几个角色的意志冲突会自然形成主线。
5. **从发声到章节展开（4 步检测）：**
   - 检测主线是否浮现——所有角色的"现在想做的事"是否指向同一个矛盾点？
   - 推导章节数——每个角色"现在想做的事"大约需要几章完成？加总得卷长
   - 检查章纲方向——从主线能拆出各阶段推进方向吗？
   - 模板可选——主线清晰则模板是辅助骨架
6. 作者根据角色发声+4 步检测结果确定卷方向。

**兜底：** 角色发声后作者仍然没方向 → 回到模板辅助找结构。

### 1.2 卷 1：展示故事结构模板

首次规划卷纲（`archives/` 为空）→ 使用结构模板提供骨架。

Agent 直接输出提案（不问"你想要哪种"），根据题材推测最适合的模板：

| 模板 | 适合 | 默认分卷 |
|------|------|---------|
| **三幕式**（setup→confrontation→resolution） | 冲突驱动、有明确对手或目标 | setup 3-4章 / confrontation 4-5章 / resolution 2-3章 |
| **起承転結**（起→承→転→結） | 东方叙事结构，慢热细腻 | 起 2-3章 / 承 3-4章 / 転 2-3章 / 結 1-2章 |
| **悬疑递进**（疑问→线索→反转→答案→余味） | 解谜向、有信息差 | 疑问 1-2章 / 线索 2-3章 / 反转 2章 / 答案 1章 / 余味 1章 |
| **人物弧线**（日常→转折→挣扎→蜕变→新常态） | 人物驱动、成长向 | 日常 2章 / 转折 2章 / 挣扎 2-3章 / 蜕变 1-2章 / 新常态 1章 |

## 2. 卷纲讨论

> 一卷只有一个主线，所有章合起来讲完这个主线。少了任何一章故事就不完整。

### 2.1 定核心冲突

从 story.md 继承本卷核心冲突，问作者"有要调整的吗？"

**底线（任一不通过禁止进入拆章节）：**
1. **核心冲突可一句说完**——"谁 + 想做什么 + 被什么阻碍"，不需要"与此同时""另一方面"
2. **对抗力量具体**——阻碍必须是具体的角色/势力/处境，不是"对抗命运""对抗社会"这类虚词

### 2.2 拆章节 + 填冲突载体

以上一步确认的模板各阶段或角色发声推导的冲突路径为骨架，逐章填充。

| 字段 | 标准 | 不合格示例 |
|------|------|-----------|
| id | "卷号-章号"（如 1-1），连续不跳号 | 1-1, 1-3（跳了 1-2） |
| title | 有信息量，能猜到本章核心事件 | "第一章""新的开始""转折" |
| summary | **三要素：谁做什么 + 冲突事件 + 结束时什么变了** | "主角继续调查"（什么都没变） |

**4 种不合格章纲，必须检测并修正：**

| 类型 | 典型句式 | 修正方向 |
|------|---------|---------|
| **主题式** | "本章讲信任"（无事件） | 把概念变成具体事件：谁在什么事上信任了谁 |
| **功能式** | "建立人物关系"（功能=展示，不是推进） | 把功能折入冲突事件：通过某件事来建立 |
| **梗概式** | "主角经历了一天的冒险"（无具体冲突点） | 指认一天里最核心的一件事 |
| **结果式** | "主角赢了比赛"（只有结果无过程） | 写出阻碍和对抗：怎么赢的、过程中遇到什么 |

**章节间必须存在因果链：** 前一章的章末变化是后一章的起点或动机。

### 2.3 新角色与设定追加

卷纲讨论中新增的角色/设定，在进入写作循环前落实：
- 新角色 → 创建 `settings/character-setting/{id}.md`
- 新设定 → 追加到 `settings/world-setting.md`
- 一致性检查 → 新增设定与已归档章节无矛盾

### 2.4 产出写入

按格式写入 `volumes/volume-{N}.md`：

```markdown
# 卷 N：{标题}

## 卷信息
- **核心冲突：** 谁 + 做什么 + 被什么阻碍
- **预估章节：** N

## 章节列表
### N-1：{有信息量的标题}
- **冲突事件：** 谁做什么 + 冲突事件 + 结束时什么变了
```

### 2.5 验收

**卷级 4 验证：**
1. **章合起来=这卷讲完？** — 逐章读所有 summary，合起来是完整故事吗？
2. **每章可追溯到卷冲突？** — 指着核心冲突，每章问"这章推进了冲突的哪一步？"
3. **每章结束后卷故事前进了？** — 章末变化是否影响了后续？
4. **标题匹配故事？** — 只看 title 能猜到这章发生什么吗？

**15 秒快速嗅探（三个全过≈合格）：**
1. 能不看文件说出核心冲突一句话吗？
2. 随便指一章，能说出"这章结束后什么东西变了"吗？
3. 随便指两章，能说出"为什么第一章之后是第二章、不是直接跳到第三章"吗？

**卷 N+1 补充：** 验证角色发声中的角色状态是否与已归档章节一致——不一致说明归档时状态更新有遗漏，需修复。
