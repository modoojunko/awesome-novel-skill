## 工作原理

```
prompt-crafter agent 生成提示词时：
    ↓
读取 anti-ai/common-rules.md（通用规则）
    ↓
读取 anti-ai/{genre}.md（题材正反例，如存在）
    ↓
合并填入 prompt.md 模块5
    ↓
subagent 写正文时执行这些规则
```

## 文件说明

### common-rules.md — 通用规则

所有题材共用。包含：
- **疲劳词阈值** — 词/单章上限/替代策略（如"突然≤3次，删掉"）
- **句式规则** — 禁止模式（如"连续4句相同主语"）
- **元叙事禁止** — 发现即删（如"本章""故事还在继续"）
- **情绪外化规则** — ❌/✅ 对照

### {genre}.md — 题材正反例

每个题材独立文件。包含：
- **高频AI病句** — 该题材最常见的AI味写法
- **正反例对照** — ❌ AI味 vs ✅ 真人感
- **写作要点** — 该题材的特殊注意事项

## 格式规范

所有题材文件遵循统一格式：

```markdown
# {题材名}反AI规则

> 适用题材：{genre-id-1}、{genre-id-2}

## 高频AI病句正反例

### 1. {问题类型}

❌ "AI味的写法"
✅ "真人感的写法"

### 2. {问题类型}

...

## 写作要点

1. **{要点标题}** — 具体说明
2. ...
```

## 如何贡献新题材

### 方式1：新建题材文件

1. 在 `memory/anti-ai/` 下新建 `{genre-id}.md`
2. 复制以下模板并填写：

```markdown
# {题材名}反AI规则

> 适用题材：{genre-id}

## 高频AI病句正反例

### 1. {问题类型}

❌ "AI味的写法"
✅ "真人感的写法"

### 2. {问题类型}

❌ "..."
✅ "..."

## 写作要点

1. **{要点}** — 说明
2. **{要点}** — 说明
```

### 方式2：扩展现有题材

已有文件的写作要点不够用时：
1. 找到对应文件
2. 在"高频AI病句正反例"下增加新的问题类型
3. 每个问题类型包含一对 ❌/✅ 对照

## 贡献指南

### 正反例原则

1. **❌ 要具体** — 给真实的AI味句子，不是抽象描述
2. **✅ 要可执行** — 给可复制的真人感写法，不是"要自然"
3. **对照要明显** — 同一场景、同一情绪，❌→✅ 转化清晰

### 写作要点原则

1. **具体不抽象** — "写手抖"不是"写紧张"
2. **可执行** — "手抖=握不住杯子"不是"要有细节"
3. **题材特色** — 每个题材的要点要反映该题材的特殊性

### 提交前自检

- [ ] 每个正反例的 ❌ 是真实的AI味句子？
- [ ] 每个正反例的 ✅ 可直接复制到正文中？
- [ ] 写作要点是否针对该题材的特殊问题？
- [ ] 是否避开了其他文件已有的重复内容？

## 示例

假设要为"游戏体育"题材新增文件：

```markdown
# 游戏体育反AI规则

> 适用题材：game-sports

## 高频AI病句正反例

### 1. 比赛胜利描写

❌ "他赢得了比赛，全场观众欢呼雀跃"
✅ "终场哨响了。他愣在原地，队友冲过来把他抱起来。他的腿软了，没站稳。"

### 2. 排比式心理

❌ "也许是压力太大，也许是对手太强，也许是自己还没准备好"
✅ "他深吸一口气。裁判在低头看表。他活动了一下脚踝，鞋带有点松。"

## 写作要点

1. **比赛写身体** — 喘、抖、腿软、口渴，不是"紧张"
2. **训练写重复** — 每天练什么？第几次失误？第几瓶水？
3. **胜利写反差** — 赢了之后干什么？不是欢呼，是愣住、腿软、找水喝
```
